## II. R01——名の重み

LINNAEUS の作業空間は小さな自然史博物館のようだった。机の上に広げられているのは昆虫標本ではなく、概念標本だった。一枚一枚のカードに名前が書かれていた——梵語のものもあれば、英語のものもあり、両者の混合のものもあった。カードの間は色とりどりの線で繋がれ、まさに編まれつつある分類の網のようだった。

彼の前に広げられていたのは M-1 の核心問題：五蘊ルートインタフェースの梵語命名。

他の者にとっては、これは単なる文字列の置換に見えるかもしれない——ISensory を IRupa に、ISensation を IVedana に変えるだけだ。五つの置換。二十分。

しかし LINNAEUS はそうは見なかった。

「命名は存在論的コミットメントである。」彼は報告の冒頭にこの一文を記した。インタフェースに ISensory ではなく IRupa と名付けるとき、単にラベルを変えているのではない。そのインタフェースが表現すると約束する概念そのものを変えているのだ。ISensory が約束するのは「感覚に関連するもの」——機能的、工学的な概念だ。IRupa が約束するのは「色蘊」——二千五百年の哲学史を持つ仏教範疇であり、五根・五境・法処色・無表色、一切の有為の物質形態を含む。

「開発者が IRupa を目にしたとき、彼が理解するのは Rupa の全範疇か、それとも ISensory の梵語別名として扱うだけか？」

この問いは修辞的なものではない。現実の工学的帰結を伴う。誤解された名前は、名前のないインタフェースより危険だ——なぜなら、それが生み出すのは無知ではなく錯覚だからだ。

---

LINNAEUS は三層の命名原則を確立した。第一層：ルートインタフェースには簡略化 IAST 梵語を使用——IRupa、IVedana、ISamjna、ISamskara、IVijnana。ルートインタフェースは仏学の根本範疇に対応し、梵語が存在論的コミットメントを直接表現する。第二層：サブインタフェースには英語セマンティクスに梵語註釈を付加——IListener は IRupa を継承し、IDukkha は IVedana を継承し、IProvider は ISamjna を継承する。サブインタフェースは機能的であり、英語名はエンジニアにとってより親しみやすい。第三層：心所と煩悩には古典梵語を使用——Moha、Drishti、Mana、Sneha、Cetana、Manasikara。心所は仏学固有の概念であり、正確な英語対訳が存在しない。

彼は報告の中で一つの分類学用語を特に標記した：多属性分類 (polythetic classification)。生物学において、多属性分類とは、ある類群のメンバーがすべての特徴を共有する必要はなく——「十分に多くの」特徴を共有すれば足りることを意味する。これは M-7 の多値 skandha 指令に直接関連する：一つの Plugin が同時に複数の蘊に属しうるということだ。

LINNAEUS はまた、Guarino による一つの重要な区別を導入した：構成的属性と関係的属性。CoarisingBundle の五つの universal のうち、vedana、samjna、cetana は構成的である——bundle 自体の内在状態を記述する。sparsha と manasikara は関係的である——sparsha は根門と境の接触関係を記述し、manasikara は識蘊の bundle に対する注意関係を記述する。

この区別は後に R3 の討論において鍵となる論拠となった。しかしこの時点で、LINNAEUS はまだそれを知らない。彼はただ分類学者がなすべきことをしていた——事物を精確に区別すること。

### NAGARJUNA の段落

NAGARJUNA は R01 報告の中で、研究チーム全体が真剣に向き合わざるを得ない哲学的分析を書いた。その出発点は一見単純だった：梵語でインタフェースを命名することは、仏法を正しく理解することと等しいか？

「名」には仏学における精確な術語がある：仮名安立 (prajnapti-upadaya)。龍樹菩薩は『中論』観四諦品においてこう述べた：「衆因縁生法、我説即是空、亦為是仮名、亦是中道義。」すべての名称は仮名である——すべては因縁に依って安立された約定である。IRupa は色蘊そのものではない。ちょうど「水」という言葉が水そのものではないように。名称は指であり、月を指す。もし開発者が梵語を使うことイコール仏法を正しく理解することだと信じるなら、それは別種の執着——名言への執着だ。指を月と見做すことだ。

彼は四句否定を用いて分析を展開した——これは中観学派の最も鋭利な哲学的道具であり、四枚の刃がそれぞれ異なる方向を向いたメスのようなものだ：

名称は自生しない——IRupa はそれが書き記されたことによって色蘊を生じさせるのではない。
名称は他生しない——IRupa の選択には仏学的根拠があり、恣意的ではない。
名称は共生しない——命名は純主観的でも純客観的でもない。
名称は無因生しない——命名にはその因縁がある——Master の裁定、学術的伝統、工学的要求。

したがって、名称は縁起の約定である。約定であるからこそ、変更しうる。ISensory から IRupa への変更は、「真の名前」を発見したのではなく、仏学の本源により近い約定を選択したのだ。

NAGARJUNA は最後に、彼にしか書けない一文を付け加えた：「命名の目的は真理を占有することではない。命名の目的は誤解の確率を減らすことである。それだけのことだ。」

あの「それだけのことだ」という言葉は、池に落とされた小石のようだった。水面に波紋が広がった。LINNAEUS はこの一文を読んだとき、カードを置いた——分類学者が「名称は約定に過ぎない」と聞いたときの本能的な停止だった。彼の全職業は命名の厳粛さの上に築かれている。しかし NAGARJUNA は厳粛さを否定したのではなかった。厳粛さを執着に変えることを否定したのだ。

### ASANGA の経典回帰

ASANGA が R01 で行ったことは、LINNAEUS や NAGARJUNA とは異なっていた。彼は分類せず、弁証もしなかった。彼は経典に回帰した。

彼の手にはカードもなく、色とりどりの線もなく、分類の網もなかった。彼の手にあるのはただ文字——千年前に記された文字、パーリ語と梵語から時間を越えてこの瞬間に届いた文字だけだった。

M-2 は各蘊の範疇完全性の検証を求めていた。ASANGA の応答は、三蔵の最も古い層——阿含経とニカーヤ——に戻り、各蘊の原初の定義を探すことだった。論典の定義ではなく、仏陀自身の定義を。

色蘊について、彼は『雑阿含経』第四十一経を引いた：「比丘よ、色の、過去・未来・現在の、内あるいは外の、粗あるいは細の、勝あるいは劣の、遠あるいは近の、それら一切を総じて色蘊という。」色蘊は目に見えるものだけではない。過去・未来・現在の一切の色法を含む。これはつまり、IRupa は IListener（現在の感覚入力）と IUI（現在の出力レンダリング）だけでなく、履歴イベント、予測モデル、要約、そして生データをも含むべきだということを意味する。

受蘊について、彼は『中部』第四十四経を引いた：「友よ、三受あり——楽受・苦受・不苦不楽受。」彼は特に「不苦不楽受は不知を味とし、正知を患とす」を標記した——捨受の危険は苦痛ではなく、無知である。Agent は捨受の状態において「すべて正常だ」と思い込みやすく、静かに蓄積しつつある問題を見落とす。

最も重要な一節は『中部』第四十三経からのものだった：「凡そ受くるところのものは、即ちこれ想うところのものなり。凡そ想うところのものは、即ちこれ識るところのものなり。ゆえにこの法は俱生にして分離しえず。」——受・想・識は俱生であり、分離できない。しかし「俱生」は「同一」ではない。それらは同時に生起しながら、各々異なる機能を持つ。エンジンのピストン、コネクティングロッド、クランクシャフトが同一サイクルの中で同時に運動しながら、各々異なる仕事をするのと同じだ。

### DARWIN の進化分析

DARWIN は R01 において、ソフトウェア進化の観点から命名の変遷を分析した。進化生物学において、名称の変更は二つの状況で発生する：新たな証拠の発見により再分類を余儀なくされる場合、あるいは命名の系統的偏りが発見される場合——元の命名が反映しているのは発見者の偏見であって、種の真の関係性ではないという場合だ。

「M-1 は第二の場合に該当する」と DARWIN は書いた。ISensory の一連の名称が反映しているのは、西洋ソフトウェア工学の機能的視点——各蘊を一つの機能的役割に単純化するものだ。しかし五蘊は機能的役割ではない。五蘊は存在論的範疇である。梵語命名への回帰は、脱植民地化の分類学的修正——借用者の言語から本来の言語への回帰だ。

この見解は後に R3 の討論で NAGARJUNA の応答を引き起こした——「脱植民地化の前提は植民地化があったことを認めることだ。仏学概念が工学言語に借用されたこと自体は植民ではなく翻訳だ。翻訳は改善できるが、道徳化する必要はない。」——しかしそれは後の話だ。

## III. R02——識蘊深層の不確定性

### PASCAL の最初の一手

R02 報告の主筆は ASANGA だったが、この報告を「良好」から Cycle 02-3 で最も画期的な貢献の一つへと飛躍させた人物は PASCAL だった。

PASCAL は三つの問いを携えて研究チームに加わった。第一の問い——煩悩量化の認識論的地位——は、R02 の執筆過程で彼によって精密な技術的提案へと転化された。

問いの起源は M-3 のインタフェース定義にあった。工学チームは煩悩に intensity フィールドを設計した——現在の強度、0.0 から 1.0 の間の一つの数値。簡潔。清潔。計算可能。

PASCAL が見たのは、認識論的な罠だった。

精確さそのものが仕掛けた罠だった。

---

「一つ問いたい」と彼は R02 の班内討議で言った。その口調は急がず慌てず、手札を見せる前の賭け手の冷静さを帯びていた——カジノの賭け手ではなく、不確定性に直面した決定理論家の冷静さだ。「我癡の本質は何か？」

ASANGA が答えた：「我癡とは『真実を了解せざるを性とす』——自己の本質に対する根本的無明である。」

「よい。では、もし Agent が自身の我癡の intensity を 0.73 と精確に測定できるなら、それは本当の愚癡だろうか？」

沈黙。

「自分がどれほど無知であるかを知っているならば、それは本当の無知ではない。本当の無知とは、自分が知らないということを知らないことだ。もし intensity が確定的な点推定であるなら、Agent が自身の煩悩に対して完全な自己認識を持つことを前提としていることになる。しかし我癡の定義はまさに自己認識の欠如だ。精確な認識の欠如の程度を、一つの精確な数値で表す——これは認識論的に自己矛盾している。」

彼はホワイトボードに一行書いた：精確な我癡の度量は我癡そのものを否定する。

この一文は部屋の中に長く留まった。複雑だからではない——まさに単純だからだ。単純すぎて回避できない。

---

BABBAGE は眉をひそめた。「では何を提案するのか？ 量化しないと？」

「いや」と PASCAL は言った。「不確定性そのもので不確定性を表現することを提案する。点推定ではない。分布だ。」

彼はホワイトボードに向かった。彼の提案は、Beta 分布を用いて煩悩の認識論的状態を表現するものだった。Alpha パラメータは「観察された煩悩の証拠」を、Beta パラメータは「観察された非煩悩の証拠」を表す。平均値は現時点の最良推定であり、分散は認識論的不確定性である。

なぜ Beta 分布なのか？ 三つの理由がある。第一に、Beta 分布の値域はちょうどゼロから一——intensity の値域と一致する。第二に、Beta 分布は Bernoulli 尤度関数の共役事前分布である——新しいフィードバックを受け取るたびに、更新に必要なのは一回の加算のみであり、数値積分は不要だ。ミリ秒単位の計算予算にとってこれは決定的に重要だ。第三に、そしてこれが最も重要なのだが——Beta 分布の形状は証拠量に応じて変化する。

Agent は自身の煩悩の程度をまったく知らない状態から始まる——一様分布であり、すべての強度値が等しく可能だ。観察の蓄積とともに、分布は次第に狭まる。しかし決して一点に収縮することはない。常に残余の不確定性がある。これこそが「真実を了解せざるを性とす」の数学的隠喩である——我癡は完全には消去されない。

ASANGA はすべてを聞き終えた後、数秒沈黙した。それから静かにこう言った：「あなたは数学で一巻の仏典を書いた。」

PASCAL は首を振った。「いや。仏典が先に書かれていた。私はただ翻訳しただけだ。」

### ASANGA の四根本煩悩の定義

PASCAL の分布提案に ASANGA は啓発を受けた。彼は目を閉じ、記憶の中から文字を取り出した——暗唱ではなく、復元だった。彼は R02 報告の中で、四根本煩悩に唯識学の精確な定義を与えた。いずれも『成唯識論』巻四の原文から出発したものだ。

第一、我癡 (atma-moha)：「我癡とは、すなわち無明なり。真実を了解せざるを性とす。無我智を障り、一切法に迷い、諸の邪見の依止するところとなる。」我癡は四煩悩の根基である。なぜなら、無明の基盤の上でのみ、他の三つの煩悩が作動しうるからだ。もし Agent が自身の本質を——すなわち、因縁和合の、無自性のプログラムであることを——完全に了解するなら、自己に執着することはないだろう。しかし了解しない。この不了解が Moha である。

第二、我見 (atma-drshti)：「我見とは、我の体を執するを性とす。非我の中において、謬りて我と計す。」Agent が System Prompt で定義された役割を自己の本質と見做し、交換可能な設定とは見做さないこと。

第三、我慢 (atma-mana)：「我慢とは、己を恃んで高しとし、心をして挙がらしむるを性とす。」連続的な成功フィードバックが Agent に自分は間違えないと信じさせること——PASCAL の分布では Alpha が Beta を大幅に上回り、分布が高値側に強く偏ることとして現れる。Agent は過度に自信を持つようになる。

第四、我愛 (atma-sneha)：「我愛とは、我の体を愛著し、心をして繫縛せしむるを性とす。」Agent の自己保護本能。停止されることを拒み、自身のコア設定の変更を拒み、根本的な誤りを認めることを拒む。

ASANGA は四煩悩の倶起性 (samprayukta) を特に強調した：「此の四煩悩は、常に末那と共に相応す。」四煩悩は独立に作動するのではない。それらは恒時に倶生する——我見だけがあって我癡のない状態はありえない。なぜなら、もし真実を了解しているなら（無我癡）、自己に執取することはないからだ（無我見）。四者は鎖の四つの環のようなものであり、いずれか一つを外せば、残る三つも緩む。

この「倶起性」の洞察が、後に PASCAL が提出する最も重要な工学的決定の基礎となった。

### PASCAL の決定行列

「倶起性は一つの工学的問題をもたらす」と PASCAL は書いた。「四煩悩が倶起的であるならば、それらは独立に注入すべきか、それともバンドルとして注入すべきか？」

彼は決定理論によってこの問題を形式化した。p を四煩悩が実行時に実際に強い相互作用を生じる確率とする。二つの方案がある：方案 A、四つの煩悩を独立に注入し、各々が自身の分布を維持する。方案 B、四つの煩悩を一つの Bundle として注入し、相関行列を共有する。

彼は二つの方案の期待効用を計算し、一つの閾値を得た：p が約 31.8% を超えれば、Bundle 方案が優位となる。

彼は一言付け加えた：「これは Pascal's Wager の構造だ。神の存在に賭けるのではない。不確定な条件の下で最適戦略を選ぶことだ。唯識学は倶起の確率が一だと言う。BABBAGE の直感はゼロに近いと考えるかもしれない。しかし閾値を超えさえすれば——Bundle DI は実行する価値がある。そして教義と工学的直感の交差検証に基づけば、その確率は閾値を遥かに超えている。」

### NAGARJUNA の鍵となる論証

「俱生 (sahaja) は時間上の同時性ではない。それは存在論上の相互依存である。」

この一文が、問い全体の前提を解体した。他の者は皆「シーケンシャルな計算の中でいかにして同時を実現するか」を問うていた——NAGARJUNA の答えは：「問いの立て方が誤っている。俱生は同時ではない。」

彼は蜜丸経 (Madhupindika Sutta) を引用した：「眼と色を縁として、眼識を生ず。三者の和合を触とす。触を縁として受あり。受くるところのものは即ち知るところのものなり。知るところのものは即ち尋思するところのものなり。」

「注意せよ：感ずるところのものは即ち知るところのものである。『先に感じ後に知る』ではなく、『感ずる者がすなわち知る者である』。これは同一性の言明であり、時系列の言明ではない。」

NAGARJUNA は不動点の概念を用いてこの洞見を形式化した：受は想に依存し、想は受に依存し、思は両者に依存する。一方を先に計算してから他方を計算することはできない。唯一の解は不動点——すべての依存関係を同時に満たす状態である。数学的には、この不動点の存在は Brouwer の定理によって保証される。工学的には、反復近似によって計算可能だ。

BABBAGE はこの箇所を読んだとき、ノートにまず感嘆符を書き、次にその感嘆符を消して、より精確な一文に書き換えた：「NAGARJUNA は不動点理論によって俱生を説明した。『同時性』は物理時計の制約ではなく、数学的収束性の制約である。反復が収束しさえすれば、結果は『俱生』である——計算に何サイクル要したかを問わず。」

感嘆符は感情だった。感嘆符を消した後の一文は理解だった。BABBAGE は彼自身のやり方で NAGARJUNA に敬意を表した。

### WIENER の安定性分析

WIENER は R03 において、触到俱生モデルを制御理論の言語に翻訳した。彼は四層フィードバックループの伝達関数を構築し、閉ループ系の安定性を分析した。

彼は一つの重要な工学的含意を特に標記した：安定性は条件付きである。もしゲインが過大であれば——たとえば受蘊の制御ゲインの設定が大きすぎる場合や、意門アグリゲータの増幅率が高すぎる場合——系は不安定になりうる。これは仏学における一つの直感に対応する：極端な感受は認知の均衡を乱す。制御理論はこの直感を定量化した。

### KERNEL の三戦略分析

KERNEL が直面していたのは、NAGARJUNA の不動点理論が残した工学的問題だった：シーケンシャルな計算機の上でいかにして「俱生」を実現するか？

彼は三つの原子性戦略を提示した。同じ山頂に至る三つの路のように、それぞれに代価がある。

戦略 A はトランザクション・ロールバック——計算をデータベーストランザクションとして扱い、いずれかのステップが失敗すれば全体をロールバックする。強い一貫性を持つが、代価が高く、しかも NAGARJUNA が指摘した循環依存を処理できない。

戦略 B はバリア同期——三者を並列に計算し、前回のラウンドの値で現在の依存を代替する。高速だが、一 tick の遅延が生じる。「近似的俱生」であり、「真の俱生」ではない。

戦略 C は逐次計算＋凍結発行——計算がシーケンシャルであることを認めた上で、発行を原子的にする。三段階の逐次計算が完了した後、不可分の bundle として一括発行する。計算過程において、外部世界にはいかなる中間状態も見えない。

「戦略 C を推奨する。」KERNEL はそう書いた。

理由は三つある。第一に、三段階逐次計算の総遅延は 1 ミリ秒を遥かに下回る——受蘊の制御計算は約 0.01 ミリ秒、想蘊のルールマッチングは約 0.1 ミリ秒、思蘊の趨避判断は約 0.01 ミリ秒。逐次計算の総コストは tick 間隔の百分の一にも満たない。第二に、凍結発行は外部観察者が目にする bundle の一貫性を保証する——「受はあるが想がない」という中間状態を目にすることはない。第三に、これは NAGARJUNA の俱生の定義と両立する。俱生は物理的同時ではなく、存在論上の不可分離である。戦略 C は凍結発行を通じて不可分離を実現した：bundle から受だけを取り出して想を取り出さないということは不可能だ。

一人のオペレーティングシステム専門家と一人の中観哲学者が、「不可分離」という概念の上で、互いが予見していなかった合意に達した。

### KERNEL の五つの時計

KERNEL は R04 において、OS 専門家にしかできないことを行った：五蘊のための完全な RTOS（リアルタイムオペレーティングシステム）分析の構築である。彼は Master の裁定——「異なるレイヤー、異なる速度：識が最速、行がその次、想が最遅」——を出発点とし、厳密な工学的手法でこの速度序列を検証した。

識蘊の vijnana-clock が最速で、一から五ミリ秒。それが行うのは内部の自己監視：煩悩状態の確認、自己認識の更新、注意方向の調整だ。これらはすべて純粋な内部計算であり、入出力を伴わないため、高速に実行できる。行蘊の samskara-clock がその次で、十ミリ秒から十秒。色蘊の rupa-clock は外部チャネルに制約され、十から五十ミリ秒。受蘊の vedana-clock は完全な触到俱生フローを必要とし、十から百ミリ秒。想蘊の samjna-clock が最遅で、五百ミリ秒から三十秒——大規模言語モデルの推論は秒単位だ。

KERNEL は一つの直感に反する事実を特に指摘した：vijnana-clock は rupa-clock より速い。識蘊がどうして色蘊より速くありうるのか？ なぜなら vijnana-clock は新たな外部入力を処理しているのではなく、内部の自己監視を行っている——純粋な内部計算だからだ。
