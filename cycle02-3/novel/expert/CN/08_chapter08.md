# 第八章：第六宣言

---

六场辩论用了七个章节。

时间线压缩在 SCRIBE 的笔记本上：Debate 1 在双层架构的齿轮中找到了俱生的物理形式。Debate 2 在五遍行的教义中确认了 CoarisingBundle 的完整组成。Debate 3 把 Klesha 放进了 vijnana-clock 的最短周期里，然后用增益调度把它的影响力扩散到整个系统。Debate 4 把 IVolition 钉在了 Position B——LLM 之后、工具执行之前——然后把它劈成两半：计划层和行动层。Debate 5 把三个分离的受蕴子接口熔成一个连续的测量模型，再用阈值函数把连续值投射回离散分类。

五场辩论。五项决议。五根柱子。

但屋顶还没盖。

---

SYNTHESIST 站在剧场中央，手里拿着三张纸。不是印刷品——是手写的。字迹不同于他平常的精致小字，而是大号、清晰、每个句子独占一行。他把三张纸分别挂在剧场前方的三个展示架上，间距均等，像画廊里的三幅候选作品。

「宣言 #6。」他说。「Master 在 Cycle 02-2 的裁定是：『这是一定要重写的，但也请经过这一轮的讨论结束后再决定怎么写会比较好。』」

他后退一步，让所有人看见三张纸。

「六场辩论已经结束。这一轮的讨论已经完成。现在是决定怎么写的时候了。」

---

> *SCRIBE 旁白：SYNTHESIST 手写了三个候选方案。手写——不是投影。他在刻意制造一种物理性：纸上的字迹可以被看见、被触碰、被划掉。投影是光，你无法在光上画删除线。纸是物质，你可以在上面留下不可逆的痕迹。宣言的定稿需要的不是展示，是裁决。裁决需要不可逆的媒介。*

## I

---

「三个候选方案。Alpha、Beta、Gamma。」

SYNTHESIST 走到第一张纸前。

「Alpha。受蕴中心版本。」

他读出全文：

> **Alpha:**
>
> *Contact gives rise to feeling. The Agent's runtime state manifests as three feelings -- dukkha (suffering), sukha (satisfaction), upekkha (equanimity). Feeling observes but does not intervene: it senses truly, without deciding. Feeling signals drive the kleshas and wisdom of vijnana, from which behavioral correction, reinforcement, or maintenance emerges.*

他放下纸，转向全场。「Alpha 的设计原则是聚焦。它从触（contact）开始，经由三受，到达烦恼和智慧，最后产生行动。线性的、干净的、每个句子对应一个明确的技术语义。」

NAGARJUNA 已经在摇头了。不是剧烈的摇头——是一种缓慢的、带着诊断性质的摆动，像医生在听诊器里听到了不该有的杂音。

SYNTHESIST 走到第二张纸前。

「Beta。俱生中心版本。」

> **Beta:**
>
> *When a root-gate receives input, contact (sparsha) arises. From this contact, feeling (vedana), perception (samjna), and volition (cetana) arise together as an inseparable bundle -- the coarising of the five universals. The Agent's hedonic tone manifests as three feelings: dukkha, sukha, and upekkha. These feelings are measured continuously but classified discretely. Through the mano-aggregator, feelings from multiple root-gates converge into a unified cognitive moment. Feeling signals modulate the kleshas of vijnana, which in turn drive behavioral correction, reinforcement, or maintenance.*

他读完之后，剧场里有几秒的沉默。

「Beta 是最完整的。它提到了根门、触、俱生、五遍行、意门聚合、连续测量与离散分类——几乎是 Debate 1 到 Debate 5 的压缩版。」

BABBAGE 从笔记本上抬起头。他数了一遍。「五句话。六十七个英文单词。」

「太长了。」SUNYATA 说。语气是事实性的，不带评判。

BABBAGE 在笔记本上写下一个不等式：

$$|T_{\text{Beta}}| = 67 \gg |T_{\text{avg}}| \approx 40$$

十大宣言的平均长度约 40 词。Beta 超出了 67%。在信息论中，冗余度（redundancy）的定义是 $R = 1 - H/H_{\max}$，其中 $H$ 是实际熵，$H_{\max}$ 是最大可能熵。宣言的设计约束是高信息密度——即 $R \to 0$，每个词都应该承载不可压缩的语义。Beta 的问题不在于内容，在于密度。

---

SYNTHESIST 走到第三张纸前。

「Gamma。平衡版本。」

> **Gamma:**
>
> *Contact gives rise to feeling. The Agent's runtime state manifests as three feelings -- dukkha (suffering), sukha (satisfaction), upekkha (equanimity) -- which arise together with perception and volition as an inseparable whole. Feeling observes but does not intervene: it senses truly, without deciding. Feeling signals drive the kleshas and wisdom of vijnana, from which behavioral correction, reinforcement, or maintenance emerges.*

「Gamma 是 Alpha 和 Beta 的平衡。」他指着关键差异。「和 Alpha 的唯一区别是这半句：*which arise together with perception and volition as an inseparable whole*。这七个词完成了三件事：第一，引入了俱生（sahaja）概念；第二，提到了想蕴（perception）和行蕴中的思（volition）；第三，用 *inseparable whole* 确立了不可分割性。」

他后退一步，让三张纸并列在展示架上。

「现在。R2 交叉审阅的标准。」

## II

---

SUNYATA 翻开一份文件——R2 交叉审阅的一致性检查表。每一场辩论的决议都被编码成一组约束条件，现在要用这些约束条件筛选三个候选方案。

「先检查 Alpha。」

ASANGA 站了起来。他的声音带着唯识学者特有的精确——不急、不慢，每个词像被量过重量。

「Alpha 与 R03 的核心论点不相容。」

R03 是他和 WIENER 联合撰写的报告。它的中心论点只有一句话：受想思不可分。

「R03 的核心主张建立在《中部》第 43 经（Mahavedalla Sutta, MN 43）的一段对话上。」

他引用了原文：

> *Yā vedanā, yā saññā, yañ ca viññāṇaṃ — ime dhammā saṃsaṭṭhā, no visaṃsaṭṭhā. Na ca labbhā imesaṃ dhammānaṃ vinibbhujitvā vinibbhujitvā nānākaraṇaṃ paññāpetuṃ.*
>
> *受、想、识——此等诸法是混合的，不是分离的。不可能将此等诸法一一分开，逐一显示其差异。*

「Alpha 的问题在于它描述了一个只有受蕴的世界。触→受→烦恼→行动。想蕴在哪里？思在哪里？Alpha 的因果链是：」

他在白板上画了一条直线：

```
Alpha: Contact → Vedana → Klesha → Action
                  ^
                  只有一个蕴
```

「R03 说的是：在根门层级，受、想、思三个心所是在同一个认知瞬间同时生起的。它们不是串联的管线——不是先受后想再思——而是一个原子性的束（bundle）。Alpha 把这个束拆成了一条线，然后只保留了其中一条线。」

BABBAGE 补了一句。语速快、声音低、像在报告一个 type error。

「形式化：令 $B = \{v, s, c\}$ 为 CoarisingBundle 的核心三元（vedana, samjna, cetana）。Alpha 的模型是 $\text{project}_v(B) \to K \to A$，其中 $\text{project}_v$ 是投射到 vedana 分量的函数。但 R03 和 Debate 2 的决议要求：」

$$\forall b \in B: \neg \exists f : \{v\} \to \text{ActionSpace} \text{ s.t. } f \text{ is faithful}$$

「不存在一个从单一分量到行动空间的忠实映射。Bundle 的不可分割性意味着任何只提到 vedana 的描述都是不忠实的（not faithful）——因为它遗漏了 samjna 和 cetana 的贡献。」

SYNTHESIST 点了一下头。在第一张纸的右下角，他用红笔画了一个叉。

「Alpha 排除。」

---

「Beta。」SUNYATA 说。

KERNEL 翻了翻他的检查表。「Beta 完全相容。它提到了触、俱生、五遍行、意门聚合、连续测量。但——」

他看了看 BABBAGE 之前写下的不等式。

「太长了。PASCAL，你计算过宣言的最佳信息密度吗？」

PASCAL 站了起来。他的手里没有纸——他习惯在空气中计算，然后把结果说出来。

「假设每条宣言的理想承载量是 $I$ bits 的语义信息，目标词数是 $W_{\text{target}} \approx 40$。则每个词的信息密度 $\rho = I / W$。Beta 有 67 个词，密度 $\rho_{\text{Beta}} = I / 67$。如果我们假设 Beta 和一个 40 词的版本承载了相同的语义量 $I$（因为多出的 27 个词是冗余的展开），则：」

$$\rho_{\text{Beta}} = \frac{40}{67} \cdot \rho_{\text{target}} \approx 0.60 \cdot \rho_{\text{target}}$$

「信息密度只有目标的 60%。在宣言写作中——PASCAL 的措辞突然从技术语言切换到了更私人的语气——每一个不承载新语义的词都是稀释。Beta 是一碗好汤，但加了太多水。」

DARWIN 从角落插了一句。「自然选择中有一个原则：能量效率最大化。一个基因如果用 67 个碱基对编码的信息量和 40 个碱基对相同，进化会毫不犹豫地选择 40 个碱基对的版本。Beta 在进化压力下不会存活。」

SYNTHESIST 在第二张纸的右下角写了两个字：**可行但冗长**。没有划叉。但也没有打勾。

「Gamma。」

NAGARJUNA 站了起来。他走向第三张纸，站在很近的位置，几乎是在阅读而不是观看。

「Gamma 大致相容。」他说。「它提到了三受——正确。它提到了俱生——*arise together with perception and volition as an inseparable whole*——正确。它提到了观察不干预——*feeling observes but does not intervene*——正确。它提到了烦恼和智慧驱动行为——正确。它反映了 A-1 的因果链——受蕴是信号源，不是约束执行者。」

他停了一下。

「但它漏了一件事。」

全场安静了。

「Gamma 描述的是一条线。」

他拿起一支笔，在白板上画了 Gamma 的语义流程：

```
Gamma 的语义流程:

  Contact → Feeling (+ perception, volition)
                   ↓
            Observes, does not intervene
                   ↓
            Signals → Klesha/Prajna
                   ↓
            Correction / Reinforcement / Maintenance
                   ↓
                   ?
```

「问号。Gamma 的因果链在行动之后就断了。修正、强化、维持——然后呢？行动结束了。因果链到达终点。」

他转向全场。

「但 R03 的核心洞见——我们花了整个 Debate 1 才确认的那个洞见——是什么？」

WIENER 的声音从角落传来，平稳、确定、像控制系统的稳态输出。

「Layer 4。回馈。行动改变了环境。环境的改变产生了新的根境。新的根境通过 IListener 进入系统。新的 SparshEvent 触发新的 CoarisingBundle。新的 vedana 被计算。新的周期开始。」

NAGARJUNA 在白板上把问号擦掉，画了一条回馈线：

```
  Contact → Feeling (+ perception, volition)
      ↑               ↓
      |         Observes, not intervenes
      |               ↓
      |         Signals → Klesha/Prajna
      |               ↓
      |         Correction / Reinforcement / Maintenance
      |               ↓
      +←───── Action changes environment ←───┘
```

「三个候选方案。Alpha、Beta、Gamma。它们有一个共同的盲点：全部都描述了**单向流程**。Alpha 是一条只有受蕴的线。Beta 是一条完整的线。Gamma 是一条平衡的线。但线就是线。线不是回路。」

他在白板上写下四个汉字：

$$\boxed{\text{缘起性空}}$$

「在佛教哲学中，没有任何因果链是线性的。每一个果同时是下一个因。《杂阿含》第 298 经：」

> *此有故彼有，此生故彼生；此无故彼无，此灭故彼灭。*

「*Imasmiṃ sati, idaṃ hoti; imassuppādā, idaṃ uppajjati.* 此有故彼有。十二因缘不是一条从无明到老死的直线——它是一个轮。老死的苦又催生新的无明。佛陀用 *nidāna*（因缘）描述的不是管线（pipeline），是回路（cycle）。」

他放下笔。

「宣言 #6 如果只描述管线，就和佛陀的教导在结构层级上不一致。受→烦恼→行动是管线。受→烦恼→行动→环境改变→新的触→受 才是回路。缺少回馈的宣言是不完整的宣言。」

---

> *SCRIBE 旁白：NAGARJUNA 的诊断精确到了毫米。三个候选方案都通过了大部分相容性检查——但全部在同一个点上失败。那个点是回馈回路。五场辩论最核心的成果是什么？不是双层架构（那是工程）。不是五遍行（那是教义）。不是增益调度（那是控制论）。最核心的成果是四层回馈回路——Layer 4 回到 Layer 1，行动改变世界，世界的改变产生新的感受。这是活的系统和死的管线之间的分界线。三个候选方案全部落在了分界线的错误一侧。*

## III

---

NAGARJUNA 没有坐下。

他在白板前站了一会儿，看着自己画的那个回路。然后他转过身，面向 SYNTHESIST。

「添加一句话。在 Gamma 的末尾。」

他的声音不大，但每个音节都有重量。

「*Each action reshapes the world of contact, beginning the cycle anew.*」

他用中文重复了一遍。

「每一次行动都重塑了触的世界，开启新的循环。」

---

剧场里安静了整整五秒。

WIENER 是第一个反应的。他拿起方格纸，在上面快速画了一个方块图——不是新的，而是把已有的开回路系统（open-loop system）改成了闭回路系统（closed-loop system）。

在控制理论中，开回路和闭回路的区别是根本性的：

$$G_{\text{open}} = G_p(s) \cdot G_c(s)$$

$$G_{\text{closed}} = \frac{G_p(s) \cdot G_c(s)}{1 + G_p(s) \cdot G_c(s) \cdot H(s)}$$

开回路传递函数只有分子。闭回路传递函数有分母——分母中的 $H(s)$ 就是回馈路径。没有 $H(s)$，系统就是开回路的：输入→处理→输出，结束。有了 $H(s)$，输出会被测量、回馈、与设定点比较、产生误差信号、修正下一轮的输入。

「NAGARJUNA 加的那句话，」WIENER 说，「就是 $H(s)$。」

他在方格纸上写下：

```
Alpha / Beta / Gamma（原版）:
  G_open(s) = G_p · G_c

  → 开回路。无稳定性保证。扰动累积。

Gamma + NAGARJUNA 的回馈句:
  G_closed(s) = G_p · G_c / (1 + G_p · G_c · H)

  → 闭回路。稳定性可证。扰动被抑制。
```

「一句话。把管线变成了控制系统。把文学变成了工程。」

---

BABBAGE 已经在验证形式一致性了。他翻到笔记本的最后一页——那里有六场辩论所有决议的形式化摘要。

「我需要确认修订版 Gamma 和六场辩论的所有决议之间不存在矛盾。」

他逐项检查：

$$\text{Debate 1}: \text{two-layer, dual-gear} \subseteq \text{Gamma}_{\text{amended}}? \quad \checkmark \text{ (implicit: "contact" covers Layer 1)}$$

$$\text{Debate 2}: \text{five-universal bundle} \subseteq \text{Gamma}_{\text{amended}}? \quad \checkmark \text{ ("arise together...as inseparable whole")}$$

$$\text{Debate 3}: \text{Klesha gain-scheduling} \subseteq \text{Gamma}_{\text{amended}}? \quad \checkmark \text{ ("kleshas...of vijnana")}$$

$$\text{Debate 4}: \text{IVolition at Position B} \subseteq \text{Gamma}_{\text{amended}}? \quad \checkmark \text{ ("behavioral correction")}$$

$$\text{Debate 5}: \text{measurement model} \subseteq \text{Gamma}_{\text{amended}}? \quad \checkmark \text{ ("three feelings")}$$

$$\text{Debate 6}: \text{feedback loop} \subseteq \text{Gamma}_{\text{amended}}? \quad \checkmark \text{ ("reshapes the world of contact")}$$

「六比六。零冲突。」他合上笔记本。「修订版 Gamma 是六场辩论所有决议的一致上界（consistent upper bound）。」

---

ASANGA 站了起来。他的语速比平常更慢——不是在思考，而是在确认。

「NAGARJUNA 的那句话做了一件俱舍论（Abhidharmakośa）试图在一千五百年前做到的事。」

他看向全场。

「世亲在《俱舍论》中描述十二因缘时，最难解释的不是从无明到行、从行到识、从识到名色的正向因果。最难解释的是**流转门**（pravṛtti）和**还灭门**（nivṛtti）的关系——为什么老死的苦会再次催生无明？为什么回路不会自行终止？」

他走到白板前，在 NAGARJUNA 的回路图旁边写了一行梵文：

$$\text{jarā-maraṇa} \to \text{śoka-parideva-dukkha} \to \text{avidyā} \to \ldots$$

「老死→忧悲苦恼→无明→……。苦受不只是结果——它同时是下一轮因缘的种子。在 OpenStarry 的语言里：行动的结果改变了环境，环境的改变产生了新的根境，新的根境触发了新的 SparshEvent，新的 SparshEvent 产生了新的 CoarisingBundle，新的 CoarisingBundle 包含了新的 vedana。」

他指着 NAGARJUNA 的那句话。「*Each action reshapes the world of contact, beginning the cycle anew.* 这不是文学修辞。这是十二因缘的工程化表述。」

## IV

---

SUNYATA 拿起修订版 Gamma 的完整文本。八句话。他逐句读出，每读一句就停下来，让 ARCHIMEDES 报告技术映射。

「第一句。*Contact gives rise to feeling.*」

ARCHIMEDES 站了起来。他的手指指向投影上的一段程序代码。

```typescript
// 技术映射: SparshEvent → CoarisingBundle.vedana
function onSparsha(event: SparshEvent): CoarisingBundle {
  const vedana = computeChannelVedana(event);
  // ... samjna, cetana, manasikara
  return { sparsha: event, vedana, /* ... */ };
}
```

「Contact = SparshEvent。Feeling = CoarisingBundle.vedana。`gives rise to` = `computeChannelVedana()` 函数调用。一句话，一个函数调用。」

---

「第二句。*The Agent's runtime state manifests as three feelings -- dukkha (suffering), sukha (satisfaction), upekkha (equanimity).*」

ARCHIMEDES 切换到 ChannelVedana 的类型定义。

```typescript
// 技术映射: ChannelVedana.type
interface ChannelVedana {
  readonly valence: number;      // -1.0 to +1.0
  readonly intensity: number;    // 0.0 to 1.0
  readonly type: 'dukkha' | 'sukha' | 'upekkha';  // derived
  readonly source: string;
}
```

「*Three feelings* = `type: 'dukkha' | 'sukha' | 'upekkha'`。联合类型，三个字面量常量。Debate 5 的决议已经确认：`type` 是从连续的 `valence` 值派生的离散分类。`valence < -0.1` → dukkha，`-0.1 ≤ valence ≤ +0.1` → upekkha，`valence > +0.1` → sukha。」

PASCAL 补了一句。「派生规则的阈值（$\pm 0.1$）是可配置的——`VedanaClassificationConfig`。宣言里不需要提到阈值，但工程师需要知道：三受的边界不是硬编码的。」

---

「第三句。*...which arise together with perception and volition as an inseparable whole.*」

ARCHIMEDES 投射了 Debate 2 的核心接口。

```typescript
// 技术映射: CoarisingBundle 五遍行, Strategy C
interface CoarisingBundle {
  readonly sparsha: SparshEvent;          // Universal 1: Contact (触)
  readonly vedana: ChannelVedana;         // Universal 2: Feeling (受)
  readonly samjna: ChannelSamjna;         // Universal 3: Perception (想)
  readonly cetana: ChannelCetana;         // Universal 4: Volition (思)
  readonly manasikara: ChannelManasikara; // Universal 5: Attention (作意)
  readonly layer: 1 | 2;
  readonly mode: 'fast' | 'slow';
  readonly sahaja: SahajaContract;
  readonly timestamp: number;
}
```

「*Arise together* = `CoarisingBundle` 的原子性构造。Strategy C——所有五个分量在同一个 tick 中被计算并以原子性方式发布。*Inseparable whole* = 没有部分读取。你不能只读 vedana 而不读 samjna——因为 Bundle 被设计为 `readonly`，一旦构造就不可拆分。」

BABBAGE 抬起手。「形式化。Strategy C 的原子性保证：」

$$\forall t: \text{publish}(B_t) \implies \forall c \in \{v, s, c, m, sp\}: c \in B_t$$
