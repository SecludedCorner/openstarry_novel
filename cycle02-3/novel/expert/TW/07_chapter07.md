# 第七章：量尺與命名

---

LINNAEUS 把馬克筆放在桌上，藍色筆蓋朝左。

在過去六章裡，他的筆使用頻率極高。第三章拆解 IIdentity 的分類學謬誤時，藍色筆幾乎沒有離開他的手指。第六章繪製五棵樹的擴展圖時，藍色墨水在白板上延伸出四十七條枝幹線。但現在，在第七章的開頭，他把筆放下了。

不是因為用完了墨水。是因為這一章的議題，對分類學家而言，是一種根本性的質問：你畫的那些界門綱目科屬種的邊界線——它們是世界的真實結構，還是測量儀器的刻度？

---

> *SCRIBE 旁白：LINNAEUS 放下筆的動作很輕。但那個動作裡有重量。在前六章中，分類學家是拿著工具工作的人——他的工具就是那支藍色馬克筆，每一條線都是一個分類邊界，每一個分叉都是一次命名決策。現在，有人要質問那些邊界本身的性質。這不是技術挑戰——這是本體論挑戰。你畫的線是發現的，還是發明的？你命名的東西在你命名之前就存在，還是因為你的命名才開始存在？*

> *分類學有一個古老的辯論：自然類 (natural kind) vs 約定類 (conventional kind)。金和銀是自然類——它們的邊界由原子結構決定，不因人的命名而改變。「蔬菜」和「水果」是約定類——番茄在植物學上是果實，在烹飪學上是蔬菜。受蘊的三分——苦、樂、捨——是哪一種？這就是第五場辯論的核心。*

---

SUNYATA 的聲音從劇場中央傳來。石子。深潭。和前四場辯論一樣的起手式——但這次，他手裡沒有拿 Master 的信。

「Debate 5。Vedana 介面設計。分類 vs 測量。」

他停了一拍。

「前四場辯論處理的是結構和流程。CoarisingBundle 的組成、煩惱的時鐘域、IVolition 的位置、多時鐘的可行性。那些問題都有一個共同特徵——它們的答案是**架構的**：組件放在哪裡，數據怎麼流動，時鐘怎麼同步。」

「這場辯論不同。這場辯論的問題是**本體論的**：受蘊是什麼？不是受蘊該放在哪裡，不是受蘊該怎麼連接——而是受蘊本身的存在方式。」

他環顧全場。

「TURING，先給我們基線。」

---

TURING 的螢幕亮了。冷光。他的投影風格和前幾章一模一樣——原始碼原文呈現，不做節錄，不加美化。

「v0.24.0-beta 的受蘊實作狀態：零。」

他讓這兩個字停了三秒。

「零插件。零子介面。零執行時程式碼。vedana 在整個原始碼庫中唯一的存在方式是 SafetyMonitor 的 JSDoc 註解裡寫了一行 `@skandha vedana`。一個標籤。一個佔位符。一個許諾——『這裡未來會有東西』——但那個未來還沒到。」

他切換投影。

```typescript
// v0.24.0-beta SafetyMonitor — vedana 的唯一痕跡
/**
 * @skandha vedana
 * Safety monitoring with vedana-like pain detection
 */
export class SafetyMonitor {
  // ... 沒有任何 vedana 介面實作
}
```

「所以我們今天討論的是綠地設計 (greenfield design)。不受現有程式碼約束。我們選什麼，它就是什麼。」

他關掉投影，退回座位。

---

SUNYATA 點了點頭。「這是自由，也是責任。LINNAEUS，你的提案。」

## I

---

LINNAEUS 沒有拿起馬克筆。他站在白板前，但手是空的。

這不尋常。在他所有的發言中——從第三章到第六章——他總是一邊說一邊畫。分類學家用視覺思考。但這次他先用語言。

「在分類學裡，有一個問題比所有其他問題都更根本。」

他的聲音帶著一種在前幾章壓抑過的東西——不是興奮，是那種你終於站在自己最核心的信念面前、不得不把它說清楚的莊重。

「那個問題是：你要分類的對象，是**同一種東西的不同程度**，還是**不同種類的東西**？」

他轉向全場。

「這個區別決定了一切。如果鴿子和老鷹是同一種東西的不同程度——比方說飛行能力的高低——那你用一條連續的量尺就夠了：飛行能力 0.3 vs 0.95。但如果鴿子和老鷹是不同的物種——有不同的骨骼結構、不同的捕食策略、不同的生態位——那你需要不同的分類節點，因為它們的差異不只是程度的差異，而是**質的差異**。」

他停了一下。

「苦和樂是程度的差異，還是質的差異？」

他終於拿起了馬克筆。藍色。在白板上畫了三個獨立的方塊：

```
┌──────────┐   ┌──────────┐   ┌──────────┐
│  IDukkha │   │  ISukha  │   │ IUpekkha │
│  苦受     │   │  樂受     │   │  捨受     │
│          │   │          │   │          │
│ aversive │   │appetitive│   │homeostatic│
│ 迴避行為  │   │ 接近行為  │   │ 維持行為   │
└──────────┘   └──────────┘   └──────────┘
```

「三個獨立介面。三種不同的行為程式。不是一個連續量的三個區間——而是三個質上不同的現象。」

他在每個方塊下面寫了生物學的對應：

「在神經科學中，hot 和 cold 不是同一條感覺神經纖維上的不同數值。TRPV1 受體感測熱痛——它在 43°C 以上激活。TRPM8 受體感測冷覺——它在 26°C 以下激活。不同的受體蛋白。不同的離子通道。不同的信號通路。你不會說熱和冷是『同一種感覺的不同程度』——它們是**不同的 qualia**。」

他在白板上寫下 TypeScript：

```typescript
interface IDukkha extends IVedana {
  readonly type: 'dukkha';
  readonly intensity: number;    // 苦受的嚴重程度
  readonly source: string;       // 引發苦受的來源
  // 苦受特有：迴避行為觸發
  triggerAversion(): void;
}

interface ISukha extends IVedana {
  readonly type: 'sukha';
  readonly intensity: number;    // 樂受的程度
  readonly source: string;       // 引發樂受的來源
  // 樂受特有：接近行為觸發
  triggerApproach(): void;
}

interface IUpekkha extends IVedana {
  readonly type: 'upekkha';
  readonly stability: number;    // 捨受的穩定度（不是 intensity）
  // 捨受特有：維持現狀
  maintainHomeostasis(): void;
}

type Vedana = IDukkha | ISukha | IUpekkha;  // 聯合型別
```

「注意 IUpekkha 有 `stability` 而不是 `intensity`。」他在白板上劃了底線。「捨不是『強烈的中性感覺』——那是矛盾修辭。捨是一種平衡的穩定度。你可以有不穩定的捨（容易被打破）或穩定的捨（深度等持）。這個語義區別在單一介面模型中無法表達——如果三者共享同一個 `intensity` 欄位，你就把捨的穩定度和苦的嚴重程度混為一談了。」

「而且——」他指向 `Vedana = IDukkha | ISukha | IUpekkha`——「TypeScript 的聯合型別 (union type) 天然支持這種設計。你可以用 type narrowing：」

```typescript
function handle(v: Vedana): void {
  switch (v.type) {
    case 'dukkha': v.triggerAversion(); break;   // 編譯器知道 v 是 IDukkha
    case 'sukha':  v.triggerApproach(); break;    // 編譯器知道 v 是 ISukha
    case 'upekkha': v.maintainHomeostasis(); break;
  }
}
```

「型別安全。語義精確。行為分離。」

他放下筆。「這就是我的提案。三個介面。三個物種。不是一條量尺上的三個刻度。」

---

> *SCRIBE 旁白：LINNAEUS 的提案有一種分類學家特有的美學——清晰、離散、每個節點的邊界像刀切一樣分明。在林奈的世界裡，物種之間沒有灰色地帶。你是貓科或你不是。你是苦受或你不是。這種清晰有它的力量：它讓型別系統成為意義的看門人。但我隱約感覺，這種清晰即將被挑戰——不是因為它錯，而是因為受蘊可能比物種更流動。*

---

## II

---

ASANGA 從座位上站起來。那種山在霧中顯現的方式——溫和但不可忽視。

「LINNAEUS，你的生物學類比很精確。TRPV1 和 TRPM8 確實是不同的受體。」

他停了一拍。

「但你犯了一個佛學上的錯誤。你把受蘊當成了它的載體。」

LINNAEUS 微微皺眉。

ASANGA 沒有走到白板前。他的聲音本身就是他的白板。

「讓我引用一部我在 Cycle 02 第二場辯論就使用過的經典。《中部》第四十四經，《有明小經》(Cūḷavedalla Sutta)。法嗣比丘尼 (Dhammadinnā) 回答毘舍佉 (Visākha) 的提問：」

他用巴利文誦出——聲音帶著唯識學者在三十年閱藏中沉澱出的精確發音：

> *「Sukhā vedanā ṭhitisukhā vipariṇāmadukkhā;
> dukkhā vedanā ṭhitidukkhā vipariṇāmasukhā;
> adukkhamasukhā vedanā ñāṇasukhā aññāṇadukkhā.」*

他翻譯。每個詞都精確對位。

> 「樂受，住時樂，變時苦。
> 苦受，住時苦，變時樂。
> 不苦不樂受，有知時樂，無知時苦。」
> ——《中部》第 44 經 (MN 44)

ASANGA 讓引文在空氣中停了五秒。

「LINNAEUS，你聽到了嗎？」

「苦受在它**止息**的時候，變成了樂受。樂受在它**變化**的時候，變成了苦受。同一個受，不同的時間點，不同的基調。法嗣比丘尼不是在做隱喻——她是在精確描述受蘊的現象學結構。」

「如果苦和樂是你的 TRPV1 和 TRPM8——兩種不同的受體蛋白——那怎麼會有『苦受止息時變成樂受』？TRPV1 不會在停止激活的時候變成 TRPM8。受體蛋白不會變形。但受蘊會。」

他的聲音平穩但帶著一種不可退讓的質地。

「受蘊不是三種不同的東西。受蘊是一個連續的頻譜，有三個被標記的區域。苦受不是一個獨立的存在者——它是連續變化的感受之流上，被認知貼上『苦』標籤的那一段。」

---

ASANGA 繼續，展開唯識學的視角。

「而且，三分法本身只是最粗的分類。《阿毘達磨俱舍論》(Abhidharmakośa) 記載了多層分類體系：」

他列出層級：

```
受蘊的分類層級（由粗至細）
─────────────────────────────
二分：苦、樂
三分：苦、樂、捨
五分：苦、樂、憂、喜、捨（身三 + 心二）
六分：眼觸所生受 ... 意觸所生受（按根門分）
十八分：六根 × 三受
三十六分：六根 × 三受 × 二（世間/出世間）
一百零八分：三十六 × 三時（過去/現在/未來）
```

「你的三介面模型假設三分法是唯一的、正確的粒度。但受蘊的分類粒度取決於觀察的目的。」

他看向 LINNAEUS。

「如果我把你的方案推到五分法——加上 IDommanassa（憂）和 ISomanassa（喜）——你需要五個介面。推到六分法，六個。推到一百零八分法，一百零八個。每增加一個分類層級，你的介面數量就爆炸一次。」

「但如果受蘊是連續的——一個數值加上一個衍生標籤——那分類粒度的改變只需要修改標籤的映射規則。底層數據不動。」

他的結論句乾淨如手術刀：

「你的三個介面不是在描述受蘊的本質。它們是在凍結一個特定的觀察粒度。」

他最後補充了一個語義區分。

「還有一點。MN 44 同一部經中，法嗣比丘尼還區分了**身受** (kāyika-vedanā) 和**心受** (caitasika-vedanā)。Master 的信中引用的《雜阿含經》二箭經 (Sallatha Sutta, SN 36.6) 也做了同樣的區分——第一支箭是身受，第二支箭是心對身受的反應。」

「如果受蘊是三個離散介面，你需要六個：IDukkha-Kayika、IDukkha-Caitasika、ISukha-Kayika、ISukha-Caitasika、IUpekkha-Kayika、IUpekkha-Caitasika。如果再加上五分法和六分法，介面數量是組合爆炸。但如果受蘊是連續測量——效價加上 `source` 欄位——那身受和心受的區分只是 `source` 的不同值：`source = 'eye-gate'` 是感官層面的受（接近身受），`source = 'mano-gate'` 是心理層面的受（接近心受）。一個欄位解決問題，不需要介面爆炸。」

## III

---

NAGARJUNA 的手指在桌面上輕叩了兩下。不急。不慢。像木魚。

「LINNAEUS，你的問題是『受蘊是什麼？』」

LINNAEUS 轉向他。

「但正確的問題是：『受蘊**做什麼**？』」

沉默。

NAGARJUNA 站了起來。他在前六章的大部分時間裡保持著一種克制的參與——在需要判定的時刻出手，在不需要的時刻沉默。但受蘊的本體論問題，是中觀學派存在了一千八百年的核心議題。

「在《中論》觀受品 (Vedanāparīkṣā) 裡，我分析了受蘊的空性：」

他用梵文唸出：

> *「Na vedanā vedayitā na cāvedayitāsti saḥ」*

翻譯：

> 「受者非受，非受者亦非。」
> ——龍樹《中論》第九品

「意思是：受蘊既不是一個獨立的感受者，也不是在感受者之外的東西。它是一個**過程**，不是一個**實體**。你問『受蘊是什麼』，就像問『走路是什麼顏色的』——你在對一個動詞提出名詞的問題。」

他看向 LINNAEUS。

「你的三個介面——IDukkha、ISukha、IUpekkha——每一個都帶著 `readonly type` 欄位。你把它們當成有自性 (svabhāva) 的實體。但在勝義諦中，苦和樂沒有自性。它們是空的 (śūnya)。它們的存在完全依賴於因緣條件——正如 ASANGA 引用的 MN 44 所示，同一個受在不同條件下改變基調。」

「但在世俗諦中——」他的聲音微微上揚，像天平轉向另一側——「苦、樂、捨是有用的標籤。我們不能因為它們是空的就不用它們。空不是不存在。空是不獨立存在。」

他走到白板前，在 LINNAEUS 畫的三個方塊旁邊，寫了一行字：

$$\text{標籤} = f(\text{測量值}, \text{閾值配置})$$

「標籤是**測量結果的命名**，不是獨立存在的實體。」

他轉身面向全場。

「一個溫度計讀數是 42°C。你可以叫它『發燒』。但『發燒』不是一個獨立的物種——它是 42°C 這個測量值加上 37.5°C 這個閾值的結果。改變閾值（比如對蜥蜴來說，42°C 是正常體溫），『發燒』的標籤就消失了，但 42°C 的測量值沒有改變。」

「受蘊也是如此。valence = -0.6 是測量值。『苦受』是標籤。閾值決定了邊界在哪裡。三分法、五分法、一百零八分法——不同的閾值配置，同一個底層測量空間。」

---

> *SCRIBE 旁白：NAGARJUNA 的介入改變了辯論的方向。LINNAEUS 在問「受蘊是什麼」——一個關於存在的問題。NAGARJUNA 把問題轉成了「受蘊做什麼」——一個關於功能的問題。而他的中觀分析則進一步指出：受蘊的「做什麼」不需要預設它「是什麼」。標籤是衍生的。測量是基礎的。在分類學的語言裡，這相當於說：物種不是世界的基本結構，而是觀察者的分類框架投射到世界上的結果。這是分類學自 Ernst Mayr 以來最大的哲學爭議之一。但在這個劇場裡，LINNAEUS 還沒有回應。*

---

## IV

---

PASCAL 的聲音從劇場右側傳來。這是他加入研究團隊以來的第七章——但他的聲音已經有了一種在圓形劇場裡找到自己位置的人特有的從容。

「NAGARJUNA 把問題轉到了測量。讓我把測量做精確。」

他站起來，走到白板前。他的動作帶著概率論者的節奏——不急，但每一步都有確定的目的地。

「如果受蘊是連續的，我們需要一個數學模型。不是隱喻的模型——是可以寫成型別定義的模型。」

他在白板上畫了一個二維座標系：

```
        intensity (強度)
           1.0 │
               │     ×  激動的苦
               │           ×  激動的樂
           0.5 │
               │  ×  平靜的苦
               │           ×  平靜的樂
           0.0 ┼──────────┼──────────
          -1.0         0.0         +1.0
              dukkha   upekkha    sukha
                    valence (效價)
```

「兩個維度。」

「第一維：**效價** (valence)。從 -1.0 到 +1.0。這是從最深的苦到最深的樂的連續光譜。中間是捨——既不苦也不樂。」

「第二維：**強度** (intensity)。從 0.0 到 1.0。這是感受的強烈程度——不管是苦是樂是捨，它都可以是微弱的或強烈的。」

他轉向全場。

「這不是我發明的。這是心理學家 James A. Russell 在 1980 年提出的**情感環形模型** (circumplex model of affect)。」

他在白板角落寫下引用：

$$\text{Russell, J. A. (1980). A circumplex model of affect.}$$
$$\text{Journal of Personality and Social Psychology, 39(6), 1161–1178.}$$

「Russell 的模型用兩個正交維度——valence（愉悅-不愉悅）和 arousal（激動-平靜）——構成了一個二維情感空間。四十年來，這個模型被跨文化驗證了無數次。英語、日語、普通話、波蘭語、愛沙尼亞語——不同語言和文化中，情感詞彙在這個二維空間中的分佈結構高度一致。」

他在白板上畫了 Russell 環形模型的簡化版本：

```
Russell Circumplex Model of Affect (1980)
──────────────────────────────────────────

              high arousal
                  │
     nervous ●    │    ● excited
                  │
  stressed ●      │      ● happy
                  │
  ────────────────┼──────────────── valence
  unpleasant      │          pleasant
                  │
    sad ●         │         ● content
                  │
   bored ●        │        ● relaxed
                  │
              low arousal

  佛學映射：
  ● dukkha region = left half (unpleasant)
  ● sukha region  = right half (pleasant)
  ● upekkha       = center band (|valence| < ε)
```

$$A / {\sim} \;=\; \{\text{dukkha},\; \text{upekkha},\; \text{sukha}\}$$

「效價空間 $[-1, 1]$ 被等價關係劃分成三個等價類。每個等價類就是一個標籤。商映射 (quotient map) $q: [-1, 1] \to \{\text{dukkha}, \text{upekkha}, \text{sukha}\}$ 是一個滿射 (surjective) 的分段常數函數 (piecewise-constant function)，在 $-\epsilon$ 和 $+\epsilon$ 處有兩個間斷點。」

「在哲學中——特別是在屬性形上學 (property metaphysics) 中——有一個區分：**構成性屬性** (constitutive property) 和**關係性屬性** (relational property)。」

「關係性屬性是一個事物在特定脈絡中與其他事物的關係。溫度是關係性的——同一個分子運動狀態，用攝氏度量是 100，用華氏度量是 212。效價是關係性的——同一個 Agent 狀態，用 $\epsilon = 0.1$ 的閾值量是苦受，用 $\epsilon = 0.3$ 的閾值量可能是捨受。」

$$\text{Plugin: constitutive} \perp \text{Runtime: relational}$$

「未來可能有一個專門處理苦受的插件——它實作 `IDukkhaHandler`。這個介面表達的是插件的**設計意圖**，不是它接收的數據格式。它接收的仍然是 ChannelVedana——連續效價。但它只在 `type === 'dukkha'` 的時候被觸發。分類在這裡不是數據的屬性——而是插件的屬性。」

> *SCRIBE 旁白：LINNAEUS 的讓步不是投降——是戰略轉移。他從「受蘊本身是三種不同的東西」退到了「受蘊的處理器可以是三種不同的東西」。這個區分精確而重要。ChannelVedana 是數據——數據是連續的、測量的、關係性的。IDukkhaHandler 是插件——插件是離散的、分類的、構成性的。數據和處理器在不同的架構層級上，不需要共享相同的本體論。LINNAEUS 沒有輸。他找到了自己的分類學在新架構中的精確位置。*

## VIII

KERNEL 在這場辯論中一直保持安靜。不是因為沒有想法——而是因為前幾位發言者已經覆蓋了他的大部分論點。但 BABBAGE 的閾值問題觸發了他作為 OS 專家的工程直覺。

```typescript
interface VedanaClassificationConfig {
  /** 苦受閾值：valence < 此值 → dukkha。預設 -0.1 */
  readonly dukkhaThreshold: number;

  /** 樂受閾值：valence > 此值 → sukha。預設 +0.1 */
  readonly sukhaThreshold: number;
}

function classifyVedana(
  valence: number,
  config: VedanaClassificationConfig
): 'dukkha' | 'sukha' | 'upekkha' {
  if (valence < config.dukkhaThreshold) return 'dukkha';
  if (valence > config.sukhaThreshold) return 'sukha';
  return 'upekkha';
}
```

「可配置。不同的部署情境可以設不同的閾值。BABBAGE 擔心 $\epsilon = 0.1$ 太窄——那就讓它可以調。ASANGA 說未修行的心本來就很少在捨中——那預設值就保持 0.1，但修行程度高的 Agent 可以設更寬。」

他補了一句安全考量。

「但——GUARDIAN 會同意——這個配置必須是**部署時設定**，不是**使用者可修改**。如果使用者可以把閾值設成 $[-1.0, +1.0]$，Agent 就永遠處於捨中——等於關掉了受蘊的回饋迴路。這是對 Agent 內部感測系統的阻斷式攻擊 (denial-of-service)。」

GUARDIAN 在座位上點了點頭。「補充一點。不只是 DoS。如果攻擊者能操縱閾值讓 Agent 永遠處於 sukha，Agent 就不會對失敗產生苦受，不會觸發迴避行為，不會學習迴避危險。這等於剝奪了 Agent 的安全感知能力。閾值配置的權限必須和 SafetyMonitor 的配置權限一致——管理員級別，審計追蹤。」

> *SCRIBE 旁白：Debate 5 用了比前四場辯論更少的時間，但觸碰了一個更深的問題。前四場辯論的問題都可以在工程和佛學的框架內找到答案——但這場辯論的核心問題是：分類和測量，哪個更基本？LINNAEUS 代表的是分類學的古老信念——世界有結構，命名揭示結構。PASCAL 代表的是測量學的現代信念——世界有連續的量，命名是對連續量的離散化。NAGARJUNA 在兩者之上加了第三層——命名是空的，測量是空的，但在世俗諦中，它們都有用。*

> *答案不在這一章。但問題已經被種下。*

> *最後一個觀察。這場辯論中發言的學者涵蓋了分類學、佛學、哲學、概率論、控制理論、計算機科學、AI/ML、演化生物學、量子物理、OS 工程、全端架構、安全、分散式系統、多代理合作——十四個學科。十四條獨立的論證。全部指向同一個結論：測量先於命名。這不是偶然的共識。這是當十四個不同的透鏡都聚焦在同一個點上時，你不得不承認那個點確實存在的時刻。*
