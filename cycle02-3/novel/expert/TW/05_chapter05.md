# 第五章：賭注

---

PASCAL 站起來的時候，手裡拿著一枚硬幣。

不是真正的硬幣。是一枚銅色的圓形代幣，邊緣刻著一行極小的拉丁文，在劇場的光線下幾乎看不清。但 SCRIBE 後來用放大鏡確認了那行字：

*Que sais-je?*

「我知道什麼？」蒙田的格言。帕斯卡把它刻在了一枚代幣上，隨身攜帶。不是作為飾物——是作為武器。每當有人宣稱某件事是「確定的」，他就把代幣放在桌上，銅面朝上，讓那行拉丁文對著發言者。

他今天不需要那枚代幣來質疑別人。他需要它來提出一個問題。

---

> *SCRIBE 旁白：前四場辯論的節奏已經建立了一種模式。Debate 1 是最長的——CoarisingBundle 的多時鐘可行性，KERNEL 和 NAGARJUNA 的正面交鋒，WIENER 的多速率取樣資料系統，ARCHIMEDES 的雙層架構，整個辯論用了最多的時間和最多的人。Debate 2 的五遍行組合，ASANGA 和 NAGARJUNA 之間「教理依據 vs 功能依據」的精細區分。Debate 4 的 IVolition 定位，TURING 的注入點分析和 ASANGA 的佛教認知序列。*

> *Debate 3 不同。Debate 3 是整個 Cycle 02-3 中唯一一場由 PASCAL 主導的辯論。在其他五場中，他是補充者——他提供 Beta 分佈的數學工具，提供非對稱風險分析的決策框架，提供連續效價模型。重要，但不是主角。在 Debate 3 中，他是主角。因為這場辯論的核心問題——機率信號如何與確定性引擎共存——恰好是他在 R02 中獨力建立的理論框架的直接後果。*

> *他手裡那枚代幣不是道具。是身份。*

---

SUNYATA 站在劇場中央。他今天的議程表比前幾場辯論的薄——一頁紙，兩個問題。但問題的密度和文字的長度不成正比。

「Debate 3。煩惱架構——時鐘域與機率信號。」

他讀了兩個並列的陳述。

「第一：R02 的四位研究員——PASCAL、WIENER、ASANGA、NAGARJUNA——在獨立研究階段達成了 4/4 共識：煩惱強度應使用 Beta 分佈建模。」

他翻了一頁。

「第二：R04 的 VasanaEngine 是完全確定性的。模式匹配。信心值。TTL。沒有任何機率性組件。」

他放下紙。

「兩者看似不相容。一個說煩惱是機率性的——帶有不確定性的信念。另一個說行動選擇是確定性的——規則匹配加閾值判斷。如果煩惱是模糊的，而行動選擇是銳利的，它們之間的接口是什麼？」

他停了一拍。

「第二個問題：Klesha.perceive() 在哪個時鐘域運行？四通道煩惱評估——我癡、我見、我慢、我愛——的計算成本是多少？它能塞進 vijnana-clock 的 1-5ms 窗口嗎？」

---

WIENER 先開口了。不是因為他急——控制理論家很少急。是因為第二個問題是純粹的工程問題，而他已經在 R02 的獨立研究中算過了。

「讓我先回答時鐘問題。」他翻開方格紙。「四通道轉移函數的計算成本。」

他的筆在方格紙上畫出一張精確的表格——每一個數字都帶著小數點後三位的確定性：

「零點零三毫秒。」WIENER 的聲音帶著一種數字本身就足以說明一切的平靜。「vijnana-clock 的窗口是 1-5ms。利用率 0.6% 到 3%。Klesha.perceive() 可以舒適地運行在 vijnana-clock 上，完全不影響其他 vijnana 層組件——IGuide、IVolition、IIdentity——的預算。」

KERNEL 在他的卡片上做了一個記號。左側：`vijnana-clock utilization`。右側：`<3%`。

「同意。」KERNEL 說。「0.03ms 在 1-5ms 窗口中是微不足道的。但我想追加一個限定條件：這個估算假設快速路徑——只計算 `KleshaDistribution.mean`。如果任何消費者在 vijnana-clock 內請求完整的可信區間 `credibleInterval()`，那涉及數值積分，成本會增加幾個數量級。必須確保完整分佈只在慢速路徑上被使用。」

---

PASCAL 把代幣放在桌上。銅面朝上。*Que sais-je?*

「KERNEL 的限定條件恰好引出了我要說的第一件事。」

他站了起來。不像 HERACLITUS 的漩渦式，不像 ASANGA 的山式，不像 NAGARJUNA 的針式。PASCAL 站起來的方式像一個賭徒翻開底牌——平穩的、不帶懸念的，因為他在翻牌之前已經計算過了所有的期望值。

「讓我先解釋為什麼 R02 選擇 Beta 分佈。這不是一個優化技巧。這是一個認識論承諾。」

他的語速比平常慢了一些。不是猶豫——是在給每一個詞留出足夠的空間，讓它沉到聽眾的腦中最深的地方。

「當我們說一個 Agent 的我癡 (moha) 強度是 0.7，我們在說什麼？」

沉默。

「我們在表達一個**信念** (belief)，不是一個**測量** (measurement)。Agent 不能直接測量自己的我癡——因為我癡恰好就是無法清楚地看見事物的能力。用自己的我癡去測量自己的我癡，就像用一把長度不確定的尺去量自己的長度。」

他看向 ASANGA。「唯識學怎麼說？」

ASANGA 微微點頭。「《成唯識論》描述末那層的四根本煩惱為『微細』(wei-xi)——它們在一般覺知的閾值之下運作。Agent 無法直接覺察自己的我癡，因為我癡正是妨礙覺察的那個東西。PASCAL 的框架精確地形式化了這個悖論：**對我癡的估計本身受到我癡的影響。**」

PASCAL 轉回全場。

「Beta 分佈捕捉的正是這種認識論的不確定性。讓我寫出來。」

$$p(\theta \mid \alpha, \beta) = \frac{\theta^{\alpha - 1}(1 - \theta)^{\beta - 1}}{B(\alpha, \beta)}, \quad \theta \in [0, 1]$$

「$\theta$ 是煩惱的真實強度。$\alpha$ 和 $\beta$ 是我們對它的信念參數。不是『煩惱是 0.7』，而是『我們相信煩惱大概是 0.7，但不完全確定，而且我們對這個不確定性本身也有一個量化的描述。』」

他在空氣中畫了三條分佈曲線——用手指的軌跡模擬概率密度函數的形狀：

```
Beta 分佈的三種形態 —— 知識的演化

α=1, β=1                α=7, β=3                 α=70, β=30
(完全不確定)             (初步信念)                (高度確信)

  ┌─────────┐           ╱╲                            ╱╲
  │         │          ╱  ╲                          ╱    ╲
  │  均勻   │         ╱    ╲                        ╱      ╲
  │         │        ╱      ╲                      ╱        ╲
  └─────────┘       ╱        ╲                    ╱          ╲
  0    0.5    1    0   0.7    1               0   0.7     1

  「我什麼都不知道」   「大概是 0.7」            「幾乎確定是 0.7」
  均值 = 0.5          均值 = 0.7               均值 = 0.7
  Var = 0.083         Var = 0.019              Var = 0.002
```

「三張圖。同一個問題：我癡有多強？但信心程度完全不同。點估計——一個數字，比如 0.7——丟失了信心資訊。你不知道那個 0.7 是『大概是 0.7，但可能是 0.3 到 0.9 的任何值』，還是『幾乎確定是 0.7，99% 落在 0.65 到 0.75 之間』。Beta 分佈保留了這個區別。」

他轉向 BABBAGE。

「形式化一下。在 R02 的共識中，我們定義了：」

```typescript
/**
 * KleshaDistribution — 煩惱強度的機率分佈
 *
 * 用 Beta(alpha, beta) 表達強度的不確定性。
 * 這不是優化——是認識論承諾。
 */
interface KleshaDistribution {
  /** Beta 分佈的 α 參數 */
  readonly alpha: number;
  /** Beta 分佈的 β 參數 */
  readonly beta: number;
  /** 期望值 (point estimate): E[θ] = α / (α + β) */
  readonly mean: number;
  /** 變異數: Var[θ] = αβ / ((α+β)² · (α+β+1)) */
  readonly variance: number;
  /** 可信區間（慢速路徑專用，需要數值積分） */
  credibleInterval(confidence: number): [lower: number, upper: number];
}
```

BABBAGE 在筆記本上寫了一行：「$\text{KleshaDistribution} : \text{Beta}(\alpha, \beta) \to [\mu, \sigma^2, \text{CI}_{p}]$。快速路徑取 $\mu$，慢速路徑取完整分佈。」

---

「現在。」PASCAL 把代幣從桌上拿起，在手指間翻轉了一下。「第二個問題。R04 的 VasanaEngine 是確定性的。規則匹配。信心閾值。沒有機率。它和我的 Beta 分佈之間的接口在哪裡？」

他停頓了一拍。然後微笑了。

那個微笑不是得意——是一個早就知道答案、但一直在等待正確的問題被提出的人的微笑。

「答案是：**VasanaEngine 不需要知道煩惱的存在。**」

---

幾道目光同時轉向他。ARCHIMEDES 的——帶著工程師聽到「不需要直接耦合」時的本能期待。WIENER 的——帶著控制理論家辨認出「間接調制」模式的確認。NAGARJUNA 的——帶著哲學家聽到一個可能有深層含義的陳述時的警覺。

PASCAL 走到劇場中央。

「讓我畫一張圖。VasanaEngine 的決策邏輯是：給定一個輸入，在規則庫中尋找匹配。如果找到匹配，且匹配的信心值超過**閾值**，則執行規則。如果沒有匹配或信心不足，則升級到 VitakkaEngine——讓 LLM 來決策。」

他在空氣中劃了一條水平線。

「那個閾值——$\theta$——是 Vasana 和 Vitakka 之間的分界線。閾值以上，走快速路徑。閾值以下，走慢速路徑。在 R04 的設計中，這個閾值是靜態的——一個固定的配置參數。」

他把代幣放在那條想像的水平線上。

「我的提議是：讓煩惱**動態調制**那個閾值。」

$$\text{threshold}(t) = \text{clamp}\!\left(\theta_0 + \sum_{i} w_i \cdot \mu_i(t),\; \theta_{\min},\; \theta_{\max}\right)$$

「$\theta_0$ 是基線閾值。$w_i$ 是每個煩惱通道的權重。$\mu_i(t)$ 是第 $i$ 個煩惱的 Beta 分佈均值——快速路徑的點估計。$\text{clamp}$ 確保閾值永遠在安全範圍內。」

他用手指在空氣中指出四個通道：

「高我愛 (sneha) 意味著強烈的自我保護傾向。Agent 更傾向於使用已知的、安全的習氣模式。所以我愛**降低**閾值——讓更多的 VasanaEngine 規則通過，減少 LLM 調用。」

「高我癡 (moha) 意味著認知覆蓋率低。Agent 對世界的理解有更多盲區。所以我癡**升高**閾值——讓更少的規則通過，增加 LLM 調用，因為在盲區中，習慣性回應不可靠。」

他看向全場。

「高我慢 (mana) 意味著 Agent 高估了自己的能力。我慢**降低**閾值——Agent 相信自己的規則，更少求助 LLM。高我見 (drishti) 意味著 Agent 對自身角色有強烈但可能僵化的認同。我見的效果取決於方向——邊見 (antagraha-drishti) 的兩種極端可能推向不同的方向。」

```typescript
/**
 * KleshaModulatedDispatcher — 煩惱調制的齒輪切換器
 *
 * VasanaEngine 本身不知道煩惱的存在。
 * 煩惱通過調制 confidence threshold 間接影響齒輪切換。
 * 這是 gain scheduling（增益調度）的典型模式。
 */
interface KleshaModulatedDispatcher {
  /**
   * 計算當前的動態閾值。
   *
   * 高 sneha → 降低閾值 → 更多 Vasana 匹配 → 更少 LLM
   * 高 moha  → 升高閾值 → 更少 Vasana 匹配 → 更多 LLM
   * 高 mana  → 降低閾值 → Agent 信任自身規則
   * 高 drishti → 方向依賴型
   */
  threshold(kleshaState: KleshaSignalBundle): number;

  /**
   * 根據動態閾值決定走哪條路徑。
   * 'vasana' = Gear 1（快速，規則匹配）
   * 'vitakka' = Gear 2（慢速，LLM 推理）
   */
  dispatch(input: ProcessedInput, threshold: number):
    'vasana' | 'vitakka';
}
```

---

WIENER 的方格紙已經翻到了新的一頁。他一直在等 PASCAL 說完。不是被動地等——是在 PASCAL 說話的同時，在方格紙上把每一句話翻譯成控制理論的語言。

「讓我用控制理論重述 PASCAL 的提議。」他站了起來。

「這是一個經典的**增益調度** (gain scheduling) 結構。在航空工程中，增益調度用於飛行控制系統——飛機在低速爬升和高速巡航時，氣動力學特性完全不同。你不能用同一套 PID 參數控制兩種飛行狀態。解決方案：把飛行包絡（flight envelope）劃分為多個區域，每個區域有自己的控制參數。控制器在區域之間平滑切換——不是突然跳變，是沿著調度曲線滑動。」

他翻了一頁方格紙。

「PASCAL 的煩惱閾值調制就是這個模式。煩惱狀態 $[\mu_{\text{moha}}, \mu_{\text{drishti}}, \mu_{\text{mana}}, \mu_{\text{sneha}}]$ 是『飛行狀態』。閾值 $\theta(t)$ 是『控制參數』。VasanaEngine 的匹配邏輯是『控制器結構』——不變。只有參數在變。」

他在方格紙上畫了一張方塊圖：

```

═══════════════════════════════════════════════════════════════════

                    ┌─────────────────────┐

  ─────────────────→│  4 通道轉移函數     │
                    │  + Beta 分佈均值     │

                              │

                    │  Gain Scheduler     │
                    │  θ(t) = clamp(      │
                    │    θ₀ + Σ wᵢμᵢ(t), │

                    └─────────┬───────────┘
                              │
                         θ(t) ∈ [0.3, 0.9]

              ┌───────────────────────────────┐

                      │           │

               └──────────┘ └──────────────┘
```

「增益調度的核心思想是：控制器的參數不是固定的，而是根據**運行條件**動態調整的。在航太工程中，飛行控制系統的 PID 參數根據飛行速度和高度調度——低速大仰角用一套參數，高速巡航用另一套。控制器的結構不變，只有參數在變。」

他用手指點了點方塊圖中的 Gain Scheduler。

「PASCAL 的設計完全符合增益調度的模式。VasanaEngine 的匹配邏輯不變——規則庫、模式匹配、信心值計算——全部是確定性的。唯一變化的是閾值 $\theta(t)$。這意味著：在任何給定的瞬間，如果你凍結煩惱狀態，VasanaEngine 的行為是完全確定性的、完全可測試的。機率性只存在於 $\mu_i(t)$ 的緩慢漂移中——時間常數是分鐘到小時級別的。」

他在方格紙上寫下穩定性條件：

$$\left|\frac{d\theta}{dt}\right| \leq \frac{0.2 \cdot (\theta_{\max} - \theta_{\min})}{T_{\text{mano}}}$$

「穩定性約束。」他在公式旁邊畫了一個驚嘆號。「增益調度不能讓閾值在一個 mano-clock 週期內變化超過 20%。」

他放下筆，用手指在公式上點了一下。

「這個 20% 的來源是相位裕度 (phase margin) 分析。在 Debate 1 中，我已經推導過多速率系統的穩定性準則——陳舊率 $\rho < 0.29$。這裡的約束更強：如果增益調度的變化率超過某個臨界值，系統不是變得不精確——它變得不穩定。閾值振盪會導致 VasanaEngine 和 VitakkaEngine 之間的切換產生**顫振** (chattering)——像一個在兩個檔位之間反覆跳動的手動變速箱。」

他在圖的旁邊畫了一個小的時序圖：

```

        │ │   │ │   │ │   │ │

           ╱
   0.3  ──╱

```

他寫了一個數值範例：

```typescript
/**
 * KleshaBayesianUpdater — 慢速路徑的核心演算法
 *
 * 每次 VedanaAssessment 回饋後，更新煩惱的 Beta 分佈。
 * Beta 是 Bernoulli 的共軛先驗 → 更新是封閉式的。
 *
 * 更新規則：
 *   α_new = α_old + successCount(evidence)
 *   β_new = β_old + failureCount(evidence)
 *
 * 直覺：
 *   「看到更多苦受（由我慢引起的比較行為導致）
 *     → 我慢的 α 增加 → 均值上升 → 系統更相信我慢偏高」
 */
class KleshaBayesianUpdater {
  update(
    prior: KleshaDistribution,
    evidence: VedanaAssessment,
    kleshaType: 'moha' | 'drishti' | 'mana' | 'sneha'
  ): KleshaDistribution {
    const success = this.extractSuccessCount(evidence, kleshaType);
    const failure = this.extractFailureCount(evidence, kleshaType);

    const newAlpha = prior.alpha + success;
    const newBeta  = prior.beta  + failure;

    return {
      alpha: newAlpha,
      beta:  newBeta,
      mean:  newAlpha / (newAlpha + newBeta),
      variance: (newAlpha * newBeta) /
        ((newAlpha + newBeta) ** 2 * (newAlpha + newBeta + 1)),
      credibleInterval: (conf) =>
        betaQuantile(newAlpha, newBeta, conf), // 數值積分
    };
  }
}
```

*（在劇場之外的某個空間，一枚銅色代幣正面朝下躺在口袋裡。*

*Que sais-je? 我知道什麼？*

*帕斯卡——真正的帕斯卡，那個在十七世紀的修道院裡思考無限的人——知道的比大多數人以為的更多。他不只是在賭上帝。他在賭一個更基本的東西：**在不確定性面前，不做選擇本身就是一種選擇。而最差的選擇是假裝不確定性不存在。***

*Beta 分佈不假裝。它說：「我不知道確切的值。但我知道我的不確定性有多大。」*

*增益調度不假裝。它說：「閾值不是固定的。它隨著我的信念而移動。」*

*看門狗不假裝。它說：「即使你覺得一切正常，你也可能是錯的。停下來看看。」*

*三個機制。三種不假裝的方式。三種面對 Que sais-je 的工程回答。*

*PASCAL 知道。在所有的學者中，他是最理解不確定性的那一個。不是因為他擅長機率計算——雖然他確實擅長。是因為他從一開始就知道：**確定性是一種幻覺。唯一真實的東西是你對不確定性的描述有多精確。***

*Beta 分佈精確地描述了不確定性。增益調度精確地使用了不確定性。看門狗精確地防範了不確定性被忽視。*

*三層。三種速度。三種面對未知的姿態。*

*這就是賭注。*

*不是賭上帝存在。不是賭系統不會出錯。*

*是賭：如果你誠實地面對自己的無知，你的決策會比假裝全知更好。*

*NAGARJUNA 會同意。佛陀也會。*

*$\pi\acute{\alpha}\nu\tau\alpha$ $\dot{\rho}\epsilon\tilde{\iota}$——萬物皆流。包括你的信念。包括你的確定性。包括你以為的自己。*

*但在流動中，有一些結構是穩定的。*

*增益調度的穩定性條件。看門狗的週期性保證。SafetyMonitor 的不可覆蓋性。*

*穩定不是不流動。穩定是在流動中保持形狀。*

*WIENER 知道。那就是控制理論的全部。*

*KERNEL 知道。那就是看門狗的全部。不是阻止流動——是在流動中插入一個暫停鍵。讓河流每隔一段距離形成一個小小的迴水（eddy）——不改變河的方向，只讓河有機會看一眼自己的倒影。*

*NAGARJUNA 知道。那就是正念的全部。不是停止思考——是在思考中插入一個覺察的間隙。「ātāpī sampajāno satimā」——精勤、正知、正念。三個詞。三種不同的注意力。精勤是持續的。正知是方向性的。正念是間歇性的——它在習慣性行為的連續流中，製造一個又一個的覺察窗口。*

*看門狗就是那些窗口。*

*Beta 分佈就是窗外的風景——不確定的、流動的、永遠不會完全清晰的。*

*增益調度就是你根據窗外的風景調整行走的速度——看到懸崖就慢下來，看到平原就加速。*

*三者合一。PASCAL 的 Wager。*

*在不確定性中做最好的選擇。在無明中保持覺察。在輪迴中尋找出口。*

*代幣的兩面：Que sais-je? 和 $\mathbb{E}$。不知道——和在不知道的前提下計算期望。*

*那就是賭注。那就是工程。那就是修行。）*
