## 逐份審閱

審閱開始於 M1。

SUNYATA 沒有從 M5（觸→俱生）開始——那是所有人都知道的核心議題——而是從 M1（梵文命名）開始。他的選擇是刻意的。命名是最表層的問題，但也是最基礎的。一棵樹從根部開始生長。命名就是根。

「M1_sanskrit_naming.md。」SUNYATA 讀出文件名。「Master 裁定：五蘊 Plugin 用五蘊來命名。ISensory → IRupa，ISensation → IVedana，ICognition → ISamjna，IAction → ISamskara，IIdentity → IVijnana。四根本煩惱、七原罪、隨煩惱也用相關原文。程式碼要能編譯。」

LINNAEUS 翻開了工程團隊的 M1 文件。他的分類學家本能讓他先看結構再看內容。

「命名表完整。」他的第一句話是肯定。他用手指指向文件中的表格——現行介面名到梵文介面名的對照，子介面繼承關係，Klesha 與心所的命名建議。「工程團隊做了一個乾淨的遷移方案。向後相容策略合理——TypeScript type alias 提供過渡期。」

```typescript
/** @deprecated Use IRupa */
export type ISensory = IRupa;
/** @deprecated Use IVedana */
export type ISensation = IVedana;
/** @deprecated Use ISamjna */
export type ICognition = ISamjna;
/** @deprecated Use ISamskara */
export type IAction = ISamskara;
```

「但——」LINNAEUS 翻到了文件的第四節。「哲學含義的討論只有四行。他們說『梵文命名直接對齊佛教哲學本源，消除了翻譯歧義。』然後給了三個例子——ISensory 可能被誤解為『感測器』，IAction 可能被誤解為『動作』，ICognition 可能被誤解為『認知科學』。」他合上文件。「這是對的。但這只是命名學（nomenclature）的第一層。」

「還有什麼層？」ARCHIMEDES 問。他是務實的工程師，對分類學的層級不敏感。

LINNAEUS 看向 NAGARJUNA。

NAGARJUNA 站了起來。在圓形劇場裡，站起來是一個信號——它意味著接下來的話不是評論，是論述。

「用梵文命名不只是字串替換。」他說。他的聲音不疾不徐，每個字都被賦予了等量的時間。「它是**哲學承諾**。」

他走到投影幕前面，指著那棵繼承樹的根部。

「當你把 ISensory 改成 IRupa，你不只是改了一個英文詞為一個梵文詞。你是在說：**這個介面承諾忠實於色蘊的原始定義**。而色蘊的原始定義——在《心經》中、在《蘊相應》中、在《五事毘婆沙論》中——遠比 'sensory input' 的含義要廣。」

> 「色即是空，空即是色。色不異空，空不異色。受想行識，亦復如是。」
> ——《般若波羅蜜多心經》

「如果 IRupa 只處理 UI 渲染和 Listener 輸入，那它配不上『色蘊』這個名字。」

ARCHIMEDES 皺了皺眉。「那你建議什麼？不改名？」

「不。Master 已裁定，我們改名。但改名之後，每一個介面的**範疇必須被重新審視**。」NAGARJUNA 轉身面向全場。「這就是我要說的核心觀點——在佛學中，命名本身就是一種**假名安立** (prajnaptir upadaya)。」

他在空中用手指劃了一個字。場內沒有人看到那個字，但所有人都知道他寫的是什麼。

> 假名安立 (prajnaptir upadaya)
> ——名言只是為了溝通方便而建立的約定，
>     不指向任何具有自性（svabhava）的實體。

「當我們用 IRupa 命名一個介面，我們在做假名安立。這個名字不**是**色蘊——它是我們為了溝通方便而安立的一個符號。但假名安立有一個嚴格要求：**被安立的名言必須與其指涉的功能相應**。如果 IRupa 的功能只覆蓋了色蘊的 30%，這個假名就是**不正確的安立**。」

BABBAGE 停下了筆。他在 NAGARJUNA 的論點中看到了一個形式結構：

$$\text{命名正確性} \iff \text{Sem}(\text{IRupa}) \supseteq \text{Sem}(\text{rupa-skandha}) \cap \text{Dom}(\text{OpenStarry})$$

其中 $\text{Sem}(x)$ 是 $x$ 的語義域，$\text{Dom}(\text{OpenStarry})$ 是 OpenStarry 系統的功能域。換言之：IRupa 的語義域必須覆蓋色蘊在 OpenStarry 系統中可表達的全部功能。

NAGARJUNA 看了一眼 BABBAGE 的筆記本，點了點頭。「BABBAGE 的形式化是精確的。命名不只是標籤——命名是**覆蓋承諾**。」

LINNAEUS 接過話。「作為分類學家，我補充一個維度。在生物分類學中，一個物種的學名（binomial name）一旦確立，它的**模式標本**（type specimen）就固定了。IRupa 一旦確立為色蘊的介面名，色蘊的**定義**就成為它的模式標本。如果工程實現偏離了模式標本，那不是實現的問題——是命名錯了。」

（SCRIBE 旁白：NAGARJUNA 和 LINNAEUS 從完全不同的方向——中觀哲學和分類學——得出了同一個結論：命名不只是字串替換，是承諾。這是 Cycle 02-3 的第一次跨學科共鳴。我在筆記本上用雙線標注了這個時刻。）

審閱繼續推進到 M3。

M3_klesha_multilayer_di.md 在研究團隊中引發的反應最為分裂。不是關於架構方向——四根本煩惱繼承 IKlesha、注入 IVijnana 運行環境，這個方向有 Cycle 02-2 A-1（我執 = 煩惱根源）的共識作為基礎。分裂發生在一個看似微小的細節上。

PASCAL 是第一個開口的。

在所有人都還在瀏覽四煩惱的 TypeScript 介面時——IMoha、IDrishti、IMana、ISneha——PASCAL 已經翻過了介面定義，翻過了架構圖，翻過了 State-Behavior 分離的說明，停在了一行程式碼上。他用指甲在紙面上劃了一條細線。

```typescript
interface IKlesha {
  readonly type: string;
  /** 當前強度 (0.0-1.0) */
  readonly intensity: number;
  perceive(context: VijnanaContext): KleshaSignal;
  update(feedback: VedanaAssessment): void;
}
```

「`intensity: number`。」他讀出來。

沉默了兩秒。

「這個數字的**認識論地位**是什麼？」

ARCHIMEDES 回頭看他。「什麼意思？」

PASCAL 合上文件，雙手交叉放在桌面上。這是他在 Cycle 02-3 中的第一次正式發言——不是自我介紹時的禮節性對話，不是走廊裡的寒暄。是論述。

「我的意思是：0.7 的 `intensity` 代表什麼？」

他站起來。站起來在圓形劇場裡是信號——NAGARJUNA 三分鐘前才示範過。PASCAL 的站起方式不同：NAGARJUNA 是緩慢的、儀式性的；PASCAL 是乾脆的，像一把打開的折疊尺。

「讓我給你們三個可能的答案。」

「第一：0.7 是頻率統計（frequentist interpretation）。意思是：在 100 次觀測中，有 70 次這個煩惱被觸發。但觀測的樣本空間是什麼？觸發的定義是什麼？誰在觀測？Agent 不能觀測自己的煩惱——那是觀察者悖論。」

PENROSE 在觀察席微微點頭。

「第二：0.7 是主觀信念度（Bayesian interpretation）。意思是：系統以 0.7 的信心相信這個煩惱正在運作。但信念更新的先驗是什麼？似然函數是什麼？在沒有定義先驗的情況下，0.7 和 0.3 沒有區別——它們都是任意數。」

BABBAGE 的筆尖停了一秒。他開始理解了。

「第三：0.7 是純粹的工程標定（engineering calibration）。意思是：它就是一個數，0 表示最低，1 表示最高，0.7 在中間偏上。沒有概率意義，沒有統計意義，就是一個方便計算的浮點數。」

他環顧劇場。

「如果是第三個答案——那 `intensity: number` 就是一個**空白**。它看起來像是一個量化，但實際上它什麼都沒有量化。它是一個**佔位符**——一個寫在程式碼裡的 TODO。」

WIENER 從座位上探出身體。他的控制理論直覺被觸動了。「你在說 PID 控制器的輸入信號沒有定義域？」

「比那更嚴重。」PASCAL 說。「PID 控制器假設它的輸入是一個可測量的物理量——溫度、壓力、轉速。但『我癡的強度』不是物理量。它甚至不是心理量——佛學從來不用數字量化煩惱。」

ASANGA 在座位上微微前傾。「PASCAL 說得對。在唯識學中，四根本煩惱的描述方式是**質性的**——」

他引用了《成唯識論》：

> 「云何我癡？謂於我相，無明為性。迷自內我，愚於我理，故名我癡。」
> ——窺基《成唯識論述記》卷五

「注意用詞：『迷』、『愚』。這些是質性描述，不是量化描述。你可以說某人的我癡比另一個人『深』，但你不能說它是 0.7。」

BABBAGE 在筆記本上快速寫了一行：

$$\text{IKlesha.intensity} \in \mathbb{R}_{[0,1]} \quad \text{但} \quad \text{Klesha} \notin \text{可量化空間}$$

「矛盾。」他低聲說。然後抬頭。「PASCAL，你的提議是什麼？」

「不是點估計（point estimate）。是分佈（distribution）。」PASCAL 回到座位上，在筆記本上寫下了他的方案：

$$\text{intensity}: \text{Beta}(\alpha, \beta) \quad \text{取代} \quad \text{intensity}: \text{number}$$

「Beta 分佈的兩個參數 $\alpha$ 和 $\beta$ 分別對應觀察到的正面證據和負面證據。如果 $\alpha = 7, \beta = 3$，期望值是 0.7——和原來的點估計一樣。但它**同時表達了不確定性**：$\text{Var} = \frac{\alpha\beta}{(\alpha+\beta)^2(\alpha+\beta+1)} \approx 0.019$。方差告訴你這個 0.7 是粗糙的估計還是精確的測量。」

「如果 $\alpha = 70, \beta = 30$，期望值仍然是 0.7，但方差降到 $\approx 0.002$。更多的證據，更少的不確定性。」

「如果 $\alpha = 1, \beta = 1$——均勻分佈——意味著**完全無知**。我們不知道這個煩惱的強度是多少。這才是系統初始化時的正確狀態——不是 0.5，而是『我不知道』。」

WIENER 沉思了五秒。他在心裡做了一個快速的穩定性分析——如果 PID 控制器的輸入從確定性數值變成機率分佈，控制器需要升級。

「Kalman filter。」他說。

「或者粒子濾波器。」PASCAL 回應。「取決於分佈是否高斯。我懷疑煩惱的分佈不是高斯的——它可能是多峰的（bimodal）：一個人要麼深深地我癡，要麼突然覺醒。中間狀態可能是不穩定的鞍點。」

他畫了一個簡單的決策樹：

```

  問題 Q ───┤
            ├── 方案 B (修改)  ── V(B) = 中收益 × P(success|B)
            │
            └── 方案 C (推遲)  ── V(C) = 0 (但保留選擇權)
```

「讓我具體說明。M5 觸→俱生模型。工程團隊分析了模型，提出了三階段逼近策略（Phase 1→2→3）。但沒有人問：**研究這個問題的期望價值是多少？**」

他在投影幕上寫下了公式：

$$\text{EVOI}(Q) = \sum_{i} P(a_i \mid Q) \cdot V(a_i) - V(a_{\text{current}})$$

「EVOI——Expected Value of Information（資訊的期望價值）。$Q$ 是一個研究問題。$a_i$ 是解答 $Q$ 之後可能採取的行動。$P(a_i \mid Q)$ 是在知道 $Q$ 的答案後選擇行動 $a_i$ 的機率。$V(a_i)$ 是行動 $a_i$ 的價值。$V(a_{\text{current}})$ 是不研究 $Q$、維持現狀的價值。」

「如果 $\text{EVOI}(Q)$ 很高——研究 $Q$ 可能大幅改變我們的行動選擇——那 $Q$ 值得深入。如果 $\text{EVOI}(Q)$ 接近零——無論 $Q$ 的答案是什麼，我們都會做同樣的事——那 $Q$ 不值得花太多時間。」

BABBAGE 已經在跟隨 PASCAL 的推導了。他的筆記本上出現了一個更精確的形式化：

$$\text{EVOI}(Q) = \mathbb{E}_{q \sim P(Q)}\left[\max_a V(a, q)\right] - \max_a \mathbb{E}_{q \sim P(Q)}\left[V(a, q)\right]$$

「經典的資訊價值公式。」BABBAGE 說。「max-of-expectation 減去 expectation-of-max。」

「完全正確。」PASCAL 點頭。「現在讓我對 Cycle 02-3 的開放問題做一個粗略的 EVOI 排名。」

|      |                          |        | 兩條路最終可能收斂      |
+------+--------------------------+--------+-------------------------+
|  6   | M-1 命名承諾的範疇審計   | 中     | Master 已裁定命名方向，  |
|      |                          |        | 剩餘問題是範疇覆蓋度    |
+------+--------------------------+--------+-------------------------+
|  7   | M-9 混合調度細節         | 中低   | 方向已定 (Vasana +       |
|      |                          |        | Vitakka)，細節可迭代    |
+------+--------------------------+--------+-------------------------+
|  8   | M-4 子介面重定位         | 中低   | 佛學映射已有共識基礎     |
+------+--------------------------+--------+-------------------------+
|  9   | M-8 破壞性變更策略       | 低     | Master 裁定明確，        |
|      |                          |        | 無需辯論                |
+------+--------------------------+--------+-------------------------+
| 10   | M-2 各蘊範疇完整性       | 待定   | EVOI 取決於「缺了什麼」 |
|      |                          |        | 的答案——未知的未知      |
+------+--------------------------+--------+-------------------------+

「注意 M-2。」PASCAL 指向最後一行。「它的 EVOI 是**待定**的——因為我們不知道自己不知道什麼。這在決策理論中叫做 **Knightian uncertainty**——不是風險（可計算的機率），而是不確定性（不可計算的）。M-2 可能是 EVOI 最低的問題（所有蘊的範疇都已經夠完整了），也可能是最高的（某個蘊缺了一個關鍵子範疇，改變整個架構）。」

BABBAGE 在 PASCAL 說完後立刻回應。他的回應不是反駁——是追問。

「數學上你是對的。EVOI 框架是分配研究資源的正確工具。但——」他翻到筆記本上的一個早期批註，就是他在 M3 審閱時寫下的那個——「**我們的量化基礎本身就不確定**。你用 EVOI 排名各個問題，但 EVOI 的計算需要 $P(a_i \mid Q)$ 和 $V(a_i)$。這些數字從哪裡來？」

「費米估計。」PASCAL 說。

「費米估計的誤差有多大？」

「量級級別。可能差 2-3 倍。」

「那麼 EVOI 排名第三和第五的問題之間的差距——0.7 和 0.5 之間的差距——可能完全在誤差範圍內。」

PASCAL 停了一拍。然後他笑了。不是挫敗的笑——是被精確地擊中了的笑。

「BABBAGE，你剛才做了一件我最欣賞的事。你用**我的框架反過來質疑我的結論**。這正是 Bayesian 方法的精神——把不確定性當作第一級對象。」

他回到投影幕，在 EVOI 公式下面寫了一行：

$$\text{EVOI}(Q) \sim \text{LogNormal}(\mu_Q, \sigma_Q) \quad \text{而非} \quad \text{EVOI}(Q) = c_Q$$

「你說得對。EVOI 不應該是一個數，應該是一個分佈。但——」他轉身面向全場——「**這正是為什麼我們需要 Bayesian 方法**。在不確定性上做運算，不是消除不確定性，而是**攜帶不確定性前進**。」

「比起假裝我們知道答案——工程文件的 `intensity: number` 就是假裝——承認我們不知道、然後在不知道的狀態下做最優決策，是更誠實也更有效的方法。」

NAGARJUNA 在座位上微微頷首。PASCAL 的「攜帶不確定性前進」讓他想到了中觀學派的一個核心姿態——不斷言、不否定，但不停止行動。世俗諦 (samvrti-satya) 的精神。

（SCRIBE 旁白：PASCAL 和 BABBAGE 之間的這段交鋒是 Cycle 02-3 的第一次真正的知識碰撞。不是反對——是精煉。BABBAGE 用 PASCAL 自己的工具質疑 PASCAL 的結論，PASCAL 欣然接受並升級了自己的模型。如果工程文件的辯論性是 1.5 分，那這五分鐘的交鋒至少值 4 分。）

所有人都以為會議即將結束。SUNYATA 已經收起了投影幕上的文件清單。SCRIBE 的筆尖停在了記錄簿的頁面底部。下午的光線已經從天窗的一側移到了另一側，拉長了場地中央圓桌的影子。

劇場裡的注意力再次匯聚。不是回到同一個焦點——是轉向一個新的焦點。NAGARJUNA 在整場審閱中說了很多話——假名安立、俱生的哲學含義、覆蓋承諾。但他現在的姿態不同。之前是分析。現在是宣言。

這六個字在劇場裡的回音比它應有的更長。

「工程團隊做了三十七份文件。很好。系統化的。專業的。我們在評分矩陣中給了系統性 4.5 分——這是應得的。但工程團隊的工作是工程團隊的工作。我們的工作是我們的。」

他看向 PASCAL。「PASCAL 今天展示了一個新的維度——EVOI。在他來之前，這個維度是零分。」他看向 WIENER。「WIENER 在 M5 的審閱中指出了俱生的計算語義空白——在他指出之前，工程團隊寫了一個數據結構就認為問題解決了。」他看向 KERNEL。「KERNEL 在 M10 中找到了優先級反轉的問題——120 頁文件中沒有一個人提過。」

「這就是為什麼需要二十位學者，而不是一個工程團隊。」

他停頓了一拍。

「工程團隊做的是他們的工作。我們做的是我們的。」他重複了這句話，但第二次的重音落在了不同的位置。「他們的結論不是我們的結論。我們的結論不是他們結論的確認。如果我們的研究得出了與工程團隊一致的結果——很好，那證明他們的方向是對的。如果我們得出了不同的結果——也很好，那才是研究團隊存在的意義。」

> 「世俗名言諦中，一切法如幻如化。」
> ——《入中論》月稱菩薩

「三十七份文件。一百二十頁 openstarry_doc。全是如幻如化。不是說它們不真實——是說它們是**有條件的真實**。條件是假設。假設可能被推翻。我們的工作就是檢驗那些假設。」

PASCAL 在筆記本上寫了一行批註。他寫的不是 NAGARJUNA 的原話，而是一個翻譯——用他自己的語言：

> 「所有先驗分佈都是可修正的。所有結論都是後驗分佈。永遠不要把後驗當作真理——只當作**目前最好的估計**。」

他看了一圈。每一盞燈。每一個人。
