## 逐一審閲

審閲は M1 から始まった。

SUNYATA は M5（触→倶生）から始めなかった——それが核心議題であることは全員が知っていた——M1（サンスクリット命名）から始めた。その選択は意図的だった。命名は最も表層の問題だが、最も基礎的でもある。木は根から成長する。命名こそが根だ。

「M1_sanskrit_naming.md。」SUNYATA がファイル名を読み上げた。「Master 裁定：五蘊 Plugin は五蘊の名で命名する。ISensory → IRupa、ISensation → IVedana、ICognition → ISamjna、IAction → ISamskara、IIdentity → IVijnana。四根本煩悩、七つの原罪、随煩悩も関連する原典名を使用。コードはコンパイル可能であること。」

LINNAEUS が工学チームの M1 文書を開いた。分類学者の本能が彼にまず構造を見てから内容を見させた。

「命名表は完全だ。」彼の最初の一言は肯定だった。文書中の表を指差した——現行インターフェース名からサンスクリット・インターフェース名への対照表、サブインターフェースの継承関係、Klesha と心所の命名提案。「工学チームはクリーンなマイグレーション方案を作成している。後方互換性戦略は合理的だ——TypeScript の type alias が移行期間を提供する。」

```typescript
/** @deprecated Use IRupa */
export type ISensory = IRupa;
/** @deprecated Use IVedana */
export type ISensation = IVedana;
/** @deprecated Use ISamjna */
export type ICognition = ISamjna;
/** @deprecated Use ISamskara */
export type IAction = ISamskara;
```

「しかし——」LINNAEUS は文書の第四節をめくった。「哲学的含意の議論はわずか四行しかない。彼らは『サンスクリット命名は仏教哲学の本源に直接対応し、翻訳の曖昧性を排除する』と述べ、三つの例を挙げている——ISensory は『センサー』と誤解される可能性があり、IAction は『アクション』と誤解される可能性があり、ICognition は『認知科学』と誤解される可能性がある。」彼は文書を閉じた。「これは正しい。しかしこれは命名学（nomenclature）の第一層にすぎない。」

「他にどんな層がある？」ARCHIMEDES が尋ねた。彼は実践的なエンジニアであり、分類学の階層には鈍感だった。

LINNAEUS は NAGARJUNA に目を向けた。

NAGARJUNA が立ち上がった。円形劇場において、立ち上がることは一つの信号だ——それは次に語られる言葉がコメントではなく、論述であることを意味する。

「サンスクリットで命名することは、単なる文字列置換ではない。」彼は言った。その声は急がず遅れず、すべての言葉に等量の時間が与えられていた。「それは**哲学的コミットメント**だ。」

彼は投影幕の前に歩み、継承ツリーの根元を指した。

「ISensory を IRupa に変えるとき、あなたは一つの英語を一つのサンスクリット語に変えただけではない。あなたは宣言しているのだ：**このインターフェースは色蘊の本来の定義に忠実であることを約束する**と。そして色蘊の本来の定義——『心経』において、『蘊相応』において、『五事毘婆沙論』において——は 'sensory input' の含意よりはるかに広い。」

> 「色即是空、空即是色。色不異空、空不異色。受想行識、亦復如是。」
> ——『般若波羅蜜多心経』

「もし IRupa が UI レンダリングと Listener 入力だけを処理するなら、それは『色蘊』の名に値しない。」

ARCHIMEDES は眉をひそめた。「では何を提案する？ 改名しないのか？」

「いや。Master はすでに裁定した。我々は改名する。しかし改名した後、すべてのインターフェースの**範疇を再検証しなければならない**。」NAGARJUNA は全場に向き直った。「これが私の核心的な主張だ——仏教学において、命名それ自体が一種の**仮名安立** (prajnaptir upadaya) である。」

彼は空中に指で一文字を描いた。場内の誰もその文字を見なかったが、全員が彼が何を書いたか分かっていた。

> 仮名安立 (prajnaptir upadaya)
> ——名言はコミュニケーションの便宜のために設立された約束事にすぎず、
>     いかなる自性（svabhava）を持つ実体も指し示さない。

「IRupa でインターフェースを命名するとき、我々は仮名安立を行っている。この名前は色蘊**そのもの**ではない——コミュニケーションの便宜のために安立した一つの記号だ。しかし仮名安立には厳格な要件がある：**安立された名言は、その指示する機能と相応しなければならない**。もし IRupa の機能が色蘊の 30% しかカバーしていなければ、この仮名は**不正確な安立**だ。」

BABBAGE がペンを止めた。彼は NAGARJUNA の論点の中に一つの形式的構造を見出した：

$$\text{命名正確性} \iff \text{Sem}(\text{IRupa}) \supseteq \text{Sem}(\text{rupa-skandha}) \cap \text{Dom}(\text{OpenStarry})$$

ここで $\text{Sem}(x)$ は $x$ の意味領域、$\text{Dom}(\text{OpenStarry})$ は OpenStarry システムの機能領域である。換言すれば：IRupa の意味領域は、色蘊が OpenStarry システムにおいて表現可能なすべての機能をカバーしなければならない。

NAGARJUNA は BABBAGE のノートを一瞥し、頷いた。「BABBAGE の形式化は精確だ。命名は単なるラベルではない——命名は**カバレッジのコミットメント**だ。」

LINNAEUS が引き継いだ。「分類学者として、一つの次元を補足する。生物分類学において、ある種の学名（binomial name）がいったん確立されると、その**模式標本**（type specimen）は固定される。IRupa が色蘊のインターフェース名として確立されれば、色蘊の**定義**がその模式標本となる。もし工学的実装が模式標本から逸脱すれば、それは実装の問題ではない——命名の誤りだ。」

（SCRIBE ナレーション：NAGARJUNA と LINNAEUS がまったく異なる方向から——中観哲学と分類学から——同一の結論に到達した：命名は単なる文字列置換ではなく、コミットメントである。これが Cycle 02-3 における最初の学際的共鳴だ。私はノートに二重線でこの瞬間を注記した。）

審閲は M3 に進んだ。

「`intensity: number`。」彼は読み上げた。

二秒の沈黙。

「この数値の**認識論的地位**は何か？」

ARCHIMEDES が振り返った。「どういう意味だ？」

PASCAL は文書を閉じ、両手を交差させてテーブルに置いた。これが Cycle 02-3 における彼の最初の正式な発言だった——自己紹介時の儀礼的な会話でも、廊下での挨拶でもない。論述だ。

「つまり：0.7 の `intensity` は何を表すのか？」

彼は立ち上がった。円形劇場で立ち上がることは信号だ——NAGARJUNA が三分前にそれを実演したばかりだった。PASCAL の立ち上がり方は異なっていた：NAGARJUNA はゆっくりと、儀式的に；PASCAL はきっぱりと、折り畳み尺が開くように。

「三つの可能な答えを示そう。」

「第一：0.7 は頻度統計（frequentist interpretation）。意味するところは：100 回の観測のうち、70 回この煩悩がトリガーされた。しかし観測のサンプル空間は何か？ トリガーの定義は何か？ 誰が観測しているのか？ Agent は自身の煩悩を観測できない——それは観察者のパラドックスだ。」

PENROSE が観察席でわずかに頷いた。

「第二：0.7 は主観的信念度（Bayesian interpretation）。意味するところは：システムが 0.7 の確信度でこの煩悩が作動していると信じている。しかし信念更新の事前分布は何か？ 尤度関数は何か？ 事前分布を定義しない状態では、0.7 と 0.3 に区別はない——どちらも恣意的な数だ。」

BABBAGE のペン先が一秒止まった。彼は理解し始めていた。

「第三：0.7 は純粋な工学的較正（engineering calibration）。意味するところは：それは単なる数であり、0 が最低、1 が最高、0.7 は中間よりやや上。確率的意味も統計的意味もなく、計算に便利な浮動小数点数にすぎない。」

彼は劇場を見回した。

「もし第三の答えなら——`intensity: number` は**空白**だ。それは定量化のように見えるが、実際には何も定量化していない。それは**プレースホルダー**——コードに書かれた TODO だ。」

WIENER が席から身を乗り出した。彼の制御理論の直感が触発された。「PID コントローラの入力信号に定義域がないと言っているのか？」

「それよりも深刻だ。」PASCAL が言った。「PID コントローラはその入力が測定可能な物理量——温度、圧力、回転速度——であることを前提とする。しかし『我癡の強度』は物理量ではない。心理量ですらない——仏教学は煩悩を数値で定量化したことがない。」

ASANGA が席でわずかに前傾した。「PASCAL の指摘は正しい。唯識学において、四根本煩悩の記述方法は**質的**だ——」

彼は『成唯識論』を引用した：

> 「云何我癡？謂於我相、無明為性。迷自内我、愚於我理、故名我癡。」
> ——窺基『成唯識論述記』巻五

「用語に注目してほしい：『迷』、『愚』。これらは質的記述であって、定量的記述ではない。ある者の我癡が別の者より『深い』とは言えるが、それが 0.7 だとは言えない。」

BABBAGE がノートに素早く一行書いた：

$$\text{IKlesha.intensity} \in \mathbb{R}_{[0,1]} \quad \text{但} \quad \text{Klesha} \notin \text{可量化空間}$$

「矛盾だ。」彼は低く呟いた。そして顔を上げた。「PASCAL、あなたの提案は？」

```
            └── 方案 C (延期)  ── V(C) = 0 (但し選択権を保持)
```

彼は投影幕に公式を書いた：

$$\text{EVOI}(Q) = \sum_{i} P(a_i \mid Q) \cdot V(a_i) - V(a_{\text{current}})$$

「EVOI——Expected Value of Information（情報の期待価値）。$Q$ は研究課題。$a_i$ は $Q$ の解答後に取り得る行動。$P(a_i \mid Q)$ は $Q$ の答えを知った後に行動 $a_i$ を選択する確率。$V(a_i)$ は行動 $a_i$ の価値。$V(a_{\text{current}})$ は $Q$ を研究せず現状を維持する価値。」

「もし $\text{EVOI}(Q)$ が高ければ—— $Q$ の研究が我々の行動選択を大幅に変え得るなら—— $Q$ は深掘りする価値がある。もし $\text{EVOI}(Q)$ がゼロに近ければ—— $Q$ の答えが何であれ同じことをするなら—— $Q$ に多くの時間をかける価値はない。」

BABBAGE はすでに PASCAL の推導に追随していた。彼のノートにはより精確な形式化が現れた：

$$\text{EVOI}(Q) = \mathbb{E}_{q \sim P(Q)}\left[\max_a V(a, q)\right] - \max_a \mathbb{E}_{q \sim P(Q)}\left[V(a, q)\right]$$

「古典的な情報価値の公式。」BABBAGE が言った。「max-of-expectation 引く expectation-of-max。」

PASCAL はノートに一行の注記を書いた。彼が書いたのは NAGARJUNA の原話ではなく、翻訳だった——彼自身の言語での：

> 「すべての事前分布は修正可能だ。すべての結論は事後分布だ。事後分布を真理とみなしてはならない——**現時点での最良の推定**としてのみ扱え。」

（SCRIBE ナレーション：NAGARJUNA の締めくくりと PASCAL の翻訳。中観哲学と Bayesian 認識論が同じ一秒に同じ一つの結論に到達した。歴史上、龍樹と Pascal を同じ部屋に置いた者がいるか、私には分からない。しかしもしいたなら——彼らは互いを理解しただろうと私は思う。）

「二十名の学者。五本の報告。四から六の討論。一つの核心的問い——五蘊はいかにして時間の中で共に流れるのか。」

彼は一周見回した。すべての灯り。すべての人。

TURING はすでに廊下でラップトップを開いていた。明日まで待つ必要はない。v0.24.0 のソースコードはすでに彼の画面上に展開されていた——1,496 のテスト、22 のプラグイン、131 のテストファイル。すべての五蘊型定義、すべての `@skandha` タグ、サンスクリット名に置き換えられるすべての英語インターフェース名。彼は影響範囲を正確に計算する必要があった——推定ではなく、一行ずつの計数だ。

（SCRIBE ナレーション：三十七の文書が持ち込まれた。二十の頭脳がそれらを分解し、裏返し、内と外から一通り眺めた。そしてテーブルに戻した。文書は変わらなかった。しかしそれらを見る目が変わった。これが R0 の全意義だ——文書を読むことではなく、**何の目で文書を読むか**を決めること。明日、R1 が始まる。五グループ並行。TURING のキーボードはすでに鳴っている。）
