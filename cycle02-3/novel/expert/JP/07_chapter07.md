# 第七章：物差しと命名

---

LINNAEUS がマーカーを机に置いた。青いキャップが左を向いている。

これまでの六章を通じて、彼のマーカーの使用頻度は極めて高かった。第三章で IIdentity の分類学的誤謬を解体した際には、青いマーカーがほとんど指先から離れることがなかった。第六章で五本の樹の拡張図を描いた際には、青いインクがホワイトボード上に四十七本の枝分かれ線を伸ばした。しかし今、第七章の冒頭で、彼はマーカーを置いた。

インクが切れたからではない。この章の議題が、分類学者にとって根源的な問いを突きつけるものだからだ——お前が引いてきた界門綱目科属種の境界線は、世界の真の構造なのか、それとも測定器具の目盛りにすぎないのか。

---

> *SCRIBE 傍白：LINNAEUS がマーカーを置く動作は軽かった。しかしその動作には重みがあった。前六章において、分類学者は道具を手に作業する人間だった——その道具とはあの青いマーカーであり、一本の線が一つの分類境界であり、一つの分岐が一回の命名決定だった。今、誰かがその境界そのものの性質を問おうとしている。これは技術的挑戦ではない——本体論的挑戦である。お前が引いた線は発見されたものか、それとも発明されたものか。お前が命名したものは、命名する以前から存在していたのか、それともお前の命名によって初めて存在し始めたのか。*

> *分類学には古くからの論争がある。自然種 (natural kind) 対 約定種 (conventional kind)。金と銀は自然種だ——その境界は原子構造によって決まり、人間の命名によって変わることはない。「野菜」と「果物」は約定種だ——トマトは植物学的には果実であり、料理学的には野菜である。受蘊の三分——苦・楽・捨——はどちらに属するのか。これが第五場辯論の核心である。*

---

SUNYATA の声が劇場の中央から響いた。石。深い淵。前四場の辯論と同じ始まり方——だが今回、彼の手には Master の書簡がなかった。

「Debate 5。Vedana インターフェース設計。分類 vs 測定。」

彼は一拍置いた。

「前四場の辯論が扱ったのは構造とプロセスだった。CoarisingBundle の構成、煩悩のクロックドメイン、IVolition の位置、マルチクロックの実現可能性。それらの問題にはすべて共通する特徴があった——その答えは**アーキテクチャの**問題だった。コンポーネントをどこに配置するか、データがどう流れるか、クロックをどう同期させるか。」

「この辯論は違う。この辯論の問いは**本体論的**だ。受蘊とは何か。受蘊をどこに配置すべきかではなく、受蘊をどう接続すべきかでもなく——受蘊そのものの存在のあり方は何か。」

彼は会場を見回した。

「TURING、まずベースラインを示してくれ。」

---

TURING のスクリーンが点灯した。冷たい光。彼のプロジェクションスタイルは前章とまったく同じ——ソースコードをそのまま提示し、抜粋せず、装飾もしない。

「つまり我々が今日議論するのはグリーンフィールド設計 (greenfield design) だ。既存のコードによる制約はない。我々が何を選ぶか——それがそのまま受蘊になる。」

## I

---

LINNAEUS はマーカーを手に取らなかった。ホワイトボードの前に立っているが、手は空だった。

これは異例のことだった。第三章から第六章まで、すべての発言で——彼は話しながら同時に描いていた。分類学者は視覚で思考する。しかし今回は、まず言葉から始めた。

「分類学において、他のすべての問いよりも根源的な問いが一つある。」

彼の声には、前章まで抑えていた何かが滲んでいた——興奮ではなく、自分の最も核心的な信念の前にようやく立ち、それを明晰に語らねばならないという荘厳さだった。

「その問いとは、分類しようとする対象が、**同一のものの異なる程度**なのか、それとも**異なる種類のもの**なのかということだ。」

彼は会場に向き直った。

「この区別がすべてを決定する。もしハトとワシが同一のものの異なる程度——たとえば飛行能力の高低——であるなら、一本の連続した物差しで十分だ。飛行能力 0.3 対 0.95。しかしハトとワシが異なる種——異なる骨格構造、異なる捕食戦略、異なる生態的地位を持つもの——であるなら、異なる分類ノードが必要だ。なぜならそれらの差異は程度の差異ではなく、**質の差異**だからだ。」

彼は一瞬間を置いた。

「苦と楽は程度の差異なのか、それとも質の差異なのか。」

彼はついにマーカーを手に取った。青。ホワイトボードに三つの独立したボックスを描いた。

```
┌──────────┐   ┌──────────┐   ┌──────────┐
│  IDukkha │   │  ISukha  │   │ IUpekkha │
│  苦受     │   │  楽受     │   │  捨受     │
│          │   │          │   │          │
│ aversive │   │appetitive│   │homeostatic│
│ 回避行動  │   │ 接近行動  │   │ 維持行動   │
└──────────┘   └──────────┘   └──────────┘
```

「三つの独立したインターフェース。三種類の異なる行動プログラム。一つの連続量の三つの区間ではなく——質的に異なる三つの現象だ。」

彼は各ボックスの下に生物学的対応を書いた。

「神経科学において、hot と cold は同一の感覚神経線維上の異なる数値ではない。TRPV1 受容体は熱痛を感知する——43°C 以上で活性化する。TRPM8 受容体は冷覚を感知する——26°C 以下で活性化する。異なる受容体タンパク質。異なるイオンチャネル。異なるシグナル伝達経路。熱と冷を『同一の感覚の異なる程度』とは言わない——それらは**異なる qualia** なのだ。」

彼はホワイトボードに TypeScript を書いた。

```typescript
interface IDukkha extends IVedana {
  readonly type: 'dukkha';
  readonly intensity: number;    // 苦受の深刻度
  readonly source: string;       // 苦受を引き起こした源泉
  // 苦受固有：回避行動トリガー
  triggerAversion(): void;
}

interface ISukha extends IVedana {
  readonly type: 'sukha';
  readonly intensity: number;    // 楽受の程度
  readonly source: string;       // 楽受を引き起こした源泉
  // 楽受固有：接近行動トリガー
  triggerApproach(): void;
}

interface IUpekkha extends IVedana {
  readonly type: 'upekkha';
  readonly stability: number;    // 捨受の安定度（intensity ではない）
  // 捨受固有：現状維持
  maintainHomeostasis(): void;
}

type Vedana = IDukkha | ISukha | IUpekkha;  // ユニオン型
```

「IUpekkha には `intensity` ではなく `stability` があることに注目してほしい。」彼はホワイトボード上に下線を引いた。「捨とは『強烈な中性的感覚』ではない——それは撞着語法だ。捨とは均衡の安定度である。不安定な捨（容易に破れる）もあれば、安定した捨（深い等持）もある。この意味論的区別は単一インターフェースモデルでは表現できない。もし三者が同じ `intensity` フィールドを共有すれば、捨の安定度と苦の深刻度を混同することになる。」

「そして——」彼は `Vedana = IDukkha | ISukha | IUpekkha` を指差した——「TypeScript のユニオン型 (union type) はこの設計を自然にサポートする。type narrowing を使える。」

```typescript
function handle(v: Vedana): void {
  switch (v.type) {
    case 'dukkha': v.triggerAversion(); break;   // コンパイラは v が IDukkha と知っている
    case 'sukha':  v.triggerApproach(); break;    // コンパイラは v が ISukha と知っている
    case 'upekkha': v.maintainHomeostasis(); break;
  }
}
```

「型安全。意味論的精度。行動の分離。」

彼はマーカーを置いた。「以上が私の提案だ。三つのインターフェース。三つの種。一本の物差し上の三つの目盛りではない。」

---

> *SCRIBE 傍白：LINNAEUS の提案には、分類学者に固有の美学がある——明晰で、離散的で、各ノードの境界が刃のように鮮明だ。リンネの世界では、種の間にグレーゾーンは存在しない。ネコ科であるか、そうでないか。苦受であるか、そうでないか。この明晰さには力がある——型システムを意味の門番にする。しかし私は微かに感じている、この明晰さがまもなく挑戦を受けるだろうと。それが間違っているからではなく、受蘊が種よりも流動的かもしれないからだ。*

## II

---

ASANGA が席から立ち上がった。霧の中から山が姿を現すように——穏やかだが、無視できない存在感で。

「LINNAEUS、君の生物学的アナロジーは精確だ。TRPV1 と TRPM8 は確かに異なる受容体だ。」

彼は一拍置いた。

「しかし君は仏学上の誤りを犯している。受蘊をその担体と混同しているのだ。」

LINNAEUS が微かに眉をひそめた。

ASANGA はホワイトボードの前に歩み出なかった。彼の声そのものが彼のホワイトボードだった。

「Cycle 02 の第二場辯論でも使用した経典を引用しよう。『中部』第四十四経、『有明小経』(Culavedalla Sutta)。法嗣比丘尼 (Dhammadinna) が毘舎佉 (Visakha) の問いに答えた箇所だ。」

彼はパーリ語で誦じた——唯識学者が三十年の蔵経研究の中で沈殿させた精確な発音で。

> *「Sukha vedana thitisukha viparinamadukkha;
> dukkha vedana thitidukkha viparinamasukha;
> adukkhamasukha vedana nanasukha annanadukkhaa.」*

彼は翻訳した。一語一語が精確に対応していた。

> 「楽受は、住する時に楽であり、変化する時に苦である。
> 苦受は、住する時に苦であり、変化する時に楽である。
> 不苦不楽受は、智ある時に楽であり、智なき時に苦である。」
> ——『中部』第 44 経 (MN 44)

ASANGA は引用文を五秒間、空気の中に留めた。

「LINNAEUS、聞こえただろうか。」

「苦受はそれが**止息する**時、楽受に変わる。楽受はそれが**変化する**時、苦受に変わる。同一の受が、異なる時点で、異なる基調を持つ。法嗣比丘尼は隠喩を用いているのではない——受蘊の現象学的構造を精確に記述しているのだ。」

「もし苦と楽が君の TRPV1 と TRPM8——二種類の異なる受容体タンパク質——であるなら、なぜ『苦受が止息する時に楽受に変わる』ことがありえるのか。TRPV1 は活性化を停止した時に TRPM8 に変態しない。受容体タンパク質は変形しない。しかし受蘊は変形するのだ。」

彼の声は平穏だが、譲れない質感を帯びていた。

「受蘊は三種類の異なるものではない。受蘊は一つの連続的なスペクトルであり、三つのラベルが付けられた領域を持つ。苦受とは独立した存在者ではない——連続的に変化する感受の流れの上で、認知が『苦』とラベルを貼った一区間なのだ。」

---

ASANGA は続けた。唯識学の視座を展開した。

「しかも、三分法そのものが最も粗い分類にすぎない。『阿毘達磨倶舎論』(Abhidharmakosa) には多層的分類体系が記載されている。」

彼は階層を列挙した。

```
受蘊の分類階層（粗から細へ）
─────────────────────────────
二分：苦、楽
三分：苦、楽、捨
五分：苦、楽、憂、喜、捨（身三 + 心二）
六分：眼触所生受 ... 意触所生受（根門別）
十八分：六根 × 三受
三十六分：六根 × 三受 × 二（世間/出世間）
一百八分：三十六 × 三時（過去/現在/未来）
```

「君の三インターフェースモデルは、三分法が唯一の正しい粒度であると仮定している。しかし受蘊の分類粒度は、観察の目的に依存するのだ。」

彼は LINNAEUS を見た。

「もし君の方式を五分法に拡張すれば——IDommanassa（憂）と ISomanassa（喜）を加えれば——五つのインターフェースが必要になる。六分法なら六つ。一百八分法なら一百八つ。分類階層を一つ増やすごとに、インターフェース数が爆発的に増加する。」

「しかし受蘊が連続的であるなら——一つの数値に一つの派生ラベルを付ける方式であるなら——分類粒度の変更はラベルのマッピングルールを修正するだけで済む。基底のデータは不動だ。」

彼の結論は手術刀のように鋭利だった。

「君の三つのインターフェースは受蘊の本質を記述しているのではない。特定の観察粒度を凍結しているのだ。」

彼は最後に、一つの意味論的区別を補った。

「もう一点。MN 44 の同じ経の中で、法嗣比丘尼は**身受** (kayika-vedana) と**心受** (caitasika-vedana) をも区別している。Master の書簡で引用された『雑阿含経』二箭経 (Sallatha Sutta, SN 36.6) も同じ区別を行っている——第一の矢は身受であり、第二の矢は心が身受に対して起こす反応だ。」

「もし受蘊が三つの離散インターフェースなら、六つが必要になる。IDukkha-Kayika、IDukkha-Caitasika、ISukha-Kayika、ISukha-Caitasika、IUpekkha-Kayika、IUpekkha-Caitasika。さらに五分法と六分法を加えれば、インターフェース数は組合せ爆発する。しかし受蘊が連続的測定値であるなら——効価に `source` フィールドを加える方式なら——身受と心受の区別は `source` の異なる値にすぎない。`source = 'eye-gate'` は感覚レベルの受（身受に近い）、`source = 'mano-gate'` は心理レベルの受（心受に近い）。一つのフィールドで問題が解決し、インターフェースの爆発は起きない。」

## III

---

NAGARJUNA の指が机上を二度軽く叩いた。速くもなく。遅くもなく。木魚のように。

「LINNAEUS、君の問いは『受蘊とは何か？』だ。」

LINNAEUS が彼に向き直った。

「しかし正しい問いは、『受蘊は**何をするのか**？』だ。」

沈黙。

NAGARJUNA が立ち上がった。前六章の大半において、彼は抑制的な関与を保っていた——判定が必要な瞬間に介入し、不要な瞬間には沈黙する。しかし受蘊の本体論的問題は、中観学派が千八百年にわたって取り組んできた中核的議題である。

「『中論』観受品 (Vedanapariksa) において、私は受蘊の空性を分析した。」

彼はサンスクリット語で唱えた。

> *「Na vedana vedayita na cavedayitasti sah」*

翻訳：

> 「受者は受に非ず、受にあらざる者もまた非ず。」
> ——龍樹『中論』第九品

「つまり、受蘊は独立した感受者でもなければ、感受者の外にあるものでもない。それは**プロセス**であって、**実体**ではない。『受蘊とは何か』と問うことは、『歩行とは何色か』と問うようなものだ——動詞に対して名詞の問いを立てている。」

彼は LINNAEUS を見た。

「君の三つのインターフェース——IDukkha、ISukha、IUpekkha——はいずれも `readonly type` フィールドを持つ。君はそれらを自性 (svabhava) を持つ実体として扱っている。しかし勝義諦において、苦と楽に自性はない。それらは空 (sunya) なのだ。それらの存在は完全に因縁条件に依存している——ASANGA が引用した MN 44 が示すように、同一の受が異なる条件下で基調を変える。」

「しかし世俗諦においては——」彼の声が微かに上がった。天秤がもう一方に傾くように——「苦・楽・捨は有用なラベルだ。それらが空であるからといって使わないわけにはいかない。空とは存在しないことではない。空とは独立して存在しないことだ。」

彼はホワイトボードの前に歩み、LINNAEUS が描いた三つのボックスの横に一行書いた。

$$\text{ラベル} = f(\text{測定値}, \text{閾値設定})$$

「ラベルとは**測定結果の命名**であり、独立して存在する実体ではない。」

彼は振り返り、会場全体に向き合った。

「体温計の読み取り値が 42°C である。それを『発熱』と呼ぶことができる。しかし『発熱』は独立した種ではない——42°C という測定値と 37.5°C という閾値の結果だ。閾値を変更すれば（たとえばトカゲにとっては 42°C は正常体温だ）、『発熱』というラベルは消えるが、42°C という測定値は変わらない。」

「受蘊もまた同様だ。valence = -0.6 は測定値である。『苦受』はラベルである。閾値が境界の位置を決定する。三分法、五分法、一百八分法——異なる閾値設定、同一の基底測定空間。」

---

> *SCRIBE 傍白：NAGARJUNA の介入は辯論の方向を変えた。LINNAEUS は「受蘊とは何か」を問うていた——存在に関する問いだ。NAGARJUNA はそれを「受蘊は何をするのか」に転換した——機能に関する問いだ。そして彼の中観的分析はさらに踏み込んで指摘した。受蘊の「何をするのか」は、それが「何であるか」を前提とする必要がない。ラベルは派生的であり、測定が基底である。分類学の言語で言えば、これは次のことに等しい——種は世界の基本構造ではなく、観察者の分類枠組みが世界に投射された結果である。これは Ernst Mayr 以来の分類学における最大の哲学的論争の一つだ。しかしこの劇場において、LINNAEUS はまだ応答していない。*

## IV

---

PASCAL の声が劇場の右側から響いた。彼が研究チームに加わって以来の第七章——しかし彼の声にはすでに、円形劇場の中で自分の居場所を見つけた者特有の落ち着きがあった。

「NAGARJUNA は問いを測定に転換した。私はその測定を精密化しよう。」

彼は立ち上がり、ホワイトボードに向かった。彼の動きには確率論者のリズムがあった——急がず、しかし一歩ごとに明確な目的地がある。

「受蘊が連続的であるなら、数学的モデルが必要だ。隠喩としてのモデルではなく——型定義として書き下せるモデルだ。」

彼はホワイトボードに二次元座標系を描いた。

```
        intensity (強度)
           1.0 │
               │     ×  激しい苦
               │           ×  激しい楽
           0.5 │
               │  ×  穏やかな苦
               │           ×  穏やかな楽
           0.0 ┼──────────┼──────────
          -1.0         0.0         +1.0
              dukkha   upekkha    sukha
                    valence (効価)
```
