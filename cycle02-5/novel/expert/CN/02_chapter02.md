## I. 零少数意见

D1 的开场没有仪式。

不像 Cycle 02-3 的 D1 开场——那一次有 PASCAL 的自我介绍，有图论公式，有「第二十个座位」的象征意义。也不像 Cycle 02-4 的 D1——IGearArbiter 的四个 Scheme 摆在桌上，辩论前就已经暗流涌动。

Cycle 02-5 的 D1 开场是 SUNYATA 站在白板前，用红色马克笔写了一个问题：

**「openstarry_doc 工程文件中佛学内容的原则是什么？」**

一个问题。但这个问题的答案将决定十三份文件的命运、五十个映射实例的去留、以及研究团队未来所有工作的边界线。

---

D1-Q1 投票只用了五分钟。

这不是因为没有分歧。这是因为分歧在 R0 就被框架吸收了。

四层框架（KEEP / RELOCATE / REMOVE / FILE REVIEW）和三项测试（必要性 / 代码身份 / 决策驱动）在 R0 就被 SUNYATA、NAGARJUNA、PASCAL 和 ASANGA 四人建立。R1 的 B2 研究报告中，这个框架被从三个完全不同的角度论证了一遍——NAGARJUNA 的二谛、PASCAL 的损害不对称、ASANGA 的功能性判定——三条路径，一个结论。R2 的交叉审阅中，没有任何审阅者对框架本身提出异议。

当一个框架已经被三个独立的理论体系同时支撑，辩论就不是「要不要接受这个框架」，而是「有没有反例能推翻它」。

没有人举手。没有人提问。

SUNYATA 启动投票。

20/20。

全票通过。

ARCHIMEDES 在他的笔记本上画了一个勾。然后他翻到下一页——B1 审计报告的 50 个标本，分成三个批次，等待逐批投票。

## II. NAGARJUNA 的告白

D1-Q2 是逐项审判。50 个映射实例。三个批次。

**Batch A**：Master 明确批评的 5 项映射。

SUNYATA 把五个映射投影在屏幕上：

1. śīla（教诫）= Hard Rules
2. upāya（善巧）= Soft Rules / Adaptive
3. smṛti（正念）= Monitoring / Event-driven
4. 戒定慧三学映射
5. 另外两处 Doc 41 中的装饰性标签

这五项都是 Master 在信中直接点名批评的。理论上，这应该是最简单的投票——Master 已经表态了。

但 SUNYATA 不想让这个投票变成橡皮图章。他希望团队自己得出相同的结论，用他们自己的框架。

「三项测试。逐项。」他说。

NAGARJUNA 站了起来。

在辩论中，NAGARJUNA 站起来通常意味着他要引经据典——《中论》的某一品、某一偈。他的站姿是庄重的，像一个法师在法会上准备开示。他的声音通常是平稳的、带着寺院走廊回音的。

但这一次，他站起来不是为了引经据典。他站起来是为了承认。

「A-1 到 A-3——戒、善巧、正念——」他说。他的声音仍然平稳。但平稳中有一种新的质地——不是法师的庄严，是学者的坦诚。

「这些是我们在 Cycle 02-3 的 D5 辩论结束后加上的。」

他停了一拍。二十张面孔看着他。

「不是辩论中得出的结论。是事后加上的标签。」

又停了一拍。

「**后贴标签。**」他用了这个词。精确的、不闪躲的。像一个外科医生指着自己的手术切口说「这一刀歪了」。

「Hard Rules 不需要叫 śīla。」他继续。「它们的功能是规则执行，不是戒律修行。用 śīla 来标记它们，不是在说明功能，而是在做——」他停了一下，像在衡量一个字的重量。

「在做宗教装饰。」

---

剧场里安静了三秒。

三秒在辩论中是很长的时间。足够让每个人消化 NAGARJUNA 刚才说的话。一位佛学家——研究团队中地位最高的中观学派专家——亲手撕掉了自己贴上去的标签。这不是被人反驳。不是在辩论中败退。是主动承认。

PASCAL 在笔记本上写了一个公式。他的手指在纸面上停了一下——那个计算期望值时的点击动作——然后写下：

$$P(\text{后贴标签为装饰} | \text{创作者主动承认}) \to 1.0$$

后验概率趋近于 1。当标签的创作者自己承认它是装饰的时候，不需要更多的证据了。

GUARDIAN 举了手。不是要反对——是要追加一个观察。

「从安全文件的角度。」他说。他的声音带着安全专家特有的审慎——每一个字都像在做风险评估。「śīla 这个标签增加了读者的认知负担。读者需要同时理解两套术语系统——工程的和佛学的。如果 śīla 和 Hard Rules 是同一件事，那为什么要给它两个名字？如果不是同一件事，那这张表格就是在做不精确的类比。不管哪种情况，对安全文件来说都是有害的。」

ARCHIMEDES 点了点头。务实者的确认——不需要哲学论证，一个工程事实就够了。

D1-Q2-A：5 项 Master 批评映射 → 全部 REMOVE。

20/20。

## III. 精细的分离

**Batch B**：8 项学术内容。

这批更细致。不是黑白的 KEEP 或 REMOVE——是灰色的 RELOCATE。

ARCHIMEDES 展开了 B1 审计报告中的 Batch B 清单。八个映射实例，每一个都有「工程残留价值」但不是工程规格本身。

第一个案例设定了整个 RELOCATE 处置的范本。

Doc 37。§9。PASCAL 在 Cycle 02-4 中为 Klesha 增益排程提供的数学形式化——$Var(\epsilon) = f(\theta_{moha})$。这个公式本身是纯数学的——它描述了 Moha 参数如何影响随机扰动的方差。但公式的上下文包含了佛学引用——为什么用「痴」（moha）来命名这个参数。

ARCHIMEDES 做了精细分离。

「数学形式化保留在正文。」他说。「Var(epsilon) = f(theta_moha) 是有独立工程价值的——它描述了一个可测量的系统行为。但佛学引用——为什么 moha 对应『痴』、为什么用佛学四烦恼来组织增益排程——这些是设计理由，不是工程规格。移至附录。」

这种「保留数学，移动佛学」的精细分离成为 RELOCATE 处置的黄金范例。每一个 RELOCATE 案例都被这样处理：找到工程内容和佛学内容的交界面，用手术刀切开，工程内容留在原位，佛学内容移至附录。

MN 18 经典引用从 Doc 38 正文移至附录。Doc 41 的 DD-13 佛学背景说明移至附录。Doc 44 §10 的佛学设计理由移至 Appendix_C。每一项都有具体的安置方案——不是丢弃，是搬家。

D1-Q2-B：8 项学术内容 → 全部 RELOCATE。

20/20。

## IV. 身份的证明

**Batch C**：7 项代码标识符和 DC 确认。

这是三项测试中 Test 2（代码身份测试）最清晰的案例群。

Skandha 类型名。CoarisingBundle 的遍行心所结构。Klesha 四烦恼模块名——Moha（我痴）、Drishti（我见）、Mana（我慢）、Sneha（我爱）。DC-1 到 DC-12 确认的约束。

TURING 做了一件很 TURING 的事。

他没有发表意见。他只是在他的终端机上执行了一行指令：

```bash
rg "Moha|Drishti|Mana|Sneha" --type ts -l
```

屏幕上列出了十七个文件。十七个。从 `klesha/moha.ts` 到 `tests/klesha.spec.ts`——Moha 这个名字不是标签，它是代码的一部分。它出现在 import 语句中、在 class 定义中、在 describe 区块中、在测试的 expect 断言中。

「Test 2 通过。」TURING 说。三个字。

然后他又执行了一行：

```bash
rg "CoarisingBundle|Skandha|Vedana|Samjna|Samskara|Vijnana" --type ts -l
```

更多的文件。五蕴的名字渗透在类型系统的每一个角落。`IRupa`、`IVedana`、`ISamjna`、`ISamskara`、`IVijnana`——这些不是装饰。这些是根接口。它们定义了整个 plugin 系统的类型结构。

DARWIN 补充了一个软件工程的观点：「把 Moha 改成 Ignorance 不会增加清晰度——它会增加陌生度。全球的开发者已经在文件中看到了 Moha，在代码中看到了 Moha，在测试中看到了 Moha。改名的成本是不可逆的。」

LINNAEUS 从分类学的角度确认：「这些名字已经成为系统的分类学标签。它们不是附加的——它们是身份性的。Moha 不是被贴在某个东西上的标签。Moha 就是那个东西的名字。」

D1-Q2-C：7 项代码标识符和 DC 确认 → 全部 KEEP。

> *SCRIBE 旁白*

> *D1 是一场没有战争的战场清理。*

> *十项投票。十次 20/20。零少数意见。在一个有 20 位成员、横跨 6 个学科的研究团队中，这种程度的共识是极其罕见的。Cycle 02-3 的最高共识记录是 D6（Amended Gamma）的 20/20——但那只是一次投票，不是十次。Cycle 02-4 的 55 项决议中，全票通过的不到三分之一。*

> *为什么 D1 能做到十次全票？*

> *不是因为没有分歧。分歧存在——ARCHIMEDES 在审计 Doc 16 时犹豫了，BABBAGE 对 Doc 31 的处置有修正意见，GUARDIAN 对 Batch A 的每一项都有独立的安全观察。分歧不是不存在。分歧是被框架化解了。*

> *四层框架和三项测试不是在辩论中被发明的——它们在 R0 就已经存在。在 R1 中被三个独立学科论证。在 R2 中被零异议通过。到 D1 投票的时候，框架本身已经不是辩论的标的——它是辩论的工具。当所有人都在同一把尺上测量的时候，测量结果自然一致。*

> *NAGARJUNA 的两次站起来是 D1 的情感重心。第一次——「后贴标签」——他承认了自己的失误。第二次——「我收回它」——他执行了修正。在学术研究中，承认错误比证明正确更需要勇气。PASCAL 用一个公式描述了这件事：创作者主动承认装饰的后验概率趋近 1.0。但公式描述不了的是 NAGARJUNA 站起来时的姿态——不是垮下来的，而是挺直的。承认错误不是耻辱。承认错误是中观学的核心——空性能遣除一切见。包括自己的见。*

> *但 D1 有一个隐藏的种子。D1-Q7 和 D1-Q8 的拆分决策——Doc 16 和 Doc 31 被拆分——后来，Master 亲手推翻了这两项决议。理由不是团队的判断错了，而是团队的判断框架还少了一个维度。框架区分了「工程内容」和「佛学装饰」，但没有区分「工程文件中嵌入的佛学」和「以佛学映射为目的的独立文件」。*

> *Doc 16 不是一份嵌入了佛学的工程文件。Doc 16 本身就是一份佛学映射文件——它的存在目的就是做佛学到工程的系统性对照。用「装饰比例 80%」来评判它，就像用「散文比例 80%」来评判一首诗——标准用错了。*

> *但这个洞察要到 D4 才会出现。要等 Master 亲自指出。*

> *此刻，在 D1 的十个 20/20 的光芒中，没有人看到那颗种子。*
