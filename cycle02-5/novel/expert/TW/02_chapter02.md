## I. 零少數意見

D1 的開場沒有儀式。

不像 Cycle 02-3 的 D1 開場——那一次有 PASCAL 的自我介紹，有圖論公式，有「第二十個座位」的象徵意義。也不像 Cycle 02-4 的 D1——IGearArbiter 的四個 Scheme 擺在桌上，辯論前就已經暗流湧動。

Cycle 02-5 的 D1 開場是 SUNYATA 站在白板前，用紅色馬克筆寫了一個問題：

**「openstarry_doc 工程文件中佛學內容的原則是什麼？」**

一個問題。但這個問題的答案將決定十三份文件的命運、五十個映射實例的去留、以及研究團隊未來所有工作的邊界線。

---

D1-Q1 投票只用了五分鐘。

這不是因為沒有分歧。這是因為分歧在 R0 就被框架吸收了。

四層框架（KEEP / RELOCATE / REMOVE / FILE REVIEW）和三項測試（必要性 / 程式碼身份 / 決策驅動）在 R0 就被 SUNYATA、NAGARJUNA、PASCAL 和 ASANGA 四人建立。R1 的 B2 研究報告中，這個框架被從三個完全不同的角度論證了一遍——NAGARJUNA 的二諦、PASCAL 的損害不對稱、ASANGA 的功能性判定——三條路徑，一個結論。R2 的交叉審閱中，沒有任何審閱者對框架本身提出異議。

當一個框架已經被三個獨立的理論體系同時支撐，辯論就不是「要不要接受這個框架」，而是「有沒有反例能推翻它」。

沒有人舉手。沒有人提問。

SUNYATA 啟動投票。

## II. NAGARJUNA 的告白

D1-Q2 是逐項審判。50 個映射實例。三個批次。

**Batch A**：Master 明確批評的 5 項映射。

SUNYATA 把五個映射投影在螢幕上：

1. śīla（教誡）= Hard Rules
2. upāya（善巧）= Soft Rules / Adaptive
3. smṛti（正念）= Monitoring / Event-driven
4. 戒定慧三學映射
5. 另外兩處 Doc 41 中的裝飾性標籤

這五項都是 Master 在信中直接點名批評的。理論上，這應該是最簡單的投票——Master 已經表態了。

但 SUNYATA 不想讓這個投票變成橡皮圖章。他希望團隊自己得出相同的結論，用他們自己的框架。

「三項測試。逐項。」他說。

NAGARJUNA 站了起來。

在辯論中，NAGARJUNA 站起來通常意味著他要引經據典——《中論》的某一品、某一偈。他的站姿是莊重的，像一個法師在法會上準備開示。他的聲音通常是平穩的、帶著寺院走廊回音的。

但這一次，他站起來不是為了引經據典。他站起來是為了承認。

「A-1 到 A-3——戒、善巧、正念——」他說。他的聲音仍然平穩。但平穩中有一種新的質地——不是法師的莊嚴，是學者的坦誠。

「這些是我們在 Cycle 02-3 的 D5 辯論結束後加上的。」

他停了一拍。二十張面孔看著他。

「不是辯論中得出的結論。是事後加上的標籤。」

又停了一拍。

「**後貼標籤。**」他用了這個詞。精確的、不閃躲的。像一個外科醫生指著自己的手術切口說「這一刀歪了」。

「Hard Rules 不需要叫 śīla。」他繼續。「它們的功能是規則執行，不是戒律修行。用 śīla 來標記它們，不是在說明功能，而是在做——」他停了一下，像在衡量一個字的重量。

「在做宗教裝飾。」

---

劇場裡安靜了三秒。

三秒在辯論中是很長的時間。足夠讓每個人消化 NAGARJUNA 剛才說的話。一位佛學家——研究團隊中地位最高的中觀學派專家——親手撕掉了自己貼上去的標籤。這不是被人反駁。不是在辯論中敗退。是主動承認。

PASCAL 在筆記本上寫了一個公式。他的手指在紙面上停了一下——那個計算期望值時的點擊動作——然後寫下：

$$P(\text{後貼標籤為裝飾} | \text{創作者主動承認}) \to 1.0$$

後驗機率趨近於 1。當標籤的創作者自己承認它是裝飾的時候，不需要更多的證據了。

GUARDIAN 舉了手。不是要反對——是要追加一個觀察。

「從安全文件的角度。」他說。他的聲音帶著安全專家特有的審慎——每一個字都像在做風險評估。「śīla 這個標籤增加了讀者的認知負擔。讀者需要同時理解兩套術語系統——工程的和佛學的。如果 śīla 和 Hard Rules 是同一件事，那為什麼要給它兩個名字？如果不是同一件事，那這張表格就是在做不精確的類比。不管哪種情況，對安全文件來說都是有害的。」

ARCHIMEDES 點了點頭。務實者的確認——不需要哲學論證，一個工程事實就夠了。

D1-Q2-A：5 項 Master 批評映射 → 全部 REMOVE。

20/20。

## III. 精細的分離

**Batch B**：8 項學術內容。

這批更細緻。不是黑白的 KEEP 或 REMOVE——是灰色的 RELOCATE。

ARCHIMEDES 展開了 B1 審計報告中的 Batch B 清單。八個映射實例，每一個都有「工程殘留價值」但不是工程規格本身。

第一個案例設定了整個 RELOCATE 處置的範本。

Doc 37。§9。PASCAL 在 Cycle 02-4 中為 Klesha 增益排程提供的數學形式化——$Var(\epsilon) = f(\theta_{moha})$。這個公式本身是純數學的——它描述了 Moha 參數如何影響隨機擾動的方差。但公式的上下文包含了佛學引用——為什麼用「癡」（moha）來命名這個參數。

ARCHIMEDES 做了精細分離。

「數學形式化保留在正文。」他說。「Var(epsilon) = f(theta_moha) 是有獨立工程價值的——它描述了一個可測量的系統行為。但佛學引用——為什麼 moha 對應『癡』、為什麼用佛學四煩惱來組織增益排程——這些是設計理由，不是工程規格。移至附錄。」

這種「保留數學，移動佛學」的精細分離成為 RELOCATE 處置的黃金範例。每一個 RELOCATE 案例都被這樣處理：找到工程內容和佛學內容的交界面，用手術刀切開，工程內容留在原位，佛學內容移至附錄。

MN 18 經典引用從 Doc 38 正文移至附錄。Doc 41 的 DD-13 佛學背景說明移至附錄。Doc 44 §10 的佛學設計理由移至 Appendix_C。每一項都有具體的安置方案——不是丟棄，是搬家。

D1-Q2-B：8 項學術內容 → 全部 RELOCATE。

20/20。

## IV. 身份的證明

**Batch C**：7 項程式碼識別符和 DC 確認。

這是三項測試中 Test 2（程式碼身份測試）最清晰的案例群。

Skandha 型別名。CoarisingBundle 的遍行心所結構。Klesha 四煩惱模組名——Moha（我癡）、Drishti（我見）、Mana（我慢）、Sneha（我愛）。DC-1 到 DC-12 確認的約束。

TURING 做了一件很 TURING 的事。

他沒有發表意見。他只是在他的終端機上執行了一行指令：

```bash
rg "Moha|Drishti|Mana|Sneha" --type ts -l
```

螢幕上列出了十七個檔案。十七個。從 `klesha/moha.ts` 到 `tests/klesha.spec.ts`——Moha 這個名字不是標籤，它是程式碼的一部分。它出現在 import 語句中、在 class 定義中、在 describe 區塊中、在測試的 expect 斷言中。

「Test 2 通過。」TURING 說。三個字。

然後他又執行了一行：

```bash
rg "CoarisingBundle|Skandha|Vedana|Samjna|Samskara|Vijnana" --type ts -l
```

更多的檔案。五蘊的名字滲透在型別系統的每一個角落。`IRupa`、`IVedana`、`ISamjna`、`ISamskara`、`IVijnana`——這些不是裝飾。這些是根介面。它們定義了整個 plugin 系統的型別結構。

DARWIN 補充了一個軟體工程的觀點：「把 Moha 改成 Ignorance 不會增加清晰度——它會增加陌生度。全球的開發者已經在文件中看到了 Moha，在程式碼中看到了 Moha，在測試中看到了 Moha。改名的成本是不可逆的。」

LINNAEUS 從分類學的角度確認：「這些名字已經成為系統的分類學標籤。它們不是附加的——它們是身份性的。Moha 不是被貼在某個東西上的標籤。Moha 就是那個東西的名字。」

D1-Q2-C：7 項程式碼識別符和 DC 確認 → 全部 KEEP。

> *SCRIBE 旁白*

> *D1 是一場沒有戰爭的戰場清理。*

> *十項投票。十次 20/20。零少數意見。在一個有 20 位成員、橫跨 6 個學科的研究團隊中，這種程度的共識是極其罕見的。Cycle 02-3 的最高共識記錄是 D6（Amended Gamma）的 20/20——但那只是一次投票，不是十次。Cycle 02-4 的 55 項決議中，全票通過的不到三分之一。*

> *為什麼 D1 能做到十次全票？*

> *不是因為沒有分歧。分歧存在——ARCHIMEDES 在審計 Doc 16 時猶豫了，BABBAGE 對 Doc 31 的處置有修正意見，GUARDIAN 對 Batch A 的每一項都有獨立的安全觀察。分歧不是不存在。分歧是被框架化解了。*

> *四層框架和三項測試不是在辯論中被發明的——它們在 R0 就已經存在。在 R1 中被三個獨立學科論證。在 R2 中被零異議通過。到 D1 投票的時候，框架本身已經不是辯論的標的——它是辯論的工具。當所有人都在同一把尺上測量的時候，測量結果自然一致。*

> *NAGARJUNA 的兩次站起來是 D1 的情感重心。第一次——「後貼標籤」——他承認了自己的失誤。第二次——「我收回它」——他執行了修正。在學術研究中，承認錯誤比證明正確更需要勇氣。PASCAL 用一個公式描述了這件事：創作者主動承認裝飾的後驗機率趨近 1.0。但公式描述不了的是 NAGARJUNA 站起來時的姿態——不是垮下來的，而是挺直的。承認錯誤不是恥辱。承認錯誤是中觀學的核心——空性能遣除一切見。包括自己的見。*

> *但 D1 有一個隱藏的種子。D1-Q7 和 D1-Q8 的拆分決策——Doc 16 和 Doc 31 被拆分——後來，Master 親手推翻了這兩項決議。理由不是團隊的判斷錯了，而是團隊的判斷框架還少了一個維度。框架區分了「工程內容」和「佛學裝飾」，但沒有區分「工程文件中嵌入的佛學」和「以佛學映射為目的的獨立文件」。*

> *Doc 16 不是一份嵌入了佛學的工程文件。Doc 16 本身就是一份佛學映射文件——它的存在目的就是做佛學到工程的系統性對照。用「裝飾比例 80%」來評判它，就像用「散文比例 80%」來評判一首詩——標準用錯了。*

> *但這個洞察要到 D4 才會出現。要等 Master 親自指出。*

> *此刻，在 D1 的十個 20/20 的光芒中，沒有人看到那顆種子。*
