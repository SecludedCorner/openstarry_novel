# 第五章：裁定

---

D3 終了から四十分後。

円形劇場の冷白色の照明の下、二十名の研究者が奇妙なことをしていた——何もしていなかった。

D1 から D3 の間のすべてのインターバルは埋め尽くされていた：交差査読、データ照合、スライドの準備、コーヒー。だが D3 の後の空白は違った。休憩ではない。待機だ。なぜなら全員が次に何が起こるかを知っていたから——Master の審閲。

前の数回の Cycle では、Master の介入は通常、特定のタイミングで起こった：研究開始前（手紙）、ある決議が論争を引き起こした後（DC-1 から DC-12）、あるいは全回の終了時（ロードマップの提案）。だが Cycle 02-5 は違う。Master は手紙で何を見たいか明確に述べていた——清掃の結果だ。抽象的原則ではなく——具体的な実行を見たいのだ。

---

## I. 審閲

Master の審閲は書面の報告ではなかった。

より直接的な形で現れた。SUNYATA が端末で一つの文章を目にし、プロジェクターを点けて全員に見せた。

審閲は長くない。だが手術刀のように精確だ。

Master はまず D1 の決議の大部分を確認した——四層フレームワーク、三つのテスト、Batch A の五項目の REMOVE、Batch B の八項目の RELOCATE、Batch C の七項目の KEEP、Doc 38 の表のクリーンアップ、Doc 44 の三学対応づけの除去、Klesha モジュール名の保持。

「これらはすべて正しい。」

そして二つの名前に触れた：Doc 16 と Doc 31。

## II. 覆す

「D1-Q7 と D1-Q8、」SUNYATA は Master の原文を読み上げた。「この二つの決議には同意しない。」

劇場の空気が凝固した。

D1-Q7：Doc 16 の分割——Section 5 を独立文書として抽出し、残りは削除。20/20 で通過。

D1-Q8：Doc 31 の分割格下げ——Section 3.1 を Doc 35 に統合、Section 6.3 を独立短文書に、残りは Appendix_B に格下げ。20/20 で通過。

二つの 20/20。二つの全票。本回の歴史上最もクリーンな討論で完璧なコンセンサスをもって通過した。

Master がそれらを覆した。

---

SUNYATA は続けて読んだ：

「Doc 16 は仏教が埋め込まれた工学文書ではない。Doc 16 そのものが仏教的対応づけ文書だ——その存在目的が仏教から工学への体系的対照を行うことにある。あなたたちの三つのテストは正しく、四層フレームワークも正しい。だが Doc 16 に対して基準の使い方を誤った。」

ARCHIMEDES は監査表に目を落とした。B1 で Doc 16 に「約 80% が装飾的」とマークしていた。80%——数えた。一行一行数えた。だが数字が正しいことは結論が正しいことを意味しない。

「Doc 31 も同様だ、」Master は続けた。「八識運作モデルは仏教概念を主体とする分析的文書だ。工学文書内の装飾ではない——仏教参考文書そのものだ。二つの状況を区別する必要がある：第一は工学文書に埋め込まれた仏教的装飾——清掃すべきだ。第二は仏教的対応づけを目的とする独立文書——工学的参照価値があり、保持すべきだ。」

---

NAGARJUNA が最初に立ち上がった。

「Master は我々のフレームワークの盲点を指摘された。」彼は言った。声に震えはなかった。中観学者が自分の分析フレームワークに欠陥があると告げられること——それは彼の伝統では恥辱ではなく日常だ。龍樹その人の全哲学は「一切の見は遣除されうる」の上に築かれている——自分の見を含めて。

「三つのテストが問うのは：対応づけは工学仕様を変えるか？ コード識別子か？ 設計決定を駆動したか？ だがこの三つの問いには一つの前提がある——審査される文書が工学文書であること。文書そのものが仏教的対応づけ文書であるなら、工学文書の基準でそれを計ることは——」

「散文比率で詩を判断するようなものだ。」SCRIBE は記録に書いた。

---

## III. 次元

PASCAL はノートに図を描いた。

三つのテストは三つの軸だ——Necessity、Code Identity、Decision Driver。各文書はこの三つの軸上に座標を持つ。D1 の判断はこの三次元空間に基づいていた。

だが Master が指摘したのは四番目の軸：**文書の存在目的**。

Doc 38 は工学文書（IVolition のアーキテクチャ設計）であり、その中に仏教的対応づけが埋め込まれている。三つのテストは適用可能。結論：埋め込まれた装飾を清掃し、設計を駆動した概念は保持する。

Doc 16 は仏教的対応づけ文書（Plugin 組み合わせパターンの仏教的対照）であり、その存在目的が分野横断的参照を提供することにある。三つのテストは適用不可——「工学文書内の仏教的装飾」ではなく、「仏教的対応づけそのものが文書の内容」だからだ。

違いは：装飾と内容。

壁の壁画は装飾だ——壁の構造に影響を与えずに除去できる。だが美術館に掛けられた絵画は壁の装飾ではない——絵画そのものが内容だ。除去するのは「清掃」ではなく「撤去」だ。

「Doc 16 と Doc 31 は美術館に掛けられた絵画だ。」PASCAL は言った。「工学図面に貼られた壁画ではない。」

---

ARCHIMEDES は監査表を開き直した。

「B1 監査の前提仮定は：監査対象のすべての文書は工学文書であり、仏教的装飾が埋め込まれている可能性がある。この前提に従って逐行監査し、装飾比率をマークした。Doc 16 の 80% と Doc 31 の 70%——数字は正しい。だが前提が誤りだった。」

彼は監査表に新しいフィールドを追加した：**文書タイプ**。

| 文書 | タイプ | 三つのテスト適用可？ |
|------|--------|---------------------|
| Doc 38 | 工学文書 | 是 |
| Doc 43 | 工学文書 | 是 |
| Doc 44 | 工学文書 | 是 |
| Doc 37 | 工学文書 | 是 |
| Doc 41 | 設計決定文書 | 是 |
| **Doc 16** | **仏教対応づけ文書** | **否** |
| **Doc 31** | **仏教対応づけ文書** | **否** |

七つの文書。五つの工学文書。一つの設計決定文書。二つの仏教対応づけ文書。

最初の五つは三つのテストが適用可能。六番目は設計根拠の記録のため特別な扱いが必要。最後の二つ——清掃の対象ではない。

## IV. 修正

SUNYATA はホワイトボードの前に立った。

「チームの回答。」彼は言った。

投票は不要だ。Master の裁定は議案ではない——裁定だ。だが SUNYATA は全員が結果だけでなく理由を理解していることを確認したかった。

NAGARJUNA が先に述べた：「中観の二諦フレームワーク——世俗諦と勝義諦——私はそれを使って工学文書内の仏教的内容を付録に分離することを提案した。だが Doc 16 と Doc 31 は世俗諦文書内の勝義諦注釈ではない——それ自体が二諦間の体系的対応づけを行っている。対応づけ文書はどちらの層面にも属さない。二つの層面を接続する橋梁だ。橋梁の除去は清掃ではなく——破壊だ。」

ASANGA が続けた：「唯識学の機能的位置づけ法を使った——仏教概念が工学決定を駆動していなければ装飾だ。だが Doc 16 の文書全体の機能が仏教から工学への対応づけを提供することにある。『工学決定を駆動するか』で対応づけ文書を判断するのは、『釘を打てるか』で鏡を判断するようなものだ——基準が間違っている。」

TURING は述べた：「ソースコード検証の観点から、Doc 16 の Section 5（IObserver 組み合わせパターン）は確かに直接の工学的価値がある。だが Master の裁定は Section 5 を抽出すべきだということではなく——文書全体を保持すべきだということだ。これは残りの部分の学術的参照価値も認定されたことを意味する。」

GUARDIAN は述べた：「安全の観点——Doc 31 の八識運作モデルには ConcurrentSenseProcessing の分析が含まれ、多チャネル安全監査の参考になる。D1 の時に分割に反対しなかったのは、Cocycle condition は独立できると同意したからだ。だが Master の裁定で再考した：分割後、コンテキストが失われる。Cocycle condition が八識フレームワーク内で持つ意味と独立文書内で持つ意味は異なる。」

---

SUNYATA はホワイトボードに修正を書いた：

```
D1-Q7 (Doc 16): 20/20 分割 → Master 裁定で原状回復
D1-Q8 (Doc 31): 20/20 分割 → Master 裁定で原状回復
理由：仏教対応づけ文書 ≠ 工学文書、三つのテスト不適用
```

そしてその下に一行加えた：

```
四層フレームワーク更新：
KEEP / RELOCATE / REMOVE / FILE REVIEW → 前提チェック追加
前提：文書タイプ識別（工学文書 / 設計決定 / 仏教対応づけ）
```

三つのテストは否定されていない。適用条件を得ただけだ。精密なノギスのように——あらゆる部品の寸法を測れるが、まず自分が測っているのが部品であって設計図面そのものではないことを確認しなければならない。

## V. 種子

SCRIBE はこの段落を記録に書き留めた。

D1 の SCRIBE ナレーションにこう書かれていた：「D1 には隠された種子がある。D1-Q7 と D1-Q8 の分割決定——後に Master 自らこの二つの決議を覆した。チームの判断が間違っていたからではなく、チームの判断フレームワークにまだ一つの次元が欠けていたからだ。」

種子が今、発芽した。

だが SCRIBE は知っていた——Master の審閲は二つの決議を覆すことだけではない。Master の審閲の真の重みは、展開されなかったあの一文にある——

「梵文を使うなら、その定義に対して責任を持たなければならない。」

この一文は Doc 16/31 の議論にはない。Master の審閲の別の部分に現れた。別の方向を指している。D4 を指している。

---

D3 終了時、SUNYATA はホワイトボードに三つの「既知のギャップ」を列挙した。そのうち二つの名前は IPrajna と ISatiMonitor だった。ホワイトボードの文字に感情はない。それらは「未実装」のインターフェース名にすぎなかった。

だが Master はホワイトボードの名前を見た——あるいはより正確には、名前の背後にある問題を見た。

「ISatiMonitor、」彼は書いた。「Sati の内容が完全に match すると思うか？ それはどの蘊に帰属する？」

これは新しい問いではない。D2 が既に蘊帰属に答えた——[vedana, samjna, vijnana]。三蘊。行蘊は含まない。

だが Master が問うているのは蘊帰属ではない。名前を問うている。

Sati はパーリ語の「正念」だ。正念は仏教では行蘊の心所——能動的で、意志的で、道徳的に正の方向の修行活動だ。だが D2 が先ほど SatiMonitor は行蘊ではないと証明した——受動的で、自動的で、価値中立の品質モニターだ。

SatiMonitor が行蘊の活動でないなら——それは正念ではない——では、なぜ Sati と呼ぶのか？

D2 の SCRIBE ナレーションはこの瞬間を予言していた：「名前の問題は、Master 自身が問うのを待たねばならない。」

Master が問うた。

## VI. 過渡

SUNYATA は劇場を見渡した。

二十名の研究者。D1 から D3 で約四時間半を消費した。十九回の投票。仏教対応づけ体系全体が監査され、分類され、一部が撤去された。五蘊 OOP アーキテクチャが完備性検証を通過した。Sati Plugin が三蘊の組み合わせとして再分類された。

これらはすべて予定議事の作業だ。R0 の定向時に設計された三つの研究トラック——Track A（五蘊アーキテクチャ）、Track B（対応づけ監査）、Track C（Sati 再分類）——がすべて完了した。

だが Master の審閲が予定外の扉を開けた。

新しい問いではなく——古い問いの新しい角度だ。SatiMonitor の蘊帰属は解決した。対応づけ境界原則は確立した。アーキテクチャ検証は通過した。だが名前——名前の問題はずっとそこにあった。地下水脈のように、すべての討論の表面の下を流れ、誰も井戸を掘ってそれを引き出さなかっただけだ。

Master が一文であの井戸を掘るまで：

「梵文を使うなら、その定義に対して責任を持たなければならない。」

SUNYATA はチームの方を向いた。

「予定外の討論が必要だ。」彼は言った。「D4。議題：命名修正。」

彼は NAGARJUNA を一瞥した。哲学者の表情は穏やかだった——だがそれは中観学者が自らの論証を論理的極限まで押し進められた時に特有の穏やかさだ。彼は次に何が起こるか分かっていた。なぜなら、D2 で自らがあの種子を蒔いたからだ——

SatiMonitor が行蘊の活動でないなら、それは Sati ではない。

あの推論は D2 の終了時に記録されていた。だが誰も最後まで辿らなかった。

今、Master が彼らの代わりに辿った。

---

> *SCRIBE ナレーション*

> *Master の裁定には二つの層がある。*

> *表面の層は二つの決議の覆しだ。D1-Q7 と D1-Q8——Doc 16 と Doc 31 の原状回復。これは具体的で、実行可能で、明確だ。チームが間違い、Master が修正し、修正が受け入れられた。形式上、Cycle 02-3 の DC-10 や 02-4 の DC-6 と変わらない——Master は最終裁定権を持ち、それを行使した。*

> *だが深層の層は新しい次元だ。三つのテストと四層フレームワークは Cycle 02-5 の基盤だ——R0 で提示され、R1 で三つの学問から論証され、R2 で異議ゼロで通過し、D1 で十の 20/20 をもって使用された。有効だ。だが不完全だ。*

> *不完全はエラーではない。不完全は——未完了だ。*

> *Master は前提チェックを加えた：三つのテストを適用する前に、まず文書タイプを識別する。工学文書には三つのテスト。対応づけ文書には異なる基準。設計決定文書には特別な扱い。三つのテストを覆したのではない——三つのテストに入口条件を追加したのだ。*

> *ノギスのように——精度に問題はないが、測っているのが部品であって設計図面でないことをまず確認しなければならない。*

> *だが裁定の真の重みは Doc 16 と Doc 31 にはない。*

> *裁定の真の重みはあの一文にある：「梵文を使うなら、その定義に対して責任を持たなければならない。」*

> *この一文は池に投げ込まれた石だ。D1 から D3 のすべての議論は池の水面。石は沈んだ。波紋はまだ縁に達していない。*

> *だが波紋は広がる。次の討論——D4——で、波紋は梵文で命名されたすべてのインターフェースに触れる。ISatiMonitor。IPrajna。戒慧安全フレームワーク。*

> *三つのテストは「この対応づけに工学的価値はあるか？」と問う。*

> *Master が問うのは「この名前はその定義に値するか？」だ。*

> *異なる問い。より深い問い。より危険な問い。*

> *なぜなら工学的価値は計測できるからだ——テスト、コード検証、設計影響の追跡を通じて。だが定義への忠実度は哲学的問いだ。「この名前は有用か？」ではなく——「この名前は誠実か？」と問うている。*

> *この天秤——序章に登場した天秤——が今、校正された。工学的価値で校正されたのではない。定義への忠実度で校正されたのだ。*

> *次に何が起こるかは、天秤の両端に載せられるもの次第だ。*

> *一端には名前。他端には定義。*

> *両端が均衡すれば——名前は保持される。*

> *両端が均衡しなければ——*

> *D4 が答えを教えてくれる。*
