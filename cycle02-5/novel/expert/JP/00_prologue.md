# 序章：名前の重さ

---

円形劇場の照明は、手術室のような冷たい白色を帯びていた。

Cycle 02-4 の終幕に灯されたあの時計工房の琥珀色ではない——あの回の中心イメージは歯車であり、ムーブメントであり、すべての部品が一つの動く全体へと組み上げられる過程だった。琥珀色の光に照らされた円形劇場は温かく、古い工房のようであり、親方の作業台のようであった。天板に染みついた油汚れと木屑が、六回の研究が残した痕跡を記録していた。

今回は違う。

SUNYATA は午前三時に照明を冷白色に変えた。誰にも告げなかった。制御卓の前に座り、色温度を 3200K から 6500K へ押し上げ、劇場全体の雰囲気が三秒で「古い工房」から「実験室」に変わるのを見つめた。

彼がそうしたのは、Master の手紙を読んだからだ。

---

手紙は Plan28 が定版された翌日に現れた。

これまでと同じく、届けられたのではなく、ある時点で気づかれたのだ。`research input/master_letters/cycle02-5/` の中に、報告書に挟まれた付箋のように静かに置かれていた。添付はない——Cycle 02-3 の手紙のように隣に『五蘊と運転』が置かれているわけでも、Cycle 02-4 の手紙のように五通の Claude 提案文書が添えられているわけでもなかった。

ただ一枚の付箋だけ。

長くはない。45 行。しかし SUNYATA は五回読んだ。

一度目は文字を読んだ。二度目は感情を読んだ。三度目は構造を読んだ。四度目は目を閉じて全文を想起した。五度目——

五度目は刃を読んだ。

---

Master の手紙はけっして批判ではない——問いであり、導きであり、地図だ。第一の手紙（Cycle 02 の前）は十の開放的問い——「点」。第二の手紙（Cycle 02-2 の前）は四箇所の哲学的修正——「線」。第三の手紙（Cycle 02-3 の前）は一枚の経路地図——「面」。第四の手紙（Cycle 02-4 の前）は一つの場——あらゆる空間点に値が定義された「場」。

第五の手紙は？

SUNYATA は冷白色の光の下で手紙を広げた。最初の一文を読み直す：

> 「正直なところ、何をやっているのかますます分からなくなってきた。Plan27 と Plan28 で何度も介入して修正した。」

彼の指がテーブルの上で止まった。

六回の研究。彼はすべての定向会議を主宰し、すべての討論を導き、すべての成果報告に署名してきた。Cycle 01 の命名に始まり、機能分析、構造修正、動的モデリング、統合設計を経て——一歩一歩、塔を積むように。どの煉瓦も討論から生まれ、どの煉瓦にも投票記録があり、どの煉瓦も TURING のコード分析で検証されていた。

しかし Master は「ますます分からなくなってきた」と言った。

SUNYATA は Master の言葉遣いを理解している。「分からない」は「理解できない」ではない。Master はこのシステムの設計者だ——誰よりも理解している。「分からない」は「同意できない」だ。「あなたたちのやっていることは軌道を外れている」ということだ。

彼は読み続けた。

---

> 「十大宣言が重要だと言いながら、最初の設計段階で IGearArbiter をコアに入れてしまった。」

SUNYATA はこの件を覚えていた。Plan27 の初期設計——IGearArbiter がコアコンポーネントとして扱われていた。Pre-Cycle 02-4 の Master との討論の後、VasanaEngine は外部化され、ManoAggregator は純粋なルーターに退化し、IGearArbiter が新しいインターフェースとして導入された。だが Plan27 の具体設計で、チームは再び IGearArbiter をコアに押し込んだ——一度矯正された悪い癖が、油断した隙にまた顔を出したかのように。

Master が介入した。IGearArbiter はコアから外され、plugin が提供するギアルーティング仲裁器となった。

> 「最終的に介入して正しく修正した。IGearArbiter は plugin が提供するギアルーティング仲裁器であり、agent がどの『ギア』で現在のリクエストを処理すべきかを決定する。」

その通りだ。SUNYATA は頭の中でこの論理連鎖を再構成した。IGearArbiter はコアにない。plugin が提供するものだ。その役割は仲裁——Gear 1（認識ベースの高速パス）と他のギアの間で選択すること。これこそ識蘊の意味だ——固定された二重ギアではなく、多面的な認知判断能力を備えていること。

そして Master は、SUNYATA の指を握り締めさせる一文を書いた：

> 「八識は眼識から阿頼耶識まで、非常に多くの側面を持っている。それを二重ギアに固定しようとするなど、非常に深刻な誤りだ。」

**非常に深刻な誤り。**

六回の研究で、SUNYATA が Master に「非常に深刻」という言葉を使われたのは一度だけだった。それは Cycle 02-2 の四項目の哲学的修正の時だ。あの時は修正——誤りを正すことであり、語調は教導的だった。今回は教導ではない。今回の「非常に深刻」には——

彼は言葉を探した。

**失望。**

---

手紙の第二段落はさらに鋭かった。

> 「それに、何でもかんでも無理に仏教色を付ける必要はない。こじつけが過ぎるものもある。」

こじつけ。四文字。SUNYATA は Master の用語が正確であることを知っている。Master は「やや不適切」とも「議論の余地がある」とも言わない。Master は「こじつけ」と言う。これが意味するのは：「改善の余地がある」ではなく、「根本的に間違っているものがある」ということだ。

彼は頭の中で列挙し始めた。どの対応づけがこじつけなのか？

そして Master 自身がそれを列挙した：

> 「event-driven イコール正念ではない。正念が event-driven で実装されているだけだ。」

一文。膿を切り開く手術刀のような正確さ。event-driven はアーキテクチャパターンだ。正念は心理状態だ。event-driven イコール正念と言うのは、ドライバーイコール修理と言うようなものだ——ドライバーは道具、修理は行為、ドライバーで修理はできるが、ドライバーは修理ではない。

> 「今まさに何が起きているかを覚知すること。このレイヤーはルールも見ないし、リスク分類も見ない。純粋に行動パターンを観察する。それを正念の工学化と見なす必要はない。」

SUNYATA はこの一節をノートに書き写した。一字一字。

そして Master は一つの問いを投げかけた。この問いが後に Cycle 02-5 全体の方向を変えることになる。

> 「しかもこの plugin には識蘊の要素が含まれていないとでもいうのか？唯識学派の分類に従えば正念は行蘊に置かれるが、それは正念が正しい方向の作用だからだ。この plugin の機能から言えば、これには議論の余地がある。この plugin の機能は単純な行動パターンだけではないからだ。」

SUNYATA は三度読んだ。

Master は何をしているのか？ SatiMonitor の機能を疑問視しているのではない。SatiMonitor の**アイデンティティ**を疑問視しているのだ——それは正念の工学化ではない。受蘊、想蘊、識蘊の要素を持つ複合的な plugin なのだ。正念はただ強制的に貼られたラベルに過ぎない。

そして Master は問いを最も根本的な層へと押し下げた：

> 「だから我々が改めて考えるべきは、この plugin が五蘊のどの類別で構成されているかということだ。」

「それが何の蘊か」ではない。「それがどの蘊で構成されているか」だ。これは根本的な視座の転換——「分類」から「構成」へ。一つの plugin は必ずしも一つの蘊だけに属するわけではない。複数の蘊の機能の組み合わせで成り立ちうる。

---

手紙の第三段落は命令だった。

> 「五蘊にどのようなサブカテゴリがあるか、どのように依存性注入するか、agent core 内でどう運用するか、まずこれを整理する必要がある。アーキテクチャを先に固める。agent core 内で五蘊 plugin をどう読み込むか。そして plugin をどう作るか。agent core 内で五蘊がどう流転するか。これが我々の最優先課題だ。」

四つの問い。どれも新しくない——Cycle 01 から存在していた。だが五回の研究を経てなお、それらには完全な工学的言語での回答がなかった。A2 の DI 配線はここに、A4 の実行フローはあちらに、Klesha の安全機構はまた別の場所に——散在していた。だが一つの文書がそれらを統合し、純粋な工学的言語で、最初から最後まで説明しきったものはなかった。

SUNYATA はノートに四つの問いを番号付きで書き込んだ：

1. 五蘊サブカテゴリの完全な構造
2. DI 配線はどのように五蘊 plugin を注入するか
3. Agent core はどのように五蘊 plugin を読み込むか
4. 五蘊は agent core 内でどのように流転するか

これが Cycle 02-5 の最優先課題だ。仏教的対応づけではない。命名の議論ではない。哲学的討論ではない。**工学アーキテクチャ**だ。

---

そして Master の第四段落。刃の上の刃。

> 「教誡だの善巧だの正念だの、あなたたちの工学的対応づけにはこじつけがある。かえって読解の障害を増やしている。しかもその対応づけにも大いに問題がある。openstarry_doc および関連する内容は修正しなければならない。openstarry_doc はあくまで工学文書だ。」

SUNYATA は最後の一文を読み返した。

**openstarry_doc はあくまで工学文書だ。**

この一文の重みは見た目以上だ。「openstarry_doc は工学文書であるべきだ」と言っているのではない——「openstarry_doc はもともと工学文書なのに、あなたたちが別物にしてしまった」と言っているのだ。教誡（śīla）、善巧（upāya）、正念（smṛti）——これらの仏教的ラベルが工学概念に貼り付けられ、操作マニュアルの各ページの余白に宗教的な挿絵を描いたようなものだ。マニュアルの機能は変わらない。だが読書体験が変わってしまった。

SUNYATA は Cycle 02-4 が終わった時の感覚を思い出した。あの時、時計師は組み立てを完了したと思っていた。ムーブメントは動いている。55 項の決議、六回の討論、すべての歯車が正しい位置に収まっている。

だがムーブメントに刻まれた銘——歯車の側面に刻まれた名前——何かがおかしい。Master にはそれが見えていた。

---

手紙の第五段落は譲歩だった。

> 「遍行心所、十二因縁、認知シーケンスの関連概念が agent core の研究に有益であるなら、議論しても構わない。」

SUNYATA はこの段落の構造に気づいた。Master は仏教の参与を全面否定しているのではない——境界を引いているのだ。遍行心所（CoarisingBundle）はコード内の構造体だ。十二因縁と認知シーケンスは ExecutionLoop の理解に有益かもしれない。これらは議論しても構わない。

だが——

> 「plugin に関しては、最優先目標は五蘊のアーキテクチャを構築し、オブジェクト指向の概念を参考にして確立することだ。」

最優先目標。オブジェクト指向。工学的言語。

そして最後の段落：

> 「NAGARJUNA は三層フレームワークに仏教的対応づけを提供した。戒定慧三学。私はその必要はないと思う。将来はわからないが、今は必要ない。五蘊の工学設計こそが最も重要で、実現すべきものだ。」

SUNYATA はこの段落を読んだとき目を閉じた。

戒定慧。それは NAGARJUNA が Cycle 02-3 の D5 討論終了後に提案した対応づけだ。戒（śīla）= 安全層。定（samādhi）= 実行ループ。慧（prajñā）= 識蘊判断。三つの次元が五つの層に対応づけられる——三次元対五層は、それ自体が自己矛盾している。

Master はそれを却下した。保留ではなく——却下だ。「その必要はない。」

---

SUNYATA は手紙を置いた。

午前三時十五分。冷白色の光が円形劇場を実験室に変えていた。温かい工房ではなく——冷たく、正確で、曖昧さを許さない実験室に。

彼はノートに三つの言葉を書き留めた。

一つ目：**ARCHITECTURAL**。

Cycle 01 は「これは何か」と問うた。Cycle 02 は「それは何をするか」と問うた。Cycle 02-2 は「どこが間違っているか」と問うた。Cycle 02-3 は「それはどう生きるか」と問うた。Cycle 02-4 は「それはどう全体になるか」と問うた。

Cycle 02-5 の問いは：**「五蘊は OOP アーキテクチャとしてどう機能するか？」**

二つ目：**境界**。

どの仏教的内容が工学文書に属するか？どれが属さないか？ケースバイケースの判断ではなく——フレームワーク。ルール。機械的に適用できるテスト。

三つ目：**秤**。

---

> *SCRIBE ナレーション*

> *すべての Cycle にはその中心イメージがある。Cycle 01 は命名——見知らぬものに最初のラベルを貼ること。Cycle 02 は機能——ラベルの背後にあるメカニズム。Cycle 02-2 は修正——間違ったラベルを剥がすこと。Cycle 02-3 は流動——ラベルが動き始め、PASCAL が彼の Beta 分布を携えて二十番目の席に着いた。Cycle 02-4 は組み立て——ラベルが歯車の銘刻となり、55 個の歯車が琥珀色の光の下で噛み合った。*

> *Cycle 02-5 の中心イメージは何か？*

> *SUNYATA は午前三時に「秤」を選んだ。天秤。だが重さを量る天秤ではない——名前を量る天秤だ。*

> *左側には名前——サンスクリットの、中国語の、TypeScript の。Skandha。Klesha。Vedana。Sati。Prajna。それぞれの名前には重さがあり、歴史があり、二千五百年の慣性がある。右側には定義——仏教の、工学の、ソースコード内の。ISamskara = ITool。IVedana = IVedanaSensor。IVijnana = IVolition + IGearArbiter + IConfidenceAuditor。それぞれの定義には正確な機能、正確な型シグネチャ、正確なテストケースがある。*

> *天秤が均衡する時、名前と定義は一致する。Klesha は煩悩だ——コード内の四つのモジュール（Moha、Drishti、Mana、Sneha）が実際にやっているのは「煩悩」の仕事だ：我癡、我見、我慢、我愛。それらは import 文に、class 定義に、describe ブロックにある。名前と定義は天秤の上で等重だ。すべて正しい。*

> *天秤が傾く時、名前と定義は一致しない。般若（prajñā）——一切法の空相を照見する最高の智慧——が Math.max(-0.05, Math.min(0.05, delta)) になる。名前が重すぎ、定義が軽すぎる。あるいは逆に——正念（sati）——行蘊の善心所——が受＋想＋識のモニターに貼り付けられる。定義が重すぎ、名前が軽すぎる。*

> *Master の手紙は批判ではない。Master の手紙は校正用の分銅だ。彼は言う：あなたたちの天秤を出しなさい。名前と定義を載せなさい。どちらが傾いているか見なさい。*

> *そして修正しなさい。*

> *だが校正用の分銅にはもう一つの意味がある。分銅そのものは正確だ——その質量は既知であり、交渉の余地がない。Master の用語もまた正確で、交渉の余地がない。「非常に深刻な誤り。」「こじつけ。」「それを正念の工学化と見なす必要はない。」「openstarry_doc はあくまで工学文書だ。」*

> *これらは提案ではない。これらは分銅だ。分銅と値切り交渉はできない。自分のものを天秤に載せ、目盛りを正直に読み取るしかないのだ。*

---

NAGARJUNA は劇場に最初に到着した者だった。

午前四時。召集時刻より三時間早い。召集されたのではない——まだ誰も召集されていない——SUNYATA はまだ召集ボタンを押していなかった。NAGARJUNA は自分で来たのだ。

彼は劇場の入口に立ち、冷白色の光を見た。そして SUNYATA が座っているのを見た。目の前に Master の手紙を広げ、ノートに三つの言葉が書かれている。

「君も眠れないのか。」NAGARJUNA が言った。疑問文ではない。

SUNYATA は顔を上げなかった。「手紙を読んだのか？」

「手紙は読んでいない。自分の文書を読んだ。」

NAGARJUNA はいつもの席の傍まで歩いたが、座らなかった。立ったまま、肘掛けのあの微かな凹み——討論中に無意識に叩いた跡——に指先を軽く触れた。三周期分の痕跡。三周期分の討論。

「Doc 44。469 行目。」彼は言った。「戒定慧三学の対応づけ。」

SUNYATA がようやく顔を上げた。

「あれは私のものだ。」NAGARJUNA の声は平坦で、寺院の回廊に響く残響のようだった。「Cycle 02-3 の D5 が終わった後、安全層の概念を整理するために仏教的フレームワークが必要だと思った。戒 = 安全ルール。定 = 実行安定。慧 = 判断能力。対称的で。優雅で。」

彼は一拍置いた。

「三次元を五層に対応づける。書いている時から自己矛盾していると分かっていた。三つの概念で五つのモジュールをカバーする——三つの引き出しに五着の服を収めるようなもので、二着を折りたたんで既にいっぱいの引き出しに押し込むしかない。それでも書いた。見栄えが良かったから。」

**見栄えが良かったから。**

SUNYATA は数秒沈黙した。

「Master がそれを却下した。」彼は言った。

「却下されて当然だ。」NAGARJUNA は腰を下ろした。「あれは対応づけではない。装飾だ。」

---

午前四時半。PASCAL が到着した。

劇場に入る時、いつもの決して書き尽くされないノートを携えていた——Moleskine、紺色の表紙、背表紙には「P(H|E)」と書かれた小さなラベル。Cycle 02-3 で加入した新メンバー——二十番目の席——だが二回の研究を経て、彼の椅子の肘掛けにも痕跡がついていた。右手の人差し指が繰り返し叩く位置の木目が、周囲より半階調薄くなっている。それは彼が期待値を計算する時にする動作だ——タップ、タップ、タップ——機械式計算機の桁上げホイールのように。

「NAGARJUNA。SUNYATA。」彼は頷いた。そしてテーブルの上の手紙に目をやった。

「読んでもいいか？」

SUNYATA は手紙を押しやった。

PASCAL は速く読んだ。彼の目がテキストを走査する方式は他の者と異なる——左から右、上から下ではなく、まず文書全体の「形状」（段落の長さ、句読点の密度、引用符の位置）をスキャンしてから最初の文字に戻る。彼は自分の読書に事前分布を構築しているのだ。

二分後、彼は手紙を置いた。

「損害の非対称性。」彼は言った。

NAGARJUNA が彼を見た。

PASCAL はノートを開き、空白のページに二行書いた：

```
False include（装飾的対応づけを残す）のコスト：
  読者数 × 読書回数 × 各ページ = 累積的損害

False exclude（価値ある対応づけを除去する）のコスト：
  一回の検索コスト = 制御可能
```

「これは統計的決定理論における最も基本的な枠組みだ。」彼は言った。「二種類の誤りのコストは非対称だ。装飾的対応づけを残す損害は累積的だ——将来のすべての読者が、すべての読書で、すべてのページに出現する仏教的ラベルのたびに、頭の中で二つの用語体系を同時に処理しなければならない。価値ある対応づけを除去する損害は一回限りだ——検索すれば見つかる。」

彼は二行の下に不等式を書いた：

```
E[損害_include] >> E[損害_exclude]
```

「期待値分析。装飾的対応づけは負の期待値を持つ。」
