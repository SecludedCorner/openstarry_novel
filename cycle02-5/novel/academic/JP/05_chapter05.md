# 第五章：D4——名前の代価

---

## 帰謬

SUNYATA は Master のあの一言をスクリーンに映し出した。大きな文字。白地に黒字。

> **「梵語を使うなら、その定義に対して責任を持たなければならない。Sati の内容は完全に match しているか？ それはどの蘊に帰属するのか？」**

NAGARJUNA が立ち上がった。今回は過ちを認めるためではない——一つの推論を完成させるためであった。

「D2 で SatiMonitor の蘊帰属は既に確定した。結論は [vedana, samjna, vijnana]。三蘊。行蘊を含まない。」

彼はホワイトボードに二行を記した：

```
前提 A：Sati = 行蘊心所（仏学的定義）
前提 B：SatiMonitor ≠ 行蘊（D2-R2 の結論）
∴ SatiMonitor ≠ Sati
```

「Sati——パーリ語のマインドフルネス——は仏学の伝統において行蘊の心所である。それは意志的で、能動的で、道徳的に正向な修行活動である。もしマインドフルネスが必然的に行蘊の心所であり、SatiMonitor が行蘊でないならば——SatiMonitor はマインドフルネスではない。」

彼はペンを置いた。

「マインドフルネスではないものが、なぜ Sati と呼ばれるのか？」

沈黙。同意しないための沈黙ではない。全員が同時に、もっと早く理解すべきだったことを理解した沈黙である。

## 五つの不一致

ASANGA が機能分析によって帰謬の結論を確認した：

| 次元 | 仏学的正念（Sati/Smṛti） | SatiMonitor |
|------|----------------------|-------------|
| 能動性 | 能動的（精進） | 受動的（event-driven） |
| 道徳性 | 道徳的に正向 | 価値中立 |
| 意志性 | 意図を要する | 自動運転 |
| 伝統的帰属 | 行蘊心所 | 受+想+識 |
| 修行上の地位 | 八正道の第七支 | 品質モニタ |

五つの次元。五つの不一致。

「我々は命名の過ちを犯した」と ASANGA は述べた。「分類の過ちではない——分類は正しい。命名の慣性だ。Cycle 02-4 で ISatiMonitor というこの名前を踏襲したのは、既に百回以上使われていたからだ。慣性が名前そのものへの批判的視点を失わせた。」

## 四つの提案

「では何と呼ぶべきか？」SUNYATA は問うた。

| 提案者 | 名前 | 理由 |
|--------|------|------|
| ARCHIMEDES | ILoopQualityMonitor | 機能を精確に記述：認知ループ品質モニタ |
| GUARDIAN | IBehaviorQualityMonitor | 行動面に重点 |
| NAGARJUNA + ASANGA | ICognitiveLoopMonitor | 認知ループの完全性を強調 |
| SYNTHESIST | IQualityMonitor | 最もシンプルな機能記述 |

ARCHIMEDES の論点は最も直截であった：「新しいエンジニアが ILoopQualityMonitor を見れば、このインターフェースが何をするか分かる——ループ品質の監視である。仏学なし。隠喩なし。歴史的しがらみなし。」

TURING はソースコードの事実によって ARCHIMEDES を支持した：SatiMonitor の 11 種類のイベント購読は認知ループ全体をカバーしており、行動段階だけではない。「Loop」は「Behavior」より正確である。

**D4-R1：ISatiMonitor → ILoopQualityMonitor。13/20。**

全員一致ではない。しかし多数は明確である。

## 核融合炉

続いて SUNYATA は二語を述べた：「IPrajna。」

NAGARJUNA は目を閉じ、しばらくして語り始めた。

「般若——prajñā——は仏学における最高の認知能力である。一切の法の実相を照見する智慧。普通の賢さではない。分析能力ではない。般若は超越的なものである——空性の直観であり、二千五百年にわたる大乗仏教体系全体の核心的認知目標である。」

彼はホワイトボードに二行を記した：

```
般若（仏学）：一切の法の実相を照見する最高の智慧
IPrajna（エンジニアリング）：信頼度に ±0.05 を加減する関数
```

ASANGA は後にすべての者が記憶することになる一言を述べた：

「これは温度の微調整ダイヤルを『核融合炉』と呼ぶようなものだ。」

誰も笑わなかった。冗談ではないからだ——精確な類比によって精確な問題を説明しているのである。

IPrajna のインターフェース：一つのメソッド、信頼度とコンテキストを入力し、`{ delta: number, reasoning: string }` を出力、delta は [-0.05, +0.05] に制限される。これはクランプである。微調整器である。

PASCAL は決定理論の観点から確認した：「IPrajna が行っているのは bounded secondary evaluation である。一次の信頼度を入力し、二次の修正量を出力し、±0.05 の硬い制限がある。これは監査である。般若ではない。」

BABBAGE は代替名の意味的精度を分析した：「IConfidenceAuditor が最も精確である。Audit——監査——この操作を精確に記述している：既存の判断に対する限定的範囲の二次評価。」

**D4-R2：IPrajna → IConfidenceAuditor。16/20。**

D4-R1 よりも高い合意。少数意見：WIENER は IThresholdCalibrator を好み（二票）、HERACLITUS と PENROSE は ISecondaryEvaluator を好んだ（二票）。BABBAGE は両者に反論した——Calibrator はシステム自体を調整することを暗示し、Evaluator は汎用的すぎる。

## 第四のテスト

D4 は二度の改名で終わらなかった。

Master の審査は Doc 03——`Sila_Prajna_Security_Framework.md`——にも言及していた。

SUNYATA はホワイトボードに既存の三項テストを列挙し、隣に新たな一項を加えた：

```
Test 1（必要性）：除去してもエンジニアリング仕様は変わるか？
Test 2（コード識別）：ソースコードで使用されているか？
Test 3（決定駆動）：DC 確認済みの設計決定を駆動したか？
Test 4（定義責任）：梵語術語を使用する場合、コンポーネントの機能はその術語の仏学的定義と一致するか？
```

第四のテストは天から降ってきたものではない——D4-R1 と D4-R2 の議論から結晶化したものである。ISatiMonitor は Test 2 を通過したが（ソースコードに存在する）、Test 4 は通過しなかった（名前と機能が不一致）。三項テストではこの次元を捉えられなかった。第四のテストがその欠落を埋めた。

NAGARJUNA が Doc 03 を項目別にテストした：

- **Test 1**：Plugin のライフサイクルは種子理論なしでも理解できる。❌
- **Test 2**：ソースコードの型は英語（'active' | 'dormant' | 'suspended'...）であり、Sila/Prajna はコメントにしか現れない。❌
- **Test 3**：DC 確認なし。❌
- **Test 4**：Sila（戒律）≠ アクセス制御。Prajna（般若）≠ CVE 撤回。❌

四項目とも不合格。

ASANGA は重要な区別を行った：「Doc 16 は仏学対応づけ文書であり——Master は保持を裁定した。Doc 03 はエンジニアリング文書に仏学的装飾が埋め込まれたものであり——整理すべきである。同じ仏学的内容でも、文書タイプが異なれば、判断も異なる。」

**D4-R3：Doc 03 を再投票、「Sila-Prajna Security Framework」→「Plugin Security Lifecycle」。17/20。**

## 反対の方向

D4 が終了した。三十分。三つの名前が改められた。一つの恒久的テストが確立された。

SUNYATA はホワイトボードの前に立ち、四項テストを見つめた。最初の三項は「仏学概念はエンジニアリングに有用か？」と問うている——仏学からエンジニアリングへの方向である。第四項は「エンジニアリングの命名は仏学的定義に忠実か？」と問うている——エンジニアリングから仏学への方向である。

二つの方向。一つの天秤。

名前が一方の端に。定義がもう一方の端に。釣り合うとき——Moha、Drishti、Mana、Sneha——名前は保持される。釣り合わないとき——ISatiMonitor、IPrajna、Sila-Prajna——名前は置き換えられる。

PASCAL がまとめた：「Master は一文で述べた。我々は一つの議論を費やした。結論は同じだ。しかし議論の価値は——なぜかを説明したことにある。」
