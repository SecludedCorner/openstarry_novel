# 第五章：D4——你的名字配得上你吗？

---

## 那个问题

SUNYATA 把师父的话打在投影幕上。大字。白底黑字。

> **"用梵文就需要对他的定义负责。你觉得 Sati 的内容完全 match 吗？"**

这是整个 Cycle 02-5 最核心的问题。

---

## 逻辑推论

哲学家 NAGARJUNA 站了起来。他在白板上写了三行字：

```
前提 1：正念（Sati）在佛学中是行蕴的活动
前提 2：SatiMonitor 不属于行蕴（D2 已经证明了）
结论 ：SatiMonitor 不是正念
```

"如果正念必然是行蕴的活动，而 SatiMonitor 不是行蕴——那 SatiMonitor 就不是正念。"

他放下笔。

"一个不是正念的东西，为什么叫 Sati？"

全场沉默。不是因为有人不同意——是因为所有人同时理解了一件他们应该更早理解的事情。

---

## 五个不一致

佛学家 ASANGA 拿出了一张对比表：

| | 佛学中的正念 | SatiMonitor |
|--|------------|-------------|
| 主动还是被动？ | 主动（需要精进） | 被动（自动运行） |
| 有道德方向吗？ | 有（道德正向） | 没有（价值中立） |
| 需要意图吗？ | 需要 | 不需要 |
| 属于哪一蕴？ | 行蕴 | 受+想+识 |
| 地位？ | 八正道第七支 | 一个品质监控器 |

五个方面。五个都不一致。

"我们犯了一个错误，"ASANGA 说。"不是分类的错误——分类是对的。是命名的错误。我们用了太久这个名字，习惯了，就忘了去检查名字本身对不对。"

---

## 新名字

SUNYATA 问："那它应该叫什么？"

四个提案中，ARCHIMEDES（工程实践专家）的得到最多支持：

**ILoopQualityMonitor** —— 认知回路品质监控器。

理由很直接："一个新来的工程师看到这个名字就知道它做什么——监控回路品质。没有佛学。没有隐喻。没有需要解释的历史。"

**13/20 通过。**

---

## 核聚变反应堆

然后 SUNYATA 说了两个字："IPrajna。"

IPrajna 是另一个用佛学术语命名的接口。般若（Prajñā）在佛学中是最高的智慧——照见一切实相的超越性认知。

NAGARJUNA 在白板上写了两行：

```
般若（佛学）：照见一切法实相的最高智慧
IPrajna（工程）：对信心度加减 0.05 的函数
```

ASANGA 说了一句后来被所有人记住的话：

"这就像把一个温度微调旋钮叫做'核聚变反应堆'。"

没有人笑。因为他说的是事实。

IPrajna 做的事情是什么？接收一个信心度数字，加减最多 0.05，然后回传结果。一个微调器。一个夹钳。这和"照见一切法实相"的距离，就像旋钮和核聚变的距离。

**16/20 通过：IPrajna → IConfidenceAuditor（信心度审计器）。**

---

## 第四项测试

D4 不只是改了两个名字。它还创造了一项新的测试标准。

在 D4 之前，团队有三项测试（必要性、程序代码识别、决策驱动）。但这三项测试有一个盲点——它们没有问："名字是否和定义一致？"

ISatiMonitor 通过了 Test 2（它在程序代码中被使用）。但它不通过一个更基本的标准——它的名字和它的功能不一致。

所以团队加了第四项测试：

> **Test 4（定义责任）：当你使用佛学梵文术语作为程序代码名字时，组件的功能是否真的匹配那个佛学术语的意思？如果不匹配，使用工程术语。**

用这个标准重新看 Doc 03（`Sila_Prajna_Security_Framework.md`）——Sila（戒律）被用来表示"访问控制"？Prajna（般若）被用来表示"安全漏洞回应机制"？四项测试全部不通过。

**17/20 通过：Doc 03 更名为 Plugin Security Lifecycle。**

---

## 天平

D4 只花了三十分钟。但它是整个 Cycle 02-5 的高潮。

之前的三项测试问的是："这个佛学概念对工程有用吗？"——从佛学到工程的方向。

第四项测试问的是："这个工程命名对佛学定义忠实吗？"——从工程到佛学的方向。

想象一架天平。名字在一端，定义在另一端。

平衡的时候——Moha（迷惑）模组确实在模拟迷惑的行为——名字保留。
不平衡的时候——SatiMonitor 不是正念、IPrajna 不是般若——名字被替换。

天平不在乎名字多古老、多崇高。天平只在乎两端是否平衡。

PASCAL 的总结："师父用了一句话。我们用了一场辩论。结论一样。但辩论的价值在于——它解释了为什么。"
