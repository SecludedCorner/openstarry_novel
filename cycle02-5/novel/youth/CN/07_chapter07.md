## 一个问题的答案

整个 Cycle 02-5 回答了一个核心问题：

**在一个同时使用佛学和工程两种语言的系统中，这两种语言的边界在哪里？**

答案可以用三条规则概括：

**规则一：程序代码名字可以用佛学术语——但名字必须配得上定义。**

Moha（迷惑）模组真的在模拟迷惑 → 保留。
SatiMonitor 不是正念 → 改名。

**规则二：设计理由可以引用佛学概念——但必须真的驱动了工程决策。**

"四烦恼同时俱起"驱动了原子性设计 → 保留。
"event-driven = 正念"没有驱动任何设计 → 移除。

**规则三：映射文件可以存在——但工程文件不应该有佛学装饰。**

Doc 16（本身就是映射文件）→ 保留。
Doc 38 中的"佛学映射"栏位（工程文件中的装饰）→ 移除。

## 名字的代价

NAGARJUNA 最后读了一遍 Doc 45。读完后他说：

"被改掉的名字——ISatiMonitor、IPrajna——它们不是坏名字。它们是放错位置的名字。正念是伟大的概念。般若是伟大的概念。但伟大的概念不应该用来命名平庸的功能。"

ASANGA 补充："当你把一个 ±0.05 的微调器叫做般若，你不仅贬低了般若——你也误导了对功能的理解。一个新来的工程师看到 IPrajna，会以为这是一个多高深的组件。打开程序代码一看——是一个 clamp 函数。期望和现实之间的落差，就是名字的代价。"

但不是所有名字都被改了。Moha、Drishti、Mana、Sneha、skandha、vedana——这些全部保留。因为它们通过了第四项测试。名字和功能一致。天平平衡。
