# 第五章：你的名字配得上你嗎？

---

這是整個 Cycle 02-5 最重要的一場辯論——D4。它不在原本的計畫中。它由 Master 的一句話觸發：

> **「用梵文就需要對他的定義負責。」**

## 名字的問題

在第三章中，我們知道了 SatiMonitor 不包含行蘊。但「Sati」在佛學中是「正念」——正念屬於行蘊。

那問題來了：

如果 SatiMonitor **不是**行蘊活動→那它**不是**正念→那它為什麼叫 **Sati**？

NAGARJUNA 用一個邏輯推理（他叫它「歸謬法」）把這個矛盾展示出來：

```
SatiMonitor 不是行蘊（D2 已證明）
正念是行蘊（佛學定義）
所以 SatiMonitor 不是正念
所以不應該叫 Sati
```

## 第一個改名：ISatiMonitor → ILoopQualityMonitor

投票：13/20。新名字的意思是「迴路品質監控器」——精確描述了這個元件做的事情。

## 第二個改名更震撼

SUNYATA 提到了另一個名字：**IPrajna**。

般若（Prajna）在佛學中是什麼？**最高的智慧。照見一切事物真相的能力。**

IPrajna 在程式中做什麼？**對一個數字加減 0.05。**

ASANGA 說了一句所有人都記住的話：

> **「這就像把一個溫度微調旋鈕叫做核融合反應爐。」**

沒有人笑。因為他說的是實話。

投票：**IPrajna → IConfidenceAuditor**，16/20。新名字的意思是「信心度審計員」——誠實地描述了這個元件的實際功能。

## 第四項測試

D4 產生了一項永久性的新規則——**定義責任測試**：

> 當你用佛學名字命名一個程式元件時，這個元件的功能必須真的符合那個佛學名字的意思。如果不符合，就用普通的工程名字。

哪些名字通過了這個測試？**Moha**（自我迷惑——程式中確實在模擬自我迷惑）。**Skandha**（蘊——程式中確實在做五蘊分類）。

哪些名字沒通過？**Sati**（正念——但 SatiMonitor 不是在做正念）。**Prajna**（最高智慧——但 IPrajna 只是加減 0.05）。

名字不是免費的。借一個偉大的名字給一個普通的功能——不會讓功能變偉大，只會讓名字貶值。
