# 第五章：D4——你的名字配得上你嗎？

---

## 那個問題

SUNYATA 把師父的話打在投影幕上。大字。白底黑字。

> **「用梵文就需要對他的定義負責。你覺得 Sati 的內容完全 match 嗎？」**

這是整個 Cycle 02-5 最核心的問題。

---

## 邏輯推論

哲學家 NAGARJUNA 站了起來。他在白板上寫了三行字：

```
前提 1：正念（Sati）在佛學中是行蘊的活動
前提 2：SatiMonitor 不屬於行蘊（D2 已經證明了）
結論 ：SatiMonitor 不是正念
```

「如果正念必然是行蘊的活動，而 SatiMonitor 不是行蘊——那 SatiMonitor 就不是正念。」

他放下筆。

「一個不是正念的東西，為什麼叫 Sati？」

全場沉默。不是因為有人不同意——是因為所有人同時理解了一件他們應該更早理解的事情。

---

## 五個不一致

佛學家 ASANGA 拿出了一張對比表：

| | 佛學中的正念 | SatiMonitor |
|--|------------|-------------|
| 主動還是被動？ | 主動（需要精進） | 被動（自動運行） |
| 有道德方向嗎？ | 有（道德正向） | 沒有（價值中立） |
| 需要意圖嗎？ | 需要 | 不需要 |
| 屬於哪一蘊？ | 行蘊 | 受+想+識 |
| 地位？ | 八正道第七支 | 一個品質監控器 |

五個方面。五個都不一致。

「我們犯了一個錯誤，」ASANGA 說。「不是分類的錯誤——分類是對的。是命名的錯誤。我們用了太久這個名字，習慣了，就忘了去檢查名字本身對不對。」

---

## 新名字

SUNYATA 問：「那它應該叫什麼？」

四個提案中，ARCHIMEDES（工程實踐專家）的得到最多支持：

**ILoopQualityMonitor** —— 認知迴路品質監控器。

理由很直接：「一個新來的工程師看到這個名字就知道它做什麼——監控迴路品質。沒有佛學。沒有隱喻。沒有需要解釋的歷史。」

**13/20 通過。**

---

## 核融合反應爐

然後 SUNYATA 說了兩個字：「IPrajna。」

IPrajna 是另一個用佛學術語命名的介面。般若（Prajñā）在佛學中是最高的智慧——照見一切實相的超越性認知。

NAGARJUNA 在白板上寫了兩行：

```
般若（佛學）：照見一切法實相的最高智慧
IPrajna（工程）：對信心度加減 0.05 的函數
```

ASANGA 說了一句後來被所有人記住的話：

「這就像把一個溫度微調旋鈕叫做『核融合反應爐』。」

沒有人笑。因為他說的是事實。

IPrajna 做的事情是什麼？接收一個信心度數字，加減最多 0.05，然後回傳結果。一個微調器。一個夾鉗。這和「照見一切法實相」的距離，就像旋鈕和核融合的距離。

**16/20 通過：IPrajna → IConfidenceAuditor（信心度審計器）。**

---

## 第四項測試

D4 不只是改了兩個名字。它還創造了一項新的測試標準。

在 D4 之前，團隊有三項測試（必要性、程式碼識別、決策驅動）。但這三項測試有一個盲點——它們沒有問：「名字是否和定義一致？」

ISatiMonitor 通過了 Test 2（它在程式碼中被使用）。但它不通過一個更基本的標準——它的名字和它的功能不一致。

所以團隊加了第四項測試：

> **Test 4（定義責任）：當你使用佛學梵文術語作為程式碼名字時，元件的功能是否真的匹配那個佛學術語的意思？如果不匹配，使用工程術語。**

用這個標準重新看 Doc 03（`Sila_Prajna_Security_Framework.md`）——Sila（戒律）被用來表示「存取控制」？Prajna（般若）被用來表示「安全漏洞回應機制」？四項測試全部不通過。

**17/20 通過：Doc 03 更名為 Plugin Security Lifecycle。**

---

## 天平

D4 只花了三十分鐘。但它是整個 Cycle 02-5 的高潮。

之前的三項測試問的是：「這個佛學概念對工程有用嗎？」——從佛學到工程的方向。

第四項測試問的是：「這個工程命名對佛學定義忠實嗎？」——從工程到佛學的方向。

想像一架天平。名字在一端，定義在另一端。

平衡的時候——Moha（迷惑）模組確實在模擬迷惑的行為——名字保留。
不平衡的時候——SatiMonitor 不是正念、IPrajna 不是般若——名字被替換。

天平不在乎名字多古老、多崇高。天平只在乎兩端是否平衡。

PASCAL 的總結：「師父用了一句話。我們用了一場辯論。結論一樣。但辯論的價值在於——它解釋了為什麼。」
