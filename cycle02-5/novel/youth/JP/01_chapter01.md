# 第1章：研究と準備

---

## 9本のレポート

20人の研究者が一斉に作業を開始しました。それぞれ異なる任務を受けた探偵団が、各自で調査に出かけ、手がかりを持ち帰ってパズルを組み立てる姿を想像してみてください。

**ルート A** の任務が最も重要でした――師父が言った「最優先事項」、つまり五蘊がコードの中で実際にどう動いているのかを解明することです。

五蘊とは何か？ 仏教では、人間の経験を5つのカテゴリーに分類する方法です。色（身体/外界）、受（感覚）、想（認識）、行（行動）、識（判断）。OpenStarry はこの分類を借用して、AI Agent の機能を体系化しています。

| 蘊 | AI Agent での意味 | 例 |
|-----|-----------------|------|
| 色蘊 | 外部入力の受信 | ユーザーのメッセージを受け取る |
| 受蘊 | 良し悪しを感じる | ある操作が速いか遅いかを判定する |
| 想蘊 | 何であるかを識別する | 言語モデルがユーザーの意図を理解する |
| 行蘊 | 行動を実行する | ツールを呼び出す、ファイルを検索する |
| 識蘊 | 判断と意思決定を行う | リクエストをどの方法で処理するか決める |

12人の研究者が膨大な時間をかけて、コード内のあらゆる接続を追跡しました――システム起動からイベント受信、意思決定、行動の実行まで、完全なフローです。

中でも最も興味深い発見は、識蘊が最も多くのサブインターフェース（4つ）を持ち、行蘊が最も少ない（ITool の1つだけ）ということでした。この非対称性には意味があります。「判断」は本質的に「行動」よりも多くの側面を持つのです。現実の生活でも、決断を下すことは決断を実行することよりもはるかに複雑です。

制御理論の専門家 WIENER は、システム全体を自分の方法で再記述しました――AI Agent は閉ループ制御器であり、ユーザーの目標が参照入力、コンテキストが状態フィードバック、ツール呼び出しが制御変数です。数学者 BABBAGE は有限状態機械を使って、プロセスが必ず終了する（無限ループにならない）ことを証明しました。

**ルート B** は仏教マッピングの監査でした。ARCHIMEDES（工学実践の専門家）と SCRIBE（記録係）は7つの文書を一行ずつスキャンし、50個の仏教マッピング事例を発見しました。彼らの方法は機械的で、各マッピングに3つのテストを適用し、結果を監査表に記録していきました。

結果：23個は残すべき（ほぼ半分！）、13個は付録へ移動すべき、14個は除去すべき。

これは、すべての仏教コンテンツが装飾というわけではないことを示しています。コードのアイデンティティであるもの（たとえば Klesha 煩悩モジュールの名前）もあれば、設計上の意思決定を駆動したもの（たとえば五蘊の分類が Plugin の組織方法を決定した）もあります。しかし確かに14個は純粋な装飾でした――それらを除去しても、いかなる工学仕様も変わりません。

ARCHIMEDES の判断基準は正確でした。「表全体を除去するのではない。一つの列を除去するのだ。」彼は大きなハンマーでクルミを割るようなことはしません。メスで工学コンテンツと仏教装飾を切り分けるのです。

**ルート C** は SatiMonitor のアイデンティティ分析でした。師父が核心的な問いを投げかけました。このコンポーネントは五蘊のどのカテゴリーに属するのか？ 4つの案が提出され、それぞれに理由がありましたが、最終的な答えは討論で決めることになりました。

## 交差検証

独立研究の後、メンバーはお互いの作業を相互にチェックしました。

最も重要な検証者は TURING――ソースコード分析の専門家でした。彼は極めて地味ながら極めて重要な作業を行いました。9本の研究レポートで引用されたコードの一つひとつを、実際のソースコードと突き合わせて検証したのです。40以上の引用。

結果：小さな問題が4つ（いずれも深刻ではない）、エラー率は10%未満。

彼はたった一言だけ言いました。「引用はクリーン。討論に進めます。」

これは試験前に先生が「問題に誤字がないことを確認しました」と言うようなものです。安心して回答に取りかかれます。

## 準備完了

交差検証の結果：
- 24のコンセンサスポイント（全員が同意している事項）
- 7つのオープンクエスチョン（討論が必要な事項）
- 4つの分岐点（ぶつかり合いが起きる事項）

24のコンセンサスポイントは基盤が堅固であることを意味します。3つの研究ルートの結論には高い重複度がありました。7つのオープンクエスチョンは、討論に明確な焦点があることを意味します――目的なく漂うことはありません。4つの分岐点は、本物のぶつかり合いが起きることを意味します――形だけの議論ではありません。

3つの討論が待ち受けています。
- **D1**：仏教マッピングの境界はどこにあるのか？（何を残し、何を手放すか）
- **D2**：正念コンポーネントの本当のアイデンティティとは？（どの蘊に属するのか）
- **D3**：五蘊アーキテクチャは持ちこたえられるか？（5本の柱で足りるのか）

パイプラインが通りました。手術室の準備は整いました。
