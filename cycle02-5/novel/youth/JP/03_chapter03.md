# 第3章：D2――正念は正念ではない

---

## あるコンポーネントのアイデンティティ・クライシス

D1 で仏教マッピングの境界問題は片づきました。D2 ではまったく異なる問いに答えなければなりません。SatiMonitor――AI Agent の行動品質を監視するコンポーネント――は、五蘊のどのカテゴリーから構成されているのか？

なぜこれが重要なのか？ 五蘊の分類は、コンポーネントがシステム内でどの位置に置かれ、何ができて何ができないかを決定するからです。分類を間違えると、温度計を「ヒーター」に分類するようなものです。期待と実際の能力がまったく噛み合わなくなります。

抽象的に聞こえますね。たとえ話で説明しましょう。

学校の廊下にモニターカメラを設置したと想像してください。このモニターは次のことができます。
- 生徒が通るのを**見る**（感覚 = 受蘊）
- 彼らが何をしているかを**識別する**――歩いている、走っている、けんかしている（認識 = 想蘊）
- 行動の品質を**判断する**――正常、異常、危険（判断 = 識蘊）

しかし、一つだけ**できない**ことがあります。行動を起こすことです。生徒を止めることも、ドアを開けることも、警報を鳴らすこともできません。観察して報告するだけです。

つまり「行蘊」（行動）の要素を持っていないのです。

## 3枚の鏡

3人の研究者が、3つの異なる角度から同じ結論に達しました。

**1枚目の鏡**――分類学者 LINNAEUS は SatiMonitor の4つの機能を列挙し、蘊の定義と一つずつ照合しました。3つの機能がそれぞれ受蘊、想蘊、識蘊に対応。4つ目の機能（スライディングウィンドウ）は単なる技術的実装で、分類には影響しません。

**2枚目の鏡**――制御理論の専門家 WIENER はもっと鋭い問いを投げかけました。「どの蘊を取り除いたら、SatiMonitor は SatiMonitor でなくなるか？」答えは――3つの蘊のどれも取り除けない。どれか一つを取れば、欠陥のあるシステムに退化してしまいます。

**3枚目の鏡**――動的システムの専門家 HERACLITUS は「退化モード」の観点から確認しました。受蘊を取り除けば、SatiMonitor はログ記録器になります。想蘊を取り除けば、生データのカウンターになります。識蘊を取り除けば、評価のない統計レポートになります。

3枚の鏡、同じ結論：受 + 想 + 識。

## ゼロコストの転換点

こんな異論が出ました。受蘊の SatiMonitor における役割は薄すぎるのではないか。本当に何かを「感じて」いるのか？

ASANGA（仏教学者）の応答は説得力がありました。「SatiMonitor はイベントを受け取るとき、ただ受動的に受け取っているわけではない。ある操作が100ミリ秒で終わった場合と10秒かかった場合とでは、SatiMonitor は異なる品質判断を下す。この判断の土台にあるのが――感覚だ。速度の感覚、連続性の感覚。感覚こそが受蘊の核心だ。」

しかし、討論に決着をつけたのは ARCHIMEDES（工学実践の専門家）のひと言でした。

「SatiMonitor が3つの蘊から構成されると宣言した場合、工学的なコストは何か？」

答え：ゼロ。

蘊の宣言はメタデータ――付加情報にすぎません。3蘊と宣言しても2蘊と宣言しても、コードは何も変わりません。ゼロコストで、より正確になる。選ばない理由がありません。

**18/20 で可決。**

これは OpenStarry 史上初の3蘊コンポーネントでした。これまですべてのコンポーネントは1つか2つの蘊にしか属していませんでした。SatiMonitor の3蘊構成は、他のコンポーネントよりも複雑な観察者であることを意味します。ただ見るだけでなく、見た後に識別し、評価するのです。

続く第3の投票はもっとシンプルでした。Agent は複数のモニターを搭載できるか？ 答えはイエスです。PluginHooks に配列スロット `monitors?: ISatiMonitor[]` を追加。異なるモニターが異なる側面に注目できるようになります。ちょうど学校に機能の異なる複数のモニターカメラを設置できるのと同じです。**20/20 全会一致で可決。**

## 問われなかった問い

D2 の終了時、記録係の SCRIBE がある観察を書き留めました。

3つのテストは多くのことを問いました。マッピングに工学的な価値はあるか？ 名前はコードの中で使われているか？ 概念は設計を駆動したか？

しかし、どのテストもこう問いませんでした。**名前は定義と一致しているか？**

もし SatiMonitor が行蘊を含まないなら――そして正念（Sati）は仏教では行蘊の活動なら――なぜ SatiMonitor は「Sati」と名付けられているのか？

この問いは D2 では誰も口にしませんでした。

それはある人物が問いかけるのを、待っていたのです。
