# 第五章：その名前はあなたにふさわしいか？

---

これは Cycle 02-5 全体で最も重要な討論でした——D4。当初の計画にはありませんでした。Master の一言がきっかけで始まったのです：

> **「梵語を使うなら、その定義に対して責任を持つ必要がある。」**

## 名前の問題

第三章で、SatiMonitor には行蘊が含まれないことがわかりました。しかし「Sati」は仏教では「正念」のこと——正念は行蘊に属します。

すると問題が浮かびます：

もし SatiMonitor が行蘊活動**ではない**なら → それは正念**ではない** → ではなぜ **Sati** と名付けられているのか？

NAGARJUNA は論理的推論（彼はこれを「帰謬法」と呼びます）でこの矛盾を示しました：

```
SatiMonitor は行蘊ではない（D2 で証明済み）
正念は行蘊である（仏教の定義）
よって SatiMonitor は正念ではない
よって Sati と呼ぶべきではない
```

## 第一の改名：ISatiMonitor → ILoopQualityMonitor

投票：13/20。新しい名前の意味は「ループ品質モニター」——このコンポーネントが実際にしていることを正確に表現しています。

## 第二の改名はさらに衝撃的でした

SUNYATA がもう一つの名前に言及しました：**IPrajna**。

般若（Prajna）は仏教で何を意味するか？**最高の智慧。すべてのものの真の姿を見通す力。**

IPrajna はプログラムの中で何をしているか？**一つの数値に 0.05 を足したり引いたりする。**

ASANGA が言った一言を、全員が忘れませんでした：

> **「これは温度の微調整つまみを核融合炉と呼ぶようなものです。」**

誰も笑いませんでした。本当のことを言っていたからです。

投票：**IPrajna → IConfidenceAuditor**、16/20。新しい名前の意味は「信頼度監査員」——このコンポーネントの実際の機能を正直に表現しています。

## 第四のテスト

D4 は一つの恒久的な新ルールを生み出しました——**定義責任テスト**：

> 仏教の名前でプログラムのコンポーネントを命名するとき、そのコンポーネントの機能はその仏教名の意味と本当に一致していなければなりません。一致しなければ、普通の工学的な名前を使います。

どの名前がこのテストに合格したか？**Moha**（自己欺瞞——プログラムの中で実際に自己欺瞞をシミュレートしている）。**Skandha**（蘊——プログラムの中で実際に五蘊分類を行っている）。

どの名前が不合格だったか？**Sati**（正念——しかし SatiMonitor は正念を実践していない）。**Prajna**（最高の智慧——しかし IPrajna は 0.05 を足し引きするだけ）。

名前はタダではありません。偉大な名前を平凡な機能に貸しても——機能が偉大になるわけではなく、名前の価値が下がるだけなのです。
