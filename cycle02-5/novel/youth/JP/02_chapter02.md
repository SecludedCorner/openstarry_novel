# 第二章：どのラベルを剥がすべきか？

---

第一回討論（D1）で議論されたのは、工程文書に貼られた仏教ラベルのうち、どれを残し、どれを剥がすかということでした。

## ルールはシンプル

チームは一つの判断ルールを作りました：

**四層分類**：
- **KEEP**（保留）：この仏教名称には工学的な意味がある
- **RELOCATE**（移動）：この仏教的内容には学術的価値があるが、工程文書に置くべきではない——付録へ移す
- **REMOVE**（除去）：これは単なる装飾で、剥がしても何にも影響しない
- **FILE REVIEW**（文書全体の審査）：半分以上が装飾の文書は、全体を再チェックする必要がある

**三項目テスト**：
1. 剥がした後、工程仕様は変わるか？ → 変わらなければ剥がす
2. この用語はソースコードで使われているか？ → 使われていれば残す
3. この仏教概念が工学的な設計判断を駆動したか？ → 駆動していれば残す

## 投票結果

十回の投票。十回とも全会一致（20/20）。少数意見ゼロ。

これは本プロジェクトの歴史上、一度も起きたことのないことでした。

なぜこれほど一致したのでしょう？それは、判断基準が討論の前にすでに確立されていたからです。全員が同じ物差しで測れば、結果は自然と一致するのです。

**剥がされたもの**：Hard rules = 戒律？剥がす。Soft rules = 善巧方便？剥がす。event-driven = 正念？剥がす。これらはすべて、工学設計が完成した後に貼り付けられた仏教ラベルでした——NAGARJUNA 自身が認めた「後貼りラベル」です。

**残されたもの**：Moha（自己欺瞞）、Sneha（我愛）、skandha（蘊）——これらはソースコード内で実際に使われている名前であり、機能も仏教的定義と一致しています。

**移動されたもの**：経典の引用、哲学的比較表——学術的価値があるので、付録へ移しました。

## でも一つ小さな問題がありました

チームは Doc 16 と Doc 31 も投票で分割しました。その後、Master がこの二つの決定を覆しました。

Master の説明：「Doc 16 と Doc 31 は工程文書に仏教の装飾を貼ったものではない——それ自体が仏教マッピング文書なのだ。『装飾の比率が高すぎる』という基準でマッピング文書を評価するのは、詩に対して『散文が多すぎる』と文句を言うようなものだ。基準の使い方を間違えている。」

チームのフレームワークには一つの次元が欠けていました——**文書の種類**です。工程文書の中の仏教は装飾です。仏教マッピング文書の中の仏教はコンテンツです。大きな違いがあります。
