# 第2章：D1――10回の全会一致

---

## 信じられない合意

D1 は歴史的な記録を打ち立てました。10の議題すべてが20票対0票で可決されたのです。

20人、6つの異なる学問分野から集まったチームにおいて、この合意レベルは極めてまれです。前回の Cycle 02-4 では55の決議があり、全会一致はその3分の1にも満たなかったのに、なぜ D1 ではそれが可能だったのでしょうか？

答えは、意見の相違がなかったからではありません。全員が同じ物差しを使ってものを測ったからです。

四層フレームワークと3つのテストは、討論の前にすでに確立されていました。投票の段階で「基準は何か」を争う必要はなく、基準はすでにあったのです。各案件をその物差しに乗せて測るだけでした。

これは、先生が事前に採点基準を公表している試験のようなものです。全員が基準を知っていれば、議論は自然と少なくなります。「この問題は何点にすべきか」を争う必要がなく、基準に照らし合わせればよいのですから。

## 装飾ラベルの除去

50個の仏教マッピングが3つのグループに分けて処理されました。

第1グループ：師父が批判した5つのマッピング。すべて除去。

哲学者 NAGARJUNA は、かつて一度もしなかったことをしました――一人称で自らの過ちを認めたのです。

「Hard Rules を śīla（戒律）と呼ぶ必要はない。その機能はルールの実行であって、修行ではない。仏教用語でラベルを貼ることは、機能を説明しているのではなく、装飾しているのだ。」

彼は正確な言葉を使いました。「後貼りラベル」と。つまり――これらの仏教用語は設計時からあったのではなく、後から付け足されたものだったのです。

数学者 PASCAL はノートにひとつの数式を書きました。ラベルの作成者自身がそれは装飾だと認めたとき、その確率はほぼ100%に近づく。これ以上の証拠は不要です。

## メスによる切り分け

第2グループ：学術的価値はあるが本文に属さない8つのマッピング。すべて付録へ移動。

ARCHIMEDES は精密なデモンストレーションを行いました。ある文書の中で、PASCAL の数学的公式は独立した工学的価値があり（本文に残す）、その横にある仏教経典の引用は設計背景であると（付録へ移動）。

「数学は残す。仏教は移す。」工学コンテンツと仏教コンテンツの境界面がメスで切り開かれ、それぞれあるべき場所に収まりました。

廃棄ではありません。引越しです。仏教のコンテンツにはそれなりの価値がありますが、工学本文の中に置くべきではありません。付録に置けば、見たい人はいつでも見つけられます。

## コードのアイデンティティ

第3グループ：すでにコードの一部となっている7つの仏教名称。すべて残す。

このグループは前の2つとはまったく異なります。前の2つは「外側に貼られたラベル」。このグループは「内側に根づいた名前」です。

TURING は何も言いませんでした。スクリーンで検索コマンドを一行走らせただけです。

結果：17ファイル。Moha（迷妄）が import 文に、class 定義に、テストに出現しています。それはコードに貼りつけられたラベルではありません――コードそのものの一部です。名前を変えるということは、17ファイルを変更するということです。

分類学者 LINNAEUS が確認しました。「これらの名前はすでにシステムのアイデンティティになっている。Moha は何かに貼られたラベルではない。Moha がそのものの名前なのだ。」

## 2つの種

D1 ではさらに2つの特殊な文書の運命が決まりました――Doc 16（仏教マッピングが80%）と Doc 31（仏教マッピングが70%）。チームはこれらを分割することに決めました。工学的価値のある段落を抽出し、残りは削除または格下げする。20/20で可決。

しかし、この決定は後に師父によって覆されました。

理由はチームの判断が間違っていたからではありません。使った基準が間違っていたのです。チームのフレームワークは「工学コンテンツ」と「仏教装飾」を区別していましたが、「工学文書に埋め込まれた仏教」と「そもそも仏教マッピング文書であるもの」を区別していませんでした。

Doc 16 は仏教が埋め込まれた工学文書ではありません。Doc 16 自体がマッピング文書なのです。「装飾割合80%」でそれを判断するのは、「散文割合80%」で詩を判断するようなものです。基準を間違えたのです。

しかし、この洞察が現れるのはもう少し先のことです。
