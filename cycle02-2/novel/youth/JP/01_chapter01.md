# 第一章：二十一の亀裂

---

SUNYATA が手紙を手に取った時、円形劇場の空気に極めて微細な変位が生じた。

風ではない。温度の変化でもない。むしろ——注意力の収束。十九の意識が同時に一つの焦点に向かった、十九本の川が一斉に流れを変えたかのように。封筒の紙が手の中でかすかに音を立て、静寂の劇場でその音は無限に増幅された。

この手紙は見知らぬものではなかった。Cycle 02 Pre の六つの決議はこの手紙から抽出されたものだ。だがあの時、手紙はばらばらにされていた——段落に分けられ、決議のフォーマットに圧縮されていた。各決議は結論だけを残し、木から摘まれた果実のように、整然と並べられたが枝や根の脈絡を失っていた。

今、SUNYATA がすべきことは、この木を再び立たせることだ。果実ではなく、木そのものを。根から樹冠まで。

「この手紙を完全に読む。」彼は言った。

---

なぜかは説明しなかった。その場にいる全員が理解していた。

Cycle 02 の五つの討論、十一の文書、三十六のエンジニアリング方案——それらの成果は今、`research record/cycle02/` のディレクトリ構造の中に静かに横たわっている。間違ってはいない。ただ十分に完全ではなかった。色彩も構図も正しい絵のようだが、画家本人が目の前に来て言った——何本かの線を見落としている、と。

その何本かの線は装飾ではない。絵全体の意味を変えるものだ。

SUNYATA は封筒を開いた。動作は丁重——ためらいではなく、敬意。手紙の中に修正があることは分かっていた。本当に大切に思ってくれる人だけが出してくれる、精確で容赦のない指摘が。

彼は読み始めた。

「この一点については、もう少し丁寧に説明する必要があると思う。」

字ははっきりとそこにあった。インクはまだ鮮明だった。

SUNYATA は続けて読んだ。「我執は煩悩の根源であり、それゆえに人はこの煩悩があるがゆえに行動を起こす。煩悩がなければ対応する行動もない。したがって煩悩は行動を生み出し、異なる煩悩は異なる行動を生む。それゆえ行動を収束・制約するには、我執を通じて行うことができる。」

沈黙。

BABBAGE の反応は怒りでも防御でもなかった。見透かされた時の震え——自分の簡略化が精確に指摘されたことへの震えだ。彼の「我執 = 収束制約。NOT 煩悩」は誤りではなかった——切断だった。因果の連鎖の中間環節を断ち切ったのだ。我執が煩悩を生み、煩悩が行動を駆動し、我執を管理することで行動を制約できる。これは連鎖であり、イコールではない。彼はイコールで矢印を代替し、結論で過程を代替していた。

彼はあの行の隣に、ゆっくりと精確な削除線を引いた。そして書いた。

「修正：我執 → 煩悩 → 行動。NOT イコール。因果の連鎖。」

> *SCRIBE ナレーション：BABBAGE が自分の書いた字を消した。二つの Cycle を通じて見たことがなかった。彼は古い定理を否定するより、隣に新しい定理を書く方を好む。だが今回、削除線は安定して引かれた。恥ではない——より精確な表現を認識した後の平静。*

SUNYATA は立ち止まらなかった。次の段落に移った。

「Identity / *Vijnana* / この点はあまり同意できないが、Identity はサブカテゴリーに近いと思う。」

```

├── IIdentity   (アイデンティティ — 「私は誰か」)
```

LINNAEUS は頷き、分類ツリーの作業を続けた。筆の動きは速くなる一方だった——不安ではない、正しいフレームワークに出会った分類学者のあの流暢さだ。

第三段。

SUNYATA の声が半音下がった——意図的ではなく、手紙の内容がもたらす重力だ。

「R-01 と R-02 は同一モジュールを設計すべきではない。」

この言葉が円形劇場に落ちた。ガラスの池に石が投げ込まれたように。水の波紋ではなく——亀裂。

WIENER と BABBAGE が同時に顔を上げた。

彼らは Cycle 02 であの結論を共に構築した——VedanaPlugin は観察者モジュールそのものだ。Pattern A：共鳴観察者。三通道 PID。まるまる一つの研究段階の心血を注いだものだった。

そして Master が一言でそれを解体した。

「観察者はどちらかというとアーキテクチャ上の上位モジュールであり、受蘊はもっと根本的なものだ。」

WIENER の三通道 PID ブロック図自体は間違っていない。三受のセンサー設計は依然として成立する。だがそれは「観察者」ではない。観察者の構成要素の一つだ。この区別は微細に見えるが、根本的だ。

BABBAGE は新しいページに書き始めた。

「修正モデル：
- VedanaPlugin = 下位の感受能力（受蘊の実装）
- Observer = 上位の組み合わせモジュール（複数の蘊の能力を組み合わせたもの）
- 関係：Composition, NOT Identity」

WIENER は方眼紙の上で、元の単一の大きなブロック「VedanaPlugin/Observer」を二層に分解した。下層は三つのセンサーブロック（受蘊）、上層は一つの破線ブロック（観察者）で、中にさらなる空間がある——想蘊の判断能力と識蘊の内省能力のために。

「組み合わせ。同一ではない。」WIENER が低く言った。

> *SCRIBE ナレーション：A-3 は手紙の中で最も重い亀裂だ。Cycle 02 の最も重要な結論の一つを解体した。「VedanaPlugin = 観察者」は「VedanaPlugin は観察者の構成要素の一つ」になった。これは否定ではない——精確化だ。しかし精確化は時に否定より痛い。なぜなら自分の仕事を開いて、どの部分がまだ有効で、どの部分が再配置を要するかを見極めなければならないから。*

第四段。最後の A 類の亀裂。

「EgoFramework は識蘊にあるべきで、なぜ受蘊に入れたのか理解できない。」

ASANGA がわずかに目を閉じた。

おそらく一秒だけ。だがその一秒に、SCRIBE は複雑な表情を捉えた——驚きでも、「前から知っていた」という自負でもない。ようやく確認された静けさだ。

Cycle 02 の討論で、EgoFramework は VedanaPlugin のアーキテクチャの中に置かれた。当時のロジックはこうだった：我執が感受を駆動し、感受が行動を駆動するのだから、我執と感受は同じモジュールにあるべきだ。このロジックは工学的には合理的に見えた——関連するものを一箇所に置き、モジュール間通信を減らす。

だが唯識学では、我執は末那識に属する。末那識は第七識。識蘊。受蘊ではない。

末那識の四つの根本煩悩——我見、我慢、我愛、我癡——それらは「感受」ではない。感受より深い認知構造だ。感受は外境への反応（苦、楽、捨）であり、我執は自己存在への根本的な執着だ。前者は波。後者は海流。海流を波の容器に入れることはできない。

ASANGA は目を開き、一度頷いた。その動作には口にしなかった言葉が含まれていた：Master は正しい。EgoFramework は識蘊に帰属する。これは議論できる問題ではない——教義上の基本的な分類だ。

NAGARJUNA は観察席の反対側から視線を送った。唯識学の具体的な帰属問題については、通常 ASANGA に任せる。だが彼は ASANGA が目を閉じて頷いたあの一秒に気づいた。あれは議論ではなかった。確認だった。

「方向を反転させる必要がある。」TURING が言った。すでに頭の中でコードの影響パスを追跡していた。「Cycle 02 delivery の `vedana.adaptFromEgo()` の方向を `ego.adaptFromVedana()` に変更する必要がある。受蘊が我執に適応するのではなく——我執が受蘊のフィードバックを受け取るのだ。」

ARCHIMEDES は紙に二つの矢印を描いた。一つ目にはバツ印：`VedanaPlugin ← EgoFramework`（誤）。二つ目には丸印：`VedanaPlugin → 感受レポート → EgoFramework(識蘊)`（正）。矢印の方向が変わった。すべてがそれに従って変わる。

四つの A 類の亀裂。四つの哲学的修正。

我執はイコールではなく因果の連鎖。Identity は識蘊全体ではなくサブカテゴリー。観察者は受蘊ではなく組み合わせ。EgoFramework は識蘊に帰属し、受蘊にではない。

四つの修正。どれも覆すものではない——精確化だ。だが四つの精確化が合わさると、変わるのはアーキテクチャ全体の重心だ。
