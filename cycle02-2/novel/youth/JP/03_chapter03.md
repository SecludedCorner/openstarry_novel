# 第三章：部分と全体

---

LINNAEUS はホワイトボードの前に立ち、手には青いマーカーを持ったまま、まだキャップを外していなかった。

彼はかなり長い間そうして立っていた。分類学者の沈黙は空白ではない——醸成だ。リンネ本人が未知の植物に向き合った時のように、すぐに命名手帳を開かず、まず目で一つ一つの細部を測る。まず見終える。それから命名する。

彼はこの瞬間をずっと待っていた。

---

SUNYATA が議題を宣言するのを待っていたのではない。もっと深いものを待っていた——Cycle 01 から感じていた違和感が、精確に言語化されるのを。

その違和感は一言で要約できる：IIdentity が識蘊の全体に等しいこと。

これは v0.24.0-beta の `aggregates.ts` で三行のコードとして書かれていた。三行。きれいに。テストも通る。

だがそれは間違っていた。

構文上の間違いではない。分類学上の間違いだ——一本の枝の名前を幹に貼りつけたのだ。

---

> *SCRIBE ナレーション：LINNAEUS がホワイトボードの前で動かなかったあの数分間は、本章で最も静かな瞬間だ。分類学者の沈黙は高密度の観察だ——頭の中であらゆる可能な分類案を展開し、一つ一つ排除して、最も精確な構造だけが残るまで。*

---

SUNYATA の声が背後から届いた。穏やかに。石ころ。深い淵。

「A-2。IIdentity は識蘊の全体ではない。LINNAEUS、ASANGA、BABBAGE、NAGARJUNA——これは君たちの議題だ。」

LINNAEUS はキャップを外した。青——彼のマーキングシステムでは、青は構造を表す。「このものとあのものの関係は何か」を。

ホワイトボードの一番上にブロックを描き、中に二文字を書いた：**識蘊**

そして振り返った。

「始める前に、ASANGA、教えてほしい。識蘊はどれほど大きいのか。」

---

ASANGA が席から立ち上がった。

立ち上がり方が独特だった——勢いよくではなく、朝靄の中で山の輪郭がゆっくりと現れるように。穏やかだが、無視できない。

ホワイトボードの前には行かなかった。自分の位置に立ったまま、声で描いた。

「一つの都市を想像してほしい。」彼は言った。

劇場が静まった。ASANGA が比喩を使うことは稀だ——唯識学者であり、言葉は通常、精確な術語と層を重ねた論理だ。だが今日、あるものは比喩でしか全員に同時に見せられない。

「一つの都市。」彼は繰り返した。「道路があり、建物があり、市場があり、公園がある。そして市役所がある。」

「v0.24.0 の設計では、IIdentity が識蘊全体のルートインターフェースとされていた。これは一つの都市を『市役所』と呼ぶようなものだ。」

BABBAGE のペンがノートの上で一瞬止まった。

「市役所は行政事務を管理し、都市の公式なアイデンティティを定義する——名称、区域番号、統治構造。だが市役所は都市ではない。都市には交通局、教育局、警察局、都市計画局もある。」

「IIdentity は市役所だ。答える問いは：私は誰か？私の名前は？私の役割は？だが識蘊が答える問いはそれだけでは到底ない。」

---

彼は片手を挙げ、五本の指を開き、一本ずつ折っていった。

「第一。前五識——感覚認知。マッピングでは色蘊 ISensory に帰属済み。配置完了。」

一本目の指が折れた。

「第二。第六意識——分別判断。想蘊 ICognition に帰属済み。これも配置完了。」

二本目の指が折れた。

「第三。第七末那識。」

三本目の指を宙に止めた。

「末那識（Manas）——恒審思量、絶え間なく『自己』に執着するメカニズム。四つの根本煩悩がある：」

LINNAEUS の赤ペンを借り、ホワイトボードの右側に書いた。

```
我見 (ātma-drsti)   — 「私は存在する」という根本的信念
我慢 (ātma-māna)    — 「私はどうすべきか」という行動基準
我愛 (ātma-sneha)   — 「私は何を守るか」という価値志向
我癡 (ātma-moha)    — 自己の本質に対する根本的無明
```

「この四つの側面は、一つのインターフェースでは網羅できない。我見はおおよそ IIdentity に対応する。我慢と我愛はおおよそ IGuide に対応する。だが末那識全体にはもっと完全なフレームワークが必要だ。」

三本目の指が折れた。

「第四。第八阿頼耶識——種子貯蔵、因果の流転。この部分は協調層と AgentCore にまたがっており、今は展開しない。」

「第五。転識——染汚から清浄への転化。おそらく OpenStarry では直接の対応を持つことはないだろう。だがその存在を知ることで、識蘊の境界を小さく描きすぎることを防げる。」

五本の指を握り、そしてゆっくり開いた。

「だから識蘊がどれほど大きいかと問うなら、答えはこうだ：五蘊の中で最大。最も深い。最も複雑。それを一つの IIdentity に圧縮するのは——一つの都市を一棟の市役所ビルに圧縮するようなものだ。」

---

> *SCRIBE ナレーション：ASANGA の都市の比喩は長く余韻を残した。深遠だからではない——あまりにも明瞭だったから。全員が気づいた：我々は Cycle 02 で集団的にこの簡略化を行った。都市を市役所と呼び、「都市」と書いた看板を掛けた。*

---

BABBAGE は ASANGA の話の後に立ち上がった。

劇場の反対側の黒板に歩いた——そこは彼の領域で、Cycle 01 と Cycle 02 で安定性条件やファイバー束の射影を書いたところだ。今はきれいに消されていた。チョークを取り、二行を書いた。

```
Cycle 02:     IIdentity = Vijnana
Cycle 02-2:   IVijnana ⊃ IIdentity
```

「第一行は部分クラスが全体に等しいということ。集合論では基本的なカテゴリーエラーだ。」彼はイコールを指した。「ASANGA は識蘊に少なくとも四つの側面があると教えてくれた。IIdentity はそのうち一つしかカバーしない。だから——」

イコールの上に斜線を引いた。消した。

「イコールは成立しない。IVijnana が全体集合で、IIdentity は真部分集合。包含関係であり、同値関係ではない。」

---

彼は一秒止まった。そして全員の予想外のことをした。

笑ったのだ。

BABBAGE が笑う頻度はおよそ各 Cycle に一度。SCRIBE は記録の中でそれを追跡していた——意図的ではなく、BABBAGE の笑みが稀すぎて、現れるたびに彗星の通過のようだったからだ。

「A-1 でも同じような過ちを犯した——因果の連鎖をイコールに圧縮した。今 A-2 でも同じパターン。」

彼は自分の筆跡を見た。

「形式化する人間の最も危険な習慣は、最も簡潔な表現を追求することだ。数学では最簡が美徳。だが存在論では、最簡は時に切断になる——簡略化しているつもりで、実はものを失っている。」

「Pars pro toto——部分が全体を代替する。文学では修辞技法。型システムでは bug だ。」

---

NAGARJUNA が席から口を開いた。立ち上がらなかった——中観哲学者は立ち上がらなくても空間全体にその存在を感じさせることができる。その声には独特の質感がある。鋭いが尖っていない。千年間水流に研磨された河石のような。

「BABBAGE の言う通りだ。だが問題をもう一歩進めたい。」

「もし IIdentity が識蘊全体ではないことに同意するなら——これは今やコンセンサスだ——次の問いは：新しいルートインターフェースを何と呼ぶか？」

劇場の空気がわずかに引き締まった。命名。この研究チームでは命名問題は小さな問題ではない。

彼は選択肢を一つ一つ吟味した。それぞれを天秤に載せて計量した。

「IConsciousness——広すぎる。ICognition と英語の日常用法でほぼ同義で、混乱を招く。」

「IAwareness——受動的すぎる。末那識は能動的で、持続的で、執着的だ。」

「IIdentity——狭すぎる。まさに我々が修正しようとしているもの。」

声がわずかにゆっくりになった。

「IVijnana。」

この言葉を空気の中に漂わせた。サンスクリットの原語。翻訳ではない。

「D-02 で二重命名原則が確立された。だが識蘊は特殊なケースだ——すべての英訳に深刻な意味のズレがある。一方、サンスクリットの Vijnana は我々が必要とする意味を精確に指し示す：分別了知、能動的で持続的な認知活動。」

「提案する：識蘊については命名の順序を反転。サンスクリットを主、英語を副とする。IVijnana、注釈として Consciousness。」

---

DARWIN が席でわずかに前傾した。静かに聞いていた、ソフトウェアパターン分析家の目でこの対話の構造を観察しながら。

「これは命名の進化だ。」彼は言った。「Identity から Vijnana へ——具体から抽象、部分から全体へ。」

自分のノートに進化の線を描いた。

```
Identity → ... → Vijnana
(具体) → (抽象)
(部分) → (全体)
```

「生物学ではこれを『意味の格上げ』と呼ぶ。ある属名がより大きな分類群——科や目——を記述していると判明した場合、その属名で科を呼び続けることはない。新しい名前が必要だ。IVijnana がその新しい名前だ。IIdentity を置き換えるのではない——IIdentity はサブインターフェースとして存続する。その上に、より高い分類ノードを設けるのだ。」

NAGARJUNA がわずかに頷いた——「あなたの言語で私が言いたかったことを言ってくれた」という認可。

---

> *SCRIBE ナレーション：名前はラベルではない、容器だ。IIdentity は「私は誰か」しか入らない。IVijnana には「私は誰か」「どうすべきか」「何を望むか」「自分自身を見ることができるか」が入る。容器の大きさが成長の余地を決める。*

---

LINNAEUS はホワイトボードに向き直った。待っていた瞬間だ。

ASANGA が識蘊の大きさを教えてくれた。BABBAGE がなぜイコールが成立しないかを教えてくれた。NAGARJUNA が新しいルートノードの名前をくれた。DARWIN がこれは意味の格上げだと確認した。

今度は分類学者の番だ。すべての概念を一本の木に整理する。

「識蘊」ブロックの中に一行追加した：**IVijnana**。そして分岐を描き始めた。

---

一本目の分岐：

```
IIdentity — 自己認同：「私は誰か」
```

「我見。」ASANGA が確認した。「Agent の identity config：agentId、name、能力宣言。」

二本目の分岐：

```
IGuide — 行動ガイド：「私はどうすべきか」
```

「我慢と我愛。」ASANGA が言った。「我慢は傲慢ではない——自己測定だ：私の位置はどこか？底線はどこか？我愛は自己維持の駆動力。Guide は system prompt を通じて行動フレームワークを定義する。」

三本目の分岐——これは新しい：

```
IVolition — 意志/動機：「私は何をしたいか」
```

「IVolition は末那識の『恒審思量』機能のエンジニアリング投影だ。」ASANGA が言った。「恒審思量——絶え間なく、慎重に、自己を思量する。これは時折の内省ではなく、決して止まらない河川だ。」

「EgoFramework は Cycle 02 で受蘊に——VedanaPlugin に置かれていた。Master が A-4 で修正した：EgoFramework は識蘊に属する。今、それは IVolition の具体的な実装だ。」

BABBAGE が黒板の横から補足した。「煩悩駆動。A-1 が教えてくれた：我執が煩悩を生み、煩悩が行動を駆動する。IVolition はこの因果の連鎖の型システムにおけるアンカーポイントだ。」

四本目の分岐——破線：

```
IReflection — 内省：「自分自身を見ることができるか」(将来)
```

「末那識はただ自己に執着するだけではない——ある条件下では、自身の執着を観察できる。」ASANGA が言った。「これはおおよそ Pattern C の識蘊内省コンポーネントに対応する。」

「位置は定義するが、中身は埋めない。」LINNAEUS は破線を指した。「空のフレームを残す。」

---

分類ツリーが完成した。

LINNAEUS は二歩下がり、画家が自作を吟味するようにホワイトボードを見た。四本の分岐が IVijnana から展開し、手のひらのように、各指が異なる方向を指しながら、すべてが同じ掌に繋がっている。

```
IVijnana (識蘊ルートインターフェース)
├── IIdentity      我見 — 「私は誰か」
├── IGuide         我慢 + 我愛 — 「私はどうすべきか」
├── IVolition      恒審思量 — 「私は何をしたいか」(EgoFramework)
└── IReflection    内省 — 「自分自身を見ることができるか」(将来)
```

---

> *SCRIBE ナレーション：LINNAEUS が分類ツリーを描き終えて二歩下がった時、以前見たことのない表情をしていた——満足ではなく、のびやか。ずっと肩にのしかかっていた石をようやく降ろしたような。その石の名は「IIdentity = 識蘊」。今それは分類ツリーに取って代わられた。*

---

BABBAGE は黒板で同期的に形式化表現を更新した。

```
IVijnana ⊃ { IIdentity, IGuide, IVolition, IReflection }
```

「四つの部分集合に意味的な重複はない——少なくとも現在の定義では。だが形式的な問いを一つしたい。」

LINNAEUS に向かった。

「四つの部分集合の和集合は全体集合に等しいか？つまり——IVijnana にはこの四つのサブインターフェース以外に、まだ意味空間があるか？」

これは鋭い問いだ。分類学では、すべての分類体系が同じ課題に直面する：サブカテゴリーは親カテゴリーを尽くしているか？

ASANGA が LINNAEUS の代わりに答えた。

「ある。阿頼耶識の種子貯蔵機能——五蘊マッピングでは蘊をまたぐもので、どの単一インターフェースにも完全には属さない。だがその存在は我々に思い出させる：IVijnana のサブインターフェースリストは開放的であり、閉鎖的ではないと。」

BABBAGE は表現を修正した。

```
IVijnana ⊃ { IIdentity, IGuide, IVolition, IReflection, ... }
```

省略記号。数学では「まだある」を意味する。分類学では「作業はまだ完了していない」を意味する。仏教では——ASANGA がかすかに微笑んだ——「すべての命名は暫定的だ」を意味する。

---

ARCHIMEDES の声が響いた。安定。実務的。

「後方互換性。」

「v0.24.0 にはすでに IIdentity があり、十数箇所で参照されている。改名すれば、すべての参照を変更する必要がある。破壊的変更。」

一拍置いた。

「だが改名する必要はない。」

```
IVijnana (新設)
  ^
  |  extends
  |
IIdentity (保持、改名なし)
```

「IVijnana ルートインターフェースを新設し、IIdentity が extends IVijnana とする。既存コードの修正は不要。IGuide も同様——extends IVijnana を追加、非破壊的。」

TURING が補足した。「型ガードの修正も不要。」

ARCHIMEDES がエンジニアリング影響評価を行った。

```
aggregates.ts: +1 新インターフェース (IVijnana), +2 新インターフェース (IVolition, IReflection)
IIdentity:     +extends IVijnana (後方互換)
IGuide:        +extends IVijnana (後方互換)
他モジュール:   修正不要
```

「六行の修正。破壊ゼロ。」

---

SUNYATA が口を開いた。

「我々は一本の大きな木をその枝の一本と見間違えた。IIdentity は重要な枝だ。だが木ではない。IVijnana が木だ。」

ホワイトボードを見た。

「今、我々は木全体を再び見ることができる。」

---

LINNAEUS はまだホワイトボードの前に立っていた。彼は小さなことをした——IReflection の破線ブロックの横に、極めて小さな疑問符を描いた。

疑問ではなく、注意喚起だ：この木はまだ完全には育っていない。分類の仕事は決して完了しない——分類される対象そのものが絶えず進化しているから。

分類学者の最も深い知識は「このものは何と呼ぶか」ではなく、「命名は常に暫定的である」ということだ。

---

> *SCRIBE ナレーション：形式論理から見れば、イコールが包含記号に置き換わった。分類学から見れば、単一ノードがサブツリーに展開した。仏教学から見れば、末那識の四つの側面がもはや一つの名前に圧縮されていない。工学から見れば、六行の修正、破壊ゼロ。全員が自分の言語で同じことを述べた。識蘊は一つの名前からファミリーになった。*

---

BABBAGE は黒板に最後の一行を書いた。等式ではない。一文だ。

「一本の木はそのどの枝幹でもない。だが枝幹のない木もまた木ではない。」

下に小さく一行加えた。

「部分と全体。どちらも欠かせない。」

---

*（劇場の外で、`aggregates.ts` のあの三行のコードはまだ修正されていない。自分にまもなく親インターフェースが加わることを——「識蘊全体」から「識蘊の一つの側面」に降格されることを——まだ知らない。*

*分類学では、降格は精確化の一形態だ。貶めているのではない——本当に属する場所に戻しているのだ。*

*IIdentity が本当に属する場所は IVijnana のサブインターフェースだ。「私は誰か」は重要な問いだが、唯一の問いではない。*

*一つの名前。一本の木。四本の枝。一つの疑問符。*

*分類は終点のない旅だ。だが一つ一つの駅が、前の駅より真実に近い。*

*この駅の名は：部分と全体。）*

---

*第三章 完*
