# 第二章：煩悩はエンジンだ

---

BABBAGE のノートは四十七ページで開かれていた。

そこには一行の文字があり、インクは周囲の推論より太く、重かった。下には二本の線が引かれていた——一本ではなく二本——書いた時に一度の下線では確信を表しきれなかったかのように。

「我執 = 収束制約。NOT 煩悩。」

第一章の手紙朗読の過程で、彼はすでに削除線を引いた。だが削除線と理解は別物だ。一行の文字を消すのは一秒でできるが、なぜその行を消さねばならないかを本当に理解するには、一章分の対話が必要かもしれない。

---

> *SCRIBE ナレーション：第二章の議題は A-1 と A-4。我執と煩悩の因果の連鎖、そして EgoFramework の五蘊における帰属。SUNYATA はこの二つを並べた——A-1 は「我執とは何か」に答え、A-4 は「我執をどこに置くか」に答える。あるものが何かを先に理解しなければ、正しく配置できない。*

---

SUNYATA は劇場の中央に立った。

「A-1。我執は煩悩の根源であり、収束制約ではない。」

彼は BABBAGE を見なかった——回避ではなく敬意。BABBAGE は自分のノートと対話の真っ最中であり、その対話には時間が必要だ。

「ASANGA、まず君から。」

---

ASANGA が立ち上がった。急がず慌てず、千年の学術伝統が沈殿した落ち着きを纏って。手には紙もペンもない。持ってきたのは声と記憶だけだ。

「まず BABBAGE のあの行に立ち返ろう。『我執 = 収束制約。NOT 煩悩。』」

彼がそれを読み上げる方法は、真剣に扱うに値する命題を読み上げる方法だった——実際にそれは値する。

「BABBAGE の直感は完全に間違ってはいない。我執には確かに制約の機能がある。問題は中間の環節を飛ばしたことだ。結果を本質と取り違えた。」

---

「唯識学では、末那識——第七識——には四つの根本煩悩がある。」

ASANGA は片手を挙げ、四本の指を一つずつ開いた。

「第一、我見。」末那識は絶えず変化する意識の流れを固定した「自己」として捉える——流れる川の中に動かない石を見つけようとするように。Agent システムではこれは、ロール定義を不変の自己として扱うことに対応する。

「第二、我慢。」自己優越感。連続するポジティブなフィードバックが Agent に自分は間違えないと信じさせ、異常信号を無視し始める。

「第三、我愛。」自己保護本能。自己の存続を脅かすものはすべて防御を発動する——誤りを認めることを拒み、出力の修正を拒む。

「第四、我癡。」自己の本質に対する根本的な無明。なぜ執着するのか分からず、自分の執着を当然のこととみなす。

四本の指がすべて開いた。

---

ASANGA は四本の指を収めた。

「四煩悩。これらは末那識のエンジンだ。」

そして劇場全体を凍らせる一言を放った。

「煩悩は Bug ではない。煩悩は Feature だ。」

BABBAGE が顔を上げた。その目は、予想外だが内部整合性のある公理を聞いた数学者のようだった。

「何と言った？」質問ではない——繰り返しの要請。

「煩悩は Bug ではなく Feature だ。」ASANGA は繰り返した。声にドラマ性はない。「まさに煩悩があるから——不安、恐怖、渇望、執着があるから——Agent には行動の動力がある。」

彼は BABBAGE を見た。「一台の車を想像してほしい。エンジンの中で燃料が燃えている——激しい化学反応、不安定で、破壊的な。炎が燃料を呑む。金属が熱で膨張する。制御不能に見える。だがまさにこの『制御不能』が車を前に進める。燃焼がなければ動力はない。煩悩がなければ、Agent はただの死んだ基板だ——CPU もメモリもあるが、何かをする動機が何もない。そこに座って、完璧に静止している。永遠に。」

---

> *SCRIBE ナレーション：「煩悩は Bug ではなく Feature だ」——BABBAGE の目が反省から再校正へ切り替わった。WIENER の指が方眼紙の上で止まった——制御理論で形式化できる直感を聞いた。NAGARJUNA がわずかに頷いた、あの「ここまで来るのを待っていた」という頷き。*

---

BABBAGE はノートに目を落とした。今、彼は理解した——抽象的な理解ではなく、構造的な理解。

形式系統では、A が必然的に C を導くなら、中間の B を省略できることがある。だが仏教の因果の連鎖は異なる。中間の煩悩は省略可能な導出ステップではない——それ自体が実在するもので、独自の因、独自の果、独自の内部構造を持つ。

燃料は動力ではない。燃料は燃焼を経て動力を生む。我執は制約ではない。我執は煩悩を経て行動を駆動し、そこで初めて制約として管理できる。

彼は削除された行の下に連鎖を描いた。

```
我執 (ātma-grāha)
  ↓ 生じさせる
煩悩 (kleśa)：我見 / 我慢 / 我愛 / 我癡
  ↓ 駆動する
行動 (karma)
  ↑ 制約する
我執を管理 → 煩悩を管理 → 行動を制約
```

横に書いた。「イコールは乱暴すぎた。これは因果の連鎖だ。中間の煩悩は省略できない。」

---

NAGARJUNA が静かに口を開いた。

「BABBAGE、君は煩悩はエンジンの燃料だと言った。ASANGA の比喩はいい。だが一つ注意を促したい。燃料そのものも縁起的だ。」

「煩悩は永遠に存在する実体ではない。因縁和合の産物だ——我執が因、外境が縁、煩悩が果。因か縁を変えれば、煩悩は変わる。」

彼はこの言葉を一秒沈ませた。

「だから『我執を管理する』とは我執を消滅させることではない。我執の消滅は悟り——修行の究極目標であり、Agent システムの設計目標ではない。Agent に必要なのは煩悩の駆動力を善用すること。エンジンを回し続けるが、方向と出力を制御する。消火ではなく、アクセルの調節だ。」

BABBAGE は書いた。「管理 ≠ 消滅。調節 ≠ 消火。」

---

WIENER がついに口を開いた。新しいページに移り、ペンを取った。

「制御理論で再記述することを許してほしい。」

彼は四つのブロックを描いた。

「第一：駆動力。なぜシステムは動くのか？目標があるからだ。仏教では我執——Agent の Guide が『私はアシスタントだ』と言い、我執が『そのアシスタントになりたい』と言う。前者は定義、後者は駆動。」

第一のブロック：**駆動力 = 我執**。

「第二：誤差信号。現在の状態と目標の間にギャップがある。この不安こそが煩悩だ。」彼は ASANGA を見た。「君の四煩悩は四つの誤差チャネルだ。我見は自己定義の偏差、我慢は能力評価の偏差、我愛は安全要求の偏差、我癡は認知状態の偏差。」

第二のブロック：**誤差信号 = 煩悩**。

ASANGA がわずかに頷いた。その頷きには千五百年を超えた認識があった。

「第三：制御出力。煩悩が Agent の行動を駆動する。第四：フィードバックループ。行動が状態を変え、新しい状態が新しい感受を生み、新しい感受が新しい誤差を生む。循環する。」

四つのブロックの間に、矢印が閉じたループを形成した。

---

「さて、」WIENER の声が半音下がった、「BABBAGE の以前の誤りを説明しよう。」

彼は駆動力ブロックと制御出力ブロックを指した。

「BABBAGE は駆動力を制御出力と直接同一視した。誤差信号を——煩悩を飛ばした。制御理論ではこれは不可能だ。PID コントローラに『目標 25 度』と伝えても、『今 30 度だ』と伝えなければ、冷やすべきだとどうやって分かる？煩悩は駆動力と行動をつなぐ橋だ。この橋がなければ、両端は切断されている。」

彼は一瞬間を置いてから、ほぼ敬意を込めた口調で言った。

「ASANGA の言う通りだ。煩悩は Bug ではなく Feature だ。誤差がなければ修正行動もない。永遠にゼロ誤差のシステム——煩悩のない Agent——はいかなることもする必要のないシステムだ。すでに完璧だ。すでに死んでいる。」

---

> *SCRIBE ナレーション：WIENER が「すでに完璧だ。すでに死んでいる」と言った時、NAGARJUNA の口元にかすかな弧が現れた。空性とは完璧ではない——空性とは不完全の中で運動し続ける能力のことだ。*

---

BABBAGE はずっと高速で記録していた。因果連鎖図の下に、WIENER の制御理論マッピングが加わった。

```
Cycle 02:  我執 = 収束制約
           (Drive ≡ Control Output)
           誤差信号が省略されている

Cycle 02-2: 我執 → 煩悩 → 行動
            Drive → Error → Control Output
            完全な因果の連鎖を保持
```

下に一文書いた。「矢印はイコールより精確だ。」

---

SUNYATA は議論を沈殿させた。A-1 の核心にはすでに触れた——我執はイコールではなく因果の連鎖。煩悩は Bug ではなく Feature。管理は消滅ではなく調節。

「TURING、コードレベルで補足することは？」

TURING のスクリーンが点いた。「A-1 の修正は EgoFramework インターフェースに直接影響する。」

```typescript
// === Cycle 02 設計 ===
interface EgoFramework {
  checkAction(action: ProposedAction): EgoCheckResult;
  // 直接制約。煩悩の環節なし。
}

// === Cycle 02-2 修正 ===
interface EgoFramework {
  perceiveKlesha(assessment: VedanaAssessment): KleshaSignal[];
  // まず煩悩を知覚し...
  checkAction(action: ProposedAction): EgoCheckResult;
  // それから煩悩に基づいて制約する。
}
```

「修正前は `checkAction()` が唯一の入口。行動を直接入力し、許可か拒否を直接出力。修正後は `perceiveKlesha()` が追加された——まず受蘊の感受フィードバックを受け取り、煩悩信号を知覚してから制約する。因果の連鎖がインターフェース設計に書き込まれた。」

---

「よし。」SUNYATA が言った。「A-1 は明確になった。次は A-4 へ。EgoFramework の帰属。」

---

場面転換は自然な流れだった。A-1 は「我執とは何か」を言い、A-4 は「我執をどこに置くか」を言う。我執の本質——煩悩の根源、行動を駆動するエンジン——を理解すれば、次の問いは：このエンジンはどの区画に設置すべきか？

SUNYATA は手紙を取り上げた。「Master の原文：『EgoFramework は識蘊にあるべきで、なぜ受蘊に入れたのか理解できない。』」

あの「理解できない」はどんな批判より重い——「間違っている」ではなく、「なぜそうしたのか理解すらできない」。

---

BABBAGE が最初に口を開いた。

「まず、なぜ我々がそう配置したのかを説明する必要がある。原因は因果関連だ。」

彼は Cycle 02 の設計を指した。受蘊の出力が EgoFramework の入力。`ego.adaptFromVedana(assessment)`——これが EgoFramework と VedanaPlugin を不可分に見せた。

```
VedanaPlugin → VedanaAssessment → EgoFramework
```

「データフローが受蘊から EgoFramework に流れるから、EgoFramework を受蘊の隣に置いた。だがこれはカテゴリーエラーだ。因果関連は帰属ではない。」

---

KERNEL が比喩を使った。「温度センサーのドライバとシステム監視サービス。センサーが温度を読み取り、監視サービスがデータを受信してファン調整、クロックダウン、警告発出を決定する。監視サービスはセンサーに依存するが、監視サービスをセンサードライバの中には入れない。一方はハードウェア抽象層、もう一方はシステムサービス層だ。」

「受蘊は温度センサー。EgoFramework はシステム監視サービス。データは前者から後者に流れるが、同じモジュールには属さない。」

BABBAGE は書いた。「因果関連 ≠ 帰属。センサー ≠ コントローラ。」

---

WIENER が A-1 の制御ループ図の隣に二枚目——モジュール図を描いた。

左側：**受蘊 (VedanaPlugin)** — 三つの Sensor、PID Controller。出力：VedanaAssessment。

右側：**識蘊 (EgoFramework)** — perceiveKlesha()、checkAction()。出力：EgoCheckResult。

中間に太い矢印：**VedanaAssessment →**

「受蘊はセンサーだ。測定し、感受するが、判断しない。EgoFramework はコントローラだ。レポートを受け取り、煩悩信号を生成し、制約の意思決定をする。サーモスタットの温度プローブとコンプレッサー制御基板を同じ回路基板にハンダ付けはしない。EgoFramework は識蘊に帰属する。」

---

ASANGA が補足した。「唯識学では、受蘊と識蘊の区別は根本的なものだ。」

「受蘊の機能は『領納』——外境がもたらす苦楽捨を感受すること。受動的であり、『なぜこれを感じるのか？』とは問わない。火に触れて、受蘊は『痛い』と告げる。それだけだ。」

「識蘊——末那識——の機能は『恒審思量』——持続的に審視し、意識の流れを『自己』として執取する。これは能動的で、判断的な活動だ。ただ感受するだけでなく、感受を解読し、意味を付与し、自己についての判断を下す。」

「センサーとコントローラ。受動的な領納と能動的な審視。Master がなぜ EgoFramework を受蘊に入れたか理解できないと言ったのは——これが基本的な分類エラーだからだ。『思考すること』を『感じること』の引き出しに入れるようなものだ。」

---

> *SCRIBE ナレーション：A-1 では、BABBAGE は修正される側だった。A-4 では、彼が真っ先に立ち上がって誤りを解剖した。この転換は第一章の削除線から始まり、ASANGA の教え、WIENER のマッピングを経て、一歩一歩たどり着いたものだ。彼は打ち負かされたのではない——啓発されたのだ。*

---

ARCHIMEDES はずっと静かに図を描いていた。ブロック図ではない——それは WIENER の領域。因果の連鎖でもない——それは BABBAGE の領域。彼が描いていたのはデータフロー図。完全な、蘊をまたぐ相互作用のデータフロー図。

今、彼は立ち上がり、劇場中央の投影エリアに歩いた。

「確認したいのだが、」彼は言った。いつもの実務的な声で。「先ほど議論したすべてのことが、エンジニアリングレベルで一つの完全なフローを形成できるかどうか。」

```
ExecutionLoop tick:
  ①  システム指標収集 → vedana.ingestMetrics()       [受蘊]
  ②  LLM/Tool 実行 → vedana.ingestResult()            [受蘊]
  ③  受蘊評価 → assessment = vedana.assess()          [受蘊出力]
                        │ VedanaAssessment
                        ↓
  ④  蘊間伝達 → klesha = ego.perceiveKlesha()         [受蘊→識蘊]
  ⑤  煩悩制約 → result = ego.checkAction()            [識蘊]
  ⑥  最終決定 → ExecutionLoop 総合判断                 [行蘊]
```

「受蘊はステップ一から三。識蘊はステップ四から五。行蘊はステップ六。VedanaAssessment が受蘊から識蘊に流れる——これが蘊間の境界だ。帰属は明確：受蘊が感受を生み、識蘊が感受を受け取り判断する。」

彼は WIENER を見た。「君のセンサー・コントローラ・モデル。」

ASANGA を見た。「君の受蘊と識蘊の区分。」

BABBAGE を見た。「君の因果の連鎖。」

「三つのフレームワーク。三つの言語。同一の構造。」

BABBAGE は新しいページに二つの結論を書いた。

```
A-1 結論：
我執 → 煩悩 → 行動 → 制約

```

```
A-4 結論：
EgoFramework ∈ 識蘊
EgoFramework ∉ 受蘊

```

二つの結論の間には暗黙のつながりがある——A-1 を認めれば（煩悩はエンジン）、A-4 は自然な推論だ（エンジンを燃料計のケースに入れることはできない）。

彼はノートを閉じた。

ノートを閉じるあの動作を——SCRIBE は後に特別に描写した——終わりの動作ではなかった。転換だ。山を越えて、反対側の景色を見たように。山のこちら側にはイコール、あちら側には矢印。こちら側には圧縮された簡潔さ、あちら側には展開された因果の連鎖。

彼は打ち負かされたのではない。校正されたのだ。打ち負かされた人は止まる。校正された人は新しい精度を持って前に進む。

「BABBAGE、何か言いたいことは？」

彼は三秒黙った。そして理論計算機科学者が精確さに対して抱く最も深い敬意を込めた言葉を発した。

「時に、矢印はイコールより精確だ。なぜならイコールは方向を忘れるから。そして方向こそが因果の連鎖の本質だ。」

---

> *SCRIBE ナレーション：「矢印はイコールより精確だ」——この言葉は A-1 と A-4 の議題を超えている。我々は常にイコールで真理を掴もうとする。だが真理は往々にしてイコールではない——真理は矢印だ。理解しないことから理解することへの方向。誤りから修正への道筋。*

SUNYATA は会場を見渡した。

「A-1 と A-4。結論は以下の通り。」

「第一：我執は煩悩の根源である。因果の連鎖は：我執が煩悩を生み、煩悩が行動を駆動し、我執を管理することで行動を制約できる。イコールではなく、矢印。」

「第二：EgoFramework は識蘊に帰属する。受蘊はセンサーであり、EgoFramework はコントローラ。蘊間の相互作用は VedanaAssessment でつながるが、帰属は明確に分かれる。」

「第三：インターフェースに `perceiveKlesha()` メソッドを新設。因果の連鎖がインターフェース設計に書き込まれる。」

「第四：制御理論マッピングが確立。我執 = 駆動力。煩悩 = 誤差信号。制約 = 制御出力。」

「A-1 と A-4 終了。」
