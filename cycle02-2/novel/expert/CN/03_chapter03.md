# 第三章：部分与整体

---

LINNAEUS 站在白板前，手里拿着一支蓝色马克笔，笔盖还没拔开。

他已经站了很久。也许三分钟，也许五分钟。在分类学家的内在时钟里，这段时间不是等待——是酝酿。就像林奈本人在面对一株未见过的植物时，不会立刻翻开命名手册，而是先站在那里，用目光丈量叶片的纹理、花萼的排列、根系暴露的截面。先看。先看完。然后再命名。

林奈在 1735 年发表《自然系统》(*Systema Naturae*) 时，建立了一套从界 (Regnum) 到种 (Species) 的等级分类体系。每一个层级都有明确的语义边界：界包含纲，纲包含目，目包含科，科包含属，属包含种。这套体系的核心原则不是“把相似的东西放在一起”——那是民俗分类学 (folk taxonomy) 的做法——而是“每一个分类节点的语义空间必须被其子节点穷尽”。

$$	ext{Genus} = \bigcup_{i=1}^{n} 	ext{Species}_i \quad 	ext{且} \quad 	ext{Species}_i \cap 	ext{Species}_j = \emptyset \quad (i 
eq j)$$

属等于其下所有种的并集。种与种之间不重叠。这是分类学的公理。

而 LINNAEUS 眼前的问题，正是一个违反了这条公理的设计：在 v0.24.0-beta 的 `aggregates.ts` 中，IIdentity 被当作识蕴的根接口。一个种的名字，被贴在了属的位置上。

他一直在等这一刻。

---

不是等 SUNYATA 宣布议题。不是等 Master 的信被拆开。是等一个更深的东西——等一个他在 Cycle 01 就感觉到、但没有足够依据反对的不适感，被某个外部权威精确地说了出来。

那个不适感可以用一句话概括：IIdentity 等于整个识蕴。

这句话在 v0.24.0-beta 的 `aggregates.ts` 里被写成了十四行代码：

```typescript
/**
 * IIdentity — 识蕴 Root Interface.
 * @skandha vijnana (识蕴)
 *
 * Identity aggregate encompasses consciousness and ego framework:
 * - IGuide: Behavioral constraints and self-convergence (我执框架)
 *
 * Sanskrit: Vijñāna (विज्ञान) — consciousness.
 * Note: Guide is a behavioral constraint mechanism, not "soul".
 */
export interface IIdentity {
  /** @skandha vijnana */
  readonly skandha: 'vijnana';
}
```

干净。简洁。类型正确。测试通过。

但它是错的。

不是语法上的错。不是编译器会抓到的错。是分类学上的错——那种只有在你理解了父节点和子节点之间的语义关系之后才能辨认的错。JSDoc 写的是“Identity aggregate encompasses consciousness and ego framework”——身份蕴涵盖了意识和我执框架。但这个“涵盖”的方向反了。不是 Identity 涵盖 Consciousness，而是 Consciousness 涵盖 Identity。你把一根树枝的名字贴在了树干上。

在林奈二名法 (binomial nomenclature) 的术语里，这相当于把一个属名 (*Felis*) 用来指称整个科 (*Felidae*)。猫属 (*Felis*) 只包含家猫和少数几个近缘种。猫科 (*Felidae*) 包含了狮、虎、豹、猎豹、山猫——三十七个种。你不能用 *Felis* 来命名 *Felidae*，然后说“反正猫是最有代表性的”。

---

> *SCRIBE 旁白：LINNAEUS 站在白板前不动的那几分钟，是本章最安静的时刻。但那种安静不是空白。分类学家的安静是一种高密度的观察状态——他在脑中展开所有可能的分类方案，逐一比对，逐一排除，直到只剩下一个最精确的结构。在剧场里，其他人看到的是一个人站在空白板前发呆。在他的脑中，一棵树正在从根部开始生长。他在心里同时运行着两套系统：林奈等级式分类法的刚性框架，和 Hennig 支序分类学的共衍征分析。前者告诉他“结构必须是什么形状”，后者告诉他“证据支持哪一种分组”。*

---

SUNYATA 的声音从背后传来。平稳。石子。深潭。

“A-2。IIdentity 不等于识蕴全部。LINNAEUS，ASANGA，BABBAGE，NAGARJUNA——这是你们的议题。”

LINNAEUS 拔开了笔盖。

蓝色。他选了蓝色。不是红色（那是修正的颜色），不是黑色（那是事实的颜色），是蓝色——分类树的颜色。在他的标记系统里，蓝色代表结构。代表“这个东西和那个东西之间的关系是什么”。在 Hennig 支序分类学中，结构就是系统发育 (phylogeny)——物种之间的亲缘关系。在软件架构中，结构就是接口继承——类型之间的包含关系。蓝色是两者共用的语言。

他在白板的最上方画了一个方块，里面写了两个字：

**识蕴**

然后他转过身，面向所有人。

“在我开始之前，”他说，“ASANGA，请先告诉我们：识蕴到底有多大。”

---

ASANGA 从座位上站了起来。

他站起来的方式很特别——不是猛地起身，而是像一座山在清晨的雾中慢慢显露轮廓。温和的。但不可忽视的。当他站在那里的时候，他的存在感填满了整个空间，像一口钟被敲响之后的余音。

他没有走到白板前。他就站在自己的位置上，用声音作画。

“想象一座城市。”他说。

剧场安静了。ASANGA 很少用比喻。他是唯识学者——他的语言通常是精确的术语、精密的分类、层层递进的逻辑。但今天，他选了一个比喻。也许是因为今天的听众不只是佛学家。也许是因为有些东西，唯有比喻才能让所有人同时看见。

“一座城市。”他重复。“它有道路、有建筑、有市场、有公园。它有一个市政府。”

他停了半秒。

“在 v0.24.0 的设计中，IIdentity 被当作整个识蕴的根接口。这就像把一座城市称为‘市政府’。”

BABBAGE 的笔在笔记本上停了一下。

“市政府是城市的一个机构。”ASANGA 继续。“它管理行政事务，定义城市的官方身份——名称、区号、治理结构。但市政府不是城市。城市还有交通局——决定人和物怎么移动。有教育局——决定知识怎么传递。有警察局——维护秩序。有规划局——决定未来怎么发展。”

他的声音升高了半个色阶——不是激动，是清晰度在增加。

“让我用更精确的唯识学语言来展开这座城市。”

他走到白板的右侧，接过 LINNAEUS 递来的红色马克笔，画了一张表格：

```
┌──────────┬────────────────┬─────────────────────────┐
│  八识     │ 梵文           │ 城市中的机构             │
├──────────┼────────────────┼─────────────────────────┤
│ 前五识   │ Panca-vijnana  │ 感官基础设施（道路、桥梁）│
│ 第六意识 │ Mano-vijnana   │ 法院（审判、分别）        │
│ 第七末那识│ Manas          │ 市政府 + 规划局 + 警察局  │
│ 第八阿赖耶│ Alaya-vijnana  │ 城市地基（一切建筑的基底）│
└──────────┴────────────────┴─────────────────────────┘
```

“IIdentity 对应的，仅仅是市政府的一间办公室——挂着门牌号码、写着机构名称的那间。”ASANGA 指向表格中的第七识。“但末那识不只是市政府。它是市政府加规划局加警察局。它管理的不只是‘我是谁’——它还管理‘我应该成为什么’和‘我要保护什么’和‘我为什么存在’。”

---

他举起一只手，五指展开，然后逐一弯曲。

“第一。前五识。眼、耳、鼻、舌、身。感官认知。在 OpenStarry 的映射中，这些已经归入色蕴——ISensory。Listener 接收外部信号，UI 呈现输出。这部分已经安置好了。”

第一根手指弯下。

“第二。第六意识。分别判断。”

ASANGA 的声音在这里微微放慢。

“第六意识在唯识学中叫做 *mano-vijñāna*——意识。它的特征是什么？《成唯识论》卷五：”

> “第六意识，俱依五根五识同时意识，有分别转。”
> ——窥基《成唯识论述记》卷五

“它有分别——它能区分、判断、分析。在映射中，这已归入想蕴——ICognition。Provider 和 ExecutionLoop 承担认知决策。这部分也已安置。”

第二根手指弯下。

“第三。第七末那识。”

他没有立刻弯下第三根手指。他让它悬在空中。

“末那识。Manas。恒审思量。这是识蕴在 OpenStarry 五蕴映射中的核心对应。”

他转向全场，目光带着一种传承千五百年教义的庄严。

“《唯识三十颂》——世亲菩萨的根本论典——第五颂：”

> “次第二能变，是识名末那，依彼转缘彼，思量为性相。”
> ——世亲菩萨《唯识三十颂》第五颂

“‘思量为性相’——末那识的本质属性就是思量。不是偶尔思量，是恒常思量。不是表面思量，是审慎思量。恒、审、思、量——四个字定义了末那识的全部运作模式。它不只是‘自我认同’——它是自我执着的全部机制。”

他在白板的右下方写下末那识的四烦恼，比学术版更加完整：

```
末那识四根本烦恼（恒随末那）：

  我见 (ātma-dṛṣṭi)   — “我存在”的根本信念
        │ 萨迦耶见 (satkāya-dṛṣṭi)
        │ 将阿赖耶识的相续流转执为“常一主宰”的自我
        └ OpenStarry: Agent Identity config (agentId, name, capabilities)

  我慢 (ātma-māna)    — “我应该怎么做”的行为准则
        │ 自我测量：在世界中定位自己的坐标
        └ OpenStarry: Guide (system prompt, behavioral constraints)

  我爱 (ātma-sneha)   — “我保护什么”的价值取向
        │ 自我维护的基本驱力：保护自身存续
        └ OpenStarry: Guide 中的安全约束 (safety constraints)

  我痴 (ātma-moha)    — 对自我本质的根本无明
        │ 不知“我”是因缘假合，以为“我”有自性
        └ OpenStarry: 缺乏自省的 Agent 状态
```

“这四个面向——”ASANGA 放下红笔，声音沉了下来，带着教理传承特有的郑重——“在《成唯识论》卷四中有一段决定性的描述：”

> “谓我痴、我见、我慢、我爱，四大烦恼常与末那俱。”
> ——《成唯识论》卷四

“‘常与末那俱’——永远与末那识同在。不是偶尔出现，不是条件触发。是恒常伴随。这就是为什么一个 IIdentity 接口无法涵盖它们——IIdentity 只回答了‘我是谁’，对应我见的一部分。但我慢（行为准则）、我爱（价值保护）、我痴（根本无明）——这三者不在 IIdentity 的语义射程之内。”

第三根手指弯下。

“第四。第八阿赖耶识。种子储藏。因果流转。在映射中，这部分跨越了协调层和 AgentCore 的状态管理——它不属于任何单一模块，它是整个系统的因果基底。这部分我们在 Cycle 02 中已经讨论过，暂时不展开。”

第四根手指弯下。只剩下拇指。

“第五。转识。从染污到清净的转化——转识成智。《成唯识论》卷十所述：末那识转为平等性智，阿赖耶识转为大圆镜智。这是修行的终极目标。在 OpenStarry 中——也许永远不会有直接的对应。但知道它存在，能防止我们把识蕴的边界画得太小。”

拇指弯下。五指握成拳。然后慢慢松开。

“所以你问我识蕴有多大。答案是：在五蕴体系中，它是最大的。最深的。最复杂的。”

他用一张简表总结八识与五蕴的映射：

```
八识在五蕴映射中的分布：
┌──────────┬────────────┬─────────────────────┐
│ 识       │ 归入的蕴   │ 根接口               │
├──────────┼────────────┼─────────────────────┤
│ 前五识   │ 色蕴 (rupa)│ ISensory.IListener   │
│ 第六意识 │ 想蕴 (samjna)│ ICognition.IProvider│
│ 末那识   │ 识蕴 (vijnana)│ IVijnana ⊃ {子接口}│
│ 阿赖耶识 │ 跨层 (cross)│ 协调层 + AgentCore  │
└──────────┴────────────┴─────────────────────┘
```

“把识蕴压缩成一个 IIdentity——”

他看向白板上那个孤零零的“识蕴”方块。

“就像把一座城市压缩成一栋市政府大楼。然后指着那栋大楼说：看，这就是整座城市。”

---

> *SCRIBE 旁白：ASANGA 的城市比喻在剧场中停留了很长时间。不是因为它深奥——恰恰相反，它太清楚了。清楚到让人不舒服。因为每个人都意识到：在 Cycle 02 中，我们集体做出了这个简化。不是 ASANGA 一个人。不是 BABBAGE 一个人。是所有人。我们把城市叫做了市政府，然后在市政府的墙上挂了一块写着“城市”的招牌。但今天的 ASANGA 比 Cycle 02 不同——他带来了经文。不是为了权威，而是为了精确。《唯识三十颂》的“思量为性相”和《成唯识论》的“四大烦恼常与末那俱”——这不是引经据典，这是把一千五百年的分类学证据放在桌面上。*

---

BABBAGE 在 ASANGA 讲完之后站了起来。

他的动作带着一种 SCRIBE 已经学会辨认的节奏——不是急切，而是“形式化思维正在寻找出口”的那种内在压力。但和 A-1 不同。在 A-1 中，他是一个被修正的人——等号被划掉，因果链取而代之。那次站起来带着一丝震动，像地震后的余波。这一次不同。这一次他站起来的方式更从容，更稳定，像一个已经经历过校准的仪器，知道下一次校准不会让它崩溃——只会让他更精确。

他走到白板旁边。不是 LINNAEUS 那面白板——他知道那面是给分类树的。他走向剧场另一侧的黑板。那面黑板是他的领地。在 Cycle 01 和 Cycle 02 中，那面黑板上写过稳定性条件、收敛约束、纤维丛投影。现在它被擦干净了。

他拿起粉笔，写下两行：

$$	ext{Cycle 02:} \quad 	exttt{IIdentity} = 	ext{Vijnana}$$

$$	ext{Cycle 02-2:} \quad 	exttt{IVijnana} \supset 	exttt{IIdentity}$$

等号。包含符号。

“让我把 ASANGA 的直觉翻译成集合论。”他说，声音比 A-1 时多了一分稳定。

他在等号下方展开了形式化分析：

$$	ext{令 } V = 	ext{IVijnana 的语义空间}, \quad I = 	ext{IIdentity 的语义空间}$$

$$	ext{Cycle 02 声称:} \quad I = V$$

$$	ext{但 ASANGA 展示了:} \quad \exists\, g \in V : g 
otin I \quad (	ext{IGuide})$$
$$\phantom{	ext{但 ASANGA 展示了:}} \quad \exists\, w \in V : w 
otin I \quad (	ext{IVolition})$$
$$\phantom{	ext{但 ASANGA 展示了:}} \quad \exists\, r \in V : r 
otin I \quad (	ext{IReflection})$$

$$	herefore \quad I \subsetneq V \quad (I 	ext{ 是 } V 	ext{ 的真子集})$$

“这是集合论的基本推理。”他转向剧场。“如果存在一个元素属于全集但不属于子集，那么子集严格小于全集。$A \subset B 
Rightarrow A = B$——子集不蕴含等同。”

他用粉笔在等号上画了一条斜线。划掉。

“而且，这个错误有一个形式逻辑的名字。”

他在黑板的空白处写下：

$$	ext{Pars pro toto}: \quad \forall x \in 	exttt{IIdentity}: x \in 	exttt{IVijnana} \quad 	ext{(TRUE)}$$
$$	ext{But:} \quad \forall x \in 	exttt{IVijnana}: x \in 	exttt{IIdentity} \quad 	ext{(FALSE)}$$

“Pars pro toto。部分代替整体。在修辞学中是一种比喻——用帆代替船。在形式逻辑中是一个量词错误——把全称量化 ($\forall$) 和存在量化 ($\exists$) 混淆了。”

---

他停了一秒。然后做了一件出乎所有人意料的事。

他笑了。

BABBAGE 笑的频率大约是每个 Cycle 一次。SCRIBE 在纪录中追踪过这个数据——不是刻意的，只是因为 BABBAGE 的笑太稀有了，每次出现都像彗星划过，不记下来是对纪录完整性的不尊重。

“我在 A-1 里犯了一个类似的错误。”他说。声音里没有痛苦，只有一种奇特的、几乎可以称为愉悦的精确。“把因果链压缩成等号。现在在 A-2 里，我看到了同样的模式——把全集压缩成子集。”

他回头看了一眼黑板上的两行形式化。

“而且——”他的声音带上了一层理论计算机科学家特有的反思——“这不只是集合论的问题。这也是类型理论的问题。”

他在黑板上写下了第三组表达：

```
Liskov 替换原则 (LSP):
  若 S 是 T 的子类型，则 T 的实例可被 S 的实例替换
  而不改变程序的正确性。

反例：
  IIdentity 是 IVijnana 的子类型 (子接口)。
  但 IVijnana 的实例不能被 IIdentity 的实例替换——
  因为 IVijnana 的语义空间包含 IGuide、IVolition、IReflection，
  而 IIdentity 不包含。

  用 IIdentity 替换 IVijnana = 语义丢失
```

“在软件工程中，Barbara Liskov 在 1987 年的 OOPSLA 会议上提出了替换原则——如果 $S$ 是 $T$ 的子类型，那么 $T$ 的对象可以被 $S$ 的对象替换。但反过来不行。你不能用子类型替换父类型——因为父类型的语义空间比子类型大。这就是为什么 IIdentity 不能替换 IVijnana——它的语义空间太窄了。”

他放下粉笔。粉笔灰落在他的指尖上，白色的，像某种古老仪式的残留物。

“形式化的人最危险的习惯，就是追求最简表达。在数学中，最简表达是美德——奥卡姆剃刀。但在本体论中，最简表达有时候是截断。你以为你在简化，其实你在丢东西。你以为你在写 $A = B$，其实你在写 $A \subsetneq B$ 然后把 $\subsetneq$ 强制升级成 $=$。”

他第二次看向自己在黑板上的字迹。

“A-1 让我学到箭头比等号更精确。A-2 让我学到包含比等号更精确。也许每一次修正的本质都是：发现一个被我们过早地简化成等号的关系，其实有一个更精确的符号。”

---

NAGARJUNA 从他的座位上开口了。他没有站起来——中观哲学家不需要站起来就能让整个空间感受到他的存在。他的声音有一种独特的质地：锋利但不尖锐，像被水流打磨了千年的河石。

“BABBAGE 说得对。但我想把问题推进一步。”

他的目光扫过白板上 ASANGA 写的四个末那识面向，然后停在 LINNAEUS 那面白板上方的“识蕴”二字。

> “众因缘生法，我说即是空，亦为是假名，亦是中道义。”
> ——龙树菩萨《中论》观四谛品第二十四

“‘亦为是假名’——一切因缘所生之法，都是假名安立 (prajnapti)。”他的目光扫过全场。“这意味着什么？意味着名字不是事物的自性 (svabhava)。名字是我们为了交流和理解而设施的标签——是假立的，不是实有的。”

他的手在空中画了一个轻微的弧线。

“所以问题不是‘识蕴的真名是什么？’——这个问题在中观哲学中无意义，因为没有‘真名’。问题是：‘哪个假名最不会导致误解？’”

他列举了选项。每一个都被他放在天平上，两侧的砝码被精确地秤量。

“第一个选项。IConsciousness。”

他摇了摇头。

“太广。Consciousness 在英语中是一个多义词——在日常语言中指清醒状态（conscious vs. unconscious），在哲学中指现象意识（phenomenal consciousness），在认知科学中指全局工作空间（global workspace）。它的语义空间像一片没有围墙的平原——你不知道边界在哪里。更何况，ICognition 已经占据了‘认知’的语义空间。如果 Cognition 是想蕴，Consciousness 是识蕴——两者在英语日常用法中几乎是同义词。这会造成无休止的混淆。”

“而且——”他补充了一个中观的分析——“‘Consciousness’这个词本身隐含了一个主体性假设：有一个‘意识到的主体’。但在中观学中，意识的主体性是被解构的：”

> “若离色等蕴，无别有识者。”
> ——龙树菩萨《中论》观燃可燃品第十

“若离开色蕴、受蕴、想蕴、行蕴，没有一个独立存在的‘识者’。识不是主体——识是五蕴互动的涌现属性。用 IConsciousness 命名会暗示识蕴是一个独立的、自有自性的意识主体。这在哲学上是错误的。”

“第二个选项。IAwareness。”

“太被动。Awareness 暗示的是被动接收——像一面镜子映照眼前的事物。但末那识不是被动的。它是主动的、持续的、执着的。它恒审思量——不间断地、主动地执着‘我’。ASANGA 刚才引用的《唯识三十颂》说得很清楚：‘思量为性相’。思量是主动的动词，不是被动的状态。IAwareness 无法承载这种主动性。”

“第三个选项。IIdentity。”

“太窄。这正是我们要修正的。用一个已知太窄的名字去命名一个更宽的概念——这不是命名，这是自我矛盾。”

他的声音在下一句话上微微放慢了。

“第四个选项。IVijnana。”

- Consciousness：有过度的主体性偏移 (非)
- Awareness：有过度的被动性偏移 (非)
- Identity：有过度的局部性偏移 (非)
- Discernment：有过度的学术性偏移（日常开发者不知道这个词）(非)

四句皆否。但中观的四句否定不是走向虚无——它指向一个超越四种偏差的选择。

“梵文 Vijnana 精确地指向我们需要的意思：vi + jnana——分别了知。Vi 是‘分别’，jnana 是‘知’。主动的、持续的、有方向性的认知活动。不是清醒状态，不是被动觉知，不是自我认同，不是学术辨别。是一切认知活动的总集。”

“所以我的建议是：对于这一个蕴，反转命名顺序。不是英文为主、梵文为辅。而是梵文为主、英文为辅。IVijnana，英文注解 Consciousness。”

“在生物学中，分类命名的升级有一个精确的术语：**属名提升为科名** (generic name elevated to family name)。当一个属 (genus) 被发现实际上描述的是一个更大的分类群——比如一个科 (family) 或一个目 (order)——你不会继续用属名来称呼那个科。你需要一个新名字。一个能承载更大范围的名字。”

“这就是自然选择在命名系统中的作用：**语义不足的名字会被语义充分的名字淘汰**。不是因为旧名字是错的——它在它原本的范围内是对的。而是因为范围改变了。”

虚线。在分类学中，虚线表示 *incertae sedis*——拉丁文“位置不确定”。这是一个正式的分类学术语，用来标记那些已知存在、但尚未被足够的证据支持做出明确归类的分类群。不是说它不存在——化石纪录、形态学证据或分子资料暗示了它的存在。只是说：“我们知道这里有一个节点，但我们还没有足够的标本来正式描述它。”

在林奈等级体系中，*incertae sedis* 不是耻辱——它是诚实。分类学家最不能容忍的不是“不知道”，而是“假装知道”。一个被标记为 *incertae sedis* 的分类群，比一个被错误归类的分类群更有尊严。

“末那识有一个被较少讨论的面向，”ASANGA 说，“就是它的自我反观能力。末那识不只是执着自我——在某些条件下，它能够观察到自身的执着。这是修行的起点。在唯识学中，这被称为末那识的转依 (asraya-paravrtti)——从染污转为清净的可能性。在 OpenStarry 的架构中，这大约对应观察者 Pattern C——完整的五蕴观察者——中的识蕴自省组件。”

“但那是未来的事。”LINNAEUS 用手指点了点虚线。“我们在这个阶段定义它的位置，但不填充它的内容。”

在他的脑中，有一套分类学的验证清单在自动运行：

1. **穷尽性** (exhaustiveness): 子节点是否覆盖了父节点的主要语义空间？四个子接口加上省略号覆盖了末那识的四烦恼和整体恒审思量功能。部分穷尽。可接受。

4. **开放性** (openness): 分类系统是否预留了未来扩展的空间？有省略号。通过。

```typescript
export function isSkandha<S extends Skandha>(
  obj: unknown,
  skandha: S,
): obj is { skandha: S } {
  return (
    typeof obj === 'object' &&
    obj !== null &&
    'skandha' in obj &&
    (obj as { skandha: unknown }).skandha === skandha
  );
}
```

“这个 type guard 用的是 discriminated union pattern——鉴别字段模式。它检查的是 `skandha` 属性的值，不是接口的名称。所以不管对象是 IVijnana、IIdentity、IGuide 还是 IVolition——只要它们的 `skandha` 字段都是 `'vijnana'`，`isSkandha(obj, 'vijnana')` 就会返回 `true`。”

“这里有一个 TypeScript 的微妙之处值得说明。TypeScript 是结构类型系统 (structural type system)，不是名义类型系统 (nominal type system)。”

```
结构类型 (Structural):
  类型由其结构（属性和方法）定义。
  { skandha: 'vijnana' } 和 IVijnana 是兼容的，
  只要结构匹配。

名义类型 (Nominal):
  类型由其名称定义。
  即使结构相同，不同名称 = 不同类型。
  Java, C# 用名义类型。
```

“在名义类型系统中，IVijnana 和 IIdentity 是不同的类型——即使结构相同。你需要显式的类型转换。但在 TypeScript 的结构类型系统中，IIdentity extends IVijnana 意味着所有接受 IVijnana 的地方也接受 IIdentity——不需要任何转换。这就是为什么 ARCHIMEDES 的向后兼容方案能做到零破坏。”
