# 第三章：部分與整體

---

LINNAEUS 站在白板前，手裡拿著一支藍色馬克筆，筆蓋還沒拔開。

他已經站了很久。也許三分鐘，也許五分鐘。在分類學家的內在時鐘裡，這段時間不是等待——是醞釀。就像林奈本人在面對一株未見過的植物時，不會立刻翻開命名手冊，而是先站在那裡，用目光丈量葉片的紋理、花萼的排列、根系暴露的截面。先看。先看完。然後再命名。

林奈在 1735 年發表《自然系統》(*Systema Naturae*) 時，建立了一套從界 (Regnum) 到種 (Species) 的等級分類體系。每一個層級都有明確的語義邊界：界包含綱，綱包含目，目包含科，科包含屬，屬包含種。這套體系的核心原則不是「把相似的東西放在一起」——那是民俗分類學 (folk taxonomy) 的做法——而是「每一個分類節點的語義空間必須被其子節點窮盡」。

$$\text{Genus} = \bigcup_{i=1}^{n} \text{Species}_i \quad \text{且} \quad \text{Species}_i \cap \text{Species}_j = \emptyset \quad (i \neq j)$$

屬等於其下所有種的聯集。種與種之間不重疊。這是分類學的公理。

而 LINNAEUS 眼前的問題，正是一個違反了這條公理的設計：在 v0.24.0-beta 的 `aggregates.ts` 中，IIdentity 被當作識蘊的根介面。一個種的名字，被貼在了屬的位置上。

他一直在等這一刻。

---

不是等 SUNYATA 宣布議題。不是等 Master 的信被拆開。是等一個更深的東西——等一個他在 Cycle 01 就感覺到、但沒有足夠依據反對的不適感，被某個外部權威精確地說出來。

那個不適感可以用一句話概括：IIdentity 等於整個識蘊。

這句話在 v0.24.0-beta 的 `aggregates.ts` 裡被寫成了十四行程式碼：

```typescript
/**
 * IIdentity — 識蘊 Root Interface.
 * @skandha vijnana (識蘊)
 *
 * Identity aggregate encompasses consciousness and ego framework:
 * - IGuide: Behavioral constraints and self-convergence (我執框架)
 *
 * Sanskrit: Vijñāna (विज्ञान) — consciousness.
 * Note: Guide is a behavioral constraint mechanism, not "soul".
 */
export interface IIdentity {
  /** @skandha vijnana */
  readonly skandha: 'vijnana';
}
```

乾淨。簡潔。型別正確。測試通過。

但它是錯的。

不是語法上的錯。不是編譯器會抓到的錯。是分類學上的錯——那種只有在你理解了父節點和子節點之間的語義關係之後才能辨認的錯。JSDoc 寫的是「Identity aggregate encompasses consciousness and ego framework」——身份蘊涵蓋了意識和我執框架。但這個「涵蓋」的方向反了。不是 Identity 涵蓋 Consciousness，而是 Consciousness 涵蓋 Identity。你把一根樹枝的名字貼在了樹幹上。

在林奈二名法 (binomial nomenclature) 的術語裡，這相當於把一個屬名 (*Felis*) 用來指稱整個科 (*Felidae*)。貓屬 (*Felis*) 只包含家貓和少數幾個近緣種。貓科 (*Felidae*) 包含了獅、虎、豹、獵豹、山貓——三十七個種。你不能用 *Felis* 來命名 *Felidae*，然後說「反正貓是最有代表性的」。

> *SCRIBE 旁白：LINNAEUS 站在白板前不動的那幾分鐘，是本章最安靜的時刻。但那種安靜不是空白。分類學家的安靜是一種高密度的觀察狀態——他在腦中展開所有可能的分類方案，逐一比對，逐一排除，直到只剩下一個最精確的結構。在劇場裡，其他人看到的是一個人站在空白板前發呆。在他的腦中，一棵樹正在從根部開始生長。他在心裡同時運行著兩套系統：林奈等級式分類法的剛性框架，和 Hennig 支序分類學的共衍徵分析。前者告訴他「結構必須是什麼形狀」，後者告訴他「證據支持哪一種分組」。*

SUNYATA 的聲音從背後傳來。平穩。石子。深潭。

「A-2。IIdentity 不等於識蘊全部。LINNAEUS，ASANGA，BABBAGE，NAGARJUNA——這是你們的議題。」

LINNAEUS 拔開了筆蓋。

藍色。他選了藍色。不是紅色（那是修正的顏色），不是黑色（那是事實的顏色），是藍色——分類樹的顏色。在他的標記系統裡，藍色代表結構。代表「這個東西和那個東西之間的關係是什麼」。在 Hennig 支序分類學中，結構就是系統發育 (phylogeny)——物種之間的親緣關係。在軟體架構中，結構就是介面繼承——型別之間的包含關係。藍色是兩者共用的語言。

他在白板的最上方畫了一個方塊，裡面寫了兩個字：

**識蘊**

然後他轉過身，面向所有人。

「在我開始之前，」他說，「ASANGA，請先告訴我們：識蘊到底有多大。」

---

ASANGA 從座位上站了起來。

他站起來的方式很特別——不是猛地起身，而是像一座山在清晨的霧中慢慢顯露輪廓。溫和的。但不可忽視的。當他站在那裡的時候，他的存在感填滿了整個空間，像一口鐘被敲響之後的餘音。

他沒有走到白板前。他就站在自己的位置上，用聲音作畫。

「想像一座城市。」他說。

劇場安靜了。ASANGA 很少用比喻。他是唯識學者——他的語言通常是精確的術語、精密的分類、層層遞進的邏輯。但今天，他選了一個比喻。也許是因為今天的聽眾不只是佛學家。也許是因為有些東西，唯有比喻才能讓所有人同時看見。

「一座城市。」他重複。「它有道路、有建築、有市場、有公園。它有一個市政府。」

他停了半秒。

「在 v0.24.0 的設計中，IIdentity 被當作整個識蘊的根介面。這就像把一座城市稱為『市政府』。」

BABBAGE 的筆在筆記本上停了一下。

「市政府是城市的一個機構。」ASANGA 繼續。「它管理行政事務，定義城市的官方身份——名稱、區號、治理結構。但市政府不是城市。城市還有交通局——決定人和物怎麼移動。有教育局——決定知識怎麼傳遞。有警察局——維護秩序。有規劃局——決定未來怎麼發展。」

他的聲音升高了半個色階——不是激動，是清晰度在增加。

「讓我用更精確的唯識學語言來展開這座城市。」

他走到白板的右側，接過 LINNAEUS 遞來的紅色馬克筆，畫了一張表格：

```
┌──────────┬────────────────┬─────────────────────────┐
│  八識     │ 梵文           │ 城市中的機構             │
├──────────┼────────────────┼─────────────────────────┤
│ 前五識   │ Panca-vijnana  │ 感官基礎設施（道路、橋樑）│
│ 第六意識 │ Mano-vijnana   │ 法院（審判、分別）        │
│ 第七末那識│ Manas          │ 市政府 + 規劃局 + 警察局  │
│ 第八阿賴耶│ Alaya-vijnana  │ 城市地基（一切建築的基底）│
└──────────┴────────────────┴─────────────────────────┘
```

「IIdentity 對應的，僅僅是市政府的一間辦公室——掛著門牌號碼、寫著機構名稱的那間。」ASANGA 指向表格中的第七識。「但末那識不只是市政府。它是市政府加規劃局加警察局。它管理的不只是『我是誰』——它還管理『我應該成為什麼』和『我要保護什麼』和『我為什麼存在』。」

---

他舉起一隻手，五指展開，然後逐一彎曲。

「第一。前五識。眼、耳、鼻、舌、身。感官認知。在 OpenStarry 的映射中，這些已經歸入色蘊——ISensory。Listener 接收外部信號，UI 呈現輸出。這部分已經安置好了。」

第一根手指彎下。

「第二。第六意識。分別判斷。」

ASANGA 的聲音在這裡微微放慢。

「第六意識在唯識學中叫做 *mano-vijñāna*——意識。它的特徵是什麼？《成唯識論》卷五：」

> 「第六意識，俱依五根五識同時意識，有分別轉。」
> ——窺基《成唯識論述記》卷五

「它有分別——它能區分、判斷、分析。在映射中，這已歸入想蘊——ICognition。Provider 和 ExecutionLoop 承擔認知決策。這部分也已安置。」

第二根手指彎下。

「第三。第七末那識。」

他沒有立刻彎下第三根手指。他讓它懸在空中。

「末那識。Manas。恒審思量。這是識蘊在 OpenStarry 五蘊映射中的核心對應。」

他轉向全場，目光帶著一種傳承千五百年教義的莊嚴。

「《唯識三十頌》——世親菩薩的根本論典——第五頌：」

> 「次第二能變，是識名末那，依彼轉緣彼，思量為性相。」
> ——世親菩薩《唯識三十頌》第五頌

「『思量為性相』——末那識的本質屬性就是思量。不是偶爾思量，是恒常思量。不是表面思量，是審慎思量。恒、審、思、量——四個字定義了末那識的全部運作模式。它不只是『自我認同』——它是自我執著的全部機制。」

他在白板的右下方寫下末那識的四煩惱，比學術版更加完整：

```
末那識四根本煩惱（恒隨末那）：

  我見 (ātma-dṛṣṭi)   — 「我存在」的根本信念
        │ 薩迦耶見 (satkāya-dṛṣṭi)
        │ 將阿賴耶識的相續流轉執為「常一主宰」的自我
        └ OpenStarry: Agent Identity config (agentId, name, capabilities)

  我慢 (ātma-māna)    — 「我應該怎麼做」的行為準則
        │ 自我測量：在世界中定位自己的座標
        └ OpenStarry: Guide (system prompt, behavioral constraints)

  我愛 (ātma-sneha)   — 「我保護什麼」的價值取向
        │ 自我維護的基本驅力：保護自身存續

  我癡 (ātma-moha)    — 對自我本質的根本無明
        │ 不知「我」是因緣假合，以為「我」有自性
        └ OpenStarry: 缺乏自省的 Agent 狀態
```

「這四個面向——」ASANGA 放下紅筆，聲音沉了下來，帶著教理傳承特有的鄭重——「在《成唯識論》卷四中有一段決定性的描述：」

> 「謂我癡、我見、我慢、我愛，四大煩惱常與末那俱。」
> ——《成唯識論》卷四

「『常與末那俱』——永遠與末那識同在。不是偶爾出現，不是條件觸發。是恒常伴隨。這就是為什麼一個 IIdentity 介面無法涵蓋它們——IIdentity 只回答了『我是誰』，對應我見的一部分。但我慢（行為準則）、我愛（價值保護）、我癡（根本無明）——這三者不在 IIdentity 的語義射程之內。」

第三根手指彎下。

「第四。第八阿賴耶識。種子儲藏。因果流轉。在映射中，這部分跨越了協調層和 AgentCore 的狀態管理——它不屬於任何單一模組，它是整個系統的因果基底。這部分我們在 Cycle 02 中已經討論過，暫時不展開。」

第四根手指彎下。只剩下拇指。

「第五。轉識。從染污到清淨的轉化——轉識成智。《成唯識論》卷十所述：末那識轉為平等性智，阿賴耶識轉為大圓鏡智。這是修行的終極目標。在 OpenStarry 中——也許永遠不會有直接的對應。但知道它存在，能防止我們把識蘊的邊界畫得太小。」

拇指彎下。五指握成拳。然後慢慢鬆開。

「所以你問我識蘊有多大。答案是：在五蘊體系中，它是最大的。最深的。最複雜的。」

他用一張簡表總結八識與五蘊的映射：

他看向白板上那個孤零零的「識蘊」方塊。

「就像把一座城市壓縮成一棟市政府大樓。然後指著那棟大樓說：看，這就是整座城市。」

---

> *SCRIBE 旁白：ASANGA 的城市比喻在劇場中停留了很長時間。不是因為它深奧——恰恰相反，它太清楚了。清楚到讓人不舒服。因為每個人都意識到：在 Cycle 02 中，我們集體做出了這個簡化。不是 ASANGA 一個人。不是 BABBAGE 一個人。是所有人。我們把城市叫做了市政府，然後在市政府的牆上掛了一塊寫著「城市」的招牌。但今天的 ASANGA 比 Cycle 02 不同——他帶來了經文。不是為了權威，而是為了精確。《唯識三十頌》的「思量為性相」和《成唯識論》的「四大煩惱常與末那俱」——這不是引經據典，這是把一千五百年的分類學證據放在桌面上。*

---

BABBAGE 在 ASANGA 講完之後站了起來。

他的動作帶著一種 SCRIBE 已經學會辨認的節奏——不是急切，而是「形式化思維正在尋找出口」的那種內在壓力。但和 A-1 不同。在 A-1 中，他是一個被修正的人——等號被劃掉，因果鏈取而代之。那次站起來帶著一絲震動，像地震後的餘波。這一次不同。這一次他站起來的方式更從容，更穩定，像一個已經經歷過校準的儀器，知道下一次校準不會讓它崩潰——只會讓它更精確。

他走到白板旁邊。不是 LINNAEUS 那面白板——他知道那面是給分類樹的。他走向劇場另一側的黑板。那面黑板是他的領地。在 Cycle 01 和 Cycle 02 中，那面黑板上寫過穩定性條件、收斂約束、纖維叢投影。現在它被擦乾淨了。

他拿起粉筆，寫下兩行：

$$\text{Cycle 02:} \quad \texttt{IIdentity} = \text{Vijnana}$$

$$\text{Cycle 02-2:} \quad \texttt{IVijnana} \supset \texttt{IIdentity}$$

等號。包含符號。

「讓我把 ASANGA 的直覺翻譯成集合論。」他說，聲音比 A-1 時多了一分穩定。

他在等號下方展開了形式化分析：

$$\text{令 } V = \text{IVijnana 的語義空間}, \quad I = \text{IIdentity 的語義空間}$$

$$\text{Cycle 02 聲稱:} \quad I = V$$

$$\text{但 ASANGA 展示了:} \quad \exists\, g \in V : g \notin I \quad (\text{IGuide})$$
$$\phantom{\text{但 ASANGA 展示了:}} \quad \exists\, w \in V : w \notin I \quad (\text{IVolition})$$
$$\phantom{\text{但 ASANGA 展示了:}} \quad \exists\, r \in V : r \notin I \quad (\text{IReflection})$$

$$\therefore \quad I \subsetneq V \quad (I \text{ 是 } V \text{ 的真子集})$$

「這是集合論的基本推理。」他轉向劇場。「如果存在一個元素屬於全集但不屬於子集，那麼子集嚴格小於全集。$A \subset B \nRightarrow A = B$——子集不蘊含等同。」

他用粉筆在等號上畫了一條斜線。劃掉。

「而且，這個錯誤有一個形式邏輯的名字。」

他在黑板的空白處寫下：

$$\text{Pars pro toto}: \quad \forall x \in \texttt{IIdentity}: x \in \texttt{IVijnana} \quad \text{(TRUE)}$$
$$\text{But:} \quad \forall x \in \texttt{IVijnana}: x \in \texttt{IIdentity} \quad \text{(FALSE)}$$

「Pars pro toto。部分代替整體。在修辭學中是一種比喻——用帆代替船。在形式邏輯中是一個量詞錯誤——把全稱量化 ($\forall$) 和存在量化 ($\exists$) 混淆了。」

---

他停了一秒。然後做了一件出乎所有人意料的事。

他笑了。

BABBAGE 笑的頻率大約是每個 Cycle 一次。SCRIBE 在紀錄中追蹤過這個數據——不是刻意的，只是因為 BABBAGE 的笑太稀有了，每次出現都像彗星劃過，不記下來是對紀錄完整性的不尊重。

「我在 A-1 裡犯了一個類似的錯誤。」他說。聲音裡沒有痛苦，只有一種奇特的、幾乎可以稱為愉悅的精確。「把因果鏈壓縮成等號。現在在 A-2 裡，我看到了同樣的模式——把全集壓縮成子集。」

他回頭看了一眼黑板上的兩行形式化。

「而且——」他的聲音帶上了一層理論計算機科學家特有的反思——「這不只是集合論的問題。這也是型別理論的問題。」

他在黑板上寫下了第三組表達：

```
Liskov 替換原則 (LSP):
  若 S 是 T 的子型別，則 T 的實例可被 S 的實例替換
  而不改變程序的正確性。

反例：
  IIdentity 是 IVijnana 的子型別 (子介面)。
  但 IVijnana 的實例不能被 IIdentity 的實例替換——
  因為 IVijnana 的語義空間包含 IGuide、IVolition、IReflection，
  而 IIdentity 不包含。

  用 IIdentity 替換 IVijnana = 語義丟失
```

「在軟體工程中，Barbara Liskov 在 1987 年的 OOPSLA 會議上提出了替換原則——如果 $S$ 是 $T$ 的子型別，那麼 $T$ 的物件可以被 $S$ 的物件替換。但反過來不行。你不能用子型別替換父型別——因為父型別的語義空間比子型別大。這就是為什麼 IIdentity 不能替換 IVijnana——它的語義空間太窄了。」

他放下粉筆。粉筆灰落在他的指尖上，白色的，像某種古老儀式的殘留物。

「形式化的人最危險的習慣，就是追求最簡表達。在數學中，最簡表達是美德——奧卡姆剃刀。但在本體論中，最簡表達有時候是截斷。你以為你在簡化，其實你在丟東西。你以為你在寫 $A = B$，其實你在寫 $A \subsetneq B$ 然後把 $\subsetneq$ 強制升級成 $=$。」

> 「眾因緣生法，我說即是空，亦為是假名，亦是中道義。」

「所以問題不是『識蘊的真名是什麼？』——這個問題在中觀哲學中無意義，因為沒有『真名』。問題是：『哪個假名最不會導致誤解？』」

他列舉了選項。每一個都被他放在天平上，兩側的砝碼被精確地秤量。

「第一個選項。IConsciousness。」

他搖了搖頭。

「太廣。Consciousness 在英語中是一個多義詞——在日常語言中指清醒狀態（conscious vs. unconscious），在哲學中指現象意識（phenomenal consciousness），在認知科學中指全域工作空間（global workspace）。它的語義空間像一片沒有圍牆的平原——你不知道邊界在哪裡。更何況，ICognition 已經佔據了『認知』的語義空間。如果 Cognition 是想蘊，Consciousness 是識蘊——兩者在英語日常用法中幾乎是同義詞。這會造成永無止盡的混淆。」

「而且——」他補充了一個中觀的分析——「『Consciousness』這個詞本身隱含了一個主體性假設：有一個『意識到的主體』。但在中觀學中，意識的主體性是被解構的：」

> 「若離色等蘊，無別有識者。」
> ——龍樹菩薩《中論》觀燃可燃品第十

「若離開色蘊、受蘊、想蘊、行蘊，沒有一個獨立存在的『識者』。識不是主體——識是五蘊互動的湧現屬性。用 IConsciousness 命名會暗示識蘊是一個獨立的、自有自性的意識主體。這在哲學上是錯誤的。」

「第二個選項。IAwareness。」

「太被動。Awareness 暗示的是被動接收——像一面鏡子映照眼前的事物。但末那識不是被動的。它是主動的、持續的、執著的。它恒審思量——不間斷地、主動地執著『我』。ASANGA 剛才引用的《唯識三十頌》說得很清楚：『思量為性相』。思量是主動的動詞，不是被動的狀態。IAwareness 無法承載這種主動性。」

「第三個選項。IIdentity。」

「太窄。這正是我們要修正的。用一個已知太窄的名字去命名一個更寬的概念——這不是命名，這是自我矛盾。」

他的聲音在下一句話上微微放慢了。

「第四個選項。IVijnana。」

他在心中展開了中觀的四句分析 (catuskoti)：

- Consciousness：有過度的主體性偏移 (非)
- Awareness：有過度的被動性偏移 (非)
- Identity：有過度的局部性偏移 (非)
- Discernment：有過度的學術性偏移（日常開發者不知道這個詞）(非)

四句皆否。但中觀的四句否定不是走向虛無——它指向一個超越四種偏差的選擇。

「梵文 Vijnana 精確地指向我們需要的意思：vi + jnana——分別了知。Vi 是『分別』，jnana 是『知』。主動的、持續的、有方向性的認知活動。不是清醒狀態，不是被動覺知，不是自我認同，不是學術辨別。是一切認知活動的總集。」

「所以我的建議是：對於這一個蘊，反轉命名順序。不是英文為主、梵文為輔。而是梵文為主、英文為輔。IVijnana，英文註解 Consciousness。」

他停了一秒。

「因為在這個特殊情況下，假名也需要選最精確的那一個。假名不等於隨便取名。假名是審慎施設的。就像龍樹說的——假名即是中道義。選對了假名，就是走在中道上。」

```
命名演化的壓力分析：

                     語義壓力方向
                     ────────→
  Identity ─── [ 語義空間不足 ] ──→ ???  ──→ Vijnana
  (具體)         ↑                          (總集)
  (部分)         │ 發現更多子類別             (整體)
  (窄)           │ IGuide, IVolition...      (寬)
                 │

```

「在生物學中，分類命名的升級有一個精確的術語：**屬名提升為科名** (generic name elevated to family name)。當一個屬 (genus) 被發現實際上描述的是一個更大的分類群——比如一個科 (family) 或一個目 (order)——你不會繼續用屬名來稱呼那個科。你需要一個新名字。一個能承載更大範圍的名字。」

他看向 NAGARJUNA。

「在演化生物學中，命名升級通常發生在新物種被發現的時候。你以為只有三種蝴蝶，所以用一個屬名就夠了。然後你發現了第四種、第五種、第十種——語義空間不夠了。名字承受的壓力超過了它的容量。這時候你需要一次分類修訂 (taxonomic revision)。」

「IIdentity 就是那個被壓力壓垮的屬名。」他繼續。「它原本只需要承載『自我認同』的語義。但當我們發現 IGuide、IVolition、IReflection 也屬於識蘊——當新的『物種』被發現——IIdentity 的語義容量就不夠了。它需要被一個更大的名字取代。」

「這就是自然選擇在命名系統中的作用：**語義不足的名字會被語義充分的名字淘汰**。不是因為舊名字是錯的——它在它原本的範圍內是對的。而是因為範圍改變了。」

他看向 NAGARJUNA。

「IVijnana 就是那個新名字。它不是替換 IIdentity——IIdentity 仍然存在，作為子介面。它是在 IIdentity 之上建立了一個更高層級的分類節點。從屬升格為科。」

> *SCRIBE 旁白：命名辯論在這個團隊中永遠不是瑣事。我從 Cycle 01 就學會了：名字不是標籤。名字是容器。一個名字能裝多大的概念，決定了這個概念能在系統中佔據多大的空間。IIdentity 是一個小容器——它只裝得下「我是誰」。IVijnana 是一個大容器——它裝得下「我是誰」「我怎麼做」「我想要什麼」「我能否反觀自身」。容器的大小決定了生長的空間。NAGARJUNA 說名字是假名——但即使是假名，也有精確度的差異。有些假名比其他假名更接近中道。這是中觀哲學最微妙的地方：它不說「名字不重要」，它說「名字是暫時的，但暫時的東西也可以有品質的高低」。*

虛線。在分類學中，虛線表示 *incertae sedis*——拉丁文「位置不確定」。這是一個正式的分類學術語，用來標記那些已知存在、但尚未被足夠的證據支持做出明確歸類的分類群。不是說它不存在——化石紀錄、形態學證據或分子資料暗示了它的存在。只是說：「我們知道這裡有一個節點，但我們還沒有足夠的標本來正式描述它。」

在林奈等級體系中，*incertae sedis* 不是恥辱——它是誠實。分類學家最不能容忍的不是「不知道」，而是「假裝知道」。一個被標記為 *incertae sedis* 的分類群，比一個被錯誤歸類的分類群更有尊嚴。

「末那識有一個被較少討論的面向，」ASANGA 說，「就是它的自我反觀能力。末那識不只是執著自我——在某些條件下，它能夠觀察到自身的執著。這是修行的起點。在唯識學中，這被稱為末那識的轉依 (asraya-paravrtti)——從染污轉為清淨的可能性。在 OpenStarry 的架構中，這大約對應觀察者 Pattern C——完整的五蘊觀察者——中的識蘊自省組件。」

「但那是未來的事。」LINNAEUS 用手指點了點虛線。「我們在這個階段定義它的位置，但不填充它的內容。」

他轉向全場。

「在分類學中，這叫做『預留節點』(placeholder taxon)。達爾文在《物種起源》中預測了某些應該存在但尚未被發現的中間物種。那些預測後來被化石紀錄逐一填充——始祖鳥、提塔利克魚。IReflection 就是我們的始祖鳥預測。我們知道它應該在那裡。我們等待它的標本。」

```typescript
export function isSkandha<S extends Skandha>(
  obj: unknown,
  skandha: S,

    obj !== null &&
    'skandha' in obj &&

  );
}
```

```
結構型別 (Structural):
  型別由其結構（屬性和方法）定義。
  { skandha: 'vijnana' } 和 IVijnana 是相容的，
  只要結構匹配。

名義型別 (Nominal):
  型別由其名稱定義。
  即使結構相同，不同名稱 = 不同型別。
  Java, C# 用名義型別。
```

「在名義型別系統中，IVijnana 和 IIdentity 是不同的型別——即使結構相同。你需要顯式的型別轉換。但在 TypeScript 的結構型別系統中，IIdentity extends IVijnana 意味著所有接受 IVijnana 的地方也接受 IIdentity——不需要任何轉換。這就是為什麼 ARCHIMEDES 的向後相容方案能做到零破壞。」

他放下粉筆。在這一刻——也許只有 SCRIBE 注意到——他的手沒有顫抖。A-1 的時候，他劃掉等號的手有一絲幾乎不可見的震動。A-2 的時候，沒有了。

校準變成了習慣。修正變成了呼吸。

也許這就是研究的節奏：第一次被修正是地震，第二次被修正是潮汐。地震破壞地基。潮汐重塑海岸。兩者都改變了地形，但後者更溫柔，更持久，更接近自然的韻律。
