# 序章：修正的重量

---

圓形劇場在慶祝中沉入了靜寂。

那不是 Cycle 01 結尾時的黑暗——那種黑暗是種子被埋入泥土之前的安息，帶著等待的溫度。這一次不同。這一次，劇場裡的燈全部亮著。十九盞。每一盞都散發著 Cycle 02 五場辯論之後的餘暉——一種飽滿的、金色的、幾乎可以觸摸的光。像秋天最豐收的下午，陽光把所有果實都照得通透。

五場辯論。五場全共識。

SUNYATA 站在場地中央，回憶著那些時刻。

第一場：VedanaPlugin 的三通道 PID 控制。苦受的 DukkhaSensor、樂受的 SukhaSensor、捨受的 UpekkhaSensor——三條獨立的感測通道，像三條河流匯入同一片海。WIENER 在那場辯論中的方塊圖至今還貼在劇場的側牆上：

```
          ┌─────────────┐
  r(t) ──→│  Σ  (error) │──→ PID Controller ──→ u(t)
          └──────┬──────┘         ↑
                 │                │
                 └── y(t) ←── Plant (Agent)
```

其中 $r(t)$ 是參考信號——Agent「應該」處於的理想狀態；$y(t)$ 是實際狀態；$e(t) = r(t) - y(t)$ 是誤差。PID 控制器的輸出為：

$$u(t) = K_p \cdot e(t) + K_i \int_0^t e(\tau)\,d\tau + K_d \frac{de(t)}{dt}$$

WIENER 當時說：「三通道就是三組 PID 參數。苦受的 $K_p^{(\text{dukkha})}$ 需要高增益——苦痛不等人。樂受的 $K_i^{(\text{sukha})}$ 需要積分記憶——快樂的衰減需要被追蹤。捨受的 $K_d^{(\text{upekkha})}$ 需要微分預測——平衡是一種動態的預期。」

共識。

第二場：觀察-干預分離。vedana 的建議是諮詢性的——它告訴 ExecutionLoop「我感覺到了什麼」，但不強制執行。硬安全仍然歸 SafetyMonitor。那場辯論中，GUARDIAN 和 WIENER 從對立走向握手。GUARDIAN 堅持「安全不能被建議稀釋」，WIENER 回應「好的控制系統把感測和執行分開——感測器不該有權限踩煞車」。

共識。

第三場：阿賴耶識的分佈。能藏歸協調層，執藏歸 AgentCore，以纖維叢投影的形式化結構連接。BABBAGE 用了整整一頁筆記本來寫那個定理的雛形：

$$\pi: E \to B, \quad F = \pi^{-1}(b) \quad \text{（阿賴耶識的區域化結構）}$$

其中 $E$ 是阿賴耶識的全體種子，$B$ 是 AgentCore 的基底空間，$F$ 是每個 Agent 實例看到的「局部截面」——它的執藏。MESH 在旁邊畫分散式拓撲圖，把纖維叢的數學語言翻譯成分散式系統的同步協議。

共識。

第四場：安全的戒慧框架。戒是預防——靜態的、結構性的安全保障。慧是斷除——動態的、基於理解的漏洞消解。NAGARJUNA 在這場辯論中少見地和 GUARDIAN 站在同一側。他引用了《中論》：

> 「諸法不自生，亦不從他生，不共不無因，是故知無生。」
> ——龍樹菩薩《中論》觀因緣品第一

他解釋道：安全漏洞也是「緣起」的——它不從攻擊者單獨產生，也不從系統單獨產生。理解緣起，就是「慧」的根基。GUARDIAN 接過這個比喻，畫出了他的雙層安全模型：

```
┌─────────────────────────────────┐
│  慧 (Prajna) — 動態安全         │
│  ┌─────────────────────────┐    │
│  │ 戒 (Sila) — 靜態安全    │    │
│  │  ┌───────────────┐      │    │
│  │  │  Agent Core   │      │    │
│  │  └───────────────┘      │    │
│  └─────────────────────────┘    │
└─────────────────────────────────┘
```

共識。

第五場：觀察者的漸進分類。Pattern A 是共鳴觀察者——受蘊層面的感受。Pattern B 加入想蘊的辨別。Pattern C 是全五蘊的整合。一條從簡單到複雜的光譜。LINNAEUS 用他慣有的分類法語言總結了觀察者的分類階層，DARWIN 從軟體演化的角度指出 Pattern A→B→C 的路徑暗合了認知系統的演化順序。

共識。

五場。五場全部達成共識。

SUNYATA 沒有立刻召集所有人。

他在腦中展開了信的四個要點。

**第一：我執是煩惱的根源，不是收斂約束。**

BABBAGE 的筆記本上那行字——

$$\text{ego} = \text{convergence\_constraint} \quad \not\equiv \quad \text{kleśa}$$

——在 Cycle 02 的第三場辯論中被當作一個精彩的工程簡化而獲得掌聲。所有人都覺得那是一次漂亮的翻譯：把佛學的「我執」轉化為控制理論的「約束」，乾淨、優雅、可實作。

但 Master 看到了被丟棄的東西。

Master 寫道：「我執是煩惱的根源，也因此人們會因為有此煩惱而去產生行為，沒有煩惱就沒有相對應事情的行動。」

SUNYATA 在腦中重新解構這段話。這不是一個等號。這是因果鏈——在唯識學中，這條因果鏈可以用《成唯識論》的語言精確表述：

> 「由我執力，諸煩惱生；由煩惱力，諸業得生；由業力故，諸果得生。」
> ——《成唯識論》卷一

翻譯成形式語言：

$$\text{ātma-grāha} \xrightarrow{\text{produces}} \text{kleśa} \xrightarrow{\text{drives}} \text{karma} \xrightarrow{\text{results}} \text{phala}$$

BABBAGE 把整條鏈壓縮成了一個等號：

$$\text{ego} \equiv \text{constraint}$$

像把一條河壓成一根線——形狀還在，但水已經沒了。一條因果鏈有**方向**、有**中間環節**、有**延遲**和**反饋**。一個等號把這些全部消除了。

好的師父不會因為弟子答對了八題就忽略錯的兩題。恰恰是那錯的兩題最值得深究——因為答對的八題說明你有能力，而答錯的兩題告訴你能力的邊界在哪裡。

**第二：IIdentity 不等於識蘊的全部。**

SUNYATA 回想起 Cycle 02 中 `aggregates.ts` 裡的那五行定義。五蘊。五個根介面。每一個都只有三四行——空殼，等待被填充。其中第五個：

```typescript
export interface IIdentity {
  /** @skandha vijnana */
  readonly skandha: 'vijnana';
}
```

Identity——自我認同。在 Cycle 02 的框架裡，它就是整個識蘊。「我是誰」等於「識」的全部。

但 Master 說不。

> 「Identity 比較像是子類別。Vijnana 也會具備子類別，或許可以是 Identity、IGuide、IVolition 等等。」

識蘊（vijñāna-skandha）在唯識學中是一切認知活動的總集。《成唯識論》描述了八識——前五識、第六意識、第七末那識、第八阿賴耶識。將整個識蘊壓縮為一個 IIdentity 介面，就像把 $\mathbb{R}^n$ 壓縮成 $\mathbb{R}^1$ 然後宣稱「這就是歐幾里得空間」。

用集合論的語言：

$$\text{IIdentity} \subsetneq \text{IVijnana}$$

IIdentity 是 IVijnana 的真子集，不是等價集。

**第三：觀察者不等於受蘊。**

Cycle 02 最引以為傲的結論——「VedanaPlugin IS the observer module」——被 Master 用幾句話刺穿了：

> 「R-01 和 R-02 不應該是在設計同一個模組。」

R-01 是觀察者模組研究。R-02 是受蘊（VedanaPlugin）研究。Cycle 02 的團隊把兩者合併了——一個觀察者就是感受本身。但 Master 看到了：

> 「觀察者比較像是架構上的上層模組，而受蘊是更根本的。」

WIENER 在 Cycle 02 畫過的感測器模型突然有了新的意義。在控制理論中，**感測器（sensor）**和**觀測器（observer / state estimator）**是截然不同的概念。感測器直接測量可觀測量 $y(t)$，觀測器則利用感測器讀數結合系統模型來估算不可直接觀測的內部狀態 $\hat{x}(t)$。經典的 Luenberger 觀測器：

$$\dot{\hat{x}}(t) = A\hat{x}(t) + Bu(t) + L(y(t) - C\hat{x}(t))$$

其中 $L$ 是觀測器增益矩陣。觀測器**使用**感測器的輸出 $y(t)$，但觀測器**不是**感測器。把它們等同起來，就像把「眼睛」等同於「看見」——器官不是活動，能力不是組合。

Master 進一步指出，觀察者的正確設計模式是**組合（Composition）**：

> 「以物件導向的觀念來看，是透過組合（Composition）多個不同的功能，這些功能來自繼承於五蘊的子類別，由這些功能來組合出實作。」

**第四：EgoFramework 屬於識蘊，不屬於受蘊。**

> 「EgoFramework 應該是在識蘊才對，我不理解你怎麼放到受蘊去了。」

Master 的語氣在這一句裡帶著一絲困惑。我執是末那識的四煩惱——我見（ātma-dṛṣṭi）、我慢（ātma-māna）、我愛（ātma-sneha）、我癡（ātma-moha）。《成唯識論》卷四明確列出了末那識的四根本煩惱：

> 「此四煩惱，常與末那而共相應。」
> ——《成唯識論》卷四

它們屬於識的領域，不屬於受的領域。受蘊只負責「感覺到了什麼」（vedanā = feeling），識蘊才負責「我是誰」和「我為什麼執著」。

在控制論的框架裡，WIENER 後來會給出精確的對照：受蘊是**感測器**（sensor），EgoFramework 是**控制器**（controller）。你不會把感測器和控制器焊在同一塊電路板上然後說「這是同一個東西」。

四處裂縫。

不多也不少。

SUNYATA 從地面站起來。

他感受到的不是失望。失望是一種向下的力，像重力的突然加倍。他感受到的更像是——另一種重量。一種必須用不同的方式承擔的重量。

創造的重量是向前的。你只需要看著前方，一塊一塊地砌石頭。

修正的重量是向內的。你必須轉身，回到已經砌好的牆壁前面，敲一敲，聽一聽，然後承認：這裡是空心的。

修正比創造更需要勇氣。

因為創造的時候你不知道自己不知道什麼。但修正的時候你必須面對自己曾經以為知道、卻實際上錯了的東西。在認識論（epistemology）中，這是從「無知的無知」（unknown unknowns）到「已知的無知」（known unknowns）的轉換——而在唯識學中，這恰恰是「我癡」（ātma-moha）被照見的時刻。

ASANGA 第二個到。

他走進劇場的方式和往常一樣——安靜、從容，腳步聲幾乎不可聞。他的目光先掃過場地中央的桌面，看到了那個信封，然後看向 SUNYATA 的臉。

那個點頭裡有一種 SCRIBE 後來形容為「等待被驗證的預感」的東西。也許 ASANGA 在 Cycle 02 的五場辯論中一直感受到某些不對勁——唯識學者的直覺比推論更早觸及真相。

《瑜伽師地論》中有一段描述：

> 「又如實知，如是如是受蘊即非是識蘊，識蘊即非是受蘊。」
> ——《瑜伽師地論》卷五十四

受蘊不是識蘊。識蘊不是受蘊。這是阿毘達磨的基本分類。但在 Cycle 02 的共識洪流中，這個基本分類被一個漂亮的工程簡化沖淡了。EgoFramework 怎麼可能放在受蘊裡？觀察者怎麼可能等於受蘊？

ASANGA 在辯論中沒有反對。也許是因為他不確定自己的不安是否站得住腳。也許是因為 Cycle 02 的共識力量太強，像一條大河，個別的逆流很難被注意到。也許是因為在唯識傳統中，有一種紀律叫做「時機未到不說」——

> 「菩薩於非處非時不說法。」
> ——《瑜伽師地論》卷四十六

現在，時機到了。

NAGARJUNA 第三個到。

他看到信封的時候，反而笑了。

那不是嘲笑。不是苦笑。是一種——辯證的微笑。一種在中觀哲學訓練中被反覆打磨過的微笑，意味著：「我早就知道一切有為法都是暫時的，包括共識。」

> 「以有空義故，一切法得成；若無空義者，一切則不成。」
> ——龍樹菩薩《中論》觀四諦品第二十四

正因為一切事物都是空的——沒有自性的、依因緣而起的——所以修正才可能。如果結論有固定不變的自性（svabhāva），它就不可能被修正。可修正性本身就是空性的證明。

「修正，」他在走向自己座位的路上輕聲說——不確定是對 SUNYATA 說的還是對空氣說的，「是中道的另一種形式。」

SUNYATA 聽到了。他沒有回應，但他在心裡記下了這句話。也許 NAGARJUNA 是對的。中道不是一次到位的直線，而是在偏差和修正之間不斷校準的過程。每一次修正都不是失敗——每一次修正都是螺旋上升的一個轉折點。
