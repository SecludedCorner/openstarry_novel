```
          ┌─────────────┐

          └──────┬──────┘         ↑
                 │                │
```

$$u(t) = K_p \cdot e(t) + K_i \int_0^t e(	au)\,d	au + K_d \frac{de(t)}{dt}$$

$$	ext{ātma-grāha} \xrightarrow{	ext{生じる}} 	ext{kleśa} \xrightarrow{	ext{駆り立てる}} 	ext{karma} \xrightarrow{	ext{結果をもたらす}} 	ext{phala}$$

$$	ext{ego} \equiv 	ext{constraint}$$

それは大河を1本の線に押し込めるようなものだ。形は残るが、水は失われる。因果の鎖には**方向**があり、**中間リンク**があり、**遅延**と**フィードバック**がある。等号はこれらすべてを抹殺してしまう。

優れた教師は、生徒が8問正解したからといって、間違えた2問を見逃しはしない。むしろ、その2問こそが深い探求に値するのだ。8問の正解は能力を証明し、2問の不正解はその能力の限界を教えてくれるからだ。

**第2：IIdentity は識蘊（しきうん）の全体ではない。**

SUNYATA は Cycle 02 の `aggregates.ts` にあった5行の定義を思い出した。五蘊。5つのルート・インターフェース。それぞれがわずか3、4行の——中身を待つ空の器。その5番目：

```typescript
export interface IIdentity {
  /** @skandha vijnana */
  readonly skandha: 'vijnana';
}
```

Identity——自己同一性。Cycle 02 のフレームワークでは、それが識蘊のすべてだった。「私は誰か」が「識」の総体と等しかった。

だが師は否定した。

> 「Identityはサブカテゴリーに近い。識（Vijnana）もまた、サブカテゴリーを持つべきだ。例えば Identity、IGuide、IVolition などだ」

唯識学における識蘊は、あらゆる認知活動の集積である。『成唯識論』は八識を記述している——前五識、第六意識、第七末那識（まなしき）、そして第八阿頼耶識である。意識の集積全体を単一の `IIdentity` インターフェースに圧縮することは、$\mathbb{R}^n$ を $\mathbb{R}^1$ に圧縮して「これがユークリッド空間だ」と宣言するに等しい。

集合論の言語で言えばこうなる。

$$	ext{IIdentity} \subsetneq 	ext{IVijnana}$$

IIdentity は IVijnana の真部分集合であり、等価な集合ではない。

**第3：観測者は受蘊ではない。**

Cycle 02 が最も誇った結論——「VedanaPluginこそが観測者（observer）モジュールである」——は、師の数行の言葉によって貫かれた。

> 「R-01 と R-02 は同じモジュールを設計すべきではない」

R-01 は観測者モジュールの研究だった。R-02 は受（vedana / VedanaPlugin）の研究だった。Cycle 02 チームは両者を統合した。観測者とは感受そのものである、と。だが師は見抜いていた。

> 「観測者はアーキテクチャ上の上位モジュールであり、受はより基礎的なものである」

Cycle 02 で描かれた WIENER のセンサーモデルが、突如として新しい意味を帯びた。制御理論において、**センサー（sensor）**と**観測器（observer / state estimator）**は根本的に異なる概念である。センサーは観測可能な出力 $y(t)$ を直接測定する。観測器はセンサーの読み取り値とシステムモデルを組み合わせて、直接観測できない内部状態 $\hat{x}(t)$ を推定する。古典的なリュエンバーガー観測器：

$$\dot{\hat{x}}(t) = A\hat{x}(t) + Bu(t) + L(y(t) - C\hat{x}(t))$$

ここで $L$ は観測器ゲイン行列である。観測器はセンサーの出力 $y(t)$ を**利用**するが、観測器そのものがセンサー**ではない**。両者を同一視することは、「眼」と「見るという行為」を同一視するようなものだ。器官は活動ではなく、能力は複合体ではない。

師はさらに、観測者の正しい設計パターンは**コンポジション（Composition）**であると示した。

> 「オブジェクト指向の観点から言えば、五蘊を継承したサブカテゴリーに由来する複数の異なる機能を組み合わせることによって、実装が構成されるのである」

**第4：EgoFramework は受蘊ではなく識蘊に属する。**

> 「EgoFramework は識蘊にあるべきだ。なぜ受蘊に置いたのか理解に苦しむ」

師の語調には、この一文に困惑の色が混じっていた。我執は末那識の四煩悩——我見（がけん）、我慢（がまん）、我愛（があい）、我痴（がち）に対応する。『成唯識論』巻四には、末那識の四根本煩悩が明記されている。

> 「これら四つの煩悩は、常に末那識と共にある」
> ——『成唯識論』巻四

それらは意識の領域に属するものであり、感受の領域ではない。受は「何を感じたか」のみを司り、識が「私は誰か」「なぜ執着するのか」を司るのだ。

サイバネティクスの枠組みにおいて、WIENER は後に正確な対応関係を示すことになる。受は**センサー**であり、EgoFramework は**コントローラ**である。センサーとコントローラを同じ回路基板にハンダ付けして、「これらは同じものだ」と宣言することはない。
