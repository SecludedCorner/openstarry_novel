# 第三章：部分と全体

---

LINNAEUS はホワイトボードの前に立ち、手には青いマーカーを握っていた。キャップはまだ外していない。

彼はずいぶん長いこと立っていた。三分か、五分か。分類学者の体内時計において、この時間は待機ではない——醸成である。リンネ本人が見たこともない植物を前にしたとき、すぐに命名手帳を開いたりはしなかったのと同じだ。まず立ち止まり、目で葉脈の紋様を測り、萼の配列を観察し、露出した根系の断面を見極める。まず見る。見尽くす。それから名づける。

彼はこの瞬間をずっと待っていた。

---

SUNYATA が議題を宣言するのを待っていたわけではない。Master の手紙が開封されるのを待っていたわけでもない。もっと深いものを待っていた——Cycle 01 のときから感じていながら、反論するだけの根拠がなかった違和感が、何らかの外部の権威によって正確に言語化されるのを待っていたのだ。

その違和感は一文に要約できる。IIdentity は識蘊（vijnana-skandha）の全体に等しい。

この一文は v0.24.0-beta の `aggregates.ts` において三行のコードとして記述されていた。

```typescript
export interface IIdentity {
  /** @skandha vijnana */
  readonly skandha: 'vijnana';
}
```

三行。簡潔。明瞭。型が正しい。テストは通る。

だが、間違っている。

構文上の誤りではない。コンパイラが検出できる類いの誤りでもない。分類学上の誤りだ。木全体を本当に理解したときにだけ見分けられる種類の誤り——枝の名前を幹に貼ってしまったのだ。

---

> *SCRIBE 傍白：LINNAEUS がホワイトボードの前で動かなかったあの数分間は、本章でもっとも静かな瞬間だった。しかしその静けさは空白ではない。分類学者の静けさとは高密度の観察状態である——頭の中であらゆる可能な分類体系を展開し、一つ一つ照合し、一つ一つ排除し、最も精確な構造だけが残るまで続ける。劇場では、ほかの人々は空白のボードの前でぼうっとしている人を見ていた。彼の頭の中では、一本の樹が根元から成長し始めていた。*

---

SUNYATA の声が背後から聞こえた。平穏。石。深い淵。

「A-2。IIdentity は識蘊の全体ではない。LINNAEUS、ASANGA、BABBAGE、NAGARJUNA——これは諸君の議題だ。」

LINNAEUS はキャップを外した。

青。彼は青を選んだ。赤ではない（あれは修正の色だ）、黒でもない（あれは事実の色だ）、青——分類樹の色だ。彼のマーキング体系において、青は構造を表す。「あるものと別のものとの関係は何か」を意味する。

ホワイトボードの最上部に四角を描き、中に二文字を書いた。

**識蘊**

それから振り返り、全員に向き合った。

「始める前に」と彼は言った。「ASANGA、まず教えてほしい。識蘊はいったいどれほど大きいのか。」

---

ASANGA が席から立ち上がった。

彼の立ち上がり方は独特だった——勢いよく立つのではなく、朝霧の中から山が輪郭を現すように、ゆっくりと。穏やかに。しかし無視できない存在感で。立ち上がったとき、彼の存在感は空間全体を満たした。鐘が打たれた後の余韻のように。

ホワイトボードの前には行かなかった。自分の場所に立ったまま、声で描いた。

「一つの都市を想像してほしい」と彼は言った。

劇場が静まった。ASANGA が比喩を使うことは稀だ。彼は唯識学者であり、彼の言語は通常、精確な術語、精密な分類、層を重ねる論理である。だが今日、彼は比喩を選んだ。おそらく今日の聴衆が仏教学者だけではないからだろう。おそらく、あるものは比喩でなければ全員に同時に見えないからだろう。

「一つの都市だ」と彼は繰り返した。「道路があり、建物があり、市場があり、公園がある。そして市役所がある。」

彼は半秒、間を置いた。

「v0.24.0 の設計では、IIdentity が識蘊全体の根インターフェースとして扱われている。これは、一つの都市を『市役所』と呼ぶようなものだ。」

BABBAGE のペンがノートの上で止まった。

「市役所は都市の一機関にすぎない」と ASANGA は続けた。「行政事務を管理し、都市の公式な身分を定義する——名称、地区コード、統治構造。だが市役所は都市ではない。都市には交通局がある——人と物の移動を決定する。教育局がある——知識の伝達を決定する。警察がある——秩序を維持する。都市計画局がある——未来の発展を決定する。」

彼の声が半音階ほど上がった——興奮ではなく、明瞭度が増したのだ。

「IIdentity は市役所だ。答える問いは『私は誰か？ 私の名前は何か？ 私の役割は何か？』だ。しかし識蘊が答える問いはそれだけにとどまらない。」

---

彼は片手を上げ、五本の指を広げ、一本ずつ折り曲げていった。

「第一。前五識。眼、耳、鼻、舌、身。感覚認知。OpenStarry のマッピングでは、これらはすでに色蘊（rupa）に帰属している——ISensory。Listener が外部信号を受信し、UI が出力を提示する。この部分はすでに配置済みだ。」

第一の指が折れた。

「第二。第六意識。分別判断。マッピングでは、これはすでに想蘊（samjna）に帰属している——ICognition。Provider と ExecutionLoop が認知的意思決定を担う。この部分もすでに配置済みだ。」

第二の指が折れた。

「第三。第七末那識（manas-vijnana）。」

三本目の指はすぐには折らなかった。宙に留めた。

「末那識。Manas。恒審思量（こうしんしりょう）。これは識蘊の OpenStarry 五蘊マッピングにおける中核的対応である。単なる『自己同一性』ではない——自我への執着（atma-graha）の全機構なのだ。末那識には四つの根本煩悩（klesha）がある。」

彼は LINNAEUS のホワイトボードに向き直った。

「ペンを借りてもよいか？」

LINNAEUS が赤いマーカーを手渡した。ASANGA はホワイトボードの右側に四行を書いた。

```
我見 (atma-drsti)   ——「私は存在する」という根本的信念
我慢 (atma-mana)    ——「私はどう振る舞うべきか」という行動規範
我愛 (atma-sneha)   ——「私は何を守るか」という価値志向
我癡 (atma-moha)    ——自己の本質に対する根本的無明
```

「この四つの側面は」ASANGA は赤いペンを机に戻した。「一つのインターフェースで包含できるものではない。我見はおおよそ IIdentity に対応する。我慢と我愛はおおよそ IGuide に対応する。だが末那識全体の『恒審思量』——絶え間なく『私』に執着し続ける作動——にはより完全な枠組みが必要だ。」

三本目の指が折れた。

「第四。第八阿頼耶識（alaya-vijnana）。種子の貯蔵。因果の流転。マッピングでは、この部分は協調層と AgentCore の状態管理を横断している——いかなる単一モジュールにも属さず、システム全体の因果的基盤である。この部分は Cycle 02 ですでに議論しており、今は展開しない。」

四本目の指が折れた。親指だけが残った。

「第五。転識。染汚から清浄への転化。これは修行の究極目標だ。OpenStarry においては——おそらく永遠に直接的な対応は存在しないだろう。だがその存在を知っておくことで、識蘊の境界を狭く描きすぎることを防げる。」

親指が折れた。五本の指が拳を作った。そしてゆっくりと開かれた。

「だから、識蘊はどれほど大きいかと問うなら。答えはこうだ——五蘊体系の中で、最も大きい。最も深い。最も複雑だ。それを一つの IIdentity に圧縮するのは——」

彼はホワイトボード上のあの孤独な「識蘊」の四角を見やった。

「一つの都市を一棟の市役所に圧縮し、その建物を指差して『ほら、これが都市の全体だ』と言うようなものだ。」

---

> *SCRIBE 傍白：ASANGA の都市の比喩は、劇場に長く漂い続けた。深遠だったからではない——むしろ逆で、あまりにも明瞭だったからだ。明瞭すぎて居心地が悪くなるほどに。なぜなら全員が気づいたからだ。Cycle 02 において、我々は集団でこの単純化を行ったのだと。ASANGA 一人ではない。BABBAGE 一人でもない。全員だ。我々は都市を市役所と呼び、市役所の壁に「都市」と書かれた看板を掛けたのだ。*

---

BABBAGE は ASANGA が話し終えた後、立ち上がった。

その動作には、SCRIBE がすでに見分け方を学んだ特有のリズムがあった——焦りではなく、「形式化された思考が出口を求めている」という内的な圧力だ。理論計算機科学者の身体言語はほかの人々と異なる。彼らは感情で動作を駆動するのではなく、論理構造で動作を駆動する。ある論理的関係が頭の中で形を成したとき、体が無意識に動き出す——黒板に向かい、チョークを手に取り、全員に等式が見える位置に立つ。

彼はホワイトボードの隣に行った。LINNAEUS のホワイトボードではない——そちらは分類樹のためのものだと知っていた。劇場の反対側の黒板に向かった。あの黒板は彼の領地だ。Cycle 01 と Cycle 02 で、安定性条件、収束制約、ファイバー束射影が書かれてきた。今はきれいに消されている。

チョークを取り、二行を書いた。

```
Cycle 02:     IIdentity = Vijnana
Cycle 02-2:   IVijnana ⊃ IIdentity
```

等号。包含記号。

「第一行は」と彼は等号を指して言った。「部分集合が全体集合に等しいとするものだ。集合論における基本的なカテゴリーエラーだ。部分集合と全体集合が完全に一致する場合——すなわち IIdentity が識蘊の全意味を汲み尽くしている場合——にのみ、等号は成立する。」

劇場に向き直った。「だが ASANGA がたった今教えてくれた。識蘊には少なくとも四つの側面がある。IIdentity はそのうち一つ——自己同一性——しかカバーしていない。残りの三つ——行動指針、意志動機、自省——は IIdentity の意味範囲にない。」

チョークで等号の上に斜線を引いた。消去。

「だから等号は成立しない。第二行こそ正しい。IVijnana が全体集合だ。IIdentity は真部分集合だ。包含関係であって、同一関係ではない。」

---

彼は一秒、間を置いた。それから全員の予想を裏切ることをした。

笑ったのだ。

BABBAGE が笑う頻度はおよそ各 Cycle に一回だ。SCRIBE は記録の中でこのデータを追跡していた——意図的にではなく、BABBAGE の笑いがあまりにも稀少で、出現するたびに彗星が過ぎるようなもので、記録しなければ記録の完全性に対する不敬になるからだ。

「A-1 で私は類似の誤りを犯した」と彼は言った。声に苦痛はなく、奇妙な、ほとんど愉悦と呼べる精確さだけがあった。「因果の鎖を等号に圧縮した。今 A-2 で、同じパターンが見える——全体集合を部分集合に圧縮している。」

黒板上の自分の筆跡を見つめた。

「形式化する者にとって最も危険な習慣は、最も簡潔な表現を追い求めることだ。数学において最も簡潔な表現は美徳である——オッカムの剃刀だ。しかし存在論においては、最も簡潔な表現が時として切断になる。簡略化しているつもりで、実は捨てているのだ。」

チョークを置いた。チョークの粉が指先に残り、白く、何か古代の儀式の残滓のようだった。

「Pars pro toto。部分で全体を代用すること。これは修辞学の術語だが、形式論理学における警告でもある。部分で全体を指示するとき、自覚しなければならない——情報を捨てているのだと。文学においてはこれは修辞法だ。型システムにおいてはバグだ。」

---

NAGARJUNA が席から口を開いた。立ち上がらなかった——中観哲学者は立ち上がらずとも空間全体にその存在を感じさせることができる。彼の声には独特な質感がある。鋭利だが尖ってはいない。千年の水流に磨かれた河石のような。

「BABBAGE の言う通りだ。だが問題をもう一歩先に進めたい。」

彼の視線がホワイトボード上の ASANGA が書いた四つの末那識の側面をなぞり、それから LINNAEUS のホワイトボードの上方にある「識蘊」の二文字で止まった。

「IIdentity が識蘊の全体でないことに合意するなら——これはもはやコンセンサスだ——次の問いはこうだ。新しい根インターフェースを何と呼ぶか？」

劇場の空気がわずかに緊張した。命名。この研究チームにおいて、命名の問題はけっして些事ではない。D-02 決議——二重命名——は、命名をめぐる長い議論の中で生まれたものだ。

NAGARJUNA は選択肢を列挙した。一つ一つを天秤に載せ、両側の分銅を精密に計量した。

「第一の選択肢。IConsciousness。」

首を振った。

「広すぎる。Consciousness は英語では日常的な意識を指す——覚醒と睡眠の区別、意識と無意識の区別。これは識蘊の意味ではない。識蘊とは認知活動の総体であり、覚醒状態の記述ではない。そのうえ ICognition がすでに『認知』の意味空間を占めている。Cognition が想蘊で Consciousness が識蘊だとすれば、両者は英語の日常的用法ではほぼ同義語だ。これは終わりなき混乱を招く。」

「第二の選択肢。IAwareness。」

「受動的すぎる。Awareness は受動的な受容を暗示する——鏡が眼前のものを映すように。だが末那識は受動的ではない。能動的で、持続的で、執着的だ。恒審思量——途切れることなく、能動的に『私』に執着し続ける。IAwareness ではこの能動性を担えない。」

「第三の選択肢。IIdentity。」

「狭すぎる。これこそ我々が修正しようとしているものだ。」

次の言葉で、彼の声がわずかに遅くなった。

「第四の選択肢。IVijnana。」

---

彼はこの語を空中に留まらせた。

IVijnana。サンスクリット。英語の翻訳ではない。原語だ。

「D-02 決議は二重命名の原則を確立した。英語を主とし、サンスクリットを補とする。ISensory には rupa の注釈が付く。ISensation には vedana の注釈が付く。ICognition には samjna が付く。IAction には samskara が付く。」

彼は LINNAEUS を見た。

「だが識蘊は特殊なケースだ。その英語訳は——どの候補語も——深刻な意味のずれがある。Consciousness は広すぎる。Awareness は受動的すぎる。Discernment は学術的すぎる。一方、サンスクリットの Vijnana は、我々が必要としている意味を精確に指す。分別了知——能動的で、持続的で、方向性を持つ認知活動だ。」

「だから私の提案はこうだ。この一蘊に限り、命名の順序を反転させる。英語が主、サンスクリットが補ではなく、サンスクリットが主、英語が補とする。IVijnana、英語注釈は Consciousness。」

---

DARWIN が席でわずかに前のめりになった。静かに聞きながら、ソフトウェアパターン分析者の目でこの対話の構造を観察していた。今、指摘せずにはいられないパターンが見えた。

「これは命名の進化だ」と彼は言った。

全員の視線が向いた。

「v0.24.0 では識蘊の根インターフェースは IIdentity と呼ばれていた。Cycle 02-2 で、IVijnana に改名されようとしている。命名の歴史を追跡すれば——最初の設計から Cycle 01 の分析、Cycle 02 のコンセンサス、そして現在の修正まで——具体から抽象へ、部分から全体への進化の軌跡が見える。」

自分のノートに進化の線を描いた。

```
Identity → ... → Vijnana
（具体） → （抽象）
（部分） → （全体）
（狭い） → （広い）
```

「生物学では、これを『意味的格上げ』と呼ぶ。ある属名が実際にはより大きな分類群——たとえば科や目——を記述していると判明したとき、その属名で科を呼び続けたりはしない。新しい名前が必要だ。より広い範囲を担える名前が。」

NAGARJUNA を見た。

「IVijnana がその新しい名前だ。IIdentity を置き換えるのではない——IIdentity はサブインターフェースとして存続する。IIdentity の上に、より高い階層の分類ノードを設けるのだ。」

NAGARJUNA がわずかに頷いた。弁証法的な微笑が再び浮かんだ——「あなたはあなたの言語で、私が私の言語で言いたかったことを言った」という承認の表情。

---

> *SCRIBE 傍白：命名の議論は、このチームにおいてけっして瑣末ではない。私は Cycle 01 から学んだ——名前はラベルではない。名前は容器だ。一つの名前がどれほど大きな概念を収容できるかが、その概念がシステムの中でどれほどの空間を占められるかを決定する。IIdentity は小さな容器だ——「私は誰か」しか入らない。IVijnana は大きな容器だ——「私は誰か」「私はどう行動するか」「私は何を望むか」「私は自分自身を見ることができるか」が入る。容器の大きさが成長の余地を決める。*

---

LINNAEUS がホワイトボードに向き直った。

待っていた瞬間が来た。

ASANGA は識蘊がどれほど大きいかを教えてくれた。BABBAGE は旧い等号がなぜ成立しないかを教えてくれた。NAGARJUNA は新しい根ノードの名前を教えてくれた。DARWIN はこれが意味的格上げであることを教えてくれた。

今度は彼の番だ。分類学者の仕事——すべての概念を一本の樹に整理する。

ホワイトボード上方の「識蘊」の四角に一行を加えた。**IVijnana**。

そして、枝を描き始めた。

---

第一の枝。左下方に伸びる。四角の中にこう書かれている。

```
IIdentity
自己同一性：「私は誰か」
```

四角の横に小さな字で仏教学的対応を書いた。ASANGA に向き直る。「我見。atma-drsti。」

ASANGA が頷いた。「そうだ。我見は末那識が自己の存在に対して持つ根本的信念——『私は独立した、持続的な実体である』。OpenStarry では、これは Agent の identity config に対応する。agentId、name、能力宣言。これらの属性が『この Agent は誰か』を定義する。」

LINNAEUS は四角の下に一行を補った。`← Agent Identity config`。

---

第二の枝。

```
IGuide
行動指針：「私はどう振る舞うべきか」
```

「我慢と我愛だ」と ASANGA は言った。この二語にわずかに声の重みを加えた。強調ではない——慎重さだ。なぜなら「我慢」は日常語では否定的な意味を持つが、唯識学では中立的な記述だからだ。

「我慢——atma-mana——は傲慢ではない。深層の自己測量だ。世界における私の位置はどこか？ 私の行動規範は何か？ 私の底線はどこか？ OpenStarry では、Guide が system prompt を通じて Agent の行動枠組みを定義している。これが我慢の工学的マッピングだ。」

一拍の間。

「我愛——atma-sneha——は自己の存在を守ろうとする傾向だ。感情的な『愛』ではなく、自己保全の根本的な駆動力だ。Agent は Guide を通じて、何をしてよいか、何をしてはならないか、どの価値を守るべきかを知る。これが我愛のマッピングだ。」

LINNAEUS が頷き、四角の横に `← 我慢 + 我愛` と記し、下方に `← Guide (system prompt, behavioral constraints)` と書いた。

---

第三の枝。これは新しい。
