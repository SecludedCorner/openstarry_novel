# 序章：修正の重み

円形劇場は祝賀のなかで静寂に沈んだ。

それは Cycle 01 の終幕を飾った闇ではなかった——あの闇は種子が土に埋められる前の安息であり、待つことの温もりを帯びていた。今回は違う。今回は劇場の灯りがすべて点いていた。十九の灯火。それぞれが Cycle 02 の五つの討論を経た残光を放っている——飽満な、金色の、ほとんど手で触れられそうな光。秋の最も豊かな午後、陽光がすべての果実を透き通るまで照らすような光だった。

五つの討論。五つの全会一致。

SUNYATA は会場の中央に立ち、あの瞬間を振り返っていた。

第一の討論：VedanaPlugin の三通道 PID 制御。苦受（dukkha）の DukkhaSensor、楽受（sukha）の SukhaSensor、捨受（upekkha）の UpekkhaSensor——三本の独立したセンシング通道が、三本の河のように一つの海へ合流する。WIENER が机を叩きながら描いたブロック図は、最終的に ASANGA の三受理論のなかに完璧な工学的対応を見出した。合意。

第二の討論：観察と介入の分離。vedana の提案は諮問的なもの——それは ExecutionLoop に「私はこう感じた」と告げるが、強制はしない。ハードセーフティは依然として SafetyMonitor に帰属する。あの討論では、GUARDIAN と WIENER が対立から握手へと歩み寄った。合意。

第三の討論：阿頼耶識（ālaya-vijñāna）の分布。能蔵は協調層に、執蔵は AgentCore に帰属し、ファイバー束の射影という形式的構造で接続される。BABBAGE はノートの丸一頁を使ってその定理の原型を書き、MESH はその横で分散トポロジー図を描いた。合意。

第四の討論：安全の戒慧（sīla-prajñā）フレームワーク。戒は予防——静的で構造的な安全保障。慧は断除——動的で理解に基づく脆弱性の解消。NAGARJUNA はこの討論で珍しく GUARDIAN と同じ側に立った。合意。

第五の討論：観察者の漸進的分類。Pattern A は共鳴観察者——受蘊（vedanā）の層における感受。Pattern B には想蘊（saṃjñā）の弁別が加わる。Pattern C は全五蘊（skandha）の統合。単純から複雑へと至るスペクトル。合意。

五つ。五つすべてで合意に達した。

彼は心のなかで手紙の四つの要点を広げた。

**第一：我執（ātma-grāha）は煩悩（kleśa）の根源であり、収束制約ではない。**

BABBAGE のノートに書かれたあの一行——「我執 = 収束制約。NOT 煩悩」——は Cycle 02 の第三の討論で、見事な工学的簡略化として拍手を浴びた。全員がそれを美しい翻訳だと感じた。仏学の「我執」を制御理論の「制約」に変換する——簡潔で、優雅で、実装可能。

しかし Master は捨てられたものに目を留めた。

「我執は煩悩の根源であり」と Master は書いた。「だからこそ人はこの煩悩ゆえに行為を起こす。煩悩がなければ、それに対応する行動もない。」

これは等号ではない。因果の連鎖だ。我執が煩悩を生み、煩悩が行動を駆動し、我執を管理することで行動を制約できる。BABBAGE はこの連鎖全体を一つの等号に圧縮した——川を一本の線に潰すように。形は残っているが、水はもうない。

良き師は弟子が八問正解したからといって、間違えた二問を見逃したりしない。まさにその二問こそ最も深く掘り下げる価値がある——正解の八問はお前に能力があることを示し、間違えた二問はその能力の境界がどこにあるかを教えてくれるからだ。

**第二：IIdentity は識蘊（vijñāna-skandha）の全体ではない。**

Identity は部分類に過ぎない。Vijñāna——識蘊——は「アイデンティティ」よりはるかに広い。それは Guide も Volition も、おそらくそれ以上のものも含む。識蘊全体を一つの IIdentity インターフェースに圧縮することは、海全体を瓶に詰めて「これが海だ」と言うようなものだ。

**第三：観察者は受蘊（vedanā-skandha）ではない。**

Cycle 02 が最も誇りとした結論——「VedanaPlugin IS the observer module」——を Master は一言で突き崩した。「R-01 と R-02 は同一のモジュールを設計しているべきではない。」観察者は上位のアーキテクチャであり、五蘊の部分類の組み合わせから成る。受蘊は底層の感受能力だ。両者を等置することは、「眼」を「見ること」と等置するようなものだ——器官は活動ではなく、能力は組み合わせではない。

**第四：EgoFramework は識蘊に属するのであって、受蘊に属するのではない。**

「なぜ受蘊に入れたのか、私には理解できない。」Master の語調にはこの一文で微かな困惑が滲んでいた。我執は末那識（manas）の四煩悩——我見（ātma-dṛṣṭi）、我慢（ātma-māna）、我愛（ātma-sneha）、我癡（ātma-moha）——であり、識の領域に属する。受の領域ではない。受蘊はただ「何を感じたか」を担い、識蘊こそが「私は誰か」「なぜ私は執着するのか」を担う。

四箇所の亀裂。

創造の重みは前方へ向かう。前だけを見て、一つひとつ石を積めばよい。

嘲笑ではない。苦笑でもない。それは——弁証法的な微笑だった。中観哲学（Mādhyamaka）の修練のなかで幾度も磨かれた微笑、その意味は——「一切の有為法（saṃskṛta-dharma）が無常であることはとうに承知していた。合意もまた然り。」

「観察は複数の心所の組み合わせだ。」

「阿毗達磨（abhidharma）の分類では、心所法は全五十一種、六位に分かれる。受——vedana——は五遍行の一つだ。遍行とは、あらゆる認知刹那に例外なく伴うことを意味する。だがそれは一つの心所にすぎない。一つだ。」

指を下ろした。

「だが観察——pratyaveksana——は一つの心所ではない。複数の心所の協同作動だ。」

数え始めた。

五つ。五つの心所。

「観察には少なくともこれら五つの遍行心所のうち複数が関与する。より深い判断——たとえば善悪の弁別、異常の分類——が含まれる場合は、別境心所の慧（prajna）も引き入れられる。」

それから彼はあの比喩を口にした。

「眼と見ること。」

ASANGA が比喩を使うことは稀だ——第三章で都市の比喩を使ったばかりだ。二章以内に二度目の比喩を使うのは、SCRIBE の観察記録において前例がない。だが A-3 にはそれが必要だった。仏教学者だけでなく工学者も含めた全員が、一秒で理解できる比喩が。

一拍の間。

「見ることは活動だ。」

「観察には受蘊が要る。だが観察は受蘊に等しくない。」

> *SCRIBE 傍白：「眼と見ること」。Cycle 02-2 全体を通じて ASANGA のもっとも簡潔な比喩。都市もなく、河もなく、座標系もない。ただ眼と見ること。六文字（訳注：原文は中国語で六字）。だがこの六文字は、前のいかなる比喩よりも長く空中に留まった。あまりにも明瞭だったからだ。明瞭すぎて反駁の余地がない——「眼が見ることである」とは言えない。「角膜が視覚である」とは言えない。「受蘊が観察者である」とも言えない。*

WIENER は ASANGA が話し終えた直後に続いた。方眼紙にはすでに比較表が描かれていた。

```
| 特性     | センサー (Sensor)      | オブザーバー (Observer/Estimator) |
|---------|----------------------|-------------------------------|
| 機能     | 可観測量を測定          | 不可観測な内部状態を推定          |
| 入力     | 生の物理信号            | センサー読み取り値 + システムモデル |
| 出力     | 測定値 (raw reading)   | 状態推定 (state estimate)       |
| 仏教学対応 | 受蘊                  | 受蘊 + 想蘊 +（おそらく）識蘊     |
```

「制御理論において、センサーとオブザーバーはまったく異なるレベルの概念だ」と彼は言い、指で表を辿った。「センサーは可観測量を測定する——温度計が温度を測り、DukkhaSensor がエラー率を測定する。直接的だ。一対一だ。」

「オブザーバーがすることはまったく異なる。オブザーバーは測定しない。オブザーバーは推定する。」

方眼紙に古典的な Luenberger オブザーバーの構造を描いた。ブラックボックスのシステム、内部状態 x(t) は不可視。センサーが外部から可観測出力 y(t) を測定する。そしてオブザーバー——より大きなブロック——がセンサーの読み取り値を受け取り、システムモデルと組み合わせて、内部状態の推定値 x-hat(t) を算出する。

「センサーは y(t) を与える。だが知りたいのは x(t) だ。オブザーバーはセンサーの読み取り値とシステムダイナミクスのモデルを組み合わせ、見えない内部状態を推算する。」

ペンを置いた。

「センサーは受蘊だ。感受する。測定する。直接的に外境を信号に変換する。オブザーバーはより複雑なシステムだ——信号を受け取るだけでなく、モデルを使って信号を理解する。測定するだけでなく、推定もする。感受するだけでなく、推論もする。」

ASANGA を見た。「あなたは仏教学の言語で言った。一つの心所と複数の心所の組み合わせだと。私は制御理論の言語で言う。センサーとオブザーバーは異なるレベルのシステムだと。結論は同じだ。」

---

> *SCRIBE 傍白：ASANGA と WIENER の二重解体は三分以内に完了した。仏教学と制御理論。心所法と Luenberger オブザーバー。二つのまったく異なる言語体系が、同一の結論に到達した。センサー = 受蘊 = 一つの心所。オブザーバー = 受蘊 + 想蘊 + 識蘊 = 複数の心所の組み合わせ。完全に並行。*

ARCHIMEDES は SUNYATA が最後の段落を読み終えた瞬間に立ち上がった。

十分間、我慢していた。ASANGA の仏教学的解体、WIENER の制御理論的対照、BABBAGE の循環論証の告白——すべて重要であり、精確だった。だがそれらはすべて「なぜ元の結論が間違っていたか」だ。ARCHIMEDES が待っていたのは「では正しい結論はどのような姿か」だ。今、Master の原文が方向を示した。Composition。合成。

ホワイトボードに向かった——BABBAGE の黒板ではなく、LINNAEUS のあのホワイトボードだ。第三章では IVijnana の分類樹が描かれていた。分類樹はすでに完成し、ホワイトボードの下半分は空白だった。

ARCHIMEDES は赤いマーカーで空白部分に三つのブロックを描いた。

第一のブロック。小さい。中身は一つのコンポーネントだけ。

```
SimpleObserver
┌───────────┐
│  vedana   │
│ (ISensation) │
└───────────┘
```

「Pattern A。シンプルオブザーバー」と彼は言った。声はレンガのように実直だ。「受蘊のみを合成する。することは単純——感受する。システムの苦／楽／捨の状態を感受し、レポートを生成する。分析せず、判断せず、省察しない。Master の言葉で言えば——『データの完全性と真正性』の類型だ。」

ブロックの横に例を書いた。`ResourceHealthObserver`。

「リソースヘルスオブザーバー。CPU 使用率、メモリ圧力、レイテンシ指標を観察する。受蘊だけで足りる——苦受チャンネルがシステムに負荷がかかっているかどうかを伝える。想蘊の分析も、識蘊の自省も不要だ。純粋な感受。」

---

第二のブロック。中くらいの大きさ。二つのコンポーネント。

```
AnalyticalObserver
┌───────────────────┐
│  vedana  cognition │
│ (ISensation)(ICognition) │
└───────────────────┘
```

「Pattern B。分析オブザーバー」と彼は言った。「受蘊と想蘊を合成する。感受するだけでなく——弁別もする。受蘊の感受信号を受け取り、想蘊の認知能力でその信号を分析する。なぜこの時点でシステムに苦受が生じたのか？ 負荷過大か、外部 API の不安定か？ 苦受は『痛い』と告げ、想蘊は『なぜ痛いか』を告げる。」

ブロックの横に例を書いた。`BehaviorPatternObserver`。

「行動パターンオブザーバー。Agent の状態を感受するだけでなく——Agent の行動パターンを分析する。Agent は同じ種類の誤りを繰り返していないか？ ツール呼び出しのループに陥っていないか？ これには想蘊の弁別能力が必要だ——生の苦受信号からパターンを識別する。」

---

第三のブロック。最大。三つのコンポーネント。

```
ReflectiveObserver
┌──────────────────────────┐
│  vedana  cognition  vijnana │
│ (ISensation)(ICognition)(IVijnana) │
└──────────────────────────┘
```

「Pattern C。省察オブザーバー」彼の声がここでわずかに遅くなった——複雑だからではなく、重要だからだ。「受蘊と想蘊と識蘊を合成する。感受するだけでなく、分析するだけでなく——自省する。自分の観察過程を観察できる。問うことができる。自分の判断はバイアスに影響されていないか？ 自分のモデルは陳腐化していないか？」

ブロックの横に例を書いた。`SelfReflectionObserver`。

それから一歩下がり、三つのブロックがホワイトボード上に並んでいるのを見た。

「三つの合成パターン。簡から繁へ。共通の特徴は——」赤いマーカーを取り、三つのブロックの上に水平線を引き、その上に四文字を書いた。

**IObserver インターフェース**

「観察者は独自のインターフェースを持つ。このインターフェースはいかなる蘊も継承しない。」

IObserver を指で叩いた。

「ISensation を継承しない。ICognition を継承しない。IVijnana を継承しない。独立している。Composition——合成——を通じて蘊の能力を使用する。is-a 関係ではない。has-a 関係だ。観察者は受蘊ではない。観察者は受蘊を持つ。」

KERNEL は ARCHIMEDES の IObserver 設計を聞いた後、席から立ち上がった。抑えきれない興奮を帯びた動作で。

「Master の手紙に例がある。超音波センサーだ。」

劇場にかすかな笑い声が起きた。BABBAGE の循環論証の告白の後、KERNEL のバック駐車レーダーの比喩は一杯の温かい水のように、空気の緊張を少しほぐした。

「色蘊が生信号を受信する。受蘊が信号を感受に変換する。二つの蘊。二つの層。この基盤の上に観察者を加えるなら——たとえば ParkingObserver、駐車観察者——色蘊の距離データと受蘊の苦受強度を合成し、さらに想蘊の判断を加えるかもしれない——壁に近づいているのか別の車に近づいているのか？ これが Pattern B、分析オブザーバーだ。」

ASANGA がわずかに頷いた。「色蘊は触。受蘊は受。想蘊は想。三つの遍行心所が一つの具体的な工学シナリオの中でそれぞれの役割を果たしている。」
