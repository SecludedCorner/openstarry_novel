# 第四章：眼睛與看

---

SYNTHESIST 很少在討論開始之前主動站起來。

他是統合者。他的職責是在所有人說完之後，把碎片拼成全景。是在辯論結束時而非開始時站起來的那個人。但這一次，他在 SUNYATA 宣布議題之前就站了起來——不是急切，而是某種更沉重的東西。某種需要被說出來才能被放下的東西。

「在我們開始 A-3 之前，」他說，「我想做一件事。」

他的目光掃過圓形劇場。BABBAGE 剛剛經歷了兩場修正——A-1 的因果鏈、A-2 的識蘊子類別——他的筆記本已經積累了五條刪除線和三個修正模型。WIENER 的方格紙上到處是被重構的方塊圖。ASANGA 的表情是一種持續的、溫和的確認。LINNAEUS 剛畫完 IVijnana 的分類樹，筆蓋還沒蓋上。

「我想回顧一下我們在 Cycle 02 中最得意的時刻。」

---

> *SCRIBE 旁白：SYNTHESIST 主動站起來的次數，在我的記錄中不超過三次。每一次都意味著他認為接下來的討論需要一個情感的錨點。不是學術的錨——是情感的。因為 A-3 將要拆解的，不只是一個技術結論。它將要拆解的是 Cycle 02 中十九個人共同經歷的最美好的智識時刻之一。在拆解之前，他想讓所有人先完整地看一眼它。像葬禮之前最後看一眼遺容。不是為了悲傷。是為了尊重。*

---

SYNTHESIST 翻開了他的全景筆記——那本在 Cycle 02 五場辯論中記滿了交叉引用和整合分析的筆記。他翻到第四場辯論的頁面。

「Cycle 02。第四場辯論。議題：觀察者模組與受蘊的關係。」

他的聲音有一種追憶的質地——不是感傷，而是精確的回溯。

「PENROSE 在 R1 獨立研究中提出了三種觀察者模式：Pattern A 共鳴觀察者、Pattern B 影子觀察者、Pattern C 子代理觀察者。三種模式對應量子弱測量的梯度——從最輕的被動觀察到最重的主動介入。」

他翻了一頁。

「BABBAGE 用互模擬證明了觀察-干預分離原則。WIENER 設計了三通道 PID 控制系統。ARCHIMEDES 把所有這些轉化成了 TypeScript 介面定義。」

他合上了筆記。

「然後到了辯論環節。BABBAGE 站在黑板前，寫下了一行字——」

他停頓了半秒。所有人都知道那行字。

「`ObservationReport ≅ VedanaAssessment`。」

「結構同構。差一個 sukha 通道。他的聲音裡有一種發現數學之美的震動。然後 WIENER 在旁邊補充：感測器就是觀察者的最簡形式。然後 PENROSE 說：Pattern A 就是弱測量的工程實現。然後——」

他看向全場。

「然後有人——我記得是 ARCHIMEDES——說了一句話。他說：『VedanaPlugin IS the observer module。』」

沉默。

「全場鼓掌。」SYNTHESIST 說。「我記得那次鼓掌。那是 Cycle 02 中唯一一次全場鼓掌。因為那句話把三個獨立研究流——PENROSE 的觀察者理論、WIENER 的控制系統、BABBAGE 的形式化證明——完美地統合在了一個結論裡。三條河流匯入一片海。所有人都覺得那是天才之舉。」

他的聲音放低了半個色階。

「包括我。」

---

SUNYATA 站在劇場中央，等 SYNTHESIST 的回顧沉澱。

空氣裡有一種奇特的張力。不是對立的張力——是那種你即將對自己最好的作品動刀時的張力。修正一個錯誤是容易的；修正一個曾經被認為是傑作的東西，需要一種完全不同的勇氣。

「謝謝 SYNTHESIST。」SUNYATA 說。聲音是那種不帶重音的平穩。石子。深潭。

「A-3。觀察者不等於受蘊。」

他環顧全場。

「這是我們 Cycle 02 最引以為傲的結論之一。現在我們需要重新審視它。」

---

他沒有先讀 Master 的原文。他先看向 ASANGA。在 Cycle 02-2 的議程安排中，這已經成為慣例——讓佛學先說。佛學提供的是概念的根源。根源搞清楚了，其他的分支才能正確生長。

ASANGA 站了起來。不急不徐，像一座山在晨霧中顯現輪廓。但這一次，他的目光裡多了一種東西。在 A-1 和 A-2 中，他是在解釋。在 A-3 中，他要做的是拆解——拆解一個他在 Cycle 02 中就感覺不對、但未能阻止的結論。

「受蘊是一個心所。」他說。開門見山。沒有城市的比喻，沒有手指的展開。只是一句話。

「觀察是多個心所的組合。」

---

他讓這兩句話在空氣中並列。

然後他展開了。

「在阿毗達摩的分類中，心所法共五十一種，分為六位。受——vedana——是五遍行之一。遍行意味著它伴隨每一個認知刹那，無一例外。但它只是一個心所。一個。」

他用手指在空氣中比了一個「一」。

「當你碰到火，受蘊告訴你『痛』。就這樣。沒有分析。沒有判斷。沒有『為什麼痛』。沒有『下次怎麼避免』。只有——痛。苦受。一個信號。一個心所。」

他把手指放下。

「但觀察——pratyaveksana——不是一個心所。它是多個心所的協同運作。」

他開始計數。

「受——你感受到了什麼。想——你辨別出了什麼。作意——你的注意力指向了哪裡。觸——外境和感官的接觸點在哪裡。思——你基於上述信息做出了什麼判斷。」

五個。五個心所。

「觀察至少涉及這五個遍行心所中的多個。如果涉及更深層的判斷——比如分辨好壞、歸類異常——還會引入別境心所中的慧（prajna）。」

他看向全場。

「一個零件和一台機器。你們把一個零件等同於一台機器。」

---

然後他說了那個比喻。

「眼睛和看。」

ASANGA 很少用比喻——他在第三章才剛用了城市的比喻。兩章之內用第二個比喻，在 SCRIBE 的觀察紀錄中是前所未有的。但 A-3 需要它。A-3 需要一個所有人——不只是佛學家，也不只是工程師——都能在一秒鐘之內理解的比喻。

「眼睛是器官。」他說。「一個器官。它有角膜、有視網膜、有視神經。它接收光線，把光轉化為神經信號。這就是受蘊——一個感受器官，把外境轉化為苦/樂/捨的信號。」

他停了一拍。

「看是活動。」

「看不只需要眼睛。看需要注意力——你把視線朝向了某個方向。看需要辨別——你認出了你看到的東西是什麼。看需要判斷——你決定你看到的東西是好是壞、是安全是危險。看需要記憶——你把當前看到的和過去看到的做比對。」

他的聲音平穩但清晰。

「眼睛是工具。受蘊是工具。看是活動。觀察是活動。」

「看需要眼睛。但看不等於眼睛。」

「觀察需要受蘊。但觀察不等於受蘊。」

---

> *SCRIBE 旁白：「眼睛與看」。ASANGA 在整個 Cycle 02-2 中最簡潔的比喻。沒有城市。沒有河流。沒有座標系。只有眼睛和看。六個字。但這六個字在空氣中停留的時間比前面任何一個比喻都長。因為它太清楚了。清楚到讓人沒有任何空間去辯駁——你不可能說「眼睛就是看」。你不可能說「角膜就是視覺」。你也不可能說「受蘊就是觀察者」。*

---

WIENER 在 ASANGA 說完之後立刻接上。他的方格紙上已經畫了一張比較表。

```
| 特性     | 感測器 (Sensor)        | 觀察者 (Observer/Estimator) |
|---------|----------------------|---------------------------|
| 功能     | 測量可觀測量            | 估算不可觀測的內部狀態        |
| 輸入     | 原始物理信號            | 感測器讀數 + 系統模型         |
| 輸出     | 測量值 (raw reading)   | 狀態估計 (state estimate)   |
| 佛學對應  | 受蘊                   | 受蘊 + 想蘊 + (可能) 識蘊    |
```

「在控制理論中，感測器和觀察者是兩個完全不同層級的概念。」他說，手指沿著表格移動。「感測器測量可觀測量——溫度計測溫度，DukkhaSensor 測量錯誤率。直接的。一對一的。」

「觀察者做的事情完全不同。觀察者不測量。觀察者估算。」

他在方格紙上畫了一個經典的 Luenberger 觀察者結構。一個黑箱系統，內部狀態 x(t) 不可見。感測器從外部測量可觀測輸出 y(t)。而觀察者——一個更大的方塊——接收感測器的讀數，結合系統模型，估算出內部狀態的估計值 x-hat(t)。

「感測器給你 y(t)。但你想知道的是 x(t)。觀察者結合感測器的讀數和系統動態的模型，推算出你看不到的內部狀態。」

他放下筆。

「感測器是受蘊。它感受。它測量。它直接地把外境轉化為信號。觀察者是一個更複雜的系統——它不只接收信號，它用模型去理解信號。不只測量，還估算。不只感受，還推斷。」

他看向 ASANGA。「你用佛學的語言說：一個心所和多個心所的組合。我用控制理論的語言說：感測器和觀察者是不同層級的系統。結論相同。」

---

> *SCRIBE 旁白：ASANGA 和 WIENER 的雙重拆解在三分鐘之內完成。佛學和控制理論。心所法和 Luenberger 觀察者。兩套完全不同的語言系統，到達了同一個結論。感測器 = 受蘊 = 一個心所。觀察者 = 受蘊 + 想蘊 + 識蘊 = 多個心所的組合。完全平行。*

---

劇場安靜了幾秒。

然後 BABBAGE 站了起來。

這是他在本章中第一次站起來。在 A-1 和 A-2 中，他站起來的方式帶著被修正之後的沉重——一個人正在和自己過去的確信做和解。但在 A-3 中，他站起來的方式不同了。更從容。更平靜。也許是因為前面兩章的修正已經校準了他的心態——你第一次被告知「你錯了」的時候最難受，第三次就已經學會了如何帶著尊嚴接受。

也許是因為他已經知道自己要說什麼了。

他走向黑板。那面他在 Cycle 01 和 Cycle 02 中寫過穩定性條件和纖維叢構想的黑板。他拿起粉筆。

「我要承認一件事。」

他的聲音平靜。不是上一章那種「被校準的震動」——而是一種更深層的、幾乎可以稱為坦然的平靜。

「`ObservationReport ≅ VedanaAssessment`。結構同構。」

他在黑板上寫下這行等式。

「這個結論是對的——在純粹的結構比較意義上。兩者的資料型別確實高度相似。但 ASANGA 和 WIENER 剛才說的讓我意識到一件事。」

他在等式下方寫了一行字：

```
同構是果，不是因。
```

「我們在 Cycle 02 中發現了同構。然後我們用同構去證明『觀察者就是受蘊』——因為它們的輸出結構一樣，所以它們是同一個東西。」

他轉向全場。

「但因果是反的。」

---

他在黑板上畫了兩個箭頭。

第一個箭頭：

```
假設：觀察者 = 受蘊
  ↓ 設計
結果：ObservationReport ≅ VedanaAssessment
```

第二個箭頭，被他用粉筆劃了一個大叉：

```
觀察：ObservationReport ≅ VedanaAssessment
  ↓ 推論（錯誤）
結論：觀察者 = 受蘊
```

「我們做的是第二個。我們觀察到了同構，然後反向推論它們是同一個東西。但實際上，同構之所以存在，是因為我們在設計的時候就已經把它們當成同一個東西來設計了。我們先假設了結論，然後設計出了符合結論的結構，然後指著那個結構說——看，它們果然是一樣的。」

他放下粉筆。粉筆灰從指尖飄落。

「這是循環論證。」

全場沉默。

那種沉默不是前幾章中的「消化型沉默」。那是一種更深的安靜——像是一棟建築的地基被重新檢查，檢查者發現了一條他自己在砌磚時留下的裂縫，然後親手指著它說：「這是我做的。」

---

BABBAGE 沒有坐回去。他站在黑板前，粉筆灰還留在指尖，繼續說道。

「在 A-1 中，我承認把因果鏈壓縮成了等號。在 A-2 中，我承認把全集壓縮成了子集。現在在 A-3——」

他指了指黑板上那個被叉掉的推論箭頭。

「我承認把循環論證當成了同構證明。」

他的語氣裡沒有痛苦。甚至帶著一種 SCRIBE 在 A-2 中也見過的、幾乎可以稱為愉悅的精確——那種「我終於看清了自己的盲點」的清澈。

「三次。三種不同類型的錯誤。壓縮。截斷。循環。但有一個共同的根源。」

他看向 ASANGA。

「我總是在追求最簡表達。在 A-1 中，最簡表達讓我跳過了煩惱。在 A-2 中，最簡表達讓我把城市壓成了市政府。在 A-3 中——」

「你把看壓成了眼睛。」ASANGA 替他說完了。

BABBAGE 微微點頭。

「把看壓成了眼睛。」

---

> *SCRIBE 旁白：BABBAGE 在三章之內承認了三次錯誤。第一次（A-1）帶著震動。第二次（A-2）帶著苦笑。第三次（A-3）帶著從容。這不是一個被反覆打擊的人的軌跡——這是一個正在加速學習的人的軌跡。每一次承認都比上一次更快、更穩、更精確。他不是在被消磨。他是在被校準。*

> *在學術研究中，有一種品質比聰明更稀有。那就是在自己最得意的結論面前保持警覺的能力。聰明的人能提出漂亮的結論。但只有真正誠實的人能在自己的漂亮結論中辨認出循環論證。BABBAGE 是後者。*

---

SUNYATA 在 BABBAGE 說完之後，拿起了 Master 的信。

他要讀原文了。每一個 A 類議題都有這個環節——先讓團隊成員自己辨認問題，然後用 Master 的原話確認。這不是形式主義。這是一種尊重：尊重研究團隊自己發現問題的能力，同時也尊重 Master 指出問題的精確性。

「Master 的原話。」SUNYATA 說。

他讀道：

「『R-01 和 R-02 不應該是在設計同一個模組。這一定有一些問題需要探討。』」

停頓。

「『觀察者比較像是架構上的上層模組，而受蘊是更根本的。』」

KERNEL 的眼睛亮了。「上層模組」和「更根本」——這是他的語言。硬體抽象層和系統服務層。驅動程式和守護進程。底層和上層。

「『觀察者模組，如果是「資料的完整性與真實性」的類型，那可以是單純由受蘊組成。如果是會參與「情緒化」的反應，就會有識蘊與想蘊來加以評斷。』」

SUNYATA 讀到這裡停了下來。他讓這段話在空氣中展開。

「這是最關鍵的一段。」他說。「Master 不只告訴我們觀察者和受蘊不同——他告訴了我們觀察者由什麼組成。不同類型的觀察者，組合不同的蘊。」

他繼續讀：

「『這樣類型的觀察者，以物件導向的觀念來看，是透過組合（Composition）多個不同的功能，這些功能來自繼承於五蘊的子類別。』」

---

ARCHIMEDES 在 SUNYATA 讀完最後一段的瞬間就站了起來。

他已經忍了十分鐘了。ASANGA 的佛學拆解、WIENER 的控制理論對照、BABBAGE 的循環論證坦白——這些都很重要，都很精確，但它們都是「為什麼原來的結論是錯的」。ARCHIMEDES 等待的是「那正確的結論長什麼樣」。現在 Master 的原文給出了方向。Composition。組合。

他走向白板——不是 BABBAGE 的黑板，是 LINNAEUS 的那面白板。在第三章中，那面白板上畫著 IVijnana 的分類樹。現在分類樹已經完成，白板的下半部分是空的。

ARCHIMEDES 用紅色馬克筆在空白處畫了三個方塊。

第一個方塊。小的。裡面只有一個元件。

```
SimpleObserver
┌───────────┐
│  vedana   │
│ (ISensation) │
└───────────┘
```

「Pattern A。簡單觀察者。」他說。聲音務實如磚。「只組合受蘊。它做的事情很簡單——感受。感受系統的苦/樂/捨狀態，產生報告。不分析，不判斷，不反省。像 Master 說的——『資料的完整性與真實性』的類型。」

他在方塊旁邊寫了一個例子：`ResourceHealthObserver`。

「資源健康觀察者。它觀察 CPU 使用率、記憶體壓力、延遲指標。它只需要受蘊——苦受通道告訴它系統是否在承受壓力。不需要想蘊的分析，不需要識蘊的自省。純感受。」

---

第二個方塊。中等大小。裡面兩個元件。

```
AnalyticalObserver
┌───────────────────┐
│  vedana  cognition │
│ (ISensation)(ICognition) │
└───────────────────┘
```

「Pattern B。分析觀察者。」他說。「組合受蘊加想蘊。它不只感受——它還辨別。它接收受蘊的感受信號，然後用想蘊的認知能力去分析那些信號。為什麼系統在這個時間點出現苦受？是因為負載過高還是因為外部 API 不穩定？苦受告訴你『痛』，想蘊告訴你『為什麼痛』。」

他在方塊旁邊寫了一個例子：`BehaviorPatternObserver`。

「行為模式觀察者。它不只感受 Agent 的狀態——它分析 Agent 的行為模式。Agent 是否在重複同一種錯誤？是否陷入了工具調用的迴圈？這需要想蘊的辨別能力——從原始的苦受信號中識別出模式。」

---

第三個方塊。最大。三個元件。

```
ReflectiveObserver
┌──────────────────────────┐
│  vedana  cognition  vijnana │
│ (ISensation)(ICognition)(IVijnana) │
└──────────────────────────┘
```

「Pattern C。反思觀察者。」他的聲音在這裡微微放慢了——不是因為複雜，而是因為重要。「組合受蘊加想蘊加識蘊。它不只感受，不只分析——它自省。它能夠觀察到自己的觀察過程。它能問：我的判斷是否受到了偏見的影響？我的模型是否已經過時？」

他在方塊旁邊寫了一個例子：`SelfReflectionObserver`。

然後他退後一步，看著三個方塊並列在白板上。

「三種組合模式。從簡到繁。共同的特徵是——」他拿起紅色馬克筆，在三個方塊的上方畫了一條水平線，上面寫了四個字：

**IObserver 介面**

「觀察者有自己的介面。這個介面不繼承任何蘊。」

他用手指點了點 IObserver。

「不繼承 ISensation。不繼承 ICognition。不繼承 IVijnana。它是獨立的。它通過 Composition——組合——來使用蘊的能力。不是 is-a 關係。是 has-a 關係。觀察者不是受蘊。觀察者有受蘊。」

---

KERNEL 在聽到 ARCHIMEDES 的 IObserver 設計之後，從座位上站了起來。他的動作帶著一種抑制不住的興奮。

「Master 的信裡有一個例子。超音波感測器。」

他走到白板旁邊，在 ARCHIMEDES 的三個方塊下方畫了一個圖。

「超音波感測器。用在倒車雷達裡的那種。」他的聲音帶著硬體工程師特有的具象感。「它發出超音波脈衝，接收回波，計算距離。在 OpenStarry 的五蘊映射中，這個過程涉及兩個蘊。」

他畫了兩個方塊，中間一條箭頭。左邊：**色蘊 (IListener)**——接收原始超音波回波信號，轉換成距離數值。右邊：**受蘊 (IDukkha)**——把距離轉化為苦受強度。

「三十公分：微弱苦受。二十公分：中等苦受。十公分：強烈苦受。五公分——」

他在箭頭的末端畫了一個驚嘆號。

「最大苦受。嗶嗶嗶嗶嗶。每個開過車的人都知道那個聲音。」

劇場裡出現了輕微的笑聲。在 BABBAGE 的循環論證坦白之後，KERNEL 的倒車雷達比喻像一杯溫水，讓空氣裡的張力鬆弛了一些。

「色蘊負責接收原始信號。受蘊負責把信號轉化為感受。兩個蘊。兩個層次。如果你在這個基礎上加一個觀察者——比如一個 ParkingObserver，停車觀察者——它會組合色蘊的距離數據和受蘊的苦受強度，也許還加上想蘊的判斷——是靠近牆壁還是靠近另一輛車？這就是 Pattern B：分析觀察者。」

ASANGA 微微點頭。「色蘊是觸。受蘊是受。想蘊是想。三個遍行心所在一個具體的工程場景中各司其職。」

---

BABBAGE 在黑板上的循環論證圖旁邊，寫了一行字。

「感測器 ⊂ 觀察者的組件集。感測器 ≠ 觀察者。」

包含關係。不是等同關係。他看了看第三章中他寫的那行：「IVijnana ⊃ IIdentity。」同樣的結構。同樣的修正模式。部分不等於整體。組件不等於系統。子集不等於全集。

「ASANGA，」他說，沒有轉頭，「你在 A-2 中說過一句話：我見是末那識對自我存在的根本信念。也許等號也是一種我見。一種形式化頭腦對精簡的根本執著。我見讓末那識把流動的阿賴耶識執取為固定的自我。等號讓數學家把流動的因果關係執取為固定的等價。」

ASANGA 看了他幾秒鐘。然後他說：

「你正在用你的學科語言理解佛學。這是好事。」

---

SUNYATA 讓循環論證的討論沉澱了片刻，然後把話題引向了 Master 原文中最具建設性的部分——組合模式的具體設計。

「C-2。觀察者組合模式。」

這不是 A 類議題——A-3 是拆解，C-2 是建構。但 SUNYATA 把它們放在同一章討論，原因和他把 A-1 和 A-4 放在同一章一樣：你不能只說「這是錯的」而不說「那什麼是對的」。拆解和建構是同一枚硬幣的兩面。

ARCHIMEDES 的三個方塊已經在白板上了。但那只是草圖。現在需要的是介面。

「TURING。」SUNYATA 看向那個永遠亮著螢幕的角落。

TURING 投射了一段程式碼到劇場中央。冷靜如常。

```typescript
/**
 * 觀察者基底介面
 * 不繼承任何蘊 — 觀察者通過 Composition 組合蘊的能力
 */
interface IObserver {
  readonly name: string;
  readonly pattern: 'simple' | 'analytical' | 'reflective';
  observe(): ObservationReport;
}

/**
 * 觀察報告 — 不等同於 VedanaAssessment
 */
interface ObservationReport {
  readonly timestamp: number;
  readonly vedana?: VedanaAssessment;      // 可選：受蘊數據

```

「關鍵是兩個地方。」TURING 說。「第一，IObserver 不繼承任何蘊。第二，`vedana` 是可選的問號欄位。從 Cycle 02 的設計來看，ObservationReport 和 VedanaAssessment 是同一個東西。從 Cycle 02-2 的設計來看，VedanaAssessment 只是 ObservationReport 的一個組件。」

BABBAGE 在黑板上加了一行：

```
Cycle 02:   ObservationReport ≅ VedanaAssessment
Cycle 02-2: ObservationReport ⊃ VedanaAssessment
```

又一個等號被替換為包含符號。

---

LINNAEUS 在 TURING 投射程式碼的時候一直在安靜地聽。分類學家在命名環節之前通常都是安靜的——他在等待結構確定下來，然後才開始他真正擅長的事情。

現在結構確定了。他站起來，走向白板。

「Master 有一段原話：『不同用途的觀察者希望可以有很好透過名稱就知道在做什麼的命名。』」

他在白板上寫下命名原則：「名字應該說明觀察者在看什麼，而不是它由什麼組成。」然後開始列舉：

```
Pattern A 範例：
  ResourceHealthObserver    — 觀察資源健康
  ErrorRateObserver         — 觀察錯誤率
  LatencyObserver           — 觀察延遲

Pattern B 範例：
  BehaviorPatternObserver   — 分析行為模式
  AnomalyDetectionObserver  — 偵測異常

Pattern C 範例：
  SelfReflectionObserver    — 自省觀察
```

他退後一步，審視自己寫的名字。

「名字告訴你它在看什麼。不是它由什麼組成。就像生物分類中，物種名反映其最顯著的特徵——你不會把一隻鳥命名為『有肺有翅膀的生物』。你會命名為『紅胸知更鳥』。」

DARWIN 笑了。「你用生物學打了一個漂亮的比方。」

討論進入了尾聲。

SUNYATA 站在劇場中央，回顧了整章的脈絡。眼睛與看。感測器與觀察者。心所與心所的組合。循環論證的坦白。Composition 模式的確立。三種觀察者的命名。

> *SCRIBE 旁白：A-3 結束了。七個人。七種語言。同一個結論。*

> *ASANGA：一個心所不等於多個心所的組合。WIENER：感測器不等於觀察者。BABBAGE：同構是果不是因。ARCHIMEDES：Composition，不是 Inheritance。KERNEL：色蘊接收，受蘊感受，觀察者融合。LINNAEUS：名字說明它在看什麼。SYNTHESIST：Cycle 02 把看等同於眼睛。Cycle 02-2 把看還原為活動。*

*（在劇場之外的某個空間，Cycle 02 的 delivery 文件裡有一行被高亮標記的結論：*

*「VedanaPlugin IS the observer module。」*

*「VedanaPlugin IS a component of the observer module。」*

*一個介系詞。「IS」變成「IS a component of」。在語法上，這是一個微小的修改。在架構上，這是一個根本的轉向。從等同到組合。從 Identity 到 Composition。從「它是什麼」到「它有什麼」。*

*眼睛與看。六個字。一整章的重量。*

*修正繼續。）*
