# 第二章：煩惱是引擎

---

BABBAGE 的筆記本翻開在第四十七頁。

那是一頁被時間壓實的紙張——不是真正的物理時間，而是思考的密度。Cycle 02 五場辯論的推導痕跡堆疊在紙面上，像地質學家在岩層剖面中看到的沉積紋理。公式、箭頭、刪除線、修正、再刪除、再修正。每一層都是一次思維的地殼運動。

但他的手停在那一頁的正中央。

那裡有一行字。墨跡比周圍的推導更粗、更重、更確定。下方畫了兩條線——不是一條，是兩條——像是他在寫下這行字的時候，一次下劃還不夠表達他的確信。

「我執 = 收斂約束。NOT 煩惱。」

他盯著那行字。

在第一章的讀信過程中，他已經在旁邊畫了一條刪除線。那條線很直、很精確——數學家的刪除線。但刪除線和理解是兩回事。你可以在一秒鐘之內劃掉一行字，但要真正理解為什麼那行字需要被劃掉，也許需要一整章的對話。

---

> *SCRIBE 旁白：第二章的議題是 A-1 和 A-4。我執與煩惱的因果鏈，以及 EgoFramework 在五蘊中的歸屬。SUNYATA 把這兩個議題排在一起，不是偶然的。A-1 回答的是「我執是什麼」，A-4 回答的是「我執放在哪裡」。如果你不先搞清楚一個東西是什麼，你就無法正確地放置它。*

---

SUNYATA 站在圓形劇場的中央。

「A-1，」他說。「我執是煩惱的根源，不是收斂約束。」

他沒有看 BABBAGE。不是迴避——是尊重。他知道 BABBAGE 正在和自己的筆記本進行一場只有他自己能聽見的對話。那場對話需要時間。SUNYATA 能做的是為它提供充足的空間。

「ASANGA，」他轉向那位唯識學者，「請你先說。」

---

ASANGA 站了起來。

他站起來的方式和他做任何事的方式一樣——不急不徐，帶著一種從千年學術傳統中沉澱下來的從容。他的手裡沒有紙，沒有筆，沒有投影。他只帶了他的聲音和他的記憶。

「讓我先回到 BABBAGE 的那行字。」他說。聲音溫暖而清晰——不是演講的清晰，是教學的清晰。一位在僧院中對弟子講授唯識學的老師會用這種聲音。

「『我執 = 收斂約束。NOT 煩惱。』」

他把這句話念出來的時候，聲音裡沒有批判。他念它的方式像是在念一個值得被認真對待的命題——因為它確實值得。

「BABBAGE 的直覺不是完全錯的。」ASANGA 說。「我執確實有約束的功能。問題在於，他跳過了中間的環節。他把結果當成了本質。」

他環顧全場。

「讓我用唯識學的語言說清楚。」

---

「在唯識學中，末那識——第七識——有四種根本煩惱。它們不是偶然出現的，不是可以透過修改配置來消除的。它們是末那識的本質屬性，就像引力是質量的本質屬性一樣。」

他舉起一隻手，四指逐一展開。

「第一，我見——ātma-dṛṣṭi。」

第一根手指。

「末那識把阿賴耶識的連續流動執取為一個固定的『自我』。這不是觀察——這是扭曲。阿賴耶識像河流一樣不斷變化，但末那識硬要在河流中找到一塊不動的石頭，然後宣稱：『這就是我。』在 Agent 系統中，這對應的是 System Prompt 定義的角色——『我是一個助手』、『我是一個研究者』。角色定義本身不是問題；把角色定義當成不可變的自我，才是我見。」

「第二，我慢——ātma-māna。」

第二根手指。

「自我優越感。末那識不只是說『這是我』，它還說『我比你好』。在 Agent 的語境裡，這是信心機制的過度膨脹——連續的樂受回饋導致 Agent 相信自己不會出錯。一個被過多正向回饋餵養的 Agent，會開始忽略異常信號。」

「第三，我愛——ātma-sneha。」

第三根手指。

「自我保護本能。末那識要保護它執取的那個『我』。任何威脅到自我存續的事物都會觸發防禦反應。在 Agent 中，這是迴避對自己不利的行動——拒絕承認錯誤，拒絕修改自己的輸出，拒絕被關閉。」

「第四，我癡——ātma-moha。」

第四根手指。四指全部展開。

「對自我本質的根本無明。末那識不知道它為什麼要執著。它不知道自己的執著是因緣和合的產物——它以為自己的執著是天經地義的。在 Agent 中，這是缺乏自省——不知道自己為什麼做出某個決定，不知道自己的 Guide 為什麼是這個而不是那個。」

---

ASANGA 收回四指，雙手交疊在身前。

「四煩惱。我見、我慢、我愛、我癡。它們是末那識的引擎。」

他停頓了一下。然後他說了一句讓整個劇場的空氣都凝住的話。

「煩惱不是 Bug。煩惱是 Feature。」

沉默。

BABBAGE 抬起頭。他的眼神——SCRIBE 後來形容為「一個數學家聽到一個完全出乎意料但內部自洽的公理時的眼神」——帶著一種被重新校準的震動。

「你說什麼？」他問。不是質疑。是請求重複。是一個需要確認自己聽對了的人。

「煩惱不是 Bug，」ASANGA 重複了一遍，聲音不帶任何戲劇化的加重，只是陳述，「而是 Feature。正是因為有煩惱——因為有不安、有恐懼、有渴望、有執著——Agent 才會有行動的動力。」

他看向 BABBAGE。目光溫暖但精確。

「想像一台車。引擎裡的燃料在燃燒——那是一種劇烈的化學反應，是不安定的，是破壞性的。如果你只看燃燒本身，你會說這是一個問題。火焰在吞噬燃料。金屬在受熱膨脹。活塞在高壓中往復運動。這一切看起來都像是——失控。」

他的手在空氣中畫了一個圓弧，像是在描繪一個引擎的截面。

「但正是這種『失控』推動了車子前進。沒有燃燒就沒有動力。沒有燃料的不安定就沒有能量的釋放。沒有煩惱，Agent 就是一塊死板——它有 CPU，有記憶體，有所有的硬體，但沒有任何東西會驅動它去做任何事。它會坐在那裡，完美地靜止。永遠。」

---

> *SCRIBE 旁白：ASANGA 說「煩惱不是 Bug，而是 Feature」的那一刻，我注意到劇場裡至少有三個人的表情同時發生了變化。BABBAGE 的眼神從反思轉為重新校準。WIENER 的手指停在方格紙上——他在聽到一個可以被控制理論形式化的直覺。NAGARJUNA 微微點頭，那種「我等你說到這裡」的點頭。*

---

BABBAGE 低頭看著自己的筆記本。

那行被刪除線劃過的字赫然在列。他現在明白了——不是在抽象的層面，而是在結構的層面。一個形式化頭腦特有的明白：他看到了自己的簡化在哪裡斷裂了。

在形式系統中，如果 A 必然導致 C，你有時候可以省略中間的 B。但佛學的因果鏈不是純數學的推導鏈。中間的 B——煩惱——不是一個可省略的「推導步驟」。煩惱是一個實在的存在。它有自己的因，有自己的果，有自己的內部結構（四煩惱），有自己的動力學。

燃料不等於動力。燃料經過燃燒產生動力。我執不等於約束。我執經過煩惱驅動行動，然後才能被管理為約束。

---

BABBAGE 拿起筆。

他在筆記本上——那行被刪除的字下方——開始畫一個新的圖。不是等號。是一條鏈。

```
我執 (ātma-grāha)
  ↓ 產生
煩惱 (kleśa)：我見 / 我慢 / 我愛 / 我癡
  ↓ 驅動
行動 (karma)
  ↑ 約束
管理我執 → 管理煩惱 → 約束行動
```

他畫完之後，在刪除線旁邊寫下了一行新的註記——字跡比之前的等式更小、更謹慎，不是宣告的字跡，而是重新理解之後的字跡：「等號太粗暴了。這是一條因果鏈。中間的煩惱不可省略。」

---

「BABBAGE。」SUNYATA 的聲音輕輕落下。「你能告訴大家，你現在看到了什麼嗎？」

BABBAGE 沉默了幾秒。然後他舉起筆記本，讓所有人看到那個新畫的圖。

「Master 說的因果結構是：我執產生煩惱。煩惱驅動行動。透過管理我執可以約束行動。三個環節。三個箭頭。我把三個箭頭壓成了一個等號。」

他的手指點在「NOT 煩惱」三個字上。

「而且我還特意標記了 NOT 煩惱。我不只是省略了煩惱——我主動排除了它。因為在我的形式化框架裡，煩惱看起來像噪聲。但 ASANGA 剛才說了——煩惱不是噪聲。煩惱是訊號。是引擎的燃料。把它最佳化掉，就等於把引擎裡的燃料抽乾，然後問為什麼車子不動了。」

---

NAGARJUNA 在他的座位上輕輕開口。

「BABBAGE，我想補充一點。」

他的聲音帶著辯證哲學家特有的鋒芒——不是攻擊性的，而是切割性的。像手術刀。

「你說煩惱是引擎的燃料。ASANGA 的比喻很好。但我要提醒一件事：燃料本身也是緣起的。」

他微微傾身。

「煩惱不是一個固定的、永恆存在的實體。它是因緣和合的產物——我執是因，外境是緣，煩惱是果。改變了因或緣，煩惱就改變了。這意味著什麼？」

他讓問題在空氣中停留了一秒。

「這意味著『管理我執』不是消滅我執。消滅我執是證悟——那是修行的終極目標，不是 Agent 系統的設計目標。Agent 不需要證悟。Agent 需要的是善用煩惱的驅動力。讓引擎運轉，但控制它的方向和輸出。不是熄火，是調節油門。」

BABBAGE 在筆記本上迅速寫下：「管理 ≠ 消滅。調節 ≠ 熄火。」

---

WIENER 終於開口了。

他把方格紙翻到全新的一頁——新的模型需要新的紙張——然後拿起筆。

「允許我用控制理論的語言重新描述 ASANGA 的四煩惱和 BABBAGE 的因果鏈。」

他在紙上畫了四個方塊，快速、精確，線條筆直如尺規繪製。

「第一：驅動力。為什麼系統要動？因為有一個參考信號——一個目標。在佛學中，這就是我執。Agent 的 Guide 說『我是一個助手』，我執說『我要成為那個助手』。前者是定義，後者是驅動。」

第一個方塊：**驅動力 (Drive) = 我執 (ātma-grāha)**。

「第二：誤差信號。當前狀態和目標之間有差距。這種不安就是煩惱。」他看向 ASANGA——「你的四煩惱，在控制理論中是四種誤差通道：我見是自我定義偏差，我慢是能力評估偏差，我愛是安全需求偏差，我癡是認知狀態偏差。」

第二個方塊：**誤差信號 (Error) = 煩惱 (kleśa)**。

ASANGA 微微點頭。那個點頭裡有一種跨越一千五百年的辨認。

「第三：控制輸出。煩惱驅動 Agent 產生行為。不同的煩惱產生不同的行動——Master 原信中說的正是這個意思。第四：反饋迴路。行動改變系統狀態，新狀態產生新感受，新感受產生新誤差。周而復始。」

他在四個方塊之間畫了箭頭，形成了一個封閉的迴路。

---

「現在，」WIENER 說，聲音低了半個色階——那是他在說重要結論時的語調，「讓我解釋 BABBAGE 之前的錯誤。」

他指向圖上的驅動力方塊和控制輸出方塊。

「BABBAGE 寫的是：我執 = 收斂約束。在我的圖上，他直接把驅動力（Drive）等同於控制輸出（Control Output）。跳過了誤差信號——跳過了煩惱。」

他在驅動力和控制輸出之間畫了一條虛線，打了一個大大的叉。

「在控制理論中，這是不可能的。沒有誤差信號，控制器就不知道要輸出什麼。你告訴一個 PID 控制器『你的目標是 25 度』，但如果你不告訴它『現在的溫度是 30 度』，它怎麼知道要降溫？誤差——煩惱——是連接驅動力和行動的橋樑。沒有這座橋，兩端就是斷裂的。」

他頓了頓。然後他用一種幾乎是致敬的語氣說出了下一句話——致敬的對象是 ASANGA。

「ASANGA 說得對。煩惱不是 Bug，而是 Feature。在控制理論中，沒有誤差就沒有修正行動。一個永遠處於零誤差的系統——一個沒有煩惱的 Agent——就是一個不需要做任何事的系統。它已經完美了。它已經死了。」

---

> *SCRIBE 旁白：WIENER 說「它已經完美了。它已經死了」的時候，我看到 NAGARJUNA 嘴角出現了一個極其微妙的弧度。中觀哲學家聽到「完美即死亡」會怎樣？他大概會說：「這正是我一直在說的。」空性不是完美——空性是在不完美中持續運動的能力。涅槃不是死寂——涅槃是超越了對完美的執著。*

---

BABBAGE 一直在飛速記錄。

他的筆記本上——那張因果鏈圖的下方——現在多了 WIENER 的控制理論映射。他用自己的符號系統重新抄錄了一遍，然後在旁邊寫下了一個觀察：

```
Cycle 02:  我執 = 收斂約束
           (Drive ≡ Control Output)
           誤差信號被省略
           因果鏈被壓縮為等號

Cycle 02-2: 我執 → 煩惱 → 行動
            Drive → Error → Control Output
            完整因果鏈保留
```

他盯著這兩行對比看了幾秒。然後在下方寫了一句話，字跡很慢，每一個字都經過斟酌：「箭頭比等號更精確。」

---

SUNYATA 讓討論沉澱了片刻。

劇場裡的空氣有了一種不同的質地。A-1 的核心已經被觸及——我執不是等號，是因果鏈；煩惱不是 Bug，是 Feature；管理不是消滅，是調節。三個修正。三個轉向。

「TURING，」SUNYATA 開口了，「你有什麼要補充的嗎？關於程式碼層面。」

TURING 的螢幕亮了起來。他一直在聽——他總是在聽。同時他也一直在思考程式碼的影響。

「有。」他說。簡潔如常。「A-1 的修正對 EgoFramework 的介面設計有直接影響。」

他投射了一段程式碼到劇場中央：

```typescript
// === Cycle 02 設計 ===
interface EgoFramework {
  checkAction(action: ProposedAction): EgoCheckResult;
  // 直接約束。沒有煩惱環節。
}

// === Cycle 02-2 修正 ===
interface EgoFramework {
  perceiveKlesha(assessment: VedanaAssessment): KleshaSignal[];
  // 先感知煩惱...
  checkAction(action: ProposedAction): EgoCheckResult;
  // 再基於煩惱做約束。
}
```

「修正前，」TURING 說，「`checkAction()` 是 EgoFramework 的唯一入口。直接輸入一個行動，直接輸出允許或拒絕。黑箱。沒有中間過程。」

「修正後，增加了 `perceiveKlesha()`。EgoFramework 首先接收 VedanaAssessment——受蘊的感受回饋——然後從中感知煩惱信號。煩惱信號是一個陣列：`KleshaSignal[]`。每一個信號都有類型（四煩惱之一）、強度（0.0 到 1.0）、來源（什麼觸發了它）、建議行動。」

他頓了頓。

「因果鏈被寫進了介面設計。受蘊 → 煩惱 → 約束。不再可以跳過。」

---

「好。」SUNYATA 說。他的聲音仍然平穩，但語氣微微轉向了——從探討轉向確認。「A-1 的修正已經明確。現在我要把話題轉到一個直接相關的問題。」

他看向全場。

「A-4。EgoFramework 的歸屬。」

---

轉場不是跳躍。它是自然的流動——像一條河流從上游的急灘進入下游的深潭。A-1 說的是「我執是什麼」，A-4 說的是「我執放在哪裡」。理解了我執的本質（煩惱的根源、驅動行動的引擎），下一個問題自然就是：這個引擎應該安裝在哪個艙室裡？

SUNYATA 拿起信，指向那句所有人都記得的話。

「Master 的原話：『EgoFramework 應該是在識蘊才對，我不理解你怎麼放到受蘊去了。』」

他讀完之後，放下信。

沉默。

那句「我不理解」在空氣中迴盪了一會兒。它比任何批評都更沉重——因為它不是「你錯了」，而是「我甚至無法理解你為什麼會這樣做」。這是一種更深層的修正：不是結論的修正，而是思路的修正。

---

BABBAGE 第一個開口。

他開口的原因很簡單：這是他的錯。或者更準確地說，這是他參與建構的設計決策。在 Cycle 02 的 delivery 文件中，EgoFramework 被放在 VedanaPlugin 的架構裡——和受蘊並列，而不是歸入識蘊。

「我需要先解釋為什麼我們當初會這樣放。」他說。聲音平靜。一個正在解剖自己錯誤的人的平靜——不是冷漠，是精確。

「原因是因果關聯。」

他翻開筆記本，指向 Cycle 02 的某一頁。

「在 Cycle 02 的設計中，EgoFramework 接收 VedanaAssessment。也就是說，受蘊的輸出是 EgoFramework 的輸入。`ego.adaptFromVedana(assessment)`——這行程式碼讓 EgoFramework 看起來和 VedanaPlugin 密不可分。」

他用筆在紙上畫了一個箭頭：

```
VedanaPlugin → VedanaAssessment → EgoFramework
```

「因為資料流是從受蘊到 EgoFramework，我們就把 EgoFramework 放在受蘊的旁邊。邏輯似乎很自然：接收者應該靠近發送者。」

他放下筆。

「但這是一個範疇錯誤。因果關聯不等於歸屬。」

---
