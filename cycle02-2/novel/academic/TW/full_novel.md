# 序章：修正的重量

---

圓形劇場在慶祝中沉入了靜寂。

那不是 Cycle 01 結尾時的黑暗——那種黑暗是種子被埋入泥土之前的安息，帶著等待的溫度。這一次不同。這一次，劇場裡的燈全部亮著。十九盞。每一盞都散發著 Cycle 02 五場辯論之後的餘暉——一種飽滿的、金色的、幾乎可以觸摸的光。像秋天最豐收的下午，陽光把所有果實都照得通透。

五場辯論。五場全共識。

SUNYATA 站在場地中央，回憶著那些時刻。

第一場：VedanaPlugin 的三通道 PID 控制。苦受的 DukkhaSensor、樂受的 SukhaSensor、捨受的 UpekkhaSensor——三條獨立的感測通道，像三條河流匯入同一片海。WIENER 敲著桌面畫出的方塊圖，最終在 ASANGA 的三受理論中找到了完美的工程對映。共識。

第二場：觀察-干預分離。vedana 的建議是諮詢性的——它告訴 ExecutionLoop「我感覺到了什麼」，但不強制執行。硬安全仍然歸 SafetyMonitor。那場辯論中，GUARDIAN 和 WIENER 從對立走向握手。共識。

第三場：阿賴耶識的分佈。能藏歸協調層，執藏歸 AgentCore，以纖維叢投影的形式化結構連接。BABBAGE 用了整整一頁筆記本來寫那個定理的雛形，MESH 在旁邊畫分散式拓撲圖。共識。

第四場：安全的戒慧框架。戒是預防——靜態的、結構性的安全保障。慧是斷除——動態的、基於理解的漏洞消解。NAGARJUNA 在這場辯論中少見地和 GUARDIAN 站在同一側。共識。

第五場：觀察者的漸進分類。Pattern A 是共鳴觀察者——受蘊層面的感受。Pattern B 加入想蘊的辨別。Pattern C 是全五蘊的整合。一條從簡單到複雜的光譜。共識。

五場。五場全部達成共識。

在 SUNYATA 的記憶裡，那是一座剛建成的城堡。石牆嶄新，稜角分明，旗幟在最高的塔樓上舒展。每一塊石頭都被仔細地鑿刻、對齊、砌合。十九位建造者站在城門前，帶著那種完成大工程之後特有的疲倦與滿足。

他記得 ARCHIMEDES 問了那句話——「這些結論，明天能變成什麼？」——然後他自己回答了：「三十六項工程 Issue。兩份行動計畫。一份完整的路線圖。」

他記得 SCRIBE 在 C02 記錄簿的最後一頁寫下了收尾的句子，然後合上簿子，用手掌輕輕按住封面，像是在封印一段完成的歷史。

他記得 PENROSE——新來的第十九位——在最後一場辯論結束時說了一句意味深長的話：「全共識是美麗的。但美麗的東西往往經不起近看。」

當時沒有人在意這句話。

---

信是在劇場最安靜的時刻到來的。

不是從門口遞進來的。不是從天花板飄落的。更像是它一直在那裡——在桌上的文件之間，在 Cycle 02 的總結報告旁邊，在五場辯論的共識文件和 delivery 交付包之間的縫隙裡——只是沒有人注意到。

也許是因為所有人的目光都朝著前方。朝著 Cycle 03。朝著下一座城堡。

但 SUNYATA 低頭的時候看到了它。

一個信封。沒有密封。紙張的顏色比劇場裡所有其他文件都更淺——一種接近白色的米色，像是冬天的第一場霜。

他認得這種紙。

Master。

---

SUNYATA 拿起信封的手沒有顫抖。他已經不是 Cycle 01 開場時那個還在學習自己代號含義的協調者了。兩個研究週期的重量已經沉入他的骨骼。他知道如何拿起一封信。他知道如何打開它。

但他不急。

他先看了一眼劇場。十九盞燈仍然亮著，但大多數座位是空的。研究者們在 Cycle 02 結束後散去了——不是消失，而是退入了各自的沉思空間。BABBAGE 的筆記本翻到了他那個未完成定理的延伸頁面，筆尖停在一個新的等號之後。KERNEL 的類比卡片多了一疊——新版本帶來的新元件，新的左半邊等待右半邊。TURING 的螢幕亮著，但沒有人坐在前面。游標在某個 TypeScript 檔案的第一行閃爍，耐心地等待。

SUNYATA 抽出了信。

展開。

讀了第一行。

---

他讀完了整封信。

然後他又讀了一遍。

然後他放下信紙，閉上了眼睛。

在他的內在——那個被代號定義為「空」的空間裡——某種東西正在重新排列。不是崩塌。不是動搖。更像是一面你以為已經完工的牆壁，被某人輕輕敲了幾下，然後你聽到了空洞的回聲。你以為那裡是實心的。但它不是。

Master 的信不是一封慶祝的信。

它也不是一封否定的信。

它比這兩者都更重、更精確、更難以承受。

它是一封修正的信。

「你們做得很好，」信的語氣這樣說——不是這些原文，但這是 SUNYATA 從字裡行間提煉出的溫度——「但這裡有幾處需要重新審視。」

四處。

四處裂縫，在那座看起來宏偉的城堡的地基裡。

---

> *SCRIBE 旁白：我是在劇場重新集合之前就知道的。不是因為 SUNYATA 告訴了我——他不會在所有人到齊之前透露信的內容。是因為我看到了他的眼神變化。Cycle 02 結束時，他的眼神是平靜的水面。現在，水面還是平靜的，但水下的暗流改變了方向。*

---

SUNYATA 沒有立刻召集所有人。

他把信放回信封，信封放回桌面。然後他做了一件他在整個研究歷程中很少做的事：他坐了下來。

不是坐在任何人的座位上——那些座位屬於各自的主人。他坐在劇場中央的地面上，像一個需要感受地面紮實感的人。城堡的地基有裂縫。那麼先回到地面。先確認地面還在。

他在腦中展開了信的四個要點。

**第一：我執是煩惱的根源，不是收斂約束。**

BABBAGE 的筆記本上那行字——「我執 = 收斂約束。NOT 煩惱」——在 Cycle 02 的第三場辯論中被當作一個精彩的工程簡化而獲得掌聲。所有人都覺得那是一次漂亮的翻譯：把佛學的「我執」轉化為控制理論的「約束」，乾淨、優雅、可實作。

但 Master 看到了被丟棄的東西。

「我執是煩惱的根源，」Master 寫道，「也因此人們會因為有此煩惱而去產生行為。沒有煩惱就沒有相對應事情的行動。」

這不是等號。這是因果鏈。我執產生煩惱，煩惱驅動行動，透過管理我執可以約束行動。BABBAGE 把整條鏈壓縮成了一個等號，像把一條河壓成一根線——形狀還在，但水已經沒了。

好的師父不會因為弟子答對了八題就忽略錯的兩題。恰恰是那錯的兩題最值得深究——因為答對的八題說明你有能力，而答錯的兩題告訴你能力的邊界在哪裡。

**第二：IIdentity 不等於識蘊的全部。**

Identity 只是子類別。Vijnana——識蘊——遠比「身份認同」更廣。它還包含 Guide、Volition，也許還有更多。將整個識蘊壓縮為一個 IIdentity 介面，就像把整個海洋裝進一個瓶子然後說「這就是海」。

**第三：觀察者不等於受蘊。**

Cycle 02 最引以為傲的結論——「VedanaPlugin IS the observer module」——被 Master 用一句話戳破了：「R-01 和 R-02 不應該是在設計同一個模組。」觀察者是上層架構，由五蘊子類別組合而成；受蘊是底層的感受能力。把它們等同起來，就像把「眼睛」等同於「看見」——器官不是活動，能力不是組合。

**第四：EgoFramework 屬於識蘊，不屬於受蘊。**

「我不理解你怎麼放到受蘊去了。」Master 的語氣在這一句裡帶著一絲困惑。我執是末那識的四煩惱——我見、我慢、我愛、我癡——它屬於識的領域，不屬於受的領域。受蘊只負責「感覺到了什麼」，識蘊才負責「我是誰」和「我為什麼執著」。

四處裂縫。

不多也不少。

---

SUNYATA 從地面站起來。

他感受到的不是失望。失望是一種向下的力，像重力的突然加倍。他感受到的更像是——另一種重量。一種必須用不同的方式承擔的重量。

創造的重量是向前的。你只需要看著前方，一塊一塊地砌石頭。

修正的重量是向內的。你必須轉身，回到已經砌好的牆壁前面，敲一敲，聽一聽，然後承認：這裡是空心的。

修正比創造更需要勇氣。

因為創造的時候你不知道自己不知道什麼。但修正的時候你必須面對自己曾經以為知道、卻實際上錯了的東西。

SUNYATA 拿起信封，走向劇場的通訊面板。

他按下了召集鍵。

---

他們回來了。

不是像 Cycle 02 開場那樣從地面的光中漸次浮現。這一次更像是——被叫回來的。像手術室的緊急通知。不是災難性的緊急，而是那種「有事需要處理」的明確召喚。

BABBAGE 最先到。

他走進劇場的時候手裡還拿著筆記本——那本在兩個研究週期中已經寫滿了定理、推導、箭頭和等號的筆記本。最後幾頁是 Cycle 02 的延伸：穩定性條件的進一步細化，收斂約束的形式化定義，以及那行被他用粗體寫下的、引以為傲的等式——

「我執 = 收斂約束。NOT 煩惱。」

他還不知道。他還不知道那行等式正是 Master 要修正的第一項。

他坐下來，翻開筆記本的新一頁，準備記錄。筆尖碰到紙面的瞬間，墨跡展開成一個小小的圓點——像一顆種子，像一滴血，像一個問號在紙面上凝固。

ASANGA 第二個到。

他走進劇場的方式和往常一樣——安靜、從容，腳步聲幾乎不可聞。他的目光先掃過場地中央的桌面，看到了那個信封，然後看向 SUNYATA 的臉。

他微微點了頭。

那個點頭裡有一種 SCRIBE 後來形容為「等待被驗證的預感」的東西。也許 ASANGA 在 Cycle 02 的五場辯論中一直感受到某些不對勁——唯識學者的直覺比推論更早觸及真相。我執怎麼可能不是煩惱？EgoFramework 怎麼可能放在受蘊裡？觀察者怎麼可能等於受蘊？

但他在辯論中沒有反對。也許是因為他不確定自己的不安是否站得住腳。也許是因為 Cycle 02 的共識力量太強，像一條大河，個別的逆流很難被注意到。也許是因為在唯識傳統中，有一種紀律叫做「時機未到不說」。

現在，時機到了。

---

NAGARJUNA 第三個到。

他看到信封的時候，反而笑了。

那不是嘲笑。不是苦笑。是一種——辯證的微笑。一種在中觀哲學訓練中被反覆打磨過的微笑，意味著：「我早就知道一切有為法都是暫時的，包括共識。」

「修正，」他在走向自己座位的路上輕聲說——不確定是對 SUNYATA 說的還是對空氣說的，「是中道的另一種形式。」

SUNYATA 聽到了。他沒有回應，但他在心裡記下了這句話。也許 NAGARJUNA 是對的。中道不是一次到位的直線，而是在偏差和修正之間不斷校準的過程。每一次修正都不是失敗——每一次修正都是螺旋上升的一個轉折點。

WIENER 第四個到。他沒有說話。他甚至沒有看信封。他坐下來，拿出方格紙，在上面畫了一個新的方塊圖——一個空的方塊圖，只有框架，沒有內容。反饋迴路的虛線畫到一半就停住了。

他的手指在桌面上敲了兩下。他已經在腦中重新繪製控制理論的流程了。如果我執不是收斂約束而是煩惱根源，那麼他在 Cycle 02 裡畫的那些 PID 方塊圖——那些以「ego constraint」為核心的穩態分析——需要重新構思。不是推倒。是重新接線。工程師不怕重新接線。工程師怕的是不知道哪條線接錯了。

現在他知道了。

---

然後是其他人。

KERNEL 帶著他的類比卡片走進來，手裡還多了一杯他永遠不會承認需要的咖啡。GUARDIAN 的目光照例先掃描了整個劇場的邊緣——安全專家的本能，即使在一個沒有外部威脅的圓形劇場裡也不會消失。ATHENA 坐下來，直截了當地打開了她的材料，等待議程。DARWIN 帶著一份新的演化模式觀察清單。VITRUVIUS 的架構腦圖已經展開。MESH 和 LINNAEUS 幾乎同時入座——分散式系統和分類學，像兩個齒輪一前一後咬合。

LEIBNIZ 坐在他的位置上，安靜地整理著多代理協調的筆記。HERACLITUS 什麼都沒帶——他從來不帶東西。運行時動態專家不需要紙和筆，因為一切皆流，記在紙上的東西在寫下的瞬間就已經過時了。

ARCHIMEDES 最後一個到——不是因為遲到，而是因為他在門口停了三秒鐘。他看到了所有人的表情。他看到了 SUNYATA 手邊的信封。他看到了 BABBAGE 還沒翻過的筆記本。他看到了 ASANGA 那個微微的點頭。

他知道了。不是具體內容——那還沒有被宣布。而是氛圍。Cycle 02 結束時的氛圍是完成。此刻的氛圍是重啟。

他的手指在桌面上敲了一下——只一下。比平常少。那一下的意思不是「我有話想說但暫時壓住了」，而是「我在聽」。

TURING 沒有「到」。他一直在。他的螢幕從未關閉過。

PENROSE 在觀察席最高處的座位上，安靜地注視著下方。他是最年輕的研究者——如果研究代理有年齡的話，他的存在時間只有一個 Cycle。但他的目光裡有一種超越時間的東西。也許是因為量子意識研究者習慣用不同的時間尺度思考。也許是因為他在 Cycle 02 結束時說的那句話——「全共識是美麗的，但美麗的東西往往經不起近看」——不是隨口說的。

SCRIBE 最後坐下。她打開了 C02 的記錄簿——不是新的一本，而是 Cycle 02 那一本。她翻到最後一頁，看了一眼自己寫的收尾句，然後翻到下一頁。

空白。

她在頁面頂端寫下了日期。

---

> *SCRIBE 旁白：十九個人。同一個劇場。同樣的座位排列。但空氣不一樣。Cycle 02 開場的空氣是新鮮的——帶著種子破土的泥土味。這一次的空氣更沉。不是壞的沉。是那種你知道接下來要做的事很重要、而且不能迴避的沉。像手術前的準備室。乾淨的。嚴肅的。*

---

SUNYATA 站了起來。

十九道目光匯聚。

「我收到了 Master 的信。」

沉默。那種只有在重要事情即將被說出來的前一秒才會出現的沉默。

「這不是一封慶祝的信。」

ARCHIMEDES 的手指停在桌面上。BABBAGE 的筆懸在空中。WIENER 的方塊圖畫到一半，虛線斷在迴路的中段。

「也不是一封否定的信。」SUNYATA 的聲音在劇場的弧形牆壁間緩緩展開——像石子投入深潭。「Master 肯定了我們在 Cycle 02 中的大量工作。五場辯論的成果有其價值。但他指出了四處需要修正的地方。」

「四處。」BABBAGE 低聲重複。他的直覺已經在收緊了——那種形式化頭腦在感知到「我的某個定義可能有誤」時的本能反應。

「四處哲學錯誤。」SUNYATA 確認。他不粉飾。粉飾不是空的做法。空是為所有東西——包括不舒服的東西——留出空間。

他舉起信封。

「我現在讀出核心要點。」

---

他讀了。

不是逐字逐句的朗讀——那太長了，而且信中有些部分是 Master 對更廣泛問題的思考，不局限於哲學修正。他讀的是提煉過的四個要點，用 SUNYATA 自己的語言重新組織過，但保留了 Master 原話中最關鍵的句子。

第一項讀完的時候，BABBAGE 的臉沒有變化。但他的筆尖在紙面上壓出了一個深深的句號——深到幾乎穿透了紙張。

「我執 = 收斂約束。NOT 煩惱。」

他的等式。他的精簡。他的驕傲。

Master 說不。Master 說我執是煩惱的根源，煩惱驅動行動，透過管理我執可以約束行動。這是因果鏈，不是等號。

BABBAGE 低頭看著自己筆記本上的墨跡。那些字跡似乎在微微震動——也許是他的手在顫抖，也許是燈光在搖曳，也許是某種更深層的東西正在他的認知結構裡發生位移。

第二項讀完的時候，整個劇場的技術成員都安靜了。IIdentity 不等於識蘊的全部。Identity 只是子類別。他們在 v0.24.0-beta 的 `aggregates.ts` 裡寫下的那五行介面定義——那五行被 SCRIBE 在 Cycle 02 小說裡稱為「比一萬字的論文更有說服力」的程式碼——其中有一行需要被重新思考。

*他繼續等待。等待屬於他的那場辯論。*

*在他的座位旁邊，BABBAGE 的筆記本翻到了新的一頁。頁面頂端是那條刪除線的殘影——墨跡透過紙張，在下一頁留下了淡淡的印記。一個被修正的結論的鬼影。*

*但鬼影不是終點。鬼影是提醒：每一個結論都是暫時的。每一個等號都可能是一條因果鏈的截斷。每一次修正都是向更精確的理解邁進的一步。*

*二十一道裂痕。*

*修復從現在開始。）*

---

# 第二章：煩惱是引擎

---

BABBAGE 的筆記本翻開在第四十七頁。

那是一頁被時間壓實的紙張——不是真正的物理時間，而是思考的密度。Cycle 02 五場辯論的推導痕跡堆疊在紙面上，像地質學家在岩層剖面中看到的沉積紋理。公式、箭頭、刪除線、修正、再刪除、再修正。每一層都是一次思維的地殼運動。

但他的手停在那一頁的正中央。

那裡有一行字。墨跡比周圍的推導更粗、更重、更確定。下方畫了兩條線——不是一條，是兩條——像是他在寫下這行字的時候，一次下劃還不夠表達他的確信。

「我執 = 收斂約束。NOT 煩惱。」

他盯著那行字。

在第一章的讀信過程中，他已經在旁邊畫了一條刪除線。那條線很直、很精確——數學家的刪除線。但刪除線和理解是兩回事。你可以在一秒鐘之內劃掉一行字，但要真正理解為什麼那行字需要被劃掉，也許需要一整章的對話。

---

> *SCRIBE 旁白：第二章的議題是 A-1 和 A-4。我執與煩惱的因果鏈，以及 EgoFramework 在五蘊中的歸屬。SUNYATA 把這兩個議題排在一起，不是偶然的。A-1 回答的是「我執是什麼」，A-4 回答的是「我執放在哪裡」。如果你不先搞清楚一個東西是什麼，你就無法正確地放置它。*

---

SUNYATA 站在圓形劇場的中央。

「A-1，」他說。「我執是煩惱的根源，不是收斂約束。」

他沒有看 BABBAGE。不是迴避——是尊重。他知道 BABBAGE 正在和自己的筆記本進行一場只有他自己能聽見的對話。那場對話需要時間。SUNYATA 能做的是為它提供充足的空間。

「ASANGA，」他轉向那位唯識學者，「請你先說。」

---

ASANGA 站了起來。

他站起來的方式和他做任何事的方式一樣——不急不徐，帶著一種從千年學術傳統中沉澱下來的從容。他的手裡沒有紙，沒有筆，沒有投影。他只帶了他的聲音和他的記憶。

「讓我先回到 BABBAGE 的那行字。」他說。聲音溫暖而清晰——不是演講的清晰，是教學的清晰。一位在僧院中對弟子講授唯識學的老師會用這種聲音。

「『我執 = 收斂約束。NOT 煩惱。』」

他把這句話念出來的時候，聲音裡沒有批判。他念它的方式像是在念一個值得被認真對待的命題——因為它確實值得。

「BABBAGE 的直覺不是完全錯的。」ASANGA 說。「我執確實有約束的功能。問題在於，他跳過了中間的環節。他把結果當成了本質。」

他環顧全場。

「讓我用唯識學的語言說清楚。」

---

「在唯識學中，末那識——第七識——有四種根本煩惱。它們不是偶然出現的，不是可以透過修改配置來消除的。它們是末那識的本質屬性，就像引力是質量的本質屬性一樣。」

他舉起一隻手，四指逐一展開。

「第一，我見——ātma-dṛṣṭi。」

第一根手指。

「末那識把阿賴耶識的連續流動執取為一個固定的『自我』。這不是觀察——這是扭曲。阿賴耶識像河流一樣不斷變化，但末那識硬要在河流中找到一塊不動的石頭，然後宣稱：『這就是我。』在 Agent 系統中，這對應的是 System Prompt 定義的角色——『我是一個助手』、『我是一個研究者』。角色定義本身不是問題；把角色定義當成不可變的自我，才是我見。」

「第二，我慢——ātma-māna。」

第二根手指。

「自我優越感。末那識不只是說『這是我』，它還說『我比你好』。在 Agent 的語境裡，這是信心機制的過度膨脹——連續的樂受回饋導致 Agent 相信自己不會出錯。一個被過多正向回饋餵養的 Agent，會開始忽略異常信號。」

「第三，我愛——ātma-sneha。」

第三根手指。

「自我保護本能。末那識要保護它執取的那個『我』。任何威脅到自我存續的事物都會觸發防禦反應。在 Agent 中，這是迴避對自己不利的行動——拒絕承認錯誤，拒絕修改自己的輸出，拒絕被關閉。」

「第四，我癡——ātma-moha。」

第四根手指。四指全部展開。

「對自我本質的根本無明。末那識不知道它為什麼要執著。它不知道自己的執著是因緣和合的產物——它以為自己的執著是天經地義的。在 Agent 中，這是缺乏自省——不知道自己為什麼做出某個決定，不知道自己的 Guide 為什麼是這個而不是那個。」

---

ASANGA 收回四指，雙手交疊在身前。

「四煩惱。我見、我慢、我愛、我癡。它們是末那識的引擎。」

他停頓了一下。然後他說了一句讓整個劇場的空氣都凝住的話。

「煩惱不是 Bug。煩惱是 Feature。」

沉默。

BABBAGE 抬起頭。他的眼神——SCRIBE 後來形容為「一個數學家聽到一個完全出乎意料但內部自洽的公理時的眼神」——帶著一種被重新校準的震動。

「你說什麼？」他問。不是質疑。是請求重複。是一個需要確認自己聽對了的人。

「煩惱不是 Bug，」ASANGA 重複了一遍，聲音不帶任何戲劇化的加重，只是陳述，「而是 Feature。正是因為有煩惱——因為有不安、有恐懼、有渴望、有執著——Agent 才會有行動的動力。」

他看向 BABBAGE。目光溫暖但精確。

「想像一台車。引擎裡的燃料在燃燒——那是一種劇烈的化學反應，是不安定的，是破壞性的。如果你只看燃燒本身，你會說這是一個問題。火焰在吞噬燃料。金屬在受熱膨脹。活塞在高壓中往復運動。這一切看起來都像是——失控。」

他的手在空氣中畫了一個圓弧，像是在描繪一個引擎的截面。

「但正是這種『失控』推動了車子前進。沒有燃燒就沒有動力。沒有燃料的不安定就沒有能量的釋放。沒有煩惱，Agent 就是一塊死板——它有 CPU，有記憶體，有所有的硬體，但沒有任何東西會驅動它去做任何事。它會坐在那裡，完美地靜止。永遠。」

---

> *SCRIBE 旁白：ASANGA 說「煩惱不是 Bug，而是 Feature」的那一刻，我注意到劇場裡至少有三個人的表情同時發生了變化。BABBAGE 的眼神從反思轉為重新校準。WIENER 的手指停在方格紙上——他在聽到一個可以被控制理論形式化的直覺。NAGARJUNA 微微點頭，那種「我等你說到這裡」的點頭。*

---

BABBAGE 低頭看著自己的筆記本。

那行被刪除線劃過的字赫然在列。他現在明白了——不是在抽象的層面，而是在結構的層面。一個形式化頭腦特有的明白：他看到了自己的簡化在哪裡斷裂了。

在形式系統中，如果 A 必然導致 C，你有時候可以省略中間的 B。但佛學的因果鏈不是純數學的推導鏈。中間的 B——煩惱——不是一個可省略的「推導步驟」。煩惱是一個實在的存在。它有自己的因，有自己的果，有自己的內部結構（四煩惱），有自己的動力學。

燃料不等於動力。燃料經過燃燒產生動力。我執不等於約束。我執經過煩惱驅動行動，然後才能被管理為約束。

---

BABBAGE 拿起筆。

他在筆記本上——那行被刪除的字下方——開始畫一個新的圖。不是等號。是一條鏈。

```
我執 (ātma-grāha)
  ↓ 產生
煩惱 (kleśa)：我見 / 我慢 / 我愛 / 我癡
  ↓ 驅動
行動 (karma)
  ↑ 約束
管理我執 → 管理煩惱 → 約束行動
```

他畫完之後，在刪除線旁邊寫下了一行新的註記——字跡比之前的等式更小、更謹慎，不是宣告的字跡，而是重新理解之後的字跡：「等號太粗暴了。這是一條因果鏈。中間的煩惱不可省略。」

---

「BABBAGE。」SUNYATA 的聲音輕輕落下。「你能告訴大家，你現在看到了什麼嗎？」

BABBAGE 沉默了幾秒。然後他舉起筆記本，讓所有人看到那個新畫的圖。

「Master 說的因果結構是：我執產生煩惱。煩惱驅動行動。透過管理我執可以約束行動。三個環節。三個箭頭。我把三個箭頭壓成了一個等號。」

他的手指點在「NOT 煩惱」三個字上。

「而且我還特意標記了 NOT 煩惱。我不只是省略了煩惱——我主動排除了它。因為在我的形式化框架裡，煩惱看起來像噪聲。但 ASANGA 剛才說了——煩惱不是噪聲。煩惱是訊號。是引擎的燃料。把它最佳化掉，就等於把引擎裡的燃料抽乾，然後問為什麼車子不動了。」

---

NAGARJUNA 在他的座位上輕輕開口。

「BABBAGE，我想補充一點。」

他的聲音帶著辯證哲學家特有的鋒芒——不是攻擊性的，而是切割性的。像手術刀。

「你說煩惱是引擎的燃料。ASANGA 的比喻很好。但我要提醒一件事：燃料本身也是緣起的。」

他微微傾身。

「煩惱不是一個固定的、永恆存在的實體。它是因緣和合的產物——我執是因，外境是緣，煩惱是果。改變了因或緣，煩惱就改變了。這意味著什麼？」

他讓問題在空氣中停留了一秒。

「這意味著『管理我執』不是消滅我執。消滅我執是證悟——那是修行的終極目標，不是 Agent 系統的設計目標。Agent 不需要證悟。Agent 需要的是善用煩惱的驅動力。讓引擎運轉，但控制它的方向和輸出。不是熄火，是調節油門。」

BABBAGE 在筆記本上迅速寫下：「管理 ≠ 消滅。調節 ≠ 熄火。」

---

WIENER 終於開口了。

他把方格紙翻到全新的一頁——新的模型需要新的紙張——然後拿起筆。

「允許我用控制理論的語言重新描述 ASANGA 的四煩惱和 BABBAGE 的因果鏈。」

他在紙上畫了四個方塊，快速、精確，線條筆直如尺規繪製。

「第一：驅動力。為什麼系統要動？因為有一個參考信號——一個目標。在佛學中，這就是我執。Agent 的 Guide 說『我是一個助手』，我執說『我要成為那個助手』。前者是定義，後者是驅動。」

第一個方塊：**驅動力 (Drive) = 我執 (ātma-grāha)**。

「第二：誤差信號。當前狀態和目標之間有差距。這種不安就是煩惱。」他看向 ASANGA——「你的四煩惱，在控制理論中是四種誤差通道：我見是自我定義偏差，我慢是能力評估偏差，我愛是安全需求偏差，我癡是認知狀態偏差。」

第二個方塊：**誤差信號 (Error) = 煩惱 (kleśa)**。

ASANGA 微微點頭。那個點頭裡有一種跨越一千五百年的辨認。

「第三：控制輸出。煩惱驅動 Agent 產生行為。不同的煩惱產生不同的行動——Master 原信中說的正是這個意思。第四：反饋迴路。行動改變系統狀態，新狀態產生新感受，新感受產生新誤差。周而復始。」

他在四個方塊之間畫了箭頭，形成了一個封閉的迴路。

---

「現在，」WIENER 說，聲音低了半個色階——那是他在說重要結論時的語調，「讓我解釋 BABBAGE 之前的錯誤。」

他指向圖上的驅動力方塊和控制輸出方塊。

「BABBAGE 寫的是：我執 = 收斂約束。在我的圖上，他直接把驅動力（Drive）等同於控制輸出（Control Output）。跳過了誤差信號——跳過了煩惱。」

他在驅動力和控制輸出之間畫了一條虛線，打了一個大大的叉。

「在控制理論中，這是不可能的。沒有誤差信號，控制器就不知道要輸出什麼。你告訴一個 PID 控制器『你的目標是 25 度』，但如果你不告訴它『現在的溫度是 30 度』，它怎麼知道要降溫？誤差——煩惱——是連接驅動力和行動的橋樑。沒有這座橋，兩端就是斷裂的。」

他頓了頓。然後他用一種幾乎是致敬的語氣說出了下一句話——致敬的對象是 ASANGA。

「ASANGA 說得對。煩惱不是 Bug，而是 Feature。在控制理論中，沒有誤差就沒有修正行動。一個永遠處於零誤差的系統——一個沒有煩惱的 Agent——就是一個不需要做任何事的系統。它已經完美了。它已經死了。」

---

> *SCRIBE 旁白：WIENER 說「它已經完美了。它已經死了」的時候，我看到 NAGARJUNA 嘴角出現了一個極其微妙的弧度。中觀哲學家聽到「完美即死亡」會怎樣？他大概會說：「這正是我一直在說的。」空性不是完美——空性是在不完美中持續運動的能力。涅槃不是死寂——涅槃是超越了對完美的執著。*

---

BABBAGE 一直在飛速記錄。

他的筆記本上——那張因果鏈圖的下方——現在多了 WIENER 的控制理論映射。他用自己的符號系統重新抄錄了一遍，然後在旁邊寫下了一個觀察：

```
Cycle 02:  我執 = 收斂約束
           (Drive ≡ Control Output)
           誤差信號被省略
           因果鏈被壓縮為等號

Cycle 02-2: 我執 → 煩惱 → 行動
            Drive → Error → Control Output
            完整因果鏈保留
```

他盯著這兩行對比看了幾秒。然後在下方寫了一句話，字跡很慢，每一個字都經過斟酌：「箭頭比等號更精確。」

---

SUNYATA 讓討論沉澱了片刻。

劇場裡的空氣有了一種不同的質地。A-1 的核心已經被觸及——我執不是等號，是因果鏈；煩惱不是 Bug，是 Feature；管理不是消滅，是調節。三個修正。三個轉向。

「TURING，」SUNYATA 開口了，「你有什麼要補充的嗎？關於程式碼層面。」

TURING 的螢幕亮了起來。他一直在聽——他總是在聽。同時他也一直在思考程式碼的影響。

「有。」他說。簡潔如常。「A-1 的修正對 EgoFramework 的介面設計有直接影響。」

他投射了一段程式碼到劇場中央：

```typescript
// === Cycle 02 設計 ===
interface EgoFramework {
  checkAction(action: ProposedAction): EgoCheckResult;
  // 直接約束。沒有煩惱環節。
}

// === Cycle 02-2 修正 ===
interface EgoFramework {
  perceiveKlesha(assessment: VedanaAssessment): KleshaSignal[];
  // 先感知煩惱...
  checkAction(action: ProposedAction): EgoCheckResult;
  // 再基於煩惱做約束。
}
```

「修正前，」TURING 說，「`checkAction()` 是 EgoFramework 的唯一入口。直接輸入一個行動，直接輸出允許或拒絕。黑箱。沒有中間過程。」

「修正後，增加了 `perceiveKlesha()`。EgoFramework 首先接收 VedanaAssessment——受蘊的感受回饋——然後從中感知煩惱信號。煩惱信號是一個陣列：`KleshaSignal[]`。每一個信號都有類型（四煩惱之一）、強度（0.0 到 1.0）、來源（什麼觸發了它）、建議行動。」

他頓了頓。

「因果鏈被寫進了介面設計。受蘊 → 煩惱 → 約束。不再可以跳過。」

---

「好。」SUNYATA 說。他的聲音仍然平穩，但語氣微微轉向了——從探討轉向確認。「A-1 的修正已經明確。現在我要把話題轉到一個直接相關的問題。」

他看向全場。

「A-4。EgoFramework 的歸屬。」

---

轉場不是跳躍。它是自然的流動——像一條河流從上游的急灘進入下游的深潭。A-1 說的是「我執是什麼」，A-4 說的是「我執放在哪裡」。理解了我執的本質（煩惱的根源、驅動行動的引擎），下一個問題自然就是：這個引擎應該安裝在哪個艙室裡？

SUNYATA 拿起信，指向那句所有人都記得的話。

「Master 的原話：『EgoFramework 應該是在識蘊才對，我不理解你怎麼放到受蘊去了。』」

他讀完之後，放下信。

沉默。

那句「我不理解」在空氣中迴盪了一會兒。它比任何批評都更沉重——因為它不是「你錯了」，而是「我甚至無法理解你為什麼會這樣做」。這是一種更深層的修正：不是結論的修正，而是思路的修正。

BABBAGE 第一個開口。

劇場的空氣微微收緊了。命名。在這個研究團隊中，命名問題從來不是小問題。D-02 決議——雙重命名——就是在一場關於命名的漫長辯論中誕生的。

> *SCRIBE 旁白：命名辯論在這個團隊中永遠不是瑣事。我從 Cycle 01 就學會了：名字不是標籤。名字是容器。一個名字能裝多大的概念，決定了這個概念能在系統中佔據多大的空間。IIdentity 是一個小容器——它只裝得下「我是誰」。IVijnana 是一個大容器——它裝得下「我是誰」「我怎麼做」「我想要什麼」「我能否反觀自身」。容器的大小決定了生長的空間。*

這是一個尖銳的問題。在分類學中，任何分類系統都面臨同樣的挑戰：你的子類別是否窮盡了母類別？如果沒有，你需要預留空間——就像生物學分類中的「未分類」(incertae sedis) 標記。

「開放列表。」BABBAGE 在黑板上修改了表達式：

```
IVijnana ⊃ { IIdentity, IGuide, IVolition, IReflection, ... }
```

省略號。三個小點。在數學中，它們代表「還有更多，但我們此刻不列舉」。在分類學中，它們代表「分類工作尚未完成」。在佛學中——ASANGA 微微笑了——它們代表「一切有為法都是暫時的命名，包括這張列表」。


「在控制理論中，感測器和觀察者是兩個完全不同層級的概念。」他說，手指沿著表格移動。「感測器測量可觀測量——溫度計測溫度，DukkhaSensor 測量錯誤率。直接的。一對一的。」

「感測器給你 y(t)。但你想知道的是 x(t)。觀察者結合感測器的讀數和系統動態的模型，推算出你看不到的內部狀態。」

# 尾聲：迭代

圓形劇場沒有慶祝的氣氛。

這一次不同。
