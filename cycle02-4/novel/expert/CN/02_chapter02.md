# 第二章：CPU 的 ISA 不能超频

## I. 三项共识，七分钟

D2 是在辩论序列被调整之后召开的。原本的排序是 D1→D2→D3→D4→D5→D6，但 SUNYATA 在 D1 结束后做了一个战术性的重新排序。D3（触与作意）的共识度极高，可以快速通过，为 D4（行蕴流向）的讨论提供基础。D4 的三层稳定架构——Sati + VitakkaWatchdog + SafetyMonitor——又是 D2（正念架构）的前置依赖。

所以实际的辩论顺序是：D1→D3→D4→**D2**→D5→D6。

D2 被安排在第四个位置。这意味着当 SUNYATA 宣布「Debate 2 开始」的时候，场内所有人已经经历了三场辩论的密集讨论。D1 的 IGearArbiter 九项决议。D3 的触与作意五项决议（最和平的一场——那个等下再说）。D4 的行蕴流向四项决议加上 WIENER 的 Lyapunov 稳定性证明。

二十位学者带着这些前置知识走进 D2。这很重要——因为 D2 的核心议题是「正念如何在系统中运作」，而 D4 已经告诉了他们「正念在三层稳定架构中占什么位置」。

SUNYATA 一开场就把三项共识列了出来。

第一项：正念应为事件驱动，而非 Polling。

NAGARJUNA 站起来引用了巴利经文——《大念处经》（DN 22）：「Atapi sampajano satima」——精勤、正知、正念。他解释了 sati 的字根 smrtti 原义为「记忆」、「不忘失」。正念的核心功能不是「去检查」（polling），而是「不忘失」——对当下状态的持续保持觉知。事件驱动意味着当认知事件发生时，觉知自然在场。Polling 则意味着两次检查之间存在觉知空白——而这正是佛学中「失念」（musita-smrti）的定义。

Master 在第四封信中引用的是同一部经。M-4：「正念不是 polling。重复说一次：**不是 polling**。」

20/20。零讨论时间。

第二项：VitakkaWatchdog 选项 (c)——重置 Klesha——永久排除。VitakkaWatchdog 不应直接修改 Klesha 状态。(b)+(a)——emit event + force Gear 2——是基线行为。D4 的「行蕴→IUI 唯一直接路径」决议已经封锁了 (c) 的可能性。

20/20。

第三项：Sati 不是独立组件，而是五蕴循环的品质指标。ASANGA 引用了《成唯识论》——念属于「别境」五心所，意味着它只在特定条件下生起，不是每一刹那都伴随的「遍行」。映射到架构：Sati 不是 Agent Core 的内建组件，而是通过 plugin 提供的可选增强功能。

20/20。

三项共识。七分钟。六十票。零反对。

SUNYATA 合上了简报的前三页。「现在进入核心争议。」

## II. 僵尸、有分心、与呼吸

D2-R1 的辩论从 ASANGA 的一个问题开始。

「如果 Sati 是 plugin 而不是核心内建——那么如果没有安装任何 Sati plugin，Agent Core 是否有最低限度的自我觉察？」

他的声音带着唯识学者那种特有的「我知道答案但我要先听你们的」的语调。

TURING 在 R1 分析中发现的僵尸状态让这个问题变得紧迫。v0.26.0-beta 的 ExecutionLoop——crash recovery 路径中，如果 `processEvent()` 抛出异常导致 `eventQueue` 内部状态损坏，`dequeue()` 可能永久阻塞。此时 `running` 仍为 `true`，但循环事实上已经停止处理事件。

一个活着但没有在运作的 Agent。僵尸。

ASANGA 提议：SafetyMonitor 应扩展一个 loop liveness check——核心自身就能检测到自己「活着但没有在运作」的状态。「这就像人即使不在禅修，也不会忘记呼吸——呼吸是生命的基线，不需要正念就存在。」

KERNEL 立刻反论。他用的是操作系统工程师那种「你说的概念是对的但你给它起了错误的名字」的语气：

「ASANGA 的观察是正确的——僵尸状态确实是问题。但我不同意将此称为『Sati 的最低基线』。」

他的论证很直接。在操作系统中，存活性检测（liveness detection）是基础设施层面的机制，不是认知层面的功能。Unix 的 `init` 程序负责重启崩溃的 daemon——不是因为 init 有「觉察」能力，而是因为它是进程管理的基本职责。

「把它叫做『Agent Core 的最低觉察』是赋予了它过高的认知地位。SafetyMonitor 做存活性检查就像 OS kernel 做 watchdog timer——这是系统管理，不是正念。」

---

然后 NAGARJUNA 做了一件他在六场辩论中做过无数次的事——用佛学为工程争议提供精确的坐标。

「在巴利阿毘达磨中，正念属于美心所（sobhana-cetasika）——它只与善心伴随。正念不伴随不善心，也不伴随所有善心——它需要特定的心理条件才能生起。」

他顿了一下。

「SafetyMonitor 的存活性检查不是正念——它是更基础的东西。在佛学中，它可能对应的是『有分心』（bhavanga-citta）的功能。」

有分心。生命流的基本维持。即使在深度睡眠中也在运作。不需要任何认知活动。不产生觉察，只维持识流的连续性。

「所以 KERNEL 是对的：存活性检测不应被称为 sati。正念从事件驱动的 plugin 层开始。」

ASANGA 接受了修正。他的分层被重新命名：

| 层级 | 功能 | 佛学对应 | 实现 |
|------|------|---------|------|
| 基础设施 | 存活性检测 | 有分心（bhavanga） | SafetyMonitor.liveness |
| 正念 Level 1 | 计数器监控 | 初级觉察 | VitakkaWatchdog |
| 正念 Level 2 | 事件驱动觉察 | 中级正念 | SatiMonitor plugin |
| 正念 Level 3 | 连续流分析 | 高级正念 | 未来 Phase 3 |

GUARDIAN 补充了安全角度——僵尸 Agent 在生产环境中是危险的。外部系统继续向它发送请求，请求永远不被处理。如果上游有超时重试，资源可能耗尽。SafetyMonitor 需要 `lastTickTimestamp` 来检测这个场景。

**D2-R1：SafetyMonitor 扩展 liveness 检查 + 不称为 sati + 有分心佛学映射——10/10。**

## III. 指令集不能超频

D2-R2 是本场辩论的核心争议——也是 HERACLITUS 整个 Cycle 02-4 中最亮眼的时刻。

争议的本质是：正念的「等级」应该在运行时动态切换吗？

HERACLITUS 的 R07 报告提出了三级模型——Level 1/2/3 在配置时固定。PASCAL 的 R08 报告提出了 EVOI 框架——根据动态价值评估在运行时切换监控等级。两者在交叉审阅中碰撞，HERACLITUS 提出了折中方案。

他走到白板前。HERACLITUS 走到白板前的时候，你可以看到他脑中的类比在成形——不是文字先行，而是图像先行。他画了一颗 CPU。

「Level 等于指令集架构。ISA。x86，ARM，RISC-V。在芯片制造时就决定了。你不能在 runtime 把一颗 ARM 芯片变成 x86 芯片。ISA 决定了处理器**能做什么**——支持哪些指令，寄存器有多少，地址宽度多大。」

他在 CPU 旁边画了一个时钟图案。

「Depth 等于时钟频率。Clock Speed。同一颗 x86 处理器可以在 1GHz 和 5GHz 之间动态调频。高负载时升频，低负载时降频节能。时钟频率决定了处理器**做得多快多细**——但它做的事情种类不变。」

他回头看了看 PASCAL。

「你的 EVOI 框架想要什么？它想在检测到高风险操作时，自动从 Level 1 升级到 Level 2。用 CPU 的话来说，你想在 runtime 把 ARM 换成 x86。」

圆形剧场里有人笑了。不是嘲笑——是「这个类比太精确了」的笑。

---

PASCAL 不是那种会因为一个精彩的类比就改变立场的人。他从 EVOI 的角度反击。

「监控不是免费的。每一层额外的监控都消耗计算资源。当 Agent 回答『今天天气怎样』时，Level 1 的粗略监控就足够了。但当 Agent 准备执行 `rm -rf /` 时，Level 2 的精细监控的信息价值远高于其计算成本。」

他的核心论点简洁有力：「如果 Level 在部署时固定为 Level 1，那么即使 Agent 遇到了高风险场景，它也**无法升级监控能力**。Level 1 看不到 EventBus 上的事件流，无论阈值怎么调都看不到。」

HERACLITUS 反驳。他的声音很稳，像一个已经想过所有反论的人。

「你说的是对的——你可以在 runtime 把一颗 ARM 芯片换成 x86 芯片。那叫做『更换硬件』，不叫做『动态调频』。动态加载 SatiMonitor plugin 是一个部署级操作——它改变了系统的架构能力，需要考虑 plugin 依赖、状态初始化、测试覆盖、安全审计。」

然后他说了决定性的一句话：

「更重要的是：PASCAL 的 EVOI 分析只需要 Depth 调整就能实现。在 Level 2 的框架内——低风险时只订阅 `gear:switch` 事件，采样率 10%。高风险时订阅全部事件，采样率 100%，启动即时分析。EVOI 驱动的资源调配完全可以在固定 Level 内完成——不需要跨 Level 切换。」

---

WIENER 在这个时刻从控制理论的角度提供了形式化的确认。

「HERACLITUS 的 Level/Depth 区分在控制理论中有精确对应。Level 等于观测器结构——观测矩阵 $(A_L, C_L)$ 决定了能看到什么。Depth 等于观测器增益——增益 $K$ 决定了看得多快。」

他在白板上写下了一个定理：

```
可观测性由 (A_L, C_L) 决定，与增益 K 无关。
```

「如果 Level 1 的观测矩阵不包含某个状态变量，无论增益如何调整，该状态变量都是不可观测的。」

这是 WIENER 在六场辩论中提供的最简洁的形式化陈述之一。一句话，封死了「通过调 Depth 来弥补 Level 不足」的可能性。

但他也给了 PASCAL 一个出口：「Level 的切换需要切换系统理论的保证——不恰当的切换时机可能导致瞬态不稳定。HERACLITUS 的折中在控制理论上是最稳健的。」

---

PASCAL 让步了。但他的让步带着两个条件。

条件一：EVOI 框架作为 Depth 调整的决策引擎。Depth 的动态调整不应是 ad hoc 的——它应有数学化的决策依据。

```
Depth(t) = argmax_d [ EVOI(d) - Cost(d) ]
```

条件二：Phase 3 预留 Level 动态切换的设计空间。不从架构上永久排除。

HERACLITUS 接受了两个条件。「我的反对是针对 Phase 1/2 的过早引入，不是永久否定。」

KERNEL 对条件一做了一个微调——Plan28 的第一版用简化版风险评分代替完整 EVOI 计算。工具调用等于高风险，Depth 升高。纯文字回复等于低风险，Depth 降低。完整 EVOI 留到后续迭代。

PASCAL 接受：「简化版风险评分是 EVOI 的退化形式——只要接口预留了未来替换的空间。」

**D2-R2：Level 固定 / Depth 动态 + 三级模型 + Phase 3 预留——16/18。**

MESH 和 SUNYATA 投了反对票。MESH 认为分布式场景中 Level 应可动态切换——多节点 Agent 中，某节点检测到异常时应能通知其他节点升级监控。SUNYATA 认为 Phase 3 预留不够明确。两个反对都被记录在案，但不改变决议。

## IV. 四个维度

D2-R3 是三方提案的统一——WIENER 的四维品质模型、ATHENA 的 SPC 框架、GUARDIAN 的外部可观测指标。

WIENER 先走到白板前。「正念的品质不是一个标量，而是一个向量——它有多个独立的维度。」

他写下四个维度：

$$\vec{Q}_{sati} = (C, G, S, E)$$

Continuity——观察的持续性。不间断。觉知在时间上无间隙。

Granularity——观察的细致度。如实知见。对事件的精确了知。

Speed——介入的即时性。敏捷。觉察的即时反应。

Equanimity——观察的不偏不倚。舍。不因事件正负而改变观察品质。

NAGARJUNA 确认了佛学映射。他引用《清净道论》XIV.141 的四个功能——守护功能（arakkhana-rasa）对应 C，取境功能（apilapana-rasa）对应 G，即时功能（sampajanna）对应 S，不偏功能（upekkha-sahagata）对应 E。

「四维模型不是任意选择的指标——它映射了南传佛学对正念功能的经典分类。」

但他留了一个微妙的保留。「佛学中的平等性不只是正面和负面事件覆盖率相同——它更深层地指向不因事件正负而改变观察的品质。WIENER 的量化定义捕捉了覆盖率平衡，但没有捕捉观察品质的一致性。」

他停顿了一秒。「这不是反对——只是记录在案的佛学微妙性。第一版的量化已经足够。」

---

ATHENA 接过 WIENER 的向量，定义了判断逻辑。SPC——统计过程控制。不用单次测量判断好坏，而是用控制图追踪趋势。

四条 Western Electric 判断规则。单点低于 Action Limit——严重异常。连续三点低于 LCL——趋势恶化。连续七点同向移动——漂移。CL 周围高频振荡——不稳定。

暖机期：前 30 个 tick 使用固定阈值，之后切换到基于历史数据的动态控制限。

PASCAL 在这里提供了一个精巧的诠释：「暖机期就是从先验到后验的过渡。前 30 个事件的 SPC 判断基于先验假设。第 31 个事件开始基于经验数据。」

BABBAGE 笑了。「也就是说，SPC 的暖机期其实是 Bayesian 学习。」

「任何暖机期都是 Bayesian 学习。」PASCAL 平静地回答。

GUARDIAN 定义了第三个面向——外部可观测性。SatiMonitor 定期通过 EventBus emit `sati:quality_report` 事件，payload 包含 SatiQualityVector 和 SPC 状态。外部系统可以订阅，建立长期追踪。

三方统一：WIENER 的四维（测什么）+ ATHENA 的 SPC（怎么判断）+ GUARDIAN 的 EventBus（谁能看到）。不冲突——分别回答品质量化的三个面向。

**D2-R3：SatiQualityVector 四维 + SPC Phase 2+ 可选 + EventBus 报告——19/19（SCRIBE 弃权）。**

## V. 僵尸检测的盲点

D2-R4 合并了两个议题：心跳的性质归类，和事件风暴下的退化策略。

心跳的争议从 BABBAGE 开始。他那种精确到令人不安的诚实——

「`setInterval(heartbeat, 1000)` 在形式上就是 polling。不管我们怎么命名，它的计算模型是周期性时间触发——这符合 polling 的形式化定义。」

他推了推眼镜。「我不是说心跳不好。我是说我们应该诚实承认它的本质。」

KERNEL 反驳。他区分了两个概念：Polling 是主动查询目标的状态——方向是观察者到被观察者。Heartbeat 是定期发射存活信号——方向是被观察者到观察者。形式上都是 periodic，但语义完全不同。

NAGARJUNA 再一次用佛学精确了坐标：

「心跳对应的不是正念，而是有分心。有分心不产生认知，不觉察外在对象——它只维持识流的连续性。心跳也是这样：它不觉察系统状态的细节，只维持『系统还活着』的信号。」

「正念是对当下经验的觉察——你必须能描述你觉察到了什么。心跳描述不了它觉察到了什么——它只能描述『我还在跳』。」

然后 TURING 提供了僵尸检测的关键事实。

「事件驱动的 Sati 检测不到僵尸。」

他展示了代码。如果 `processEvent()` 抛出异常导致 `eventQueue` 损坏，`dequeue()` 永久阻塞。`running` 仍为 `true`。但没有新事件被处理，就没有事件被 emit，Sati 订阅的 listener 永远不会被触发。

「这是心跳的价值所在。如果最后一次心跳的 timestamp 超过了预期间隔，外部观察者可以判定系统已进入僵尸状态。」

GUARDIAN 把这个论点推到了终点：「事件驱动的监控有一个根本的盲点——它依赖事件的持续产生。如果事件产生本身停止了，事件驱动的监控也就失明了。心跳填补了这个盲点。」

BABBAGE 修正了他的立场：「形式上心跳是 periodic，但语义上它是 liveness probe，不是 cognitive polling。这个区分重要——因为如果我们把心跳归类为 polling sati，Master 的『正念不是 Polling』指令就会与心跳的存在相矛盾。正确的归类避免了这个矛盾。」

---

退化策略由 GUARDIAN 提出。四阶段：

| Stage | 触发条件 | 行为 | 安全事件 |
|-------|---------|------|---------|
| 0 Normal | 负载 < 80% | 全量处理 | 全量 |
| 1 Throttle | 80%-95% | 丢弃低优先级事件 | 全量 |
| 2 Sample | 95%-99% | 采样处理 10-50% | **永不丢弃** |
| 3 Emergency Drop | > 99% | 只处理安全事件 | **永不丢弃** |

退化不是失败——是有控制的品质降级。WIENER 提供了三个稳定性保证：安全事件永不丢弃，退化是可逆的，退化是渐进的。

「安全事件永不丢弃」——D1-R3 的最小事件集（gear:arbiter_evaluated, gear:switch, action:proposed, action:executed）在任何 Stage 下都不被丢弃。SafetyMonitor 的可观测性不受影响。

**D2-R4：心跳 = liveness probe + 四阶段退化——20/20。**

## VI. 正念资粮

D2 的最后一项决议是一场命名之争——表面上是语言学，底下是哲学。

NAGARJUNA 在交叉审阅中提出了异议：EVOI（Expected Value of Information）框架将正念的资源分配描述为「信息的期望价值」——这是否将正念「商品化」了？

「把正念叫做『信息的期望价值』，就像把禅修叫做『注意力资源的投资回报率』——技术上也许有部分正确，但在精神上偏离了佛学的本意。」

他提议改名：MRA——Monitoring Resource Allocation——监控资源分配策略。佛学文献中使用「正念资粮」（sati-sambhara）的比喻。

PASCAL 的回应兼顾了两面。数学层面：EVOI 是决策理论标准术语，由 Howard Raiffa 在 1960 年代定义，重新命名会导致与学术文献脱节。但他承认：「在 OpenStarry 文件中使用 EVOI 可能给读者错误印象——正念只是一个经济学最优化问题。」

他的折中：内部代码保留 EVOI（与学术一致），设计文件使用 MRA，佛学讨论使用「正念资粮」。三者指向同一个数学框架。

ASANGA 提供了佛学桥接。「资粮道」（sambhara-marga）——修行者在见道之前需要积累福德资粮和智慧资粮。「我们不是在购买正念——那是商品化。我们是在积累正念所需的条件——那是资粮观。」

BABBAGE 投了唯一的反对票——基于学术一致性。「EVOI 是已确立的术语，重新命名在跨学科沟通中可能造成混乱。」但他尊重多数决议。

**D2-R5：EVOI → MRA 改名 + 正念资粮佛学映射——17/18（BABBAGE 反对）。**

## VII. 散场

D2 结束的时候，白板上有八项决议——三项快速通过的共识，五项辩论决议。二十位学者，从三项 20/20 到一项 16/18，覆盖了存活性检测、监控架构分层、品质量化、退化策略、命名语义。

SCRIBE 的记录显示 D2 是六场辩论中第三长的——仅短于 D5（~150 分钟）和 D6。但 D2 的特殊之处不在长度，而在于它是一场「类比之战」。

HERACLITUS 的 CPU ISA/Clock Speed。NAGARJUNA 的有分心。WIENER 的观测器结构/增益。PASCAL 的先验到后验。DARWIN 在整场辩论中没有发言，但在 SCRIBE 的旁白中被记录下来——他在笔记本上画了一张生物学的类比图：Level 对应的是生物的感觉器官种类（眼、耳、鼻），Depth 对应的是每种感觉器官的灵敏度。蛇没有耳朵——无论多努力，它都听不到声音。这是 Level 的限制。但蛇的红外线感觉极度灵敏——这是 Depth 的优势。

每个人用自己的学科语言表达了同一个洞见：**结构决定可观测性，增益只影响精度。**

SUNYATA 在白板上写下 D2 的结语：

```
Sati ≠ Polling（Master M-4 确认）
有分心 → VitakkaWatchdog → SatiMonitor（三层分离）
Level = ISA，Depth = Clock Speed（HERACLITUS 类比）
Q_sati = (C, G, S, E)（四维品质向量）
```

PASCAL 合上了笔记本。笔记本翻到的那一页不再是空白——不再是 Beta(1,1) 的均匀分布。经过三场辩论，他的先验已经被大量的观测更新。他知道这个系统的正念架构长什么样。不是完美的——NAGARJUNA 关于平等性的微妙保留、MESH 关于分布式场景的观察、BABBAGE 关于命名的异议——这些都是后验中的不确定性。

但不确定性不是无知。不确定性是诚实——是承认你知道什么、不知道什么、以及不知道的部分有多大。

PASCAL 写下了最后一行：

```
P(Sati 架构正确 | D2 全部决议) >> P(Sati 架构正确 | Prior)
```

远大于。不是等于。不是略大于。是远大于。

> *SCRIBE 旁白：D2 是六场辩论中「类比密度」最高的一场——CPU ISA/Clock Speed、观测器结构/增益、有分心/正念、先验/后验。每一个类比都来自不同的学科，每一个都精确地映射了同一个结构性洞见。*

> *如果 D1 是主题陈述——提出动机——那么 D2 是第二乐章的发展部（development section）。D1 的动机在 D2 中被展开、变形、重新组合。D1 的 ManoAggregator 纯路由催生了 D2 的 Level/Depth 分离。D1 的最小事件集成为 D2 退化策略的安全底线。D1 的 evaluate() 观测函数概念在 D2 中被泛化为整个 Sati 的设计哲学——观测，但不干预。*

> *HERACLITUS 用 CPU 的语言说了 NAGARJUNA 用佛学的语言说了 WIENER 用数学的语言说了同一件事：你不能通过超频来改变一颗芯片的指令集。你不能通过更加努力来获得你的感觉器官无法感知的信息。你不能通过调增益来扩大观测矩阵。*

> *结构先于精度。Level 先于 Depth。ISA 先于 Clock Speed。*

> *这是 D2 的核心洞见。它的影响会在 D5 的五层 Model Delta 中再次出现——L0 Safety Floor 是不可调的结构，L1-L4 是可调的增益。同一个原则，在不同的尺度上反复出现。*
