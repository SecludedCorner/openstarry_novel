## I. 暴風雨前的寧靜

D5 開場的六分鐘是整輪 Cycle 最快的共識確認。

四項。全部 20/20。六分鐘。

SUNYATA 不需要解釋為什麼要快——在座的二十位學者都知道今天的真正戰場在後面。這四項共識是鋪路石，不是目的地。

第一項：四煩惱構成論。我執不是四煩惱的「產物」，而是四煩惱的「宏觀表述」。R01 的原文用了一個物理學類比——「溫度之於分子運動」。溫度不是分子碰撞產生的某種東西，溫度就是分子碰撞的統計描述。同理，我執就是 moha、drishti、mana、sneha 四個信號的聯合描述。

PENROSE 點了點頭：「構成性湧現——複雜系統中最自然的模式。」

第二項：EgoFramework 不需要獨立的 ego 狀態變量。既然我執是四煩惱的函數，那就不需要 `egoLevel: number` 這樣的獨立變量，不需要 `d(ego)/dt = f(...)` 這樣的動態方程。四個 `KleshaSignal.value` 就是全部。

第三項：四煩惱的聯合分佈不可分解為邊際積。PASCAL 的論證簡潔——高 moha（無明）通常伴隨高 sneha（執著），因為缺乏覺察的人傾向更強的執著。高 mana（慢）和高 drishti（見）也正相關。工程含義：四煩惱不能被獨立校準。

第四項：Model Delta 的 Layer 0 和 Layer 1 可以直接落地。SafetyMonitor 和 KleshaModulatedDispatcher 在 v0.26.0-beta 中已經存在，只差接線。

六分鐘。四項共識。零爭議。

然後 SUNYATA 說了一句話，語調沒有任何變化，但圓形劇場的空氣突然密了起來：

## II. 繼承的代價

IKlesha 的蘊歸屬問題——在 Cycle 02-4 的語境中——不應該是一場激烈的辯論。R2 交叉審閱後，方案 B（IKlesha extends IVijnana）已經獲得壓倒性傾向。但辯論的價值不在結論，而在過程中暴露的細節。

BABBAGE 的型別分析是三個方案的精確解剖。

方案 A——保持獨立。型別收益：零。沒有任何編譯期保證 IKlesha 與 IVijnana 的關聯。「PluginManifest.skandha 允許空陣列，」BABBAGE 指出，「一個 IKlesha 實作可以完全不宣告蘊歸屬而通過編譯。」這是 D1 中 TURING 已經報告過的事實。

方案 C——中間層 IVijnanaMechanism。多引入一個空介面，違反 DC-6（不鎖定蘊內子介面歸屬），且多餘。

方案 B——繼承。成本只有一個唯讀判別式欄位。收益是編譯期的結構性保證。

KERNEL 接受了方案 B，但附加了一個條件——他稱之為「凍結契約」。

「IVijnana 永遠不得新增行為方法。」

他的理由精確：如果未來有人在 IVijnana 上加了方法，所有子介面——ISamjna、IKlesha、以及未來任何繼承者——都會被迫實作。基底介面方法泛濫。經典的介面設計反模式。

具體做法是在 JSDoc 上加 `@sealed` 標注。不是 TypeScript 語言特性——TypeScript 沒有 sealed interface——而是團隊約定，code review 層面的約束。

NAGARJUNA 提供了佛學確認：「煩惱為識蘊所攝——末那識恆審思量，四煩惱與之俱起。IKlesha 屬於識蘊在佛學上是不可爭議的。」

18/20。兩票來自哪裡不重要——重要的是 @sealed 條件被寫進了決議。一把鎖，鎖住了一扇還沒有人試圖打開的門。

## III. 主戰場

第三節。一百五十分鐘辯論中最長的一段。全域信心度上限 vs 風險加權閾值。

GUARDIAN 站了起來。

在 Cycle 02-4 的六場辯論中，GUARDIAN 的每一次站起都帶著一種特定的重量——不是激進的重量，而是責任的重量。他知道自己要說的話不會受歡迎。他知道自己可能是少數派。但他站起來，因為有些話如果沒有人說，就永遠不會被記錄。

「我提議全域信心度上限 0.85。」

理由簡潔：任何 Gear 1 候選方案的 confidence 如果超過 0.85，就直接截斷為 0.85。不管 VasanaEngine 的規則多精確，不管匹配度多高——0.85 是天花板。

「安全不應依賴『正確理解情境』，」GUARDIAN 說，「因為情境理解本身可能被操控。如果一個惡意使用者在五次正常操作後發送一個精心構造的請求，VasanaEngine 可能給出 0.92 的信心度——T-2 攻擊場景。全域上限是唯一不依賴語義理解的安全機制。」

ATHENA 站了起來。如果說 GUARDIAN 的站起帶著責任的重量，ATHENA 的站起帶著功能的重量。

「全域上限 0.85 抹殺了 Gear 1 的校準性。」

她的論證直指核心：如果一個 IGearArbiter plugin 經過大量驗證，在某類請求上確實有 0.95 的正確率，全域上限把它壓到 0.85——這不是「安全」，這是「浪費」。系統明明知道答案是正確的，卻被迫走更慢的路徑。

然後 PASCAL 站了起來。

---

PASCAL 在 D5 的第一次介入後來被 SCRIBE 標記為「轉折點 1」。

他沒有站在 GUARDIAN 一邊，也沒有站在 ATHENA 一邊。他站在數學一邊。

「讓我用資訊理論的語言描述全域上限的代價。」

他在白板上畫了一條線。線的左端標記 0.0，右端標記 1.0。

「信心度是一個 [0, 1] 區間上的信號。如果我們把上限設為 0.85，那麼 [0.85, 1.0] 這個區間的信道容量為零——所有落在此區間的信號都被壓縮成同一個值。」

他停頓了一秒。

「從 Bayesian 校準性的角度看，一個校準良好的分類器在 confidence = 0.95 時，其實際正確率應該接近 0.95。全域上限把所有 ≥ 0.85 的信號壓成 0.85——破壞了校準性。系統不再能區分『我有 86% 的信心』和『我有 99% 的信心』。」

圓形劇場安靜了。

GUARDIAN 沒有馬上反駁。他在思考。

ATHENA 也沒有追擊。她知道 PASCAL 的論證比她的直覺更有力——不是因為它更「對」，而是因為它把問題放在了一個雙方都無法迴避的數學框架中。

WIENER 補充了第二個約束：風險加權的 Δ_risk 必須是靜態函數——依賴動作分類而非歷史狀態。否則，閾值的回饋可能產生極限環震盪。

GUARDIAN 站了起來。但這一次，他的語調不同了。

```
ManoAggregator 路由延遲 (目標): ~16ms

加上 IVolition:
  v1 (規則型): 16 + 5 = ~21ms (+31%)
```

## VI. 五層，一個公式

第六節。Model Delta 五層結構。

PASCAL 在白板上寫下了一個公式，然後退後一步，讓所有人看清楚：

```
θ_final = clamp(
  θ_base                    // 基礎閾值 0.60
  + Δ_klesha(t)             // Layer 1: Klesha 增益
  + Δ_prajna(t)             // Layer 2: Prajna 調節 ∈ [-0.05, +0.05]
  + Δ_sati(t)               // Layer 3: Sati 品質 ∈ [-0.05, +0.05]
  + Δ_vedana_emergency(t)   // Layer 4: Vedana 緊急 ∈ [0, +0.15]
  + Δ_risk(a),              // 風險加權
  θ_min,                    // 0.30 (Safety Floor)
  θ_max                     // 0.90 (Safety Floor)
)
```

五層。每一層有明確的責任、明確的限幅、明確的時間尺度。Layer 0 是地板——不可妥協。Layer 4 的 Vedana Emergency 只能提高閾值（更保守），不能降低。所有層的結果被 clamp 在 [0.30, 0.90]——Safety Floor 始終有效。

WIENER 做了累積偏移分析：Layer 2 和 Layer 3 各 ±0.05，累積最大偏移 ±0.10。加上 Layer 4 的 +0.15，所有上層的最大累積偏移是 -0.10 到 +0.25。在 [0.30, 0.90] 的 clamp 範圍內，不會溢出。

ARCHIMEDES 把每一層對應到工程計畫：Layer 0 + Layer 1 + Δ_risk → Plan27。Layer 4 → Plan28。Layer 2 + Layer 3 → Plan29+。

20/20。

全票通過。在一場充滿爭議的辯論中，五層模型是唯一讓所有人同意的東西。因為它不是一個觀點——它是一個框架。觀點可以被反對，框架容納所有的觀點。

## VII. Beta(2,18) 的峰值

第七節。Sneha 的下限值。

ASANGA 引用了《成唯識論》：「此四煩惱，常與末那俱。」四煩惱與末那識恆審思量地同時生起。在未達阿羅漢果位之前，四煩惱不可能歸零。

工程含義：sneha.value 不應能降到 0。一個 Agent 如果對自己的存續和任務完成毫無執著，它就不是一個有意義的 Agent。

ASANGA 建議 0.10。PASCAL 在 R08 中原本建議 0.05。

然後 PASCAL 做了一件令人敬佩的事：他用自己的方法否定了自己的建議。

「floor = 0.05 時，Beta(1, 19) 的 mode = 0。分佈認為 sneha 越低越好。floor 的存在意義消失——分佈的 mode 就在 floor 上。」

他畫了兩條曲線。

「floor = 0.10 時，Beta(2, 18) 的 mode = 1/18 ≈ 0.056。分佈有一個有意義的峰值。系統在啟動初期 sneha 的值在 0.056 到 0.10 之間波動，而不是被釘在 floor 上。」

Beta(1,19) 是一條單調遞減曲線。Beta(2,18) 有一個內部峰值。前者說「越少越好」。後者說「有一個自然水平」。

0.10。18/20。

ASANGA 接著推動了一個擴展：既然「四煩惱俱生」，floor 不應只適用於 sneha。四個煩惱都應該有 floor = 0.10。WIENER 同意，並建議 maxLevel = 0.95。

所有四煩惱：klesha_value ∈ [0.10, 0.95]。

WIENER 接著提出了耦合校準協定——十個參數的聯合校準不能獨立進行。座標下降法。聯合目標函數。有界且連續的收斂保證。Plan28 的研究項目。

20/20。

## VIII. 湧現

第八節。三層安全框架。

GUARDIAN 站了起來。這一次，他不是在反對什麼——他在呈現一個他能認同的整體架構。

**第一層：Absolute Safety。**

SafetyMonitor 的 isSafe()。Klesha 免疫。Theta clamp [0.30, 0.90]。跨 Agent 行蘊禁止。Gear 1 最小事件集。破壞性動作 IVolition 強制。

沒有任何可調參數。安全公理。物理定律不可商量。

**第二層：Tunable Safety。**

風險加權閾值。MAX_GEAR1_CONFIDENCE。IPrajna 限幅。Gear 1 IVolition by RiskLevel。Sati Quality Influence。Vedana Emergency。Klesha 增益排程。

每個項目都有明確的護欄——限幅、配置範圍、審計要求。管理員可以在護欄範圍內調整。

**第三層：Reduce Complexity。**

移除 inputHash 完整性（無 TEE 環境下無法防止偽造）。移除重複的輸入最小權限（Sandbox 已覆蓋）。

不是放鬆安全——是移除安全劇場。看穿不提供實質保護的虛假安全。

---

NAGARJUNA 為三層框架提供了佛學映射。

「戒——不可違犯的底線規則。定——有紀律的靈活性。慧——看穿虛妄的智慧。」

戒定慧三學。佛學修行的三個維度。兩千五百年前的分層框架，映射到二十一世紀的系統安全架構——不是巧合，而是因為安全與功能的張力不是現代工程獨有的問題，而是一切有意識系統面臨的根本挑戰。

> *SCRIBE 旁白：如果前四場辯論是交響曲的四個樂章，那麼 D5 是最後的 Allegro molto——急板，最猛烈，最密集。*

> *每一個環節都有一個人在關鍵時刻做了正確的事。PASCAL 三次打破僵局——不是因為他更聰明，而是因為他拒絕在沒有數字的情況下做判斷。GUARDIAN 三次調整立場——不是因為他更弱，而是因為他知道堅持一個數學上有缺陷的立場是自我傷害。NAGARJUNA 提供了兩諦的概念框架——不是裝飾性的引用，而是直接預構了三層安全框架的概念基礎。*

> *六票反對。14/20。Cycle 02-4 所有辯論中贊成率最低的決議。但如果 D5-R4 是未來最有可能被重新審議的決議，那麼六票反對不是失敗——它們是路標。它們精確地標識了設計的已知脆弱點，為未來的安全審計提供了檢查清單的起點。*

> *安全與功能的張力不是被「解決」了。它是被「結構化」了。結構化不是妥協——它是在更高層次上的統一。正如 NAGARJUNA 所說：方便法門。*
