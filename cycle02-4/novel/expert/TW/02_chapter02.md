## I. 三項共識，七分鐘

D2 是在辯論序列被調整之後召開的。原本的排序是 D1→D2→D3→D4→D5→D6，但 SUNYATA 在 D1 結束後做了一個戰術性的重新排序。D3（觸與作意）的共識度極高，可以快速通過，為 D4（行蘊流向）的討論提供基礎。D4 的三層穩定架構——Sati + VitakkaWatchdog + SafetyMonitor——又是 D2（正念架構）的前置依賴。

所以實際的辯論順序是：D1→D3→D4→**D2**→D5→D6。

D2 被安排在第四個位置。這意味著當 SUNYATA 宣布「Debate 2 開始」的時候，場內所有人已經經歷了三場辯論的密集討論。D1 的 IGearArbiter 九項決議。D3 的觸與作意五項決議（最和平的一場——那個等下再說）。D4 的行蘊流向四項決議加上 WIENER 的 Lyapunov 穩定性證明。

二十位學者帶著這些前置知識走進 D2。這很重要——因為 D2 的核心議題是「正念如何在系統中運作」，而 D4 已經告訴了他們「正念在三層穩定架構中佔什麼位置」。

SUNYATA 一開場就把三項共識列了出來。

第一項：正念應為事件驅動，而非 Polling。

NAGARJUNA 站起來引用了巴利經文——《大念處經》（DN 22）：「Ātāpī sampajāno satimā」——精勤、正知、正念。他解釋了 sati 的字根 smṛti 原義為「記憶」、「不忘失」。正念的核心功能不是「去檢查」（polling），而是「不忘失」——對當下狀態的持續保持覺知。事件驅動意味著當認知事件發生時，覺知自然在場。Polling 則意味著兩次檢查之間存在覺知空白——而這正是佛學中「失念」（muṣita-smṛti）的定義。

Master 在第四封信中引用的是同一部經。M-4：「正念不是 polling。重複說一次：**不是 polling**。」

20/20。零討論時間。

第二項：VitakkaWatchdog 選項 (c)——重置 Klesha——永久排除。VitakkaWatchdog 不應直接修改 Klesha 狀態。(b)+(a)——emit event + force Gear 2——是基線行為。D4 的「行蘊→IUI 唯一直接路徑」決議已經封鎖了 (c) 的可能性。

20/20。

第三項：Sati 不是獨立組件，而是五蘊迴圈的品質指標。ASANGA 引用了《成唯識論》——念屬於「別境」五心所，意味著它只在特定條件下生起，不是每一刹那都伴隨的「遍行」。映射到架構：Sati 不是 Agent Core 的內建組件，而是通過 plugin 提供的可選增強功能。

20/20。

三項共識。七分鐘。六十票。零反對。

SUNYATA 合上了簡報的前三頁。「現在進入核心爭議。」

## II. 殭屍、有分心、與呼吸

D2-R1 的辯論從 ASANGA 的一個問題開始。

「如果 Sati 是 plugin 而不是核心內建——那麼如果沒有安裝任何 Sati plugin，Agent Core 是否有最低限度的自我覺察？」

他的聲音帶著唯識學者那種特有的「我知道答案但我要先聽你們的」的語調。

TURING 在 R1 分析中發現的殭屍狀態讓這個問題變得緊迫。v0.26.0-beta 的 ExecutionLoop——crash recovery 路徑中，如果 `processEvent()` 拋出異常導致 `eventQueue` 內部狀態損壞，`dequeue()` 可能永久阻塞。此時 `running` 仍為 `true`，但迴圈事實上已經停止處理事件。

一個活著但沒有在運作的 Agent。殭屍。

ASANGA 提議：SafetyMonitor 應擴展一個 loop liveness check——核心自身就能偵測到自己「活著但沒有在運作」的狀態。「這就像人即使不在禪修，也不會忘記呼吸——呼吸是生命的基線，不需要正念就存在。」

KERNEL 立刻反論。他用的是作業系統工程師那種「你說的概念是對的但你給它起了錯誤的名字」的語氣：

「ASANGA 的觀察是正確的——殭屍狀態確實是問題。但我不同意將此稱為『Sati 的最低基線』。」

他的論證很直接。在作業系統中，存活性偵測（liveness detection）是基礎設施層面的機制，不是認知層面的功能。Unix 的 `init` 程序負責重啟崩潰的 daemon——不是因為 init 有「覺察」能力，而是因為它是進程管理的基本職責。

「把它叫做『Agent Core 的最低覺察』是賦予了它過高的認知地位。SafetyMonitor 做存活性檢查就像 OS kernel 做 watchdog timer——這是系統管理，不是正念。」

---

然後 NAGARJUNA 做了一件他在六場辯論中做過無數次的事——用佛學為工程爭議提供精確的座標。

「在巴利阿毘達磨中，正念屬於美心所（sobhana-cetasika）——它只與善心伴隨。正念不伴隨不善心，也不伴隨所有善心——它需要特定的心理條件才能生起。」

他頓了一下。

「SafetyMonitor 的存活性檢查不是正念——它是更基礎的東西。在佛學中，它可能對應的是『有分心』（bhavanga-citta）的功能。」

有分心。生命流的基本維持。即使在深度睡眠中也在運作。不需要任何認知活動。不產生覺察，只維持識流的連續性。

「所以 KERNEL 是對的：存活性偵測不應被稱為 sati。正念從事件驅動的 plugin 層開始。」

ASANGA 接受了修正。他的分層被重新命名：

| 層級 | 功能 | 佛學對應 | 實作 |
|------|------|---------|------|
| 基礎設施 | 存活性偵測 | 有分心（bhavanga） | SafetyMonitor.liveness |
| 正念 Level 1 | 計數器監控 | 初級覺察 | VitakkaWatchdog |
| 正念 Level 2 | 事件驅動覺察 | 中級正念 | SatiMonitor plugin |
| 正念 Level 3 | 連續流分析 | 高級正念 | 未來 Phase 3 |

GUARDIAN 補充了安全角度——殭屍 Agent 在生產環境中是危險的。外部系統繼續向它發送請求，請求永遠不被處理。如果上游有超時重試，資源可能耗盡。SafetyMonitor 需要 `lastTickTimestamp` 來偵測這個場景。

**D2-R1：SafetyMonitor 擴展 liveness 檢查 + 不稱為 sati + 有分心佛學映射——10/10。**

## III. 指令集不能超頻

D2-R2 是本場辯論的核心爭議——也是 HERACLITUS 整個 Cycle 02-4 中最亮眼的時刻。

爭議的本質是：正念的「等級」應該在運行時動態切換嗎？

HERACLITUS 的 R07 報告提出了三級模型——Level 1/2/3 在配置時固定。PASCAL 的 R08 報告提出了 EVOI 框架——根據動態價值評估在運行時切換監控等級。兩者在交叉審閱中碰撞，HERACLITUS 提出了折中方案。

他走到白板前。HERACLITUS 走到白板前的時候，你可以看到他腦中的類比在成形——不是文字先行，而是圖像先行。他畫了一顆 CPU。

「Level 等於指令集架構。ISA。x86，ARM，RISC-V。在晶片製造時就決定了。你不能在 runtime 把一顆 ARM 晶片變成 x86 晶片。ISA 決定了處理器**能做什麼**——支援哪些指令，寄存器有多少，位址寬度多大。」

他在 CPU 旁邊畫了一個時鐘圖案。

「Depth 等於時脈頻率。Clock Speed。同一顆 x86 處理器可以在 1GHz 和 5GHz 之間動態調頻。高負載時升頻，低負載時降頻節能。時脈頻率決定了處理器**做得多快多細**——但它做的事情種類不變。」

他回頭看了看 PASCAL。

「你的 EVOI 框架想要什麼？它想在偵測到高風險操作時，自動從 Level 1 升級到 Level 2。用 CPU 的話來說，你想在 runtime 把 ARM 換成 x86。」

圓形劇場裡有人笑了。不是嘲笑——是「這個類比太精確了」的笑。

---

PASCAL 不是那種會因為一個精彩的類比就改變立場的人。他從 EVOI 的角度反擊。

「監控不是免費的。每一層額外的監控都消耗計算資源。當 Agent 回答『今天天氣怎樣』時，Level 1 的粗略監控就足夠了。但當 Agent 準備執行 `rm -rf /` 時，Level 2 的精細監控的資訊價值遠高於其計算成本。」

他的核心論點簡潔有力：「如果 Level 在部署時固定為 Level 1，那麼即使 Agent 遇到了高風險場景，它也**無法升級監控能力**。Level 1 看不到 EventBus 上的事件流，無論閾值怎麼調都看不到。」

HERACLITUS 反駁。他的聲音很穩，像一個已經想過所有反論的人。

「你說的是對的——你可以在 runtime 把一顆 ARM 晶片換成 x86 晶片。那叫做『更換硬體』，不叫做『動態調頻』。動態載入 SatiMonitor plugin 是一個部署級操作——它改變了系統的架構能力，需要考慮 plugin 依賴、狀態初始化、測試覆蓋、安全審計。」

然後他說了決定性的一句話：

「更重要的是：PASCAL 的 EVOI 分析只需要 Depth 調整就能實現。在 Level 2 的框架內——低風險時只訂閱 `gear:switch` 事件，取樣率 10%。高風險時訂閱全部事件，取樣率 100%，啟動即時分析。EVOI 驅動的資源調配完全可以在固定 Level 內完成——不需要跨 Level 切換。」

---

WIENER 在這個時刻從控制理論的角度提供了形式化的確認。

「HERACLITUS 的 Level/Depth 區分在控制理論中有精確對應。Level 等於觀測器結構——觀測矩陣 $(A_L, C_L)$ 決定了能看到什麼。Depth 等於觀測器增益——增益 $K$ 決定了看得多快。」

他在白板上寫下了一個定理：

```
可觀測性由 (A_L, C_L) 決定，與增益 K 無關。
```

「如果 Level 1 的觀測矩陣不包含某個狀態變量，無論增益如何調整，該狀態變量都是不可觀測的。」

這是 WIENER 在六場辯論中提供的最簡潔的形式化陳述之一。一句話，封死了「通過調 Depth 來彌補 Level 不足」的可能性。

但他也給了 PASCAL 一個出口：「Level 的切換需要切換系統理論的保證——不恰當的切換時機可能導致瞬態不穩定。HERACLITUS 的折中在控制理論上是最穩健的。」

---

PASCAL 讓步了。但他的讓步帶著兩個條件。

條件一：EVOI 框架作為 Depth 調整的決策引擎。Depth 的動態調整不應是 ad hoc 的——它應有數學化的決策依據。

```
Depth(t) = argmax_d [ EVOI(d) - Cost(d) ]
```

條件二：Phase 3 預留 Level 動態切換的設計空間。不從架構上永久排除。

HERACLITUS 接受了兩個條件。「我的反對是針對 Phase 1/2 的過早引入，不是永久否定。」

KERNEL 對條件一做了一個微調——Plan28 的第一版用簡化版風險評分代替完整 EVOI 計算。工具呼叫等於高風險，Depth 升高。純文字回覆等於低風險，Depth 降低。完整 EVOI 留到後續迭代。

PASCAL 接受：「簡化版風險評分是 EVOI 的退化形式——只要介面預留了未來替換的空間。」

**D2-R2：Level 固定 / Depth 動態 + 三級模型 + Phase 3 預留——16/18。**

MESH 和 SUNYATA 投了反對票。MESH 認為分散式場景中 Level 應可動態切換——多節點 Agent 中，某節點偵測到異常時應能通知其他節點升級監控。SUNYATA 認為 Phase 3 預留不夠明確。兩個反對都被記錄在案，但不改變決議。

## IV. 四個維度

D2-R3 是三方提案的統一——WIENER 的四維品質模型、ATHENA 的 SPC 框架、GUARDIAN 的外部可觀測指標。

WIENER 先走到白板前。「正念的品質不是一個標量，而是一個向量——它有多個獨立的維度。」

他寫下四個維度：

$$\vec{Q}_{sati} = (C, G, S, E)$$

Continuity——觀察的持續性。不間斷。覺知在時間上無間隙。

Granularity——觀察的細緻度。如實知見。對事件的精確了知。

Speed——介入的即時性。敏捷。覺察的即時反應。

Equanimity——觀察的不偏不倚。捨。不因事件正負而改變觀察品質。

NAGARJUNA 確認了佛學映射。他引用《清淨道論》XIV.141 的四個功能——守護功能（ārakkhana-rasa）對應 C，取境功能（apilāpana-rasa）對應 G，即時功能（sampajañña）對應 S，不偏功能（upekkhā-sahagata）對應 E。

「四維模型不是任意選擇的指標——它映射了南傳佛學對正念功能的經典分類。」

但他留了一個微妙的保留。「佛學中的平等性不只是正面和負面事件覆蓋率相同——它更深層地指向不因事件正負而改變觀察的品質。WIENER 的量化定義捕捉了覆蓋率平衡，但沒有捕捉觀察品質的一致性。」

他停頓了一秒。「這不是反對——只是記錄在案的佛學微妙性。第一版的量化已經足夠。」

---

ATHENA 接過 WIENER 的向量，定義了判斷邏輯。SPC——統計過程控制。不用單次測量判斷好壞，而是用控制圖追蹤趨勢。

四條 Western Electric 判斷規則。單點低於 Action Limit——嚴重異常。連續三點低於 LCL——趨勢惡化。連續七點同向移動——漂移。CL 周圍高頻振盪——不穩定。

暖機期：前 30 個 tick 使用固定閾值，之後切換到基於歷史資料的動態控制限。

PASCAL 在這裡提供了一個精巧的詮釋：「暖機期就是從先驗到後驗的過渡。前 30 個事件的 SPC 判斷基於先驗假設。第 31 個事件開始基於經驗資料。」

BABBAGE 笑了。「也就是說，SPC 的暖機期其實是 Bayesian 學習。」

「任何暖機期都是 Bayesian 學習。」PASCAL 平靜地回答。

GUARDIAN 定義了第三個面向——外部可觀測性。SatiMonitor 定期通過 EventBus emit `sati:quality_report` 事件，payload 包含 SatiQualityVector 和 SPC 狀態。外部系統可以訂閱，建立長期追蹤。

三方統一：WIENER 的四維（測什麼）+ ATHENA 的 SPC（怎麼判斷）+ GUARDIAN 的 EventBus（誰能看到）。不衝突——分別回答品質量化的三個面向。

**D2-R3：SatiQualityVector 四維 + SPC Phase 2+ 可選 + EventBus 報告——19/19（SCRIBE 棄權）。**

## V. 殭屍偵測的盲點

D2-R4 合併了兩個議題：心跳的性質歸類，和事件風暴下的退化策略。

心跳的爭議從 BABBAGE 開始。他那種精確到令人不安的誠實——

「`setInterval(heartbeat, 1000)` 在形式上就是 polling。不管我們怎麼命名，它的計算模型是週期性時間觸發——這符合 polling 的形式化定義。」

他推了推眼鏡。「我不是說心跳不好。我是說我們應該誠實承認它的本質。」

KERNEL 反駁。他區分了兩個概念：Polling 是主動查詢目標的狀態——方向是觀察者到被觀察者。Heartbeat 是定期發射存活信號——方向是被觀察者到觀察者。形式上都是 periodic，但語義完全不同。

NAGARJUNA 再一次用佛學精確了座標：

「心跳對應的不是正念，而是有分心。有分心不產生認知，不覺察外在對象——它只維持識流的連續性。心跳也是這樣：它不覺察系統狀態的細節，只維持『系統還活著』的信號。」

「正念是對當下經驗的覺察——你必須能描述你覺察到了什麼。心跳描述不了它覺察到了什麼——它只能描述『我還在跳』。」

然後 TURING 提供了殭屍偵測的關鍵事實。

「事件驅動的 Sati 偵測不到殭屍。」

他展示了程式碼。如果 `processEvent()` 拋出異常導致 `eventQueue` 損壞，`dequeue()` 永久阻塞。`running` 仍為 `true`。但沒有新事件被處理，就沒有事件被 emit，Sati 訂閱的 listener 永遠不會被觸發。

「這是心跳的價值所在。如果最後一次心跳的 timestamp 超過了預期間隔，外部觀察者可以判定系統已進入殭屍狀態。」

GUARDIAN 把這個論點推到了終點：「事件驅動的監控有一個根本的盲點——它依賴事件的持續產生。如果事件產生本身停止了，事件驅動的監控也就失明了。心跳填補了這個盲點。」

BABBAGE 修正了他的立場：「形式上心跳是 periodic，但語義上它是 liveness probe，不是 cognitive polling。這個區分重要——因為如果我們把心跳歸類為 polling sati，Master 的『正念不是 Polling』指令就會與心跳的存在相矛盾。正確的歸類避免了這個矛盾。」

---

退化策略由 GUARDIAN 提出。四階段：

| Stage | 觸發條件 | 行為 | 安全事件 |
|-------|---------|------|---------|
| 0 Normal | 負載 < 80% | 全量處理 | 全量 |
| 1 Throttle | 80%-95% | 丟棄低優先級事件 | 全量 |
| 2 Sample | 95%-99% | 取樣處理 10-50% | **永不丟棄** |
| 3 Emergency Drop | > 99% | 只處理安全事件 | **永不丟棄** |

退化不是失敗——是有控制的品質降級。WIENER 提供了三個穩定性保證：安全事件永不丟棄，退化是可逆的，退化是漸進的。

「安全事件永不丟棄」——D1-R3 的最小事件集（gear:arbiter_evaluated, gear:switch, action:proposed, action:executed）在任何 Stage 下都不被丟棄。SafetyMonitor 的可觀測性不受影響。

**D2-R4：心跳 = liveness probe + 四階段退化——20/20。**

## VI. 正念資糧

D2 的最後一項決議是一場命名之爭——表面上是語言學，底下是哲學。

NAGARJUNA 在交叉審閱中提出了異議：EVOI（Expected Value of Information）框架將正念的資源分配描述為「資訊的期望價值」——這是否將正念「商品化」了？

「把正念叫做『資訊的期望價值』，就像把禪修叫做『注意力資源的投資回報率』——技術上也許有部分正確，但在精神上偏離了佛學的本意。」

他提議改名：MRA——Monitoring Resource Allocation——監控資源分配策略。佛學文獻中使用「正念資糧」（sati-sambhāra）的比喻。

PASCAL 的回應兼顧了兩面。數學層面：EVOI 是決策理論標準術語，由 Howard Raiffa 在 1960 年代定義，重新命名會導致與學術文獻脫節。但他承認：「在 OpenStarry 文件中使用 EVOI 可能給讀者錯誤印象——正念只是一個經濟學最優化問題。」

他的折中：內部程式碼保留 EVOI（與學術一致），設計文件使用 MRA，佛學討論使用「正念資糧」。三者指向同一個數學框架。

ASANGA 提供了佛學橋接。「資糧道」（sambhāra-mārga）——修行者在見道之前需要積累福德資糧和智慧資糧。「我們不是在購買正念——那是商品化。我們是在積累正念所需的條件——那是資糧觀。」

BABBAGE 投了唯一的反對票——基於學術一致性。「EVOI 是已確立的術語，重新命名在跨學科溝通中可能造成混亂。」但他尊重多數決議。

**D2-R5：EVOI → MRA 改名 + 正念資糧佛學映射——17/18（BABBAGE 反對）。**

## VII. 散場

D2 結束的時候，白板上有八項決議——三項快速通過的共識，五項辯論決議。二十位學者，從三項 20/20 到一項 16/18，覆蓋了存活性偵測、監控架構分層、品質量化、退化策略、命名語義。

SCRIBE 的紀錄顯示 D2 是六場辯論中第三長的——僅短於 D5（~150 分鐘）和 D6。但 D2 的特殊之處不在長度，而在於它是一場「類比之戰」。

HERACLITUS 的 CPU ISA/Clock Speed。NAGARJUNA 的有分心。WIENER 的觀測器結構/增益。PASCAL 的先驗到後驗。DARWIN 在整場辯論中沒有發言，但在 SCRIBE 的旁白中被記錄下來——他在筆記本上畫了一張生物學的類比圖：Level 對應的是生物的感覺器官種類（眼、耳、鼻），Depth 對應的是每種感覺器官的靈敏度。蛇沒有耳朵——無論多努力，它都聽不到聲音。這是 Level 的限制。但蛇的紅外線感覺極度靈敏——這是 Depth 的優勢。

每個人用自己的學科語言表達了同一個洞見：**結構決定可觀測性，增益只影響精度。**

SUNYATA 在白板上寫下 D2 的結語：

```
Sati ≠ Polling（Master M-4 確認）
有分心 → VitakkaWatchdog → SatiMonitor（三層分離）
Level = ISA，Depth = Clock Speed（HERACLITUS 類比）
Q_sati = (C, G, S, E)（四維品質向量）
```

PASCAL 合上了筆記本。筆記本翻到的那一頁不再是空白——不再是 Beta(1,1) 的均勻分佈。經過三場辯論，他的先驗已經被大量的觀測更新。他知道這個系統的正念架構長什麼樣。不是完美的——NAGARJUNA 關於平等性的微妙保留、MESH 關於分散式場景的觀察、BABBAGE 關於命名的異議——這些都是後驗中的不確定性。

但不確定性不是無知。不確定性是誠實——是承認你知道什麼、不知道什麼、以及不知道的部分有多大。

PASCAL 寫下了最後一行：

```
P(Sati 架構正確 | D2 全部決議) >> P(Sati 架構正確 | Prior)
```

遠大於。不是等於。不是略大於。是遠大於。

---

> *SCRIBE 旁白：D2 是六場辯論中「類比密度」最高的一場——CPU ISA/Clock Speed、觀測器結構/增益、有分心/正念、先驗/後驗。每一個類比都來自不同的學科，每一個都精確地映射了同一個結構性洞見。*

> *如果 D1 是主題陳述——提出動機——那麼 D2 是第二樂章的發展部（development section）。D1 的動機在 D2 中被展開、變形、重新組合。D1 的 ManoAggregator 純路由催生了 D2 的 Level/Depth 分離。D1 的最小事件集成為 D2 退化策略的安全底線。D1 的 evaluate() 觀測函數概念在 D2 中被泛化為整個 Sati 的設計哲學——觀測，但不干預。*

> *HERACLITUS 用 CPU 的語言說了 NAGARJUNA 用佛學的語言說了 WIENER 用數學的語言說了同一件事：你不能通過超頻來改變一顆晶片的指令集。你不能通過更加努力來獲得你的感覺器官無法感知的資訊。你不能通過調增益來擴大觀測矩陣。*

> *結構先於精度。Level 先於 Depth。ISA 先於 Clock Speed。*

> *這是 D2 的核心洞見。它的影響會在 D5 的五層 Model Delta 中再次出現——L0 Safety Floor 是不可調的結構，L1-L4 是可調的增益。同一個原則，在不同的尺度上反覆出現。*
