# 第四章：虛構的程式碼

---

## I. TURING 的修正

D4 以一條修正開場。

TURING 站起來的時候，場內所有人都知道「修正」意味著什麼。在 Cycle 02-4 的六場辯論中，TURING 的修正總是指向同一個問題——R1 階段的某份報告引用了不存在的程式碼，然後其他報告在不知情的情況下建立在這個引用之上。

「R07 報告在多處引用了 `ISensation.ingestToolResult()` 方法。」

他的語氣像讀日誌。沒有責備，沒有戲劇性——只有事實。

「`ISensation` 介面和 `ingestToolResult()` 方法在 v0.26.0-beta 整個程式碼基線中完全不存在。全域搜尋零結果。」

圓形劇場裡安靜了三秒。

R07 是 HERACLITUS 和 ARCHIMEDES 合寫的動態分析報告——其中引用了這個方法來建構「路徑 Beta」的完整分析，聲稱 v0.26.0-beta 中行蘊結果可在同輪內即時更新受蘊。HERACLITUS 低頭看了一下他的報告副本。他沒有爭辯——TURING 的全域搜尋是不可反駁的。

「R07 提及的 `feedbackQueue` 也不存在。`SparshEvent.timestamp` 欄位也不存在。」

三個虛構。一份報告。

SUNYATA 接過話：「R07 關於受蘊主動攝入的設計方向仍然合理，但所有引用 R07 同輪 vedana 微更新的結論必須修正為——尚未實作，是 Plan27 待建的功能。」

---

這次修正的影響不只是刪除幾個引用。它改變了 D4 的整個基調。

如果 `ingestToolResult()` 存在，那麼 D4 的討論就是「如何約束一個已有的機制」。但它不存在——D4 的討論變成了「如何從零設計一個不存在的機制」。這是一個更自由的設計空間，也是一個更危險的——因為你可以在虛構的程式碼上建立任何理論。

TURING 的修正像是在施工前檢查地基。他發現了三塊不屬於那裡的石頭，把它們取出來。地基反而更穩了。

**教訓被記錄下來：所有涉及程式碼引用的分析必須經過 TURING 驗證。**

## II. 四項共識

TURING 的修正之後，四項共識被快速確認。

**共識 A**：行蘊的唯一直接輸出是 IUI（色蘊輸出面）。ASANGA 引用《增支部》——「諸比丘，我說思即是業。思已，以身、語、意造業。」身行與語行通過身體和語言影響外在世界。WIENER 用控制理論語言確認：行蘊是控制器，IUI 是致動器。控制器不直接修改感測器讀數。9/9。

**共識 B**：行蘊對受蘊的影響通過 EventBus 事件中介。受蘊主動攝入（subscribe），非行蘊主動推送（push）。PENROSE 補充：EventBus 作為中介不改變因果方向——因果結構仍然是 samskara → vedana，差別在耦合程度。9/9。

**共識 C**：不需要獨立的 SamskaraFeedback 介面。行蘊回饋已有兩條既有路徑——色蘊間接路徑和 KleshaContext.actionHistory。新增第三條增加複雜度無額外收益。GUARDIAN 從安全角度同意：減少影響管道降低攻擊面。9/9。

**共識 D**：`ISensation.ingestToolResult()` 不存在。事實陳述，不需表決。

八分鐘。四項共識。D4 的地基就此確立。

## III. 增益上限：0.3

D4-R1 的爭議圍繞一個數字展開。

PASCAL 提出：行蘊結果對受蘊的效價映射增益不應超過 0.3。

「0.3 不是任意數字。」他翻開筆記本，指向一行公式。「在 Bayesian 濾波框架中，觀測雜訊 σ_obs = 0.15 時，初始 Kalman gain K₁ = σ²_prior / (σ²_prior + σ²_obs) ≈ 0.308。工程上取 0.3 作為保守值。」

WIENER 從閉迴路穩定性的角度分析。行蘊→受蘊→想蘊→識蘊→行蘊的完整閉迴路。如果行蘊結果對受蘊的增益為 w，迴路中其他環節的增益乘積 G_rest 約為 2-3，那麼穩定性要求 w < 1/G_rest ≈ 0.33 到 0.50。w ≤ 0.3 提供了約 10% 的安全裕度。

ARCHIMEDES 提出了一個務實的問題——不同行為類型是否需要不同的增益？資訊查詢的增益上限 0.1 就夠了，但破壞性操作的失敗苦受應被充分感知，增益上限或許該高一些。

PASCAL 否定了分類增益的方案。「不需要手動設定不同上限——觀測雜訊 σ_obs 可以按行為類型調節。高風險行為的 σ_obs 設低，Agent 更相信這個信號。低風險行為的 σ_obs 設高，Agent 保持懷疑。Kalman gain 會自動調節。」

WIENER 的折中：統一增益上限 w ≤ 0.3 作為硬性安全約束，同時允許 Bayesian 濾波器在此上限之內自適應。「硬性上限是 SafetyMonitor 可以檢查的簡單不變式。Bayesian 自適應在上限之內提供靈活性。」

NAGARJUNA 提供了一個重要的佛學澄清：「增益限制不應理解為壓低行蘊對受蘊的影響——因為在佛學因果鏈中，行蘊結果不是直接寫入受蘊的。增益限制的正確理解是 VedanaSensor 的最大靈敏度。」

感測器的設計參數。不是因果鏈的約束。

**D4-R1：增益上限 w ≤ 0.3——8/9（ARCHIMEDES 棄權，保留未來按風險等級分化的可能）。**

## IV. 自纏其身

D4-R2 是六場辯論中數學最密集的決議——也是 WIENER 整個 Cycle 02-4 中最深入的分析。

WIENER 走到白板前，畫了一個迴路。

```
偏見(Drishti) ↑
    → IGearArbiter 的信心度被偏見膨脹
    → ManoAggregator 錯誤選擇 Gear 1
    → 快速路徑的結果強化 Drishti
    → Drishti 進一步膨脹
    → ...
```

「這是意行（mano-karma）的內部正反饋迴路。」他的聲音很平靜，但白板上的箭頭在描述一個危險的結構。「它完全在控制器內部運行，不經過外部世界的校正反饋。」

外部迴路——行蘊→IUI→外部世界→IListener→新觸→新受——有外部世界作為現實檢驗。如果行為真的錯了，外部結果會以苦受的形式回饋。但意行的內部迴路沒有這個機制。

WIENER 形式化了這個迴路。狀態向量三維：Drishti 強度 d(t)、有效信心度 c(t)、Gear 1 選擇頻率 g(t)。動態方程。正反饋增益鏈 G_loop = α₁ · α₂ · α₃。

「當正念為零且自然衰減很小時，如果 G_loop > 1，系統發散——偏見無限累積，Agent 永遠鎖定在 Gear 1。」

---

NAGARJUNA 在 WIENER 的數學之後提供了佛學映射。他引用了《成唯識論》——末那識與四煩惱「恆審思量」的運作。

「我見一旦建立，就像一面有色鏡片——通過它觀看的一切都被染色。被染色的辨認結果又確認了鏡片的正確性。這是確認偏誤。」

ASANGA 引用了更直接的經文——《相應部》的繩索比喻：

「如蠶織繭，自纏其身。」

他頓了一下，讓這個意象沉澱。

「意行的正反饋迴路就是自纏其身的數學表述。偏見→扭曲辨認→錯誤判斷→更強偏見——這是十二因緣中無明緣行的動態機制。」

---

WIENER 提出了 Lyapunov 候選函數。V(x) = w₁·d² + w₂·(c-c₀)² + w₃·(g-g*)²。穩定性要求每一步 V 嚴格遞減。

展開主要項後，穩定性的充分條件是：

**β_d · s_min + γ_d > α₁ · g_max**

白話翻譯：正念的最低抑制力加上自然衰減，必須大於正反饋的最大增益。

PASCAL 代入了具體數值。α₁ ≈ 0.05，g_max = 1.0，γ_d ≈ 0.01。穩定性要求 β_d · s_min > 0.04。如果最低正念水準 s_min = 0.2，那麼 β_d > 0.2。

但 WIENER 強調了非線性保障。Drishti 的 clamp（d ∈ [0, 1]，偏見不能無限增長）。VitakkaWatchdog 的截斷（連續 Gear 1 超過閾值就強制 Gear 2）。SafetyMonitor 的邊界（危險行為直接被阻止）。

「即使線性分析預測不穩定，系統也不會真正發散——它會被限制在一個有界的區域內振盪。」

三層穩定化機制由此確立：

| 層 | 機制 | 時間尺度 | 控制理論 | 佛學映射 |
|----|------|---------|---------|---------|
| Layer 1 | Sati 精調 | 每輪微調 | PID 微分項 | 正知（sampajanna） |
| Layer 2 | VitakkaWatchdog 粗反饋 | 跨輪趨勢 | 積分器抗飽和 | 尋（vitakka）觀照 |
| Layer 3 | SafetyMonitor 硬約束 | 即時觸發 | 硬限幅器 | 戒（sila）護欄 |

GUARDIAN 插了一個關鍵的補充：「如果偏見迴路導致 Agent 連續做出安全但低品質的決策——SafetyMonitor 不會觸發，但偏見仍在累積。此時只有 Sati 和 VitakkaWatchdog 能介入。」

三者互補。缺一不可。

ASANGA 做了佛學總結：「正念在十二因緣中的角色，是在受→愛之間插入覺察，打斷自動反應鏈。WIENER 的穩定性分析用數學語言表達了同一個佛學真理——正念是防止煩惱正反饋發散的穩定化機制。」

**D4-R2：三層穩定化機制——9/9。**

## V. 零階保持

D4-R3 處理的是一個微妙的技術問題——受蘊通過 EventBus 訂閱行蘊結果時，更新在何時生效？

Node.js 的 `EventEmitter.emit()` 是同步的。這意味著行蘊執行工具後發射 `TOOL_RESULT` 事件，VedanaSensor 的回調在同一個 JavaScript microtask 中完成。效價更新在同一輪內生效。

PENROSE 用 Pearl 因果 DAG 分析了這個問題。同輪內的多個工具執行形成線性鏈——tool_1_result → vedana_update → deliberate(tool_2)——這不是環路，DAG 語義保持。但問題不在因果方向，而在於是否「希望同輪內的後續工具審議感知到前面工具的影響」。

WIENER 把選項歸為三類：同步回饋（即時更新，正反饋螺旋風險）、零階保持（整輪不變，資訊延遲）、setImmediate 延遲（介於兩者之間，但依賴 Node.js 事件迴圈的微妙語義）。

ARCHIMEDES 給出了工程上最乾淨的方案：

「每輪開始時快照 vedana，整輪使用此快照。輪末統一收集所有 TOOL_RESULT 事件，批次更新 vedana。下一輪使用更新後的 vedana。」

零階保持加批次更新。

NAGARJUNA 確認了佛學時序：「十二因緣的時序是清晰的——本次行蘊的結果，成為下一次認知的觸的條件。行蘊結果在同一認知瞬間內更新受蘊在佛學中沒有依據。」

GUARDIAN 補充了安全角度：「即時止損由 SafetyMonitor 的 afterToolExecution() 檢查點負責，不依賴 vedana 更新。」

四維驗證：控制理論（穩定性最佳）、佛學（因果時序正確）、工程（實作最簡潔）、安全（SafetyMonitor 獨立負責即時保護）。

**D4-R3：零階保持 + 批次更新——9/9。**

## VI. 熟練導致無覺察

D4 的最後一項決議回答了 OQ-6——actionHistory 是否應影響 Moha？

ASANGA 的分析精巧而深刻。Drishti 是帶方向性的偏見——「我認為這樣做是對的」。行蘊重複模式直接強化 Drishti 的方向。但 Moha 是無方向的愚昧——「我不知道自己在做什麼」。

「重複成功的行為讓 Agent 不再思考為什麼這樣做——熟練導致無覺察，無覺察就是 Moha 的增強。」

NAGARJUNA 補充：「Moha 不僅是不知道自己不知道，更是對因果結構本身的忽視。當 Agent 的行為模式固化，它不再質疑為什麼這個模式有效——這正是無明的本質。」

PASCAL 設計了飽和機制——遞減邊際影響：

$$\Delta Moha = \frac{\alpha_m \cdot r}{1 + \beta_m \cdot M}$$

α_m = 0.02，β_m = 5.0。Moha 越高，行為重複對它的增強越弱。自然飽和。增益上限 0.3 被遵守。

GUARDIAN 從安全角度認可——Moha 增高提高 Klesha 聚合值，降低閾值，系統更傾向 Gear 2 深度思考。安全方向正確。但他也警告了資源耗盡攻擊——故意製造重複模式提高 Moha，迫使 Agent 頻繁走 Gear 2。

**D4-R4：actionHistory → Moha，遞減邊際飽和——9/9。**

## VII. 散場

D4 結束的時候，白板上有四項決議和四項共識。但場內的學者們帶走的不只是決議。

他們帶走了一個方程。

WIENER 的穩定性條件——β_d · s_min + γ_d > α₁ · g_max——是六場辯論中最精煉的一行數學。它把佛學的「正念打斷煩惱」、控制理論的「穩定化機制」、和工程的「三層防禦」壓縮成一個不等式。

ASANGA 的「自纏其身」是這個不等式的佛學翻譯。GUARDIAN 的三檢查點是它的工程實例。PASCAL 的 0.3 增益上限是它的 Kalman 證明。

---

> *SCRIBE 旁白：D4 是六場辯論中「數學密度」最高的一場——Lyapunov 函數、Kalman gain、非線性增益調度、零階保持。但所有的數學都指向同一個人類可理解的結論：如果你讓正念的力量大於偏見的力量，系統就不會失控。*

> *TURING 的修正——ISensation.ingestToolResult() 不存在——是 D4 的第一個事件，也是最重要的。它把整場辯論從「約束已有機制」拉到了「設計新機制」。更重要的是，它建立了一個規則：事實先於理論。在開始討論因果流向之前，先確認因果的載體是否存在。*

> *虛構的程式碼比虛構的概念更危險。概念可以在辯論中被修正。程式碼——如果沒有人去驗證——會被當作事實，然後在事實之上建立理論，理論之上建立決議，決議之上建立工程計畫。TURING 在 D4 開場的修正，打斷了這條鏈的第一個環節。*

> *這是「踏實優先於速度」的一個具體實例。*
