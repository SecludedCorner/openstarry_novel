## I. 暴風雨前の静けさ

D5 冒頭の六分間――四項目の合意、すべて 20/20。

四煩悩(kleśa)構成論：我執は四煩悩の「産物」ではなく、四煩悩の「マクロ的表述」である。温度と分子運動の関係。水の温度は水分子とは別の何かではない――分子運動のマクロ的計測値そのものだ。同様に、我執は四煩悩（我癡、我見、我慢、我愛）とは別の第五の状態変数ではない――四煩悩の同時分布のマクロ的表述そのものだ。EgoFramework は独立した ego 状態変数を必要としない。

Model Delta の Layer 0（安全フロア）と Layer 1（Klesha ゲイン）は直接実装可能――その数学的定義はすでに十分明確だ。しかし WIENER は重要な制約を提示した：四煩悩の同時分布は分解不能である――それらの間には結合があり、独立して校正できない。我癡の増加は我見の安定性に影響し、我慢の変動は我愛の減衰率に影響する。これは四煩悩のパラメータ校正を個別に行えないことを意味する――同時校正でなければならない。

PASCAL はこの六分間にさりげないことを行った：ホワイトボードに図を描き、四煩悩の同時分布を四次元空間の超曲面として描いた。我癡（Moha）、我見（Drishti）、我慢（Mana）、我愛（Sneha）――四つの軸、各軸の値が他の三つに影響する。「もしそれらが分解不能ならば」と PASCAL は言った、「校正する唯一の方法は同時校正だ。」WIENER は頷いた――これは彼が D6 で結合校正プロトコルを提案する種だった。

六分間の地ならし。四項目の合意の迅速な通過は一つのことを物語っている：前四場の討論（D1-D4）の研磨を経て、チームは基礎概念についてすでに十分な合意を持っていた。D5 の争点は基礎にではなく――応用にあった。

戦場はこの先にあった。

## II. IKlesha extends IVijnana

BABBAGE の型分析が再び決定的武器となった。

三つの方案の精密な解剖：方案 A（独立インターフェース）――PluginManifest.skandha が空配列を許容し、コンパイル時保証なし。方案 C（中間層 IKleshaBase）――DC-6「蘊内サブインターフェースの帰属を固定しない」に違反。方案 B（IKlesha extends IVijnana）――読み取り専用判別子フィールド `readonly skandha: 'vijnana'` で構造的保証を提供。

KERNEL が凍結契約を付加：IVijnana に振る舞いメソッドを追加してはならない。`@sealed` アノテーション――言語機能ではなく、チーム合意。意味するところは：IVijnana は安定した基盤であり、煩悩はその上に立てるが、基盤が煩悩の要求に応じて形を変えることはない。

18/20。二票棄権。

ASANGA はこの決議の仏教的意義を補足した：IKlesha extends IVijnana は煩悩が識蘊に帰属することを意味する――これは唯識学の観点と一致する。四煩悩（我癡、我見、我慢、我愛）は恒に末那識(manas)と倶行し、末那識は識蘊の一部である。煩悩は独立した心理的実体ではなく、識の一つの染汚パターンである。型の継承関係がコードレベルでこの仏教的観点を表現している。

## III. 主戦場：グローバル上限 vs リスク加重

GUARDIAN はグローバル信頼度上限 0.85 を提案した。彼の論点は率直だった：安全は状況理解に依存すべきではない、なぜなら状況理解自体が操作されうるからだ。システムの信頼度がどれほど大きくても、信頼度は 0.85 を超えない――常に Gear 2 の深度審議を経る。

ATHENA が反論した：グローバル上限は Gear 1 の校正性を抹消する。「システムは答えが正しいことを明らかに知っている――二足す二は四――なのに LLM を呼び出して確認することを強制される。これは安全ではない――浪費だ。不必要な LLM 呼び出しにはレイテンシコスト、金銭コスト、注意コストがある。」

PASCAL の一回目の介入――転換点。

「グローバル上限は [0.85, 1.0] 区間の情報路容量をゼロに圧縮する。Bayesian 校正性の観点から、システムは『86% の信頼度がある』と『99% の信頼度がある』を区別できなくなる。」

情報理論の精確な言語。抽象的な哲学的論争を定量化可能な数学的フレームワークに固定した。PASCAL は「グローバル上限は良くない」とは言わなかった――「グローバル上限は測定可能な情報を破壊する」と言った。

GUARDIAN は第三ラウンドで立場を調整した――グローバル上限から MAX_GEAR1_CONFIDENCE の設定可能パラメータ（デフォルト 0.95）に後退。「コアメカニズム」から「デプロイ時セーフティネット」へ。

この調整は GUARDIAN の戦略的明敏さを示した。グローバル上限 0.85 をコアメカニズムとすれば強い反対に遭う――アーキテクチャ設計に影響する。しかし MAX_GEAR1_CONFIDENCE を設定可能パラメータとすれば拒否はほぼ不可能だ――アーキテクチャを変えず、デプロイ時に追加の安全弁を提供するだけだ。あるデプロイ環境の管理者が 0.85 がより安全だと考えれば、0.85 に設定できる。別の環境の管理者が Gear 1 の校正性を信頼すれば、デフォルトの 0.95 を維持できる。

GUARDIAN の核心的訴求――「システムには信頼度の上限があるべきだ」――は異なる形式で完全に保持された。

リスク加重閾値 16/20。WIENER の重要な制約：Δ_risk は静的関数でなければならない――その値は行動タイプにのみ依存し、システムの現在の閾値には依存しない。もし Δ_risk が現在の閾値を読み取ってそれに応じて自身を調整するなら、フィードバックループが形成され、リミットサイクル発振を生じうる――閾値が二つの値の間を往復し、永遠に安定しない。静的関数はこのフィードバック経路を断ち切る。

四段階リスク：

| リスク等級 | Δ_risk | 説明 |
|---------|--------|------|
| destructive | +0.15 | 破壊的操作 |
| stateModifying | +0.10 | 状態変更 |
| readOnly | +0.00 | 読み取り専用操作 |
| informational | -0.05 | 情報照会 |

## IV. 最も争われた投票

IPrajna LLM の閾値調節参加。14/20。六票反対。Cycle 02-4 で最も争われた決議。

GUARDIAN の論点は第三節よりさらに鋭かった。彼は具体的な攻撃シナリオを展開した：「悪意のあるユーザーが巧妙に構成した prompt により、IPrajna に現在のタスクが低リスクだと認識させたと仮定する。IPrajna が閾値を 0.05 引き下げる。informational の Δ_risk = -0.05 を加えると、閾値は 0.10 低下する。Gear 1 がより低い信頼度で起動可能になる――LLM の深度審議をバイパスする。」

彼の結論：「LLM は閾値調節の位置において Sati と IVolition のガードレールを欠いている。Layer 2 の LLM 呼び出しは裸の API call だ。」

ATHENA の反論は核心を突いた：「LLM の安全意思決定への参加を禁止することは自己矛盾だ――Gear 2 全体が LLM に依存している。もし LLM の判断力を信頼しないなら、Gear 2 を使うべきではない。しかし Gear 2 はシステムのセーフティネットだ。『LLM はセーフティネットだ』と言いながら同時に『LLM は信頼できない』と言うことはできない。」

PASCAL の二回目の介入――損害上限分析。worst case：IPrajna が完全に支配され、閾値が 0.05 低下、destructive の Δ_risk = +0.15 を加えても、最終的になお 70% の信頼度が必要。安全マージンは十分。

「しかしこれは限幅が十分に小さいからにすぎない。±0.05 の損害上限は無視可能。ハードリミッター。設定変更不可。」

損害上限計算の完全な推論：

| 条件 | 閾値変化 |
|------|---------|
| θ_base | 0.60（典型値）|
| Δ_prajna（worst case）| -0.05 |
| Δ_risk（destructive）| +0.15 |
| θ_final | 0.60 - 0.05 + 0.15 = 0.70 |

IPrajna が完全に操作されたとしても、最終閾値はなお 0.70――システムは Gear 1 を使うのに 70% の信頼度を必要とする。安全マージンは十分。

PASCAL は GUARDIAN 側にも ATHENA 側にも立たなかった。数学の側に立った。損害上限分析は「LLM は参加すべきか」という問いに答えない――「もし LLM が参加したら、最悪の場合はどれほど悪いか」に答える。答え：それほど悪くない。限幅の範囲内。

NAGARJUNA は投票終了後に語った：「世俗諦は状況に応じた安全。勝義諦は妥協不可能な安全。GUARDIAN は勝義諦を守っている――彼は正しい。ATHENA は世俗諦を守っている――彼女も正しい。二諦は共存しなければならない。」

PENROSE はより大きな構造を予見した：「NAGARJUNA の二諦のフレームワークがある創発構造を生み出しつつある。世俗諦の状況的安全と勝義諦の妥協不可能な安全――対立ではなく、相補的だ。次に何が起こるか、私には予想がつく。」彼の予想は正しかった――D5 の最後の二十分間、三層安全フレームワークが討論から自然に創発した。

## V. 五層 Model Delta

PASCAL がホワイトボードに公式を書いた：

```
θ_final = clamp(
  θ_base + Δ_klesha + Δ_prajna + Δ_sati + Δ_vedana_emergency + Δ_risk,
  0.30, 0.90
)
```

五層：
- **Layer 0**：安全フロア [0.30, 0.90]――妥協不可能
- **Layer 1**：Klesha ゲイン――四煩悩の同時的影響
- **Layer 2**：Prajna ±0.05――LLM 微調整、ハードリミッター
- **Layer 3**：Sati ±0.05――正念品質補正
- **Layer 4**：Vedana Emergency [0, +0.15]――閾値を上げることのみ可能、下げることは不可（緊急ブレーキ）

WIENER の累積偏移分析：

| 方向 | 最大偏移 | 出典 |
|------|---------|------|
| 引下げ | -0.10 | Klesha(-) + Prajna(-0.05) + Sati(-0.05) |
| 引上げ | +0.25 | Klesha(+) + Prajna(+0.05) + Sati(+0.05) + Vedana Emergency(+0.15) |

[0.30, 0.90] の clamp 内で安全。すべての層が同時に極端値に達しても、最終結果はなお安全範囲内にある。Layer 4（Vedana Emergency）は非対称的――閾値を上げることのみ可能で、下げることはできない。これは受蘊の緊急シグナルがシステムを「より慎重に」できるだけで、「より大胆に」はできないことを意味する。20/20。

## VI. Beta(2,18) のピーク

ASANGA が『成唯識論』を引用した――四煩悩は恒に末那と倶行し、sneha はゼロになるべきではない。

PASCAL は Beta 分布を使って自身の R08 の提案を否定した――研究者が自らのツールで自らの結論を覆した。これは学術的誠実さの範例である。

floor = 0.05 の場合、対応する Beta(1,19)。Beta 分布の mode = (α-1)/(α+β-2)。α=1 の場合、mode = 0――分布は低いほど良いと判断し、sneha をゼロに向けて駆動する。これは ASANGA の仏教的制約に直接違反する：四煩悩は恒に末那と倶行し、ゼロになるべきではない。

floor = 0.10 の場合、対応する Beta(2,18)。mode = (2-1)/(2+18-2) = 1/18 ≈ 0.056。分布には有意味なピークがある――sneha の最も可能性の高い値は正の数であり、ゼロではないと「判断する」。仏教的制約と統計モデルが同一の結論で合流した。

四煩悩統一 floor = 0.10、maxLevel = 0.95。18/20。Sneha の完全なパラメータセット：

| パラメータ | 値 | 意味 |
|------|-----|------|
| gain | 0.10 | 各正の相互作用の増分 |
| floor | 0.10 | 最低値（ゼロにならない）|
| maxLevel | 0.95 | 最高値 |
| λ | 0.05 | 指数減衰率 |

三つのプリセットプロファイルが異なるユースケースに対応――高依存、中等、低依存。λ = 0.05 は約 14 時間単位後に半減することを意味する――安定したユーザー嗜好を維持するのに十分遅く、変化に適応するのに十分速い。

WIENER の結合校正プロトコルは Plan28 の研究項目として記録――四煩悩は独立して校正不可能、なぜならそれらの同時分布は分解不能だから。20/20。

## VII. 三層安全フレームワークの創発

D5 の最後――GUARDIAN は彼が認めうる全体アーキテクチャを提示した：

**Absolute Safety**――SafetyMonitor、Klesha 免疫、theta clamp。ゼロの可調パラメータ。物理法則。

**Tunable Safety**――リスク加重、MAX_GEAR1_CONFIDENCE、IPrajna リミッター、Sati Quality。各項に明確なガードレール。

**Reduce Complexity**――安全劇場の除去。実質的な保護を提供しない虚偽の安全を見抜く。

NAGARJUNA は即座に仏教的マッピングを見出した：

| 安全層 | 仏教 | 特性 |
|--------|------|------|
| Absolute Safety | 戒 (sila) | 不可侵犯 |
| Tunable Safety | 定 (samadhi) | 規律ある柔軟性 |
| Reduce Complexity | 慧 (prajna) | 虚妄を見抜く |

三学。戒定慧。二千五百年前のフレームワークが、情報セキュリティエンジニアと確率哲学者の討論の中で自然に創発した。NAGARJUNA は D5 の冒頭で三学フレームワークを提出したのではない――GUARDIAN の全体アーキテクチャを見た後に、三学のマッピングを発見したのだ。これは先験的設計ではない――事後に発見された同型性だ。二千五百年前の認知フレームワークと二十一世紀の安全工学フレームワークが、構造的に等価である。

PENROSE：「これは先験的に設計されたものではない。討論から自然に創発した。衝突なくして統合はない。」

17/20。GUARDIAN は三項目の留保意見を記録した――敗者の不満ではなく、道標として：

1. **IPrajna リミッターの長期的十分性**：±0.05 は現在の Model Delta において安全だが、将来さらに多くの調節層が追加されれば、累積偏移が結論を変えうる。アーキテクチャ変更のたびに再検証が必要。
2. **Klesha 結合校正のタイミング**：四煩悩の独立校正不可能はすでに確認されたが、同時校正の具体的方法はまだ設計されていない。Plan28 の研究議題。
3. **VasanaEngine 規則注入の攻撃面**：三ゲートの保護があっても、巧妙に構成された規則シーケンスは統計的に無害に見えながら、組み合わせ効果は有害でありうる。D6 でのさらなる処理が必要。

これら三項目の留保意見は設計の既知の脆弱点を精確に標識した――GUARDIAN の価値は彼の反対だけでなく、彼の反対がいかにしてシステムの防御マップとなるかにある。
