## I. TURING 的修正

D4 以一條修正開場。

TURING 站起來：「R07 報告在多處引用了 `ISensation.ingestToolResult()` 方法。這個方法在 v0.26.0-beta 整個程式碼基線中完全不存在。全域搜尋零結果。」

三個虛構——ISensation、feedbackQueue、SparshEvent.timestamp。一份報告。HERACLITUS 沒有爭辯——TURING 的全域搜尋是不可反駁的。

這次修正改變了 D4 的基調。如果行蘊可以在同輪內即時更新受蘊（如 R07 所述），穩定性分析是一回事。如果行蘊只能在下一輪才影響受蘊（實際狀況），穩定性分析是完全不同的另一回事。回饋延遲從零變成一個完整迴圈——這是控制理論中足以改變系統穩定性結論的差異。

HERACLITUS 的反應令人印象深刻——沒有辯解，沒有「我的分析仍然有效」的辯護。他立刻接受了修正，重新調整了行蘊的影響模型。學術誠實不僅是承認錯誤——是在錯誤被指出的三秒鐘內就調整方向。

教訓被寫入記錄：**所有涉及程式碼引用的分析必須經過 TURING 驗證。** 這條規則從 D4 開始生效，在 D5 和 D6 中被嚴格執行——每當有人引用程式碼，TURING 都會同步進行全域搜尋確認。

## II. 行蘊的邊界

四項快速共識確立了行蘊的嚴格限制：

1. 行蘊只能直接影響 IUI（使用者介面蘊）——所有對其他蘊的影響必須透過 EventBus 間接發生。
2. 行蘊不直接修改識蘊或想蘊的狀態。
3. 意行通過 EventBus 跨迴圈影響受蘊。
4. 行蘊結果觸發新觸（意門自觸）延後到 Phase 2。

這些限制構成了行蘊的「不可為」清單。行蘊是手——它可以推動外界（IUI），但不能直接伸進大腦（識蘊、想蘊）去改變思考方式。要改變思考方式，它必須先改變外部狀態，讓外部狀態在下一輪成為新的輸入。

LEIBNIZ 從多代理合作的角度確認了這個限制的必要性：如果行蘊能直接修改其他蘊的狀態，就等於一個模組可以繞過介面直接修改另一個模組的內部狀態。在分散式系統中，這是導致競爭條件和不可重現 bug 的主要原因。

NAGARJUNA 的佛學確認更為根本：行蘊直接修改識蘊或想蘊，等於「果直接修改因」。在緣起觀中，因果鏈必須經過中介——行動改變環境，環境在下一剎那成為新的緣起條件。跳過中介就跳過了因果。

這四項約束在 D4 後續的所有討論中被頻繁引用——它們是行蘊模型的基本定理，所有後續推導都建立在這個基礎上。ARCHIMEDES 把它們稱為「行蘊的四條公理」——不需要證明，也不能違反的基本規則。

值得注意的是第四項約束——意門自觸的延後。行蘊的結果可以觸發新的觸（sparsha），就像一個動作的後果可以成為新的感知對象。但這個機制涉及因果循環：行蘊 → 新觸 → 新受 → 新想 → 新行蘊。在 Phase 1 的線性管道中實作這種循環需要額外的設計，因此被延後到 Phase 2。

## III. 增益上限與穩定性

D4-R1：samskara → vedana 回饋增益上限 w ≤ 0.3。

PASCAL 提供了 Bayesian 依據。Kalman 增益的推導：在離散時間 Kalman 濾波器中，增益 K = P·H'·(H·P·H' + R)⁻¹。當過程噪聲 Q 遠大於觀測噪聲 R（高不確定性/高觀測品質），增益趨近於上界。PASCAL 在 OpenStarry 的典型操作條件下估算了這個上界：K₁ ≈ 0.308。

工程上取 0.3 作為保守值——略低於理論上界，留出安全邊際。8/9（ARCHIMEDES 棄權——他認為具體數值應按部署環境差異化配置，0.3 作為預設值沒問題，但不應被硬編碼）。

D4-R2：三層穩定化機制——本輪最深入的跨學科成果之一。

| 層 | 機制 | 佛學映射 | 特性 |
|----|------|---------|------|
| Layer 1 | Sati 精調（每輪微調）| 正知 (sampajanna) | 連續、小幅 |
| Layer 2 | VitakkaWatchdog 粗反饋（跨輪趨勢）| 尋 (vitakka) 觀照 | 間歇、中幅 |
| Layer 3 | SafetyMonitor 硬約束（即時觸發）| 戒 (sila) 護欄 | 離散、大幅 |

WIENER 的 Lyapunov 穩定性分析提供了數學保證。定義 Lyapunov 函數 V = Σ(x_i - x_i*)²，其中 x_i 是各蘊的狀態，x_i* 是期望的平衡點。穩定性條件：

```
k_positive_feedback × w_samskara < k_stabilization × sati_min
```

其中 k_positive_feedback 是正反饋增益（行蘊重複模式的強化效果），w_samskara 是行蘊權重（≤ 0.3），k_stabilization 是穩定化增益（三層機制的組合效果），sati_min 是最低正念水平。

具體地說，如果一個行動產生正面回饋、使得系統更傾向重複該行動，三層穩定化機制確保這個正反饋環不會失控：
- Layer 1（Sati）持續微調——每輪小幅修正
- Layer 2（VitakkaWatchdog）監控趨勢——偏差累積時中幅介入
- Layer 3（SafetyMonitor）硬切——一切失敗時大幅修正

只要三層機制的組合效果足以抵消正反饋的最大增益，系統就是全域漸近穩定的。9/9。

## IV. 零階保持

D4-R3：受蘊在迴圈開始時快照（Zero-Order Hold），迴圈結束時批次更新。

這是 TURING 修正的直接後果——既然行蘊不能在同輪內即時更新受蘊，那就明確規定：受蘊在每個迴圈的開始時做一次快照，整個迴圈中所有蘊都使用這個快照，迴圈結束時再把累積的變化一次性寫入。

四維驗證使這項決議成為跨學科共識的典範：

- **控制理論**（WIENER）：穩定性最佳——消除了迴圈內的即時反饋路徑。在離散控制系統中，零階保持是標準方法。
- **佛學**（NAGARJUNA）：因果時序正確——受→想→行的因果鏈不被中途修改干擾。一個剎那的受蘊決定了這個剎那的想蘊和行蘊——不能因為行蘊的結果反過來修改同一剎那的受蘊。
- **工程**（ARCHIMEDES）：實作最簡——無須處理迴圈內的競爭條件。快照一次，使用一輪，寫入一次。
- **安全**（GUARDIAN）：SafetyMonitor 獨立負責即時保護——它不受零階保持的約束。如果出現真正的緊急情況，SafetyMonitor 可以在任何時刻介入，不必等待迴圈結束。

四個學科。四個獨立的論證。同一個結論。9/9。

## V. 蠶蟲自纏

D4-R4 回答了 OQ-6：行蘊重複模式增強 Moha（我癡），採用遞減邊際飽和模型。

ASANGA 引用了唯識學的意象：「如蠶作繭，自纏其身」——出自《成唯識論》。蠶蟲吐絲結繭，把自己封在裡面。每一根絲都很細，但累積起來就是一個繭。行蘊的重複模式就像吐出的絲——一個 Agent 如果反覆使用同一種策略，這個策略就會逐漸成為它的「繭」。它會越來越傾向於使用這個策略，越來越難以嘗試新的策略。

DARWIN 從軟體模式的角度確認了這個現象的普遍性：「這就是路徑依賴（path dependency）。在軟體系統中，早期的架構決策會限制後期的可能性。在認知系統中，早期的行為模式會限制後期的行為空間。遞減邊際模型正確地捕捉了這個特性。」

數學上：ΔMoha = α_m · r / (1 + β_m · M)。

公式的意義：r 是重複次數，M 是當前 Moha 水平。分子 α_m · r 隨重複線性增長，但分母 1 + β_m · M 也在增長，使得增量越來越小。早期重複影響大——第一次重複一個模式，Moha 增加最多。但隨著 Moha 累積，新的重複帶來的增量遞減。就像一個已經很深的習慣——再重複一次帶來的固化效果，比前幾次要小得多。

WIENER 確認這個模型的穩定性：遞減邊際保證了 Moha 有自然的飽和上限——它不會無限增長。安全上限：actionHistory 貢獻 ≤ 30%，確保行蘊重複模式不能主宰整個 Moha 的值。9/9。

## VI. D4 的跨學科密度

D4 是數學密度最高的一場辯論。四個核心概念——Kalman 增益（PASCAL）、Lyapunov 函數（WIENER）、零階保持（ARCHIMEDES/WIENER）、遞減邊際（PASCAL/ASANGA）——來自四個不同的學科，卻在同一個問題上匯合：行蘊如何在不破壞系統穩定性的前提下影響其他蘊？

答案不是一個公式，而是一套互鎖的約束：
- 增益上限（w ≤ 0.3）限制了單次影響的幅度
- 零階保持消除了迴圈內的即時反饋
- 三層穩定化確保了長期的全域漸近穩定
- 遞減邊際防止了 Moha 的無限累積

四個約束。四個學科。一個穩定的系統。

D4 的另一個隱含貢獻：它建立了 TURING 在辯論中的「事實核查」角色。從 D4 開始，每當有人引用程式碼，TURING 會同步進行全域搜尋。這個非正式的制度在 D5 和 D6 中被嚴格執行，確保了後續辯論的所有程式碼引用都有事實基礎。TURING 在 D6 中發現的三項工程備忘（injectPrompt 安全缺陷、VedanaRegistry 重複檢查缺失、isSahajaValid 死代碼）都是這個角色的產物。

> *SCRIBE 旁白：D4 的數學密度是六場辯論中最高的——Kalman 增益、Lyapunov 函數、零階保持、遞減邊際。但最深刻的時刻不是數學，而是 TURING 的三秒沉默——那三秒，整場辯論的基礎被從虛構的程式碼上移到了真實的程式碼上。事實先於理論。這個教訓在 D5 和 D6 中持續迴響——每當有人引用程式碼，TURING 都會默默打開搜尋窗口。*
