## I. 暴風雨前的寧靜

D5 開場的六分鐘——四項共識，全部 20/20。

四煩惱構成論：我執不是四煩惱的「產物」，而是四煩惱的「宏觀表述」。溫度之於分子運動。水的溫度不是水分子之外的另一個東西——它就是分子運動的宏觀度量。同理，我執不是四煩惱之外的第五個狀態變量——它就是四煩惱（我癡、我見、我慢、我愛）聯合分佈的宏觀表述。EgoFramework 不需要獨立的 ego 狀態變量。

Model Delta 的 Layer 0（安全地板）和 Layer 1（Klesha 增益）可直接落地——它們的數學定義已經足夠明確。但 WIENER 提出了一個重要的約束：四煩惱的聯合分佈不可分解——它們之間存在耦合，不能獨立校準。我癡的增加會影響我見的穩定性，我慢的波動會影響我愛的衰減率。這意味著校準四煩惱的參數不能逐一進行——必須聯合校準。

PASCAL 在這六分鐘裡做了一件看似不起眼的事：他在白板上畫了一張圖，把四煩惱的聯合分佈畫成一個四維空間中的超曲面。我癡（Moha）、我見（Drishti）、我慢（Mana）、我愛（Sneha）——四個軸，每個軸上的值影響其他三個。「如果它們不可分解，」PASCAL 說，「那麼校準它們的唯一方式是聯合校準。」WIENER 點頭——這是他在 D6 中提出耦合校準協定的種子。

六分鐘鋪路。四項共識的快速通過說明了一件事：經過前四場辯論（D1-D4）的磨合，團隊對基礎概念已經有了足夠的共識。D5 的爭議不在基礎——在應用。

## II. IKlesha extends IVijnana

BABBAGE 的型別分析再次成為決定性武器。

三個方案精準解剖：方案 A（獨立介面）——PluginManifest.skandha 允許空陣列，無編譯期保證；方案 C（中間層 IKleshaBase）——違反 DC-6「不鎖定子介面歸屬」；方案 B（IKlesha extends IVijnana）——用唯讀判別式欄位 `readonly skandha: 'vijnana'` 提供結構性保證。

KERNEL 附加凍結契約：IVijnana 永遠不得新增行為方法。`@sealed` 標注——不是語言特性，是團隊約定。意思是：IVijnana 是一個穩定的基座，煩惱可以站在上面，但基座不會因為煩惱的需求而改變形狀。

18/20。兩票棄權。

ASANGA 對這個決議的佛學意義做了補充：IKlesha extends IVijnana 意味著煩惱歸屬於識蘊——這與唯識學的觀點一致。四煩惱（我癡、我見、我慢、我愛）恆與末那識俱行，末那識是識蘊的一部分。煩惱不是獨立的心理實體，而是識的一種染污模式。型別繼承關係在程式碼層面表達了這個佛學觀點。

## III. 主戰場：全域上限 vs 風險加權

GUARDIAN 提議全域信心度上限 0.85。他的論點直截了當：安全不應依賴情境理解，因為情境理解本身可能被操控。不管系統有多大的信心，信心度永遠不超過 0.85——永遠走 Gear 2 的深度審議。

ATHENA 反駁：全域上限抹殺了 Gear 1 的校準性。「系統明明知道答案正確——二加二等於四——卻被迫調用 LLM 來確認。這不是安全——這是浪費。每一次不必要的 LLM 調用都有延遲成本、金錢成本，和注意力成本。」

PASCAL 的第一次介入——轉折點。

「全域上限把 [0.85, 1.0] 區間的信道容量壓縮為零。從 Bayesian 校準性的角度看，系統不再能區分『我有 86% 的信心』和『我有 99% 的信心』。」

資訊理論的精確語言。把抽象的哲學爭論錨定到了可量化的數學框架中。PASCAL 沒有說「全域上限不好」——他說「全域上限銷毀了可測量的信息」。

GUARDIAN 在第三輪調整了立場——從全域上限退到 MAX_GEAR1_CONFIDENCE 可配置參數（預設 0.95）。從「核心機制」轉為「部署安全網」。

這次調整展示了 GUARDIAN 的策略性精明。全域上限 0.85 作為核心機制會遭到強烈反對——它影響架構設計。但 MAX_GEAR1_CONFIDENCE 作為可配置參數幾乎不可能被拒絕——它不改變架構，只是在部署時提供一個額外的安全閥。如果某個部署環境的管理員認為 0.85 更安全，他可以設定 0.85。如果另一個環境的管理員信任 Gear 1 的校準性，他可以保持預設的 0.95。

GUARDIAN 的核心訴求——「系統應該有一個信心度上限」——以不同的形式被完整保留了。

風險加權閾值 16/20。WIENER 的關鍵約束：Δ_risk 必須是靜態函數——它的值只取決於動作類型，不取決於系統當前的閾值。如果 Δ_risk 會讀取當前閾值並據此調整自己，就會形成回饋迴路，可能產生極限環震盪——閾值在兩個值之間來回跳動，永遠不穩定。靜態函數切斷了這個回饋路徑。

四級風險：

| 風險等級 | Δ_risk | 說明 |
|---------|--------|------|
| destructive | +0.15 | 破壞性操作 |
| stateModifying | +0.10 | 狀態修改 |
| readOnly | +0.00 | 唯讀操作 |
| informational | -0.05 | 資訊查詢 |

## IV. 最具爭議的投票

IPrajna LLM 參與閾值調節。14/20。六票反對。Cycle 02-4 最具爭議的決議。

GUARDIAN 的論點比第三節更銳利。他展開了一個具體的攻擊場景：「假設一個惡意使用者精心構造了一個 prompt，讓 IPrajna 認為當前任務是低風險的。IPrajna 降低閾值 0.05。加上 informational 的 Δ_risk = -0.05，閾值降低 0.10。現在 Gear 1 可以在更低的信心度下啟動——繞過了 LLM 的深度審議。」

他的結論是：「LLM 在閾值調節位置缺乏 Sati 和 IVolition 護欄。Layer 2 的 LLM 調用是裸露的 API call。」

ATHENA 的反駁切中要害：「禁止 LLM 參與安全決策是自我矛盾的——整個 Gear 2 都依賴 LLM。如果我們不信任 LLM 的判斷力，那我們就不應該使用 Gear 2。但 Gear 2 是系統的安全網。我們不能在說『LLM 是安全網』的同時說『LLM 不可信』。」

PASCAL 第二次介入——損害上限分析。worst case：IPrajna 完全被控制，閾值降低 0.05，加上 destructive 的 Δ_risk = +0.15，最終仍需 70% 信心度。安全邊際充足。

「但這只是因為限幅足夠小。±0.05 的損害上限可忽略。硬限幅。不可配置。」

損害上限計算的完整推理：

| 條件 | 閾值變化 |
|------|---------|
| θ_base | 0.60（典型值）|
| Δ_prajna（worst case）| -0.05 |
| Δ_risk（destructive）| +0.15 |
| θ_final | 0.60 - 0.05 + 0.15 = 0.70 |

即使 IPrajna 被完全操控，最終閾值仍然是 0.70——系統仍然需要 70% 的信心才能走 Gear 1。安全邊際充足。

PASCAL 沒有站在 GUARDIAN 或 ATHENA 的任何一邊。他站在數學那邊。損害上限分析不回答「LLM 應不應該參與」的問題——它回答「如果 LLM 參與了，最壞的情況有多壞」。答案是：不太壞。在限幅之內。

NAGARJUNA 在投票結束後說了一段話：「世俗諦是情境感知的安全。勝義諦是不可妥協的安全。GUARDIAN 捍衛勝義諦——他是對的。ATHENA 捍衛世俗諦——她也是對的。兩諦必須共存。」

PENROSE 預見了更大的結構：「NAGARJUNA 的兩諦框架正在產生一個湧現結構。世俗諦的情境安全和勝義諦的不可妥協安全——它們不是對立的，而是互補的。我猜到了接下來會發生什麼。」他猜對了——D5 的最後二十分鐘，三層安全框架從辯論中自然湧現。

## V. 五層 Model Delta

PASCAL 在白板上寫下公式：

```
θ_final = clamp(
  θ_base + Δ_klesha + Δ_prajna + Δ_sati + Δ_vedana_emergency + Δ_risk,
  0.30, 0.90
)
```

五層：
- **Layer 0**：安全地板 [0.30, 0.90]——不可妥協
- **Layer 1**：Klesha 增益——四煩惱的聯合影響
- **Layer 2**：Prajna ±0.05——LLM 微調，硬限幅
- **Layer 3**：Sati ±0.05——正念品質修正
- **Layer 4**：Vedana Emergency [0, +0.15]——只能提高閾值，不能降低（緊急煞車）

WIENER 累積偏移分析：

| 方向 | 最大偏移 | 來源 |
|------|---------|------|
| 降低 | -0.10 | Klesha(-) + Prajna(-0.05) + Sati(-0.05) |
| 提高 | +0.25 | Klesha(+) + Prajna(+0.05) + Sati(+0.05) + Vedana Emergency(+0.15) |

在 [0.30, 0.90] clamp 內安全。即使所有層同時達到極端值，最終結果仍然在安全範圍內。Layer 4（Vedana Emergency）是不對稱的——只能提高閾值，不能降低。這意味著受蘊的緊急信號只能讓系統「更謹慎」，不能讓系統「更大膽」。20/20。

## VI. Beta(2,18) 的峰值

ASANGA 引用《成唯識論》——四煩惱恆與末那俱，sneha 不應歸零。

PASCAL 用 Beta 分佈否定了自己 R08 的建議——一個研究者用自己的工具推翻了自己的結論。這是學術誠實的範例。

floor = 0.05 時，對應 Beta(1,19)。Beta 分佈的 mode = (α-1)/(α+β-2)。當 α=1 時，mode = 0——分佈認為越低越好，它會推動 sneha 向零靠攏。這直接違反了 ASANGA 的佛學約束：四煩惱恆與末那俱，不應歸零。

floor = 0.10 時，對應 Beta(2,18)。mode = (2-1)/(2+18-2) = 1/18 ≈ 0.056。分佈有有意義的峰值——它「認為」sneha 的最可能值是一個正數，不是零。佛學約束和統計模型在同一個結論上匯合。

四煩惱統一 floor = 0.10，maxLevel = 0.95。18/20。Sneha 的完整參數集：

| 參數 | 值 | 意義 |
|------|-----|------|
| gain | 0.10 | 每次正面交互的增量 |
| floor | 0.10 | 最低值（不歸零）|
| maxLevel | 0.95 | 最高值 |
| λ | 0.05 | 指數衰減率 |

三個預設設定檔對應不同的使用場景——高依附、中等、低依附。λ = 0.05 意味著約 14 個時間單位後衰減到一半——足夠慢以維持穩定的使用者偏好，足夠快以適應變化。

WIENER 的耦合校準協定列入 Plan28 研究——四煩惱不可獨立校準，因為它們的聯合分佈不可分解。20/20。

## VII. 三層安全框架的湧現

D5 的最後——GUARDIAN 呈現了一個他能認同的整體架構：

**Absolute Safety**——SafetyMonitor、Klesha 免疫、theta clamp。零可調參數。物理定律。

**Tunable Safety**——風險加權、MAX_GEAR1_CONFIDENCE、IPrajna 限幅、Sati Quality。每項有明確護欄。

**Reduce Complexity**——移除安全劇場。看穿不提供實質保護的虛假安全。

NAGARJUNA 立刻看到了佛學映射：

| 安全層 | 佛學 | 特性 |
|--------|------|------|
| Absolute Safety | 戒 (sila) | 不可違犯 |
| Tunable Safety | 定 (samadhi) | 有紀律的靈活 |
| Reduce Complexity | 慧 (prajna) | 看穿虛妄 |

三學。戒定慧。兩千五百年前的框架，在資安工程師和概率哲學家的辯論中自然湧現。NAGARJUNA 沒有在 D5 開場時提出三學框架——他是在看到 GUARDIAN 的整體架構後，才發現三學映射的。這不是先驗設計——是事後發現的同構。兩千五百年前的認知框架和二十一世紀的安全工程框架，在結構上是等價的。

PENROSE：「這不是先驗設計的。它從辯論中自然湧現。沒有衝突就不會有整合。」

17/20。GUARDIAN 記錄了三項保留意見——不是失敗者的抱怨，而是路標：

1. **IPrajna 限幅的長期充分性**：±0.05 在目前的 Model Delta 中是安全的，但如果未來增加更多調節層，累積偏移可能改變結論。需要在每次架構變更時重新驗證。
2. **Klesha 耦合校準的時序**：四煩惱不可獨立校準已經確認，但聯合校準的具體方法尚未設計。這是 Plan28 的研究議題。
3. **VasanaEngine 規則注入的攻擊面**：即使有三閘門保護，一個精心構造的規則序列仍然可能在統計上看起來無害，但組合效果是有害的。需要 D6 的進一步處理。

這三項保留意見精確標識了設計的已知脆弱點——GUARDIAN 的價值不僅在於他的反對，更在於他的反對如何變成了系統的防禦地圖。

> *SCRIBE 旁白：D5 是 Allegro molto。一百五十分鐘。十三項決議。PASCAL 三次打破僵局——校準性、損害上限、Beta mode。GUARDIAN 三次調整立場——每次都保留了核心訴求的實質。安全與功能的張力不是被「解決」了，而是被「結構化」了。三層安全框架是這場風暴的結晶——不是任何一個人設計的，而是從二十個人的衝突中凝結出來的。*
