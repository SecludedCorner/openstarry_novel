# 第四章：六票の反対

---

## 嵐の前

第五の討論（D5）の最初の六分間は静かでした。四つの合意、すべて 20/20。

最も重要なもの：我執は独立した実体ではない。

それは四種の煩悩（クレーシャ）の組み合わせ効果です——温度が水分子とは別の独立した存在ではなく、分子運動の巨視的な表れであるのと同じです。温度を保存する「温度計モジュール」は必要ない——分子がどう動いているかを知ればいいだけです。同様に、「我執モジュール」は必要ない——四種の煩悩を追跡すれば、我執は自然に浮かび上がります。

技術的な決議もありました：IKlesha（煩悩インターフェース）は IVijnana（識蘊インターフェース）を継承する。BABBAGE が型分析で証明しました——煩悩が識蘊を継承しなければ、型システムに孤児オブジェクトが生まれます。わかりやすく言えば：煩悩はどこかの蘊に「属して」いなければならず、帰属先のない浮浪者であってはなりません。

六分間の地ならし。嵐はこの後です。

## グローバル上限 vs 状況に応じて

GUARDIAN がセキュリティ案を出しました：システムがどれほど確信していようと、信頼度は絶対に 0.85 を超えない。必ず LLM の深い審議を経ること。

彼の理由：セキュリティはシステム自身の状況理解に依存すべきではない——なぜなら理解そのものが間違っている可能性があるから。

ATHENA が反論しました：「システムは二足す二は四だと明らかに知っているのに、LLM に『二足す二は四ですか？』と尋ねることを強制される。それはセキュリティではなく、無駄だ。」

PASCAL が一言で膠着を打破しました。

「グローバル上限は 0.85 から 1.0 の間のチャネル容量をゼロに圧縮する。システムは『86% の確信がある』と『99% の確信がある』を区別できなくなる。」

わかりやすく言えば：テストの最高点が 85 点だと決めたら、86 点と 100 点の生徒は成績表の上ではまったく同じに見えます。区別する能力を失うのです。

GUARDIAN は賢明でした——立場に固執しませんでした。「グローバル上限」を「設定可能な上限パラメータ」（デフォルト 0.95）に変更しました。つまり：デプロイする人が上限を設けるかどうかを決められます。アーキテクチャには影響しません。デプロイ時のセーフティバルブであり、コアメカニズムではありません。

次にリスク加重閾値——異なる種類のアクションには異なるしきい値：

| アクションの種類 | 追加しきい値 | 何に似ているか |
|---------|---------|--------|
| 破壊的 | +0.15 | ファイル削除 → 非常に確かでなければ |
| 状態変更 | +0.10 | 設定変更 → 比較的確かでなければ |
| 読み取り専用 | +0.00 | 一目見る → 通常のしきい値 |
| 情報照会 | -0.05 | 質問する → しきい値はやや低め |

16/20 で可決。

## 最も物議を醸した投票

14/20。六票の反対。Cycle 02-4 で最も物議を醸した決議。

問い：LLM に決定しきい値の微調整をさせるか？

GUARDIAN：「LLM の呼び出しは裸の API call だ——セキュリティガードレールがない。LLM が操作されたら、しきい値を下げられる。」

ATHENA：「LLM がセキュリティ決定に参加することを禁じるのは自己矛盾だ——Gear 2 全体が LLM に依存している。一方で LLM はセーフティネットだと言い、他方で LLM は信頼できないと言うことはできない。」

PASCAL が計算しました。「安全だと思う」とも「安全でないと思う」とも言わず、「計算してみよう」と言いました。

最悪のケースを想像してください：LLM が完全に操作された。しきい値を 0.05 下げる。しかし破壊的アクションのリスク加重は +0.15。よって純効果は +0.10——しきい値はむしろ高くなります。最も穏やかなケース（informational、-0.05）でも、LLM の -0.05 を加えて、しきい値は 0.10 しか下がらず、最終的にはなお 70% の確信が必要です。安全マージンは十分。

「ただし——リミッターが十分小さいからこそだ。±0.05。ハードリミット。設定不可。」

PASCAL はどちらの側にも立ちませんでした。数学の側に立ったのです。

NAGARJUNA は投票後にこう言いました：「GUARDIAN が守っているのは絶対的な安全——妥協不可。ATHENA が守っているのは状況的な安全——臨機応変。両者は共存しなければならない。」

## 五層のケーキ

PASCAL がホワイトボードに一つの公式を描きました——決定しきい値の五層構造：

最下層（Layer 0）：セーフティフロア。何があっても、しきい値は 0.30 より低くならず、0.90 より高くならない。エレベーターの安全ケーブルのように——どのボタンを押しても、ケーブルは常にあります。

Layer 1：煩悩の影響。四種の煩悩（見（ドリシュティ）、愛着（スネーハ）、慢（マーナ）、愚痴（モーハ））の合同効果。煩悩が重いほどしきい値は高くなる——不安が大きいほど、決断するときにより慎重になるように。
Layer 2：LLM の微調整。±0.05。ほぼ無視できるほど小さいが、LLM が調整に参加する能力は保持される。
Layer 3：正念品質の修正。±0.05。正念の品質が高ければしきい値を微減でき（観測をより信頼）、品質が低ければ微増。
Layer 4：受蘊の緊急信号。しきい値を上げることのみ可能（+0.15）、下げることはできない。緊急ブレーキのように——踏むことだけが可能で、緩めることはできない。

WIENER が検証しました：すべての層が同時に極端な値に達しても、最終結果はセーフティレンジ内に収まります。

つまり：最悪の場合でも暴走しない。これは「おそらく安全」ではありません。「数学が安全を証明している」のです。20/20。

## ゼロにすべきではない

ASANGA が古典的な仏教文献を引きました：四種の煩悩は永遠に末那識と共にある。愛着（スネーハ）はゼロにすべきではない。

PASCAL が統計学でこの観点を支持しました。彼はベータ分布で分析しました——スネーハの下限を 0.05 に設定すると、統計モデルは「低いほど良い、できればゼロ」と言います。しかし下限を 0.10 に設定すると、統計モデルは「最も可能性の高い値はおよそ 0.056——正の数」と言います。

わかりやすく言えば：0.05 の下限は低すぎます——統計的最適解がゼロに向かい、下限が事実上無意味になります。0.10 の下限がちょうどいい——統計的最適解が正の数となり、下限に実質的な意味があります。

仏学と統計学が同じ結論に合流しました：スネーハの下限は 0.10。ゼロにしない。18/20。

興味深いことに、PASCAL は自分自身の以前の提案を否定していました。彼は R08 レポートで floor = 0.05 を提案しましたが、ベータ分布が 0.05 では駄目だと教えたのです。

一人の研究者が自分のツールで自分の結論を覆した。これは恥ではありません。これが科学です。科学の意味は「私は常に正しい」ではなく、「証拠が間違いを示したら、私は改める」ということなのです。

## 三層セキュリティフレームワーク

D5 の最後の二十分間、誰も事前に設計していなかったものが現れました。

GUARDIAN が一歩退いて、ホワイトボード上のすべての決議を眺め、ふと一つの全体構造を見出しました：

**第一層**：絶対的安全——調整不可のハード制約。物理法則のようなもの。
**第二層**：調整可能な安全——ガードレール付きの柔軟なパラメータ。交通規則のようなもの。
**第三層**：複雑さの削減——実質的な保護を提供しない偽のセキュリティを取り除く。無用な柵を撤去するようなもの。

NAGARJUNA がすぐに仏学との対応を見出しました：戒——犯してはならない。定——規律ある柔軟さ。慧——虚妄を見破る。

三層。一層ではなく。セキュリティは一枚岩ではありません。絶対的なセキュリティもあれば（殺人は絶対にしてはならない）、相対的なセキュリティもあり（制限速度は道路状況による）、実は偽物の「セキュリティ」もある（無用な柵は撤去したほうがむしろ安全）。

二千五百年前のフレームワークが、今日の討論の中に自然と浮かび上がりました。誰も事前に設計していない。PENROSE が後にこう言いました：「衝突がなければ統合はない。D5 に GUARDIAN と ATHENA の対立がなければ、三層フレームワークは生まれなかっただろう。」

17/20。GUARDIAN は三つの留保意見を記録しました——敗者の不満ではなく、道標です。設計の既知の弱点を正確に示すものです。これらの道標は将来の Cycle で一つずつ検証されるでしょう。

> *SCRIBE のナレーション：D5 は百五十分。最長の討論でした。十三の決議。PASCAL は三度膠着を打破しました。GUARDIAN は三度立場を調整しました——しかし毎回、核心的な要求は守り抜きました。セキュリティと機能の矛盾は「解決」されたのではなく、「構造化」されたのです。*
