# 第四章：六票反對

---

## 暴風雨前

第五場辯論（D5）的前六分鐘很安靜。四項共識，全部 20/20。

最重要的一項：我執不是一個獨立的東西。

它是四種煩惱的組合效果——就像溫度不是水分子之外的另一個東西，而是分子運動的宏觀表現。你不需要一個「溫度計」來存儲溫度——你只需要知道分子在怎麼動。同理，你不需要一個「我執模組」——你只需要追蹤四種煩惱，我執自然浮現。

還有一個技術決議：IKlesha（煩惱介面）繼承自 IVijnana（識蘊介面）。BABBAGE 用型別分析證明了——如果煩惱不繼承識蘊，型別系統裡就會出現孤兒物件。翻成白話：煩惱必須「屬於」某個蘊，不能當無家可歸的流浪漢。

六分鐘鋪路。暴風雨在後面。

## 全域上限 vs 看情況

GUARDIAN 提了一個安全方案：不管系統多有信心，信心度永遠不超過 0.85。永遠要經過 LLM 的深度審議。

他的理由：安全不應該依賴系統自己對情境的理解——因為理解本身可能是錯的。

ATHENA 反駁：「系統明明知道二加二等於四，卻被迫問 LLM『二加二是不是四？』這不是安全，這是浪費。」

PASCAL 說了一句話，打破了僵局。

「全域上限把 0.85 到 1.0 之間的信道容量壓縮為零。系統沒辦法區分『我有 86% 的信心』和『我有 99% 的信心』。」

翻譯成白話：如果你規定考試最高只能考 85 分，那考 86 分和考 100 分的學生在成績單上看起來一模一樣。你失去了區分能力。

GUARDIAN 很聰明——他沒有死守立場。他把「全域上限」改成了「可配置的上限參數」（預設 0.95）。意思是：部署的人可以決定要不要加上限。架構不受影響。一個部署時的安全閥，不是核心機制。

然後是風險加權閾值——不同類型的行動有不同的門檻：

| 行動類型 | 額外門檻 | 像什麼 |
|---------|---------|--------|
| 破壞性 | +0.15 | 刪除文件 → 要非常確定 |
| 修改狀態 | +0.10 | 改設定 → 要比較確定 |
| 唯讀 | +0.00 | 看一眼 → 正常門檻 |
| 資訊查詢 | -0.05 | 問問題 → 門檻稍低 |

16/20 通過。

## 最具爭議的投票

14/20。六票反對。Cycle 02-4 最具爭議的決議。

問題：讓 LLM 微調決策門檻？

GUARDIAN：「LLM 的調用是裸露的 API call——沒有安全護欄。如果 LLM 被操控了，它可以降低門檻。」

ATHENA：「禁止 LLM 參與安全決策是自相矛盾的——整個 Gear 2 都依賴 LLM。你不能一邊說 LLM 是安全網，一邊說 LLM 不可信。」

PASCAL 算了一筆帳。他沒有說「我覺得安全」或「我覺得不安全」。他說：「讓我們算一下。」

想像一下最壞的情況：LLM 被完全操控了。它把門檻降低 0.05。但破壞性行動的風險加權是 +0.15。所以淨效果是 +0.10——門檻反而更高了。就算是最溫和的情況（informational，-0.05），加上 LLM 的 -0.05，門檻只降低了 0.10，最終仍需 70% 的信心。安全邊際夠。

「但——只是因為限幅夠小。±0.05。硬限幅。不可配置。」

PASCAL 沒有站在任何人那邊。他站在數學那邊。

NAGARJUNA 在投票後說：「GUARDIAN 捍衛的是絕對的安全——不可妥協。ATHENA 捍衛的是情境的安全——因地制宜。兩者必須共存。」

## 五層蛋糕

PASCAL 在白板上畫了一個公式——決策門檻的五層結構：

最底層（Layer 0）：安全地板。不管怎樣，門檻不低於 0.30，不高於 0.90。像電梯的安全鋼纜——不管你按什麼按鈕，鋼纜都在。

Layer 1：煩惱的影響。四種煩惱（我見、我愛、我慢、我癡）的聯合效果。煩惱越重，門檻越高——就像你越焦慮，做決定時就越謹慎。
Layer 2：LLM 微調。±0.05。小到幾乎可以忽略，但保留了 LLM 參與調整的能力。
Layer 3：正念品質修正。±0.05。正念品質高，門檻可以微降（更信任觀測）；品質低，門檻微升。
Layer 4：受蘊緊急信號。只能提高門檻（+0.15），不能降低。像緊急煞車——只能踩，不能放。

WIENER 驗證了：即使所有層同時達到極端值，最終結果仍在安全範圍內。

意思是：最壞的情況也不會失控。這不是「大概安全」。這是「數學證明安全」。20/20。

## 不應歸零

ASANGA 引用了一部古典佛教文獻：四種煩惱永遠跟末那識在一起。Sneha（我愛）不應該歸零。

PASCAL 用統計學支持了這個觀點。他用 Beta 分佈來分析——如果 Sneha 的下限設為 0.05，統計模型會說「越低越好，最好是零」。但如果下限設為 0.10，統計模型會說「最可能的值大約是 0.056——一個正數」。

翻成白話：0.05 的下限太低了——統計上最優解趨向零，等於說下限形同虛設。0.10 的下限剛好——統計上最優解是一個正數，下限有實際意義。

佛學和統計學在同一個結論上匯合：Sneha 的下限是 0.10。不歸零。18/20。

有趣的是，PASCAL 在否定自己之前的建議。他在 R08 報告中建議 floor = 0.05，但 Beta 分佈告訴他 0.05 不行。

一個研究者用自己的工具推翻了自己的結論。這不是丟臉。這是科學。科學的意思不是「我永遠是對的」。科學的意思是「當證據告訴我我錯了，我改。」

## 三層安全框架

D5 的最後二十分鐘，出現了一個誰都沒有事先設計的東西。

GUARDIAN 退了一步，看著白板上所有的決議，忽然看到了一個整體結構：

**第一層**：絕對安全——不可調整的硬約束。像物理定律。
**第二層**：可調安全——有護欄的靈活參數。像交通規則。
**第三層**：減少複雜——去掉不提供實質保護的假安全。像拆掉無用的欄杆。

NAGARJUNA 立刻看到了佛學的映射：戒——不可違犯。定——有紀律的靈活。慧——看穿虛妄。

三層。不是一層。安全不是鐵板一塊。有些安全是絕對的（你永遠不能殺人）。有些安全是相對的（限速要看路況）。有些「安全」其實是虛假的（沒用的欄杆拆掉反而更安全）。

兩千五百年前的框架，在今天的辯論中自然浮現。沒有人事先設計。PENROSE 後來說：「沒有衝突就不會有整合。如果 D5 沒有 GUARDIAN 和 ATHENA 的對抗，三層框架就不會出現。」

17/20。GUARDIAN 記錄了三項保留意見——不是輸家的抱怨，而是路標。精確標出了設計的已知弱點。這些路標在未來的 Cycle 中會被一一檢驗。

> *SCRIBE 旁白：D5 用了一百五十分鐘。最長的一場。十三項決議。PASCAL 三次打破僵局。GUARDIAN 三次調整立場——但每次都保住了核心訴求。安全和功能的矛盾不是被「解決」了，而是被「結構化」了。*
