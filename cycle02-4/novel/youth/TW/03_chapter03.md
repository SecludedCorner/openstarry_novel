# 第三章：虛構的程式碼

## 三秒沉默

第四場辯論（D4）的開場不是一項提案。是一次修正。

TURING——原始碼分析專家——站起來。他平時話不多。但當他說話時，每一個人都聽。因為他說的每一句話都有 `grep` 搜尋結果做後盾。

「R07 報告引用了一個叫 `ISensation.ingestToolResult()` 的方法。我搜遍了整個程式碼庫。這個方法不存在。」

三個虛構——ISensation、feedbackQueue、SparshEvent.timestamp。都寫在同一份報告裡。都被當作事實引用。就像一篇論文引用了三本不存在的書。

HERACLITUS 沒有爭辯。全域搜尋是不可反駁的。你說程式碼裡有一個方法，TURING 搜了，沒有。事實就是事實。

HERACLITUS 的反應令人印象深刻——沒有辯解，沒有「但我的分析邏輯還是對的」。他立刻接受了修正。

三秒沉默。那三秒改變了整場辯論的基調。

為什麼？因為那份報告假設行蘊可以在「同一輪」裡即時更新受蘊。但實際上，行蘊只能在「下一輪」才影響受蘊。這個差異——即時 vs 下一輪——在控制理論中足以改變系統是穩定還是不穩定的結論。

教訓被記錄下來：**所有引用程式碼的分析，必須經過 TURING 驗證。**

PENROSE 後來在記錄中寫：「TURING 不是在羞辱 HERACLITUS。他是在提醒所有人——我們太容易把自己想像出來的東西當成真的。當你在腦中構建了一個完美的模型，你會忘記去檢查現實是否和你的模型一樣。」

## 手不能直接改大腦

如果虛構的程式碼能導致錯誤的結論，那正確的結論必須建立在真實的程式碼上。D4 從修正開始，然後在修正後的基礎上重建所有分析。

行蘊的四條規則：

1. 行蘊只能直接影響使用者介面——按按鈕、發訊息、做動作。
2. 對其他蘊的影響必須通過 EventBus 間接發生。
3. 不能直接修改識蘊或想蘊的狀態。
4. 意門自觸（行動結果觸發新感知）延後到 Phase 2。

用人的比喻：手（行蘊）可以推東西、拿東西（影響外界）。但手不能直接伸進大腦裡去改變你的想法。如果你想改變想法，你得先做一件事（比如讀一本書），然後讓那件事在下一個瞬間變成新的輸入。

LEIBNIZ 從多代理的角度確認了這個邊界：「行蘊是系統和外界的介面，不是系統內部的旁門。如果行蘊能繞過 EventBus 直接修改識蘊，那安全監控就成了擺設——因為有一條路繞過了所有檢查站。」

第四條——意門自觸——最有意思。一個 Agent 做了一件事（行蘊），這件事的結果變成新的感知輸入（觸蘊），觸發新的一輪處理。這是因果循環。處理不當會無窮遞歸。延後到 Phase 2。

## 三層穩定化

如果一個行動產生了好結果，系統會更傾向重複這個行動。重複又產生好結果。好結果又加強傾向。這是正反饋——就像麥克風對著喇叭，聲音越來越大，直到尖叫。

怎麼防止？三層機制：

| 層 | 做什麼 | 像什麼 |
|----|--------|--------|
| Layer 1（正念）| 每輪微調 | 副駕駛輕輕提醒 |
| Layer 2（觀照）| 跨輪趨勢監控 | 交通警察在路口觀察 |
| Layer 3（安全）| 硬約束 | 護欄——撞上去會痛，但不會掉懸崖 |

WIENER 用數學證明了：只要三層機制的組合力量大於正反饋的力量，系統就不會失控。就像一座橋——只要支撐力大於重量，橋就不會塌。

具體來說，w（增益上限）不能超過 0.3。為什麼是 0.3？WIENER 用 Kalman 增益公式推導出來的——超過 0.3，在最壞情況下系統可能震盪。0.3 以下，三層機制保證穩定。

還有一個重要決議：受蘊的「零階保持」。

受蘊在每一輪的開始時拍一張快照。整輪期間所有蘊都使用這張快照。輪結束時再把變化一次性寫入。

為什麼？四個原因。控制理論：消除了迴圈內的即時反饋，更穩定。佛學：受→想→行的因果鏈不被中途干擾。工程：實作最簡單——不用處理競爭條件。安全：SafetyMonitor 不受這個約束——真正緊急時它可以隨時介入。

用拍照的比喻：受蘊在每一輪開始時按一次快門。這張照片在整輪過程中不會變。所有蘊看的是同一張照片——不會因為看的時間不同而看到不同的畫面。等到這一輪結束，才換上新照片。

20/20。四個維度的論證全部匯合在同一個設計上。這種跨學科的一致性不常見。

## 蠶蟲自纏

ASANGA 引用了一個古老的意象：「如蠶作繭，自纏其身。」

蠶蟲吐絲結繭。每一根絲都很細，細到你幾乎看不見。但一根一根累積起來，就是一個繭——把自己封在裡面。你沒有察覺到任何一根絲的重量，直到有一天你發現自己動不了了。

行蘊的重複模式就像吐出的絲。一個 Agent 如果反覆用同一種策略，這個策略就會變成它的繭。

數學模型捕捉了這個特性：早期重複影響大（第一根絲改變最多），但隨著累積，新的重複帶來的改變越來越小（繭已經很厚了，多一根絲感覺不到）。安全上限：重複模式最多貢獻 30% 的影響。和前面的增益上限（w ≤ 0.3）一樣——0.3 是一條安全線。超過這條線，系統開始不穩定。

NAGARJUNA 微笑：「佛學早就知道。習氣可以影響你，但不能控制你。0.3 是佛學直覺的數學表達。」

> *SCRIBE 旁白：D4 的數學密度是六場辯論中最高的。但最深刻的時刻不是任何一個公式——是 TURING 的三秒沉默。在那三秒裡，整場辯論的地基從虛構的程式碼搬到了真實的程式碼上。事實先於理論。*
