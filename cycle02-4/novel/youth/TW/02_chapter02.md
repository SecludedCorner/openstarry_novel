# 第二章：你不能超頻一顆 CPU 的指令集

---

## 正念不是鬧鐘

第二場辯論（D2）開場七分鐘就通過了三項共識。最重要的一項：正念不是 polling。

Polling 是什麼？就像設一個鬧鐘，每五秒鐘問一次「現在怎麼樣了？」。問完就不管了，等下一次鬧鐘響。很機械。很間歇。兩次鬧鐘之間發生的事，全部漏掉。

正念不是這個。

正念是持續的覺察——就像一個真正專注開車的人，他不是每五秒看一次路。他一直在看。他注意到前面的車剎車了、旁邊有個小孩衝出來了、紅綠燈變黃了——不是因為他每五秒檢查一次，而是因為他一直在覺察。

SafetyMonitor 的定時檢查更像什麼？更像心跳。你不需要「注意」心臟在跳——它自己在跳。心跳是生存條件，不是認知品質。一個昏迷的人有心跳但沒有正念。

NAGARJUNA 用了一個佛學概念來說明這個區別：有分心（bhavanga）——無意識的基線存在。SafetyMonitor 的定時檢查就是有分心的數位版本。

ASANGA 補充：「有分心維持生命但不產生覺知。正念產生覺知。這是根本差異。」翻成白話：你睡著的時候心臟在跳，但你不知道自己在睡。正念是你醒著的時候知道自己醒著。

20/20。然後又是 20/20。七分鐘，三個滿分。起手式乾淨利落。

## CPU 不能在跑的時候變架構

D2 最激烈的辯論：正念的 Level 和 Depth 要分開嗎？

HERACLITUS 給了一個超棒的類比。

Level 是 CPU 的指令集架構——像 x86 或 ARM。部署時就決定了。你不能讓一顆 x86 的 CPU 在跑的時候變成 ARM。

Depth 是時鐘速度——可以超頻或降頻。你可以讓 CPU 跑快一點或慢一點。

正念的 Level 決定了 Agent 能覺察什麼——能監控哪些事件類型。Depth 決定了覺察的精細度——看得多細、反應多快。

Level 固定意味著安全保證——惡意輸入不能讓系統突然「學會」監控新的事件類型，然後利用這個新能力做壞事。如果有人能在運行時改變 Level，等於打開了一扇後門。

PASCAL 提議讓 Level 可以動態切換——被否決了。16/18。

不是因為想法不好，而是因為改 Level 等於改架構。就像你不能在高速公路上行駛的時候換引擎。你得先停下來，進維修站，做完整的檢查，然後才能上路。

這是 PASCAL 加入團隊後第一個被否決的提案。他的反應值得記錄：沒有爭辯，沒有「但是我覺得⋯⋯」。直接接受。然後調整策略——從此他不再提架構方案，改為提供數學工具。從「我認為應該這樣」到「數學告訴我們如果這樣會怎樣」。這個轉變讓他在 D5 中三次打破僵局。

PENROSE 後來評論：「PASCAL 在 D2 學到了一件事——在一個有二十個人的團隊裡，最有影響力的不是提出最多方案的人，而是提供最好工具的人。」

## 正念的四把尺

SatiQualityVector——正念品質的四個維度：

| 維度 | 像什麼 |
|------|--------|
| C（連續性）| 有沒有漏掉該看的東西 |
| G（精細度）| 看得有多細 |
| S（速度）| 從發生到注意到，花了多久 |
| E（均衡性）| 是不是壞消息看得比好消息仔細 |

E 維度最有意思。如果一個系統對壞消息特別敏感、對好消息不太在意，它的判斷就會偏掉——像一個只看新聞負面報導的人，會覺得世界比實際上更糟。

19/19。全票通過。

WIENER 從控制理論的角度解讀：C 是觀測器的覆蓋率（有沒有監測到所有該監測的）；G 是解析度（看得多清楚）；S 是延遲（反應多快）；E 是偏差（有沒有看走眼）。四個維度完整描述了一個觀測器的品質。

NAGARJUNA 微笑：「佛學在兩千五百年前就定義了這四個維度。現在控制理論用不同的符號說了同樣的話。」

## 最安靜的四十五分鐘

第三場辯論（D3）是六場中最安靜的。四十五分鐘。五項決議。零爭議。

為什麼安靜？因為 D1 和 D2 已經把該爭的爭完了。退化行為、事件驅動、Level 固定——這些基礎決定了 D3 的方向。地基穩，蓋牆就快。

SparshEvent——觸的事件格式——被定義為三個必要欄位：root（感官來源）、object（感知對象）、consciousness（識的類型）。佛學叫「三和合」——根、境、識的交匯。

LINNAEUS——分類學專家——做了一個精確的區分：這三個欄位是「定義性的」——沒有它們就不是觸。timestamp 是「附隨性的」——有它更方便，但沒它觸還是觸。intensity（強度）也是附隨性的。

就像一杯咖啡。咖啡豆和水是定義性的——沒有它們就不是咖啡。杯子是附隨性的——你可以用碗喝，它還是咖啡。溫度也是附隨性的——冰的熱的都是咖啡。

另一個重要決議：CoarisingBundle（俱生束）裡，觸是唯一的「必要頻道」。受蘊、想蘊、作意是「參考頻道」——有的話更好，沒有也不阻止系統運作。就像開車時，方向盤是必要的（沒有方向盤你不能開車），但導航和音樂是參考的（沒有它們你還是可以開）。

還有一個新東西：ChannelManasikara——作意的事件頻道。

作意是什麼？是「注意到某件事」的那個動作。你的眼睛掃過房間，注意到桌上有杯水——那個「注意到」就是作意。它不是感覺（那是受），不是辨認（那是想），它是讓你的注意力轉向那裡的東西。

在系統裡，作意決定了「下一步該處理什麼」。它是資源分配器——有限的注意力，要投向哪裡？

> *SCRIBE 旁白：D3 安靜不是因為簡單。是因為地基打得好——D1 的退化行為表和之前 Cycle 的觸定義已經把路鋪平了。最安靜的辯論，往往建立在最深的共識之上。*
