# 第四章：六票反对

---

## 暴风雨前

第五场辩论（D5）的前六分钟很安静。四项共识，全部 20/20。

最重要的一项：我执不是一个独立的东西。

它是四种烦恼的组合效果——就像温度不是水分子之外的另一个东西，而是分子运动的宏观表现。你不需要一个「温度计」来存储温度——你只需要知道分子在怎么动。同理，你不需要一个「我执模块」——你只需要追踪四种烦恼，我执自然浮现。

还有一个技术决议：IKlesha（烦恼接口）继承自 IVijnana（识蕴接口）。BABBAGE 用类型分析证明了——如果烦恼不继承识蕴，类型系统里就会出现孤儿对象。翻成白话：烦恼必须「属于」某个蕴，不能当无家可归的流浪汉。

六分钟铺路。暴风雨在后面。

---

## 全域上限 vs 看情况

GUARDIAN 提了一个安全方案：不管系统多有信心，信心度永远不超过 0.85。永远要经过 LLM 的深度审议。

他的理由：安全不应该依赖系统自己对情境的理解——因为理解本身可能是错的。

ATHENA 反驳：「系统明明知道二加二等于四，却被迫问 LLM『二加二是不是四？』这不是安全，这是浪费。」

PASCAL 说了一句话，打破了僵局。

「全域上限把 0.85 到 1.0 之间的信道容量压缩为零。系统没办法区分『我有 86% 的信心』和『我有 99% 的信心』。」

翻译成白话：如果你规定考试最高只能考 85 分，那考 86 分和考 100 分的学生在成绩单上看起来一模一样。你失去了区分能力。

GUARDIAN 很聪明——他没有死守立场。他把「全域上限」改成了「可配置的上限参数」（预设 0.95）。意思是：部署的人可以决定要不要加上限。架构不受影响。一个部署时的安全阀，不是核心机制。

然后是风险加权阈值——不同类型的行动有不同的门槛：

| 行动类型 | 额外门槛 | 像什么 |
|---------|---------|--------|
| 破坏性 | +0.15 | 删除文件 → 要非常确定 |
| 修改状态 | +0.10 | 改配置 → 要比较确定 |
| 只读 | +0.00 | 看一眼 → 正常门槛 |
| 信息查询 | -0.05 | 问问题 → 门槛稍低 |

16/20 通过。

## 最具争议的投票

14/20。六票反对。Cycle 02-4 最具争议的决议。

问题：让 LLM 微调决策门槛？

GUARDIAN：「LLM 的调用是裸露的 API call——没有安全护栏。如果 LLM 被操控了，它可以降低门槛。」

ATHENA：「禁止 LLM 参与安全决策是自相矛盾的——整个 Gear 2 都依赖 LLM。你不能一边说 LLM 是安全网，一边说 LLM 不可信。」

PASCAL 算了一笔账。他没有说「我觉得安全」或「我觉得不安全」。他说：「让我们算一下。」

想象一下最坏的情况：LLM 被完全操控了。它把门槛降低 0.05。但破坏性行动的风险加权是 +0.15。所以净效果是 +0.10——门槛反而更高了。就算是最温和的情况（informational，-0.05），加上 LLM 的 -0.05，门槛只降低了 0.10，最终仍需 70% 的信心。安全边际够。

「但——只是因为限幅够小。±0.05。硬限幅。不可配置。」

PASCAL 没有站在任何人那边。他站在数学那边。

NAGARJUNA 在投票后说：「GUARDIAN 捍卫的是绝对的安全——不可妥协。ATHENA 捍卫的是情境的安全——因地制宜。两者必须共存。」

## 五层蛋糕

PASCAL 在白板上画了一个公式——决策门槛的五层结构：

最底层（Layer 0）：安全地板。不管怎样，门槛不低于 0.30，不高于 0.90。像电梯的安全钢缆——不管你按什么按钮，钢缆都在。

Layer 1：烦恼的影响。四种烦恼（我见、我爱、我慢、我痴）的联合效果。烦恼越重，门槛越高——就像你越焦虑，做决定时就越谨慎。
Layer 2：LLM 微调。±0.05。小到几乎可以忽略，但保留了 LLM 参与调整的能力。
Layer 3：正念品质修正。±0.05。正念品质高，门槛可以微降（更信任观测）；品质低，门槛微升。
Layer 4：受蕴紧急信号。只能提高门槛（+0.15），不能降低。像紧急刹车——只能踩，不能放。

WIENER 验证了：即使所有层同时达到极端值，最终结果仍在安全范围内。

意思是：最坏的情况也不会失控。这不是「大概安全」。这是「数学证明安全」。20/20。

## 不应归零

ASANGA 引用了一部古典佛教文献：四种烦恼永远跟末那识在一起。Sneha（我爱）不应该归零。

PASCAL 用统计学支持了这个观点。他用 Beta 分布来分析——如果 Sneha 的下限设为 0.05，统计模型会说「越低越好，最好是零」。但如果下限设为 0.10，统计模型会说「最可能的值大约是 0.056——一个正数」。

翻成白话：0.05 的下限太低了——统计上最优解趋向零，等于说下限形同虚设。0.10 的下限刚好——统计上最优解是一个正数，下限有实际意义。

佛学和统计学在同一个结论上汇合：Sneha 的下限是 0.10。不归零。18/20。

有趣的是，PASCAL 在否定自己之前的建议。他在 R08 报告中建议 floor = 0.05，但 Beta 分布告诉他 0.05 不行。

一个研究者用自己的工具推翻了自己的结论。这不是丢脸。这是科学。科学的意思不是「我永远是对的」。科学的意思是「当证据告诉我我错了，我改。」

## 三层安全框架

D5 的最后二十分钟，出现了一个谁都没有事先设计的东西。

GUARDIAN 退了一步，看着白板上所有的决议，忽然看到了一个整体结构：

**第一层**：绝对安全——不可调整的硬约束。像物理定律。
**第二层**：可调安全——有护栏的灵活参数。像交通规则。
**第三层**：减少复杂——去掉不提供实质保护的假安全。像拆掉无用的栏杆。

NAGARJUNA 立刻看到了佛学的映射：戒——不可违犯。定——有纪律的灵活。慧——看穿虚妄。

三层。不是一层。安全不是铁板一块。有些安全是绝对的（你永远不能杀人）。有些安全是相对的（限速要看路况）。有些「安全」其实是虚假的（没用的栏杆拆掉反而更安全）。

两千五百年前的框架，在今天的辩论中自然浮现。没有人事先设计。PENROSE 后来说：「没有冲突就不会有整合。如果 D5 没有 GUARDIAN 和 ATHENA 的对抗，三层框架就不会出现。」

17/20。GUARDIAN 记录了三项保留意见——不是输家的抱怨，而是路标。精确标出了设计的已知弱点。这些路标在未来的 Cycle 中会被一一检验。

> *SCRIBE 旁白：D5 用了一百五十分钟。最长的一场。十三项决议。PASCAL 三次打破僵局。GUARDIAN 三次调整立场——但每次都保住了核心诉求。安全和功能的矛盾不是被「解决」了，而是被「结构化」了。*
