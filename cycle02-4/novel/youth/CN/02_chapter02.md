# 第二章：你不能超频一颗 CPU 的指令集

---

## 正念不是闹钟

第二场辩论（D2）开场七分钟就通过了三项共识。最重要的一项：正念不是 polling。

Polling 是什么？就像设一个闹钟，每五秒钟问一次「现在怎么样了？」。问完就不管了，等下一次闹钟响。很机械。很间歇。两次闹钟之间发生的事，全部漏掉。

正念不是这个。

正念是持续的觉察——就像一个真正专注开车的人，他不是每五秒看一次路。他一直在看。他注意到前面的车刹车了、旁边有个小孩冲出来了、红绿灯变黄了——不是因为他每五秒检查一次，而是因为他一直在觉察。

SafetyMonitor 的定时检查更像什么？更像心跳。你不需要「注意」心脏在跳——它自己在跳。心跳是生存条件，不是认知品质。一个昏迷的人有心跳但没有正念。

NAGARJUNA 用了一个佛学概念来说明这个区别：有分心（bhavanga）——无意识的基线存在。SafetyMonitor 的定时检查就是有分心的数字版本。

ASANGA 补充：「有分心维持生命但不产生觉知。正念产生觉知。这是根本差异。」翻成白话：你睡着的时候心脏在跳，但你不知道自己在睡。正念是你醒着的时候知道自己醒着。

20/20。然后又是 20/20。七分钟，三个满分。起手式干净利落。

## CPU 不能在跑的时候变架构

D2 最激烈的辩论：正念的 Level 和 Depth 要分开吗？

HERACLITUS 给了一个超棒的类比。

Level 是 CPU 的指令集架构——像 x86 或 ARM。部署时就决定了。你不能让一颗 x86 的 CPU 在跑的时候变成 ARM。

Depth 是时钟速度——可以超频或降频。你可以让 CPU 跑快一点或慢一点。

正念的 Level 决定了 Agent 能觉察什么——能监控哪些事件类型。Depth 决定了觉察的精细度——看得多细、反应多快。

Level 固定意味着安全保证——恶意输入不能让系统突然「学会」监控新的事件类型，然后利用这个新能力做坏事。如果有人能在运行时改变 Level，等于打开了一扇后门。

PASCAL 提议让 Level 可以动态切换——被否决了。16/18。

不是因为想法不好，而是因为改 Level 等于改架构。就像你不能在高速公路上行驶的时候换引擎。你得先停下来，进维修站，做完整的检查，然后才能上路。

这是 PASCAL 加入团队后第一个被否决的提案。他的反应值得记录：没有争辩，没有「但是我觉得……」。直接接受。然后调整策略——从此他不再提架构方案，改为提供数学工具。从「我认为应该这样」到「数学告诉我们如果这样会怎样」。这个转变让他在 D5 中三次打破僵局。

PENROSE 后来评论：「PASCAL 在 D2 学到了一件事——在一个有二十个人的团队里，最有影响力的不是提出最多方案的人，而是提供最好工具的人。」

## 正念的四把尺

SatiQualityVector——正念品质的四个维度：

| 维度 | 像什么 |
|------|--------|
| C（连续性）| 有没有漏掉该看的东西 |
| G（精细度）| 看得有多细 |
| S（速度）| 从发生到注意到，花了多久 |
| E（均衡性）| 是不是坏消息看得比好消息仔细 |

E 维度最有意思。如果一个系统对坏消息特别敏感、对好消息不太在意，它的判断就会偏掉——像一个只看新闻负面报道的人，会觉得世界比实际上更糟。

19/19。全票通过。

WIENER 从控制理论的角度解读：C 是观测器的覆盖率（有没有监测到所有该监测的）；G 是分辨率（看得多清楚）；S 是延迟（反应多快）；E 是偏差（有没有看走眼）。四个维度完整描述了一个观测器的品质。

NAGARJUNA 微笑：「佛学在两千五百年前就定义了这四个维度。现在控制理论用不同的符号说了同样的话。」

## 最安静的四十五分钟

第三场辩论（D3）是六场中最安静的。四十五分钟。五项决议。零争议。

为什么安静？因为 D1 和 D2 已经把该争的争完了。退化行为、事件驱动、Level 固定——这些基础决定了 D3 的方向。地基稳，盖墙就快。

SparshEvent——触的事件格式——被定义为三个必要字段：root（感官来源）、object（感知对象）、consciousness（识的类型）。佛学叫「三和合」——根、境、识的交汇。

LINNAEUS——分类学专家——做了一个精确的区分：这三个字段是「定义性的」——没有它们就不是触。timestamp 是「附随性的」——有它更方便，但没它触还是触。intensity（强度）也是附随性的。

就像一杯咖啡。咖啡豆和水是定义性的——没有它们就不是咖啡。杯子是附随性的——你可以用碗喝，它还是咖啡。温度也是附随性的——冰的热的都是咖啡。

另一个重要决议：CoarisingBundle（俱生束）里，触是唯一的「必要频道」。受蕴、想蕴、作意是「参考频道」——有的话更好，没有也不阻止系统运作。就像开车时，方向盘是必要的（没有方向盘你不能开车），但导航和音乐是参考的（没有它们你还是可以开）。

还有一个新东西：ChannelManasikara——作意的事件频道。

作意是什么？是「注意到某件事」的那个动作。你的眼睛扫过房间，注意到桌上有杯水——那个「注意到」就是作意。它不是感觉（那是受），不是辨认（那是想），它是让你的注意力转向那里的东西。

在系统里，作意决定了「下一步该处理什么」。它是资源分配器——有限的注意力，要投向哪里？

> *SCRIBE 旁白：D3 安静不是因为简单。是因为地基打得好——D1 的退化行为表和之前 Cycle 的触定义已经把路铺平了。最安静的辩论，往往建立在最深的共识之上。*
