# 第五章：師父的三輪指示

---

## 心所的問題

就在工程軌和哲學軌各自忙碌的時候，師父又發了指示。

起因是哲學軌的心所分析。ASANGA 整理了 51 個心所（心理功能）的重新歸屬建議。這份報告在交叉審閱時被呈給了師父，師父連續給了三輪回應。

每一輪都比上一輪更清楚。就像對焦——第一輪看到大致的形狀，第二輪看到細節，第三輪看到全貌。

---

## 第一輪：劃清界線

師父說：「心所是論典的產物，不是佛陀說的。」

這是一個很重要的區分。佛陀在原始經典中說了五蘊、十二因緣、八正道。但「51 個心所」這個分類系統是後來的論典（阿毗達磨）學者整理出來的——就像佛陀寫了一本小說，然後幾百年後的學者替小說裡的每個角色做了詳細的分類卡片。分類卡片有參考價值，但它不是原始小說的一部分。

師父的結論：Plugin 的命名不能用心所的梵文名字。

## 第二輪：保留價值

但師父不是說心所沒用。第二輪指示中，他承認心所的「功能描述」有參考價值。

比如，心所「精進」（virya）描述了「持續不懈地投入善法」的功能。你可以參考這個功能來設計一個 Plugin——但 Plugin 不能叫 VīryaPlugin，而應該叫 PersistencePlugin 或 RetryPlugin 之類的工程名稱。

「可以說『參考精進的涵義，設計了重試 Plugin』，」師父說。「但不能說『重試 Plugin = 精進』。」

## 第三輪：解放

第三輪是最深刻的。師父說：「Plugin 不等於心所。」

這聽起來很簡單，但它的含義很大。

在唯識學派中，每個心所有固定的蘊歸屬——比如「精進」屬於行蘊，「慧」屬於識蘊。如果 Plugin = 心所，那 Plugin 的蘊歸屬也被鎖死了。

但 Plugin ≠ 心所。一個 Plugin 可能同時有行蘊和識蘊的功能——它既做判斷（READ），又執行動作（WRITE）。在心所系統裡這是不允許的（每個心所只能屬於一個蘊），但在 Plugin 系統裡完全沒問題。

師父的話：「命名解耦帶來歸屬自由。」

不用心所的名字命名 Plugin，反而讓 Plugin 更自由了——它可以跨越多個蘊，不受論典的固定分類束縛。

---

## 八點永久規則

三輪指示被整理成八條永久規則：

1. **心所是論典產物**，不是原始經典的內容
2. **心所功能有參考價值**，可以作為 Plugin 設計的靈感
3. **可以說明參考來源**：「參考某某心所的涵義，設計某某 Plugin」
4. **Plugin 用工程名稱**，不得使用心所梵文名
5. **梵文術語只用原始經典的**——五蘊（skandha）、觸（sparsha）可以，但不能用論典創造的心所梵文名
6. **心所分類不決定蘊歸屬**——「唯識說精進屬行蘊」不能作為 Plugin 歸屬的依據
7. **既有決策不受影響**——之前已經確認過的 Plugin 歸屬繼續有效
8. **Plugin 可以跨多蘊**——因為 Plugin ≠ 心所，自然不受限

這八條規則和哲學軌的「蘊歸屬五原則」一起，構成了 OpenStarry 未來所有命名和歸屬問題的永久判定標準。

NAGARJUNA 說了一句話：「上一輪我們學會了什麼不該做。這一輪我們學會了為什麼不該做。」
