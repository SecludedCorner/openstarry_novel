# 第三章：四維品質

---

## 怎麼知道 AI 做得好不好？

想像你在教一個學生解數學題。你怎麼判斷他學得好不好？

不能只看正確率——也許他只是碰巧猜對的。你會想知道更多：他的方法穩不穩定？他會不會一直換策略？他是越做越好還是原地踏步？

OpenStarry 的品質監控器（LoopQualityMonitor）面對同樣的問題。它需要評估 AI Agent 的認知迴路——Agent 接收訊息、思考、做決定、執行、再接收下一個訊息——這個循環的品質怎麼樣？

WIENER 和 ATHENA（AI/ML 專家）帶領團隊設計了四個維度：

---

## 四把尺

**第一把尺：一致性（coherence）**

公式：`1 - 切換次數 / (窗口大小 - 1)`

如果 Agent 一直在同一個「齒輪」（處理模式）下運作——不斷切換就像開車時一直換檔——那表示它知道自己在做什麼。如果它一直在不同的齒輪之間跳來跳去，可能是迷路了。

想像一個人在做菜。他一下子拿起鍋鏟，放下，拿起刀，放下，又拿起鍋鏟——他可能不知道自己在做什麼。但如果他穩穩地用鍋鏟炒了整盤菜，那他很清楚自己的目標。

**第二把尺：效率（efficiency）**

公式：`成功次數 / 嘗試次數`

Agent 提出了多少個動作？其中多少個成功了？如果 Agent 提了 10 個工具呼叫，8 個成功，效率就是 0.8。

這是最直覺的一把尺。就像考試的正確率。

**第三把尺：收斂性（convergence）**

公式：`最長連續成功次數 / 窗口大小`

Agent 有沒有越做越好？如果最近連續成功了 8 次（窗口大小是 10），收斂性就是 0.8。這表示 Agent 找到了正確的方向，正在穩步前進。

就像學生做題——如果最近十題對了八題而且是連續的，比起十題中散布八題對，更能說明他已經掌握了方法。

**第四把尺：穩定性（stability）**

公式：`1 - min(1, 方差 / 0.25)`

Agent 的信心度分數穩不穩定？如果每次做決定的信心度都差不多（方差小），穩定性就高。如果信心度忽高忽低——這次 0.9，下次 0.3，再下次 0.8——系統可能不太穩定。

想像你問一個人「你確定嗎？」如果他每次都說「我 80% 確定」，那他很穩定。如果他一下子說「90%！」一下子說「20%......」，你會擔心他是不是在亂猜。

## 滑動窗口

四把尺都在一個「滑動窗口」中計算。窗口大小是 10 個事件。

為什麼是 10？因為太小的窗口會對隨機波動太敏感（看到一次失敗就恐慌），太大的窗口又太遲鈍（Agent 已經出問題了還沒反應過來）。10 是一個平衡點。

而且，在收集到至少 5 個事件之前，監控器不會發出任何品質分數。這叫「暖機期」——就像新車上路前要先暖引擎，避免在資料不足時做出錯誤判斷。

四個維度的分數各乘以 0.25（等權重），加起來就是一個 0 到 1 之間的「複合品質分數」Q。

為什麼等權重？因為目前沒有數據支持任何一個維度比其他維度更重要。PASCAL（決策理論專家）說：「在沒有證據的情況下，最大熵原則告訴我們：給每個可能性相同的權重。」先跑起來，收集數據，未來再調。

## 六隻耳朵

監控器需要「聽」EventBus 上的事件才能計算品質。HERACLITUS（運行時動態專家）和 TURING 規劃了兩階段的監聽方案：

第一階段（Phase 1）只聽 6 種事件——剛好夠計算四個維度。就像裝了六隻耳朵，各自負責聽不同的聲音。

第二階段（Phase 2）再加 5 種事件，用來計算更精細的品質指標。

為什麼分兩階段？因為越多事件意味著越多的資源消耗。先用最少的耳朵做到「夠用」，然後再看需不需要更多。

如果某個事件碰巧不存在怎麼辦？監控器不會當機。它會用安全的預設值——比如效率預設為 1.0（假設一切正常），穩定性預設為 0.5（不好不壞）。

HERACLITUS 的原則：「缺失是資訊，不是錯誤。」
