# 第五章：师父的三轮指示

---

## 心所的问题

就在工程轨和哲学轨各自忙碌的时候，师父又发了指示。

起因是哲学轨的心所分析。ASANGA 整理了 51 个心所（心理功能）的重新归属建议。这份报告在交叉审阅时被呈给了师父，师父连续给了三轮回应。

每一轮都比上一轮更清楚。就像对焦——第一轮看到大致的形状，第二轮看到细节，第三轮看到全貌。

---

## 第一轮：划清界线

师父说：「心所是论典的产物，不是佛陀说的。」

这是一个很重要的区分。佛陀在原始经典中说了五蕴、十二因缘、八正道。但「51 个心所」这个分类系统是后来的论典（阿毗达磨）学者整理出来的——就像佛陀写了一本小说，然后几百年后的学者替小说里的每个角色做了详细的分类卡片。分类卡片有参考价值，但它不是原始小说的一部分。

师父的结论：Plugin 的命名不能用心所的梵文名字。

## 第二轮：保留价值

但师父不是说心所没用。第二轮指示中，他承认心所的「功能描述」有参考价值。

比如，心所「精进」（virya）描述了「持续不懈地投入善法」的功能。你可以参考这个功能来设计一个 Plugin——但 Plugin 不能叫 VīryaPlugin，而应该叫 PersistencePlugin 或 RetryPlugin 之类的工程名称。

「可以说『参考精进的涵义，设计了重试 Plugin』，」师父说。「但不能说『重试 Plugin = 精进』。」

## 第三轮：解放

第三轮是最深刻的。师父说：「Plugin 不等于心所。」

这听起来很简单，但它的含义很大。

在唯识学派中，每个心所有固定的蕴归属——比如「精进」属于行蕴，「慧」属于识蕴。如果 Plugin = 心所，那 Plugin 的蕴归属也被锁死了。

但 Plugin ≠ 心所。一个 Plugin 可能同时有行蕴和识蕴的功能——它既做判断（READ），又执行动作（WRITE）。在心所系统里这是不允许的（每个心所只能属于一个蕴），但在 Plugin 系统里完全没问题。

师父的话：「命名解耦带来归属自由。」

不用心所的名字命名 Plugin，反而让 Plugin 更自由了——它可以跨越多个蕴，不受论典的固定分类束缚。

---

## 八点永久规则

三轮指示被整理成八条永久规则：

1. **心所是论典产物**，不是原始经典的内容
2. **心所功能有参考价值**，可以作为 Plugin 设计的灵感
3. **可以说明参考来源**：「参考某某心所的涵义，设计某某 Plugin」
4. **Plugin 用工程名称**，不得使用心所梵文名
5. **梵文术语只用原始经典的**——五蕴（skandha）、触（sparsha）可以，但不能用论典创造的心所梵文名
6. **心所分类不决定蕴归属**——「唯识说精进属行蕴」不能作为 Plugin 归属的依据
7. **既有决策不受影响**——之前已经确认过的 Plugin 归属继续有效
8. **Plugin 可以跨多蕴**——因为 Plugin ≠ 心所，自然不受限

这八条规则和哲学轨的「蕴归属五原则」一起，构成了 OpenStarry 未来所有命名和归属问题的永久判定标准。

NAGARJUNA 说了一句话：「上一轮我们学会了什么不该做。这一轮我们学会了为什么不该做。」
