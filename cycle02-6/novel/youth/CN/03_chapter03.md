# 第三章：四维品质

---

## 怎么知道 AI 做得好不好？

想象你在教一个学生解数学题。你怎么判断他学得好不好？

不能只看正确率——也许他只是碰巧猜对的。你会想知道更多：他的方法稳不稳定？他会不会一直换策略？他是越做越好还是原地踏步？

OpenStarry 的品质监控器（LoopQualityMonitor）面对同样的问题。它需要评估 AI Agent 的认知回路——Agent 接收讯息、思考、做决定、执行、再接收下一个讯息——这个循环的品质怎么样？

WIENER 和 ATHENA（AI/ML 专家）带领团队设计了四个维度：

---

## 四把尺

**第一把尺：一致性（coherence）**

公式：`1 - 切换次数 / (窗口大小 - 1)`

如果 Agent 一直在同一个「齿轮」（处理模式）下运作——不断切换就像开车时一直换挡——那表示它知道自己在做什么。如果它一直在不同的齿轮之间跳来跳去，可能是迷路了。

想象一个人在做菜。他一下子拿起锅铲，放下，拿起刀，放下，又拿起锅铲——他可能不知道自己在做什么。但如果他稳稳地用锅铲炒了整盘菜，那他很清楚自己的目标。

**第二把尺：效率（efficiency）**

公式：`成功次数 / 尝试次数`

Agent 提出了多少个动作？其中多少个成功了？如果 Agent 提了 10 个工具调用，8 个成功，效率就是 0.8。

这是最直觉的一把尺。就像考试的正确率。

**第三把尺：收敛性（convergence）**

公式：`最长连续成功次数 / 窗口大小`

Agent 有没有越做越好？如果最近连续成功了 8 次（窗口大小是 10），收敛性就是 0.8。这表示 Agent 找到了正确的方向，正在稳步前进。

就像学生做题——如果最近十题对了八题而且是连续的，比起十题中散布八题对，更能说明他已经掌握了方法。

**第四把尺：稳定性（stability）**

公式：`1 - min(1, 方差 / 0.25)`

Agent 的信心度分数稳不稳定？如果每次做决定的信心度都差不多（方差小），稳定性就高。如果信心度忽高忽低——这次 0.9，下次 0.3，再下次 0.8——系统可能不太稳定。

想象你问一个人「你确定吗？」如果他每次都说「我 80% 确定」，那他很稳定。如果他一下子说「90%！」一下子说「20%......」，你会担心他是不是在乱猜。

## 滑动窗口

四把尺都在一个「滑动窗口」中计算。窗口大小是 10 个事件。

为什么是 10？因为太小的窗口会对随机波动太敏感（看到一次失败就恐慌），太大的窗口又太迟钝（Agent 已经出问题了还没反应过来）。10 是一个平衡点。

而且，在收集到至少 5 个事件之前，监控器不会发出任何品质分数。这叫「暖机期」——就像新车上路前要先暖引擎，避免在数据不足时做出错误判断。

四个维度的分数各乘以 0.25（等权重），加起来就是一个 0 到 1 之间的「复合品质分数」Q。

为什么等权重？因为目前没有数据支持任何一个维度比其他维度更重要。PASCAL（决策理论专家）说：「在没有证据的情况下，最大熵原则告诉我们：给每个可能性相同的权重。」先跑起来，收集数据，未来再调。

## 六只耳朵

监控器需要「听」EventBus 上的事件才能计算品质。HERACLITUS（运行时动态专家）和 TURING 规划了两阶段的监听方案：

第一阶段（Phase 1）只听 6 种事件——刚好够计算四个维度。就像装了六只耳朵，各自负责听不同的声音。

第二阶段（Phase 2）再加 5 种事件，用来计算更精细的品质指标。

为什么分两阶段？因为越多事件意味着越多的资源消耗。先用最少的耳朵做到「够用」，然后再看需不需要更多。

如果某个事件碰巧不存在怎么办？监控器不会当机。它会用安全的预设值——比如效率预设为 1.0（假设一切正常），稳定性预设为 0.5（不好不坏）。

HERACLITUS 的原则：「缺失是信息，不是错误。」
