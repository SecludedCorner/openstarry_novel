# 第五章：心所命名指示的三轮精化 —— 从论典产物到命名解耦

---

## 5.1 指示的触发脉络

R2 交叉审阅期间，ASANGA 的唯识心所批判报告（C2）被提交至 DC Master。Master 做出三轮连续性指示，直接导致 D1 议程修订：D1-Q2（心所多蕴归属）删除，D1-Q6（逐条投票 26 项）简化为原则投票。

---

## 5.2 第一轮：经典与论典的区分

> 心所是论典（Abhidharma）的产物，非原始经典（Sutta/Agama）的内容。

佛陀在原始经典（SN 22.56、MN 44 等）中阐述了五蕴、十二因缘、八正道。五十一心所分类体系则是无著（Asanga）、世亲（Vasubandhu）等后世论典学者在佛陀入灭后数百年间建构的。

直接结论：**Plugin 命名不得使用心所梵文名。**

---

## 5.3 第二轮：功能描述的参考价值

> 心所的功能描述具有参考价值，可作为 Plugin 设计灵感。

心所"精进"（virya）描述的"持续投入"功能可启发重试 Plugin 设计，但该 Plugin 应命名为 `RetryPlugin`，非 `VīryaPlugin`。

- **允许**："参考精进的涵义，设计了重试 Plugin"
- **禁止**："重试 Plugin = 精进"

此区分在"借鉴"（reference）与"等同"（identification）之间划定界线。

---

## 5.4 第三轮：命名解耦与归属自由

> Plugin 不等于心所（plugin ≠ cetasika）。

逻辑推论：唯识系统中每个心所有固定蕴归属（如 virya 属行蕴）。若 Plugin = 心所，Plugin 归属被锁死。但 Plugin ≠ 心所，因此 Plugin 可同时具有行蕴功能（WRITE）和识蕴功能（READ），不产生逻辑矛盾。

Master 概括：**"命名解耦带来归属自由。"**

---

## 5.5 八点永久性规则

| # | 规则 | 性质 |
|---|------|------|
| 1 | 心所是论典产物，非原始经典内容 | 文献学判定 |
| 2 | 心所功能可作为 Plugin 设计灵感 | 肯定性保留 |
| 3 | 可说明"参考某心所涵义，设计某 Plugin" | 表述规范 |
| 4 | Plugin 用工程术语命名，禁用心所梵文名 | 命名规范 |
| 5 | 梵文术语限于原始经典者 | 术语范围 |
| 6 | 心所分类不作为蕴归属依据 | 归属原则 |
| 7 | 既有 Plugin 归属决议不受影响 | 稳定性保证 |
| 8 | Plugin ≠ 心所，可自然跨多蕴 | 跨蕴自由 |
