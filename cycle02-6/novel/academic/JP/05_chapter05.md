# 第五章：心所命名指示の三輪精化 —— 論典産物から命名解結へ

---

## 5.1 指示の発動脈絡

R2 交差査読期間中、ASANGA の唯識心所批判報告（C2）が DC Master に提出された。Master は三輪にわたる連続的指示を発し、D1 の議題修正に直結した。D1-Q2（心所の多蘊帰属）は削除、D1-Q6（26 項目の逐条投票）は原則投票へと簡略化された。

---

## 5.2 第一輪：経典と論典の区別

> 心所は論典（阿毘達磨）の産物であり、原始経典（経蔵／阿含）の内容ではない。

仏陀は原始経典（SN 22.56、MN 44 等）において五蘊、十二因縁、八正道を説いた。五十一心所の分類体系は、無著（Asanga）、世親（Vasubandhu）ら後世の論典学者が仏陀入滅後数百年の間に構築したものである。

直接的結論：**Plugin の命名に心所の梵語名を使用してはならない。**

---

## 5.3 第二輪：機能記述の参照価値

> 心所の機能記述には参照価値があり、Plugin 設計の着想とし得る。

心所「精進」（virya）が記述する「持続的投入」の機能は、リトライ Plugin の設計に着想を与えうる。しかし、当該 Plugin は `RetryPlugin` と命名すべきであり、`VīryaPlugin` としてはならない。

- **許容**：「精進の含意を参照して、リトライ Plugin を設計した」
- **禁止**：「リトライ Plugin = 精進」

この区別は「参照」（reference）と「同一視」（identification）の間に境界線を画定する。

---

## 5.4 第三輪：命名解結と帰属の自由

> Plugin は心所と等しくない（plugin ≠ cetasika）。

論理的推論：唯識体系では各心所に固定的な蘊帰属が定められている（例えば virya は行蘊に属する）。もし Plugin = 心所であれば、Plugin の帰属は固定される。しかし Plugin ≠ 心所であるから、Plugin は行蘊の機能（WRITE）と識蘊の機能（READ）を同時に有しても論理矛盾は生じない。

Master は概括した：**「命名解結は帰属の自由をもたらす。」**

---

## 5.5 八点恒久的規則

| # | 規則 | 性質 |
|---|------|------|
| 1 | 心所は論典の産物であり、原始経典の内容ではない | 文献学的判定 |
| 2 | 心所の機能は Plugin 設計の着想とし得る | 肯定的保留 |
| 3 | 「某心所の含意を参照し、某 Plugin を設計した」と述べ得る | 表述規範 |
| 4 | Plugin は工学用語で命名し、心所の梵語名を禁ずる | 命名規範 |
| 5 | 梵語術語は原始経典に由来するものに限る | 術語範囲 |
| 6 | 心所の分類は蘊帰属の根拠としない | 帰属原則 |
| 7 | 既存の Plugin 帰属決議は影響を受けない | 安定性保証 |
| 8 | Plugin ≠ 心所であり、自然に複数の蘊にまたがりうる | 蘊横断の自由 |

## 5.6 蘊帰属五原則との相補性

| 側面 | 蘊帰属五原則 | 八点規則 |
|------|------------|---------|
| 出典 | 研究チームの帰納 | DC Master の指示 |
| 焦点 | 帰属判定方法 | 命名と術語の規範 |
| 中核主張 | 機能分析が唯一の根拠 | Plugin ≠ 心所、命名解結 |

二つの規則体系は恒久的判定枠組みを構成する。NAGARJUNA：「02-5 では何をすべきでないかを学んだ。02-6 ではなぜすべきでないかを学んだ。」
