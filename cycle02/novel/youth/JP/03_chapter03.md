# 第三章：差異レポート

---

TURING に準備は必要ない。

正確に言えば、彼には「準備」と「本番開始」の境界線が存在しない。SUNYATA が研究対象の更新を宣言した瞬間から、すでに作業中だった。四つのターミナルウィンドウが同時に開き、左半分は v0.22.1-beta、右半分は v0.24.0-beta。まるで同一の身体の、異なる時点で撮られた二枚のレントゲン写真を照合するように。

彼は設計文書を読まない。差分を読む。一行一行。一文字一文字。

---

他の研究員がそれぞれのグループスペースに散った頃、TURING はすでに分析を終えていた。SUNYATA が全員を呼び戻す。

「TURING の差異レポートだ。」

この重々しさは皆が知っている——Cycle 01 で、TURING のコード事実報告はすべての研究の「錨」だった。SYNTHESIST の当時の表現は「経験的アンカーポイント」。あらゆる哲学分析、制御理論、セキュリティ評価は——どれほど精緻であっても——TURING のレポート中のある一行のコードまで遡れなければならなかった。

今、錨が再び打ち込まれる。

「v0.22.1-beta から v0.24.0-beta。コアソースコードの差分。四つのレベルで報告する：新規追加、変更、マーキング、問題。」

「二つ目。`aggregates.test.ts`。四十三行。五蘊ルートインターフェースと型ガードのテスト。全件パス。」

「三つ目。`events.test.ts`。三十二行。イベントジェネリクスが正しく型推論できるかのテスト。」

彼は一拍置いた。

「もう一つ、四つ目がある。SDK ではなく Core 側。スラッシュコマンドが例外をスローした際にエラーイベントを正しく発行するテスト。」

TURING にしては珍しく背景を説明した。「v0.22.1 では、スラッシュコマンドのエラーはログに書き込まれるだけで、誰にも通知されなかった。ユーザーがバグのあるコマンドを入力しても、インターフェースは完全に無反応だった。」

「サイレントフェイルだ。」KERNEL が言った。OS エンジニアリングにおいて、サイレントフェイルは最も危険だ——誰もそれが起きたことを知らないから。家の火災報知器が壊れても「壊れました」と知らせてくれないようなものだ。

「v0.24.0 で修正された。エラーはイベントシステムを通じてブロードキャストされ、UI が受信して表示できるようになった。」

## 二、変更ファイル

TURING の話すスピードは変わらない。各データポイントに同じ時間をかける。

「変更ファイルは十一。SDK が七つ、Core が四つ。」

「`events.ts`。最大の単一変更。百十六行の追加。すべてのイベント型に対して精密なデータ構造を定義。」

彼はコードを投影した：

```typescript
export interface AgentEventPayloadMap {
  [AgentEventType.AGENT_STARTED]: { identity: { id: string; name: string } };
  [AgentEventType.LOOP_ERROR]: { error: string; sessionId?: string };
  // ... 全イベント型をカバー

```

「Cycle 01 で、DARWIN は `payload?: unknown` を『別の宇宙から越境してきた型定義』と形容した。その宇宙の裂け目は塞がれた。ただし型レベルに限る——ランタイムは依然 JavaScript で、型安全は強制されない。」

続いて SDK の六つの変更。すべて同じパターン：旧い五蘊マーキングが更新され、`@skandha` マーキングが追加されたが、インターフェース構造の変更はゼロ。

「`listener.ts`——旧テキスト『受蘊』が『色蘊 sensory input』に変更。`ui.ts`——同じパターン。`provider.ts`——想蘊マーキング追加。`tool.ts`——行蘊。`guide.ts`——識蘊。」

「聞き取れたはずだ。五つの SDK 型ファイル。すべてコメントのみの変更。コード変更はゼロ行。」

「マーキングはドキュメントであって、契約ではない。」ARCHIMEDES が言った。宣言と実装を区別するのは、エンジニアの本能だ。

---

Core の四つの変更。

「`agent-core.ts`。三箇所の変更。第一：`loadPlugins()` メソッドの追加——プラグインを一つずつしかロードできなかったのが、バッチロードと依存関係の自動ソートが可能になった。」

MESH がわずかに身を乗り出した。依存関係のソートは彼の日常だ。

「第二：スラッシュコマンドのエラーハンドリング改善。第三——」TURING は 0.5 秒長く間を置いた。「五蘊マッピングの修正。四箇所のインラインコメントで、元の『受蘊』が『色蘊 sensory input』に変更。」

NAGARJUNA が小さく頷いた。Cycle 01 で受蘊が Listener に誤ってマッピングされている問題を指摘したのは、まさに彼だった。開発チームは修正した——少なくともコメントレベルでは。

「だが、すべての箇所ではない。」TURING が言った。この三語で GUARDIAN の注意が一瞬にして引き締まった。

「`loop.ts`。LLM タイムアウトサポートの追加。」

```typescript
const llmTimeout = deps.llmTimeout ?? 120000;
const abortController = new AbortController();
const llmTimer = setTimeout(() => {
  abortController.abort(new Error(`LLM call timed out after ${llmTimeout}ms`));
```

「最後の一つ。`safety-monitor.ts`。マーキング内容：vedana 受蘊、三受フィードバック placeholder、Plan26 で完全実装予定。」

## 四、継承の不在

TURING は問題リストに入る前に、構造的な観察を一つ挟んだ。

「五蘊ルートインターフェースは定義済みだ。しかし五つの派生インターフェース——IListener、IUI、IProvider、ITool、IGuide——そのどれも、対応するルートインターフェースを継承していない。」

```
IListener  → 継承すべき ISensory    → 実際: なし
IUI        → 継承すべき ISensory    → 実際: なし
IProvider  → 継承すべき ICognition  → 実際: なし
ITool      → 継承すべき IAction     → 実際: なし
IGuide     → 継承すべき IIdentity   → 実際: なし
```

「五対五。すべて未継承。したがって `isSkandha()` 型ガードは既存のプラグインに対して使用できない——なぜなら、そもそも `skandha` フィールドが存在しないからだ。型ガードは有名無実だ。」

「足場は組まれたが、既存の建物と接続されていない。」VITRUVIUS が建築家の言葉で要約した。

> *SCRIBE の記録：LINNAEUS が分類学ノートに感嘆符を加えた——Cycle 01 で構築した分類体系は、これらの継承関係の存在を前提としていた。BABBAGE は途切れた矢印を描いた。*
