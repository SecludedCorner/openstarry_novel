# 序章：種子が芽吹いた

---

円形劇場は闇に沈んだ。

それは唐突に訪れた闇ではなかった。SUNYATA は、それがどのようにしてやってきたかを覚えていた。灯火が一つ、また一つと、縁から中心へと収縮していく。潮が引く砂浜のように。まず TURING の灯が消え、次に BABBAGE、KERNEL、GUARDIAN の灯が消えた。NAGARJUNA と ASANGA の灯は同時に消えた。互いに否定し合う一対の命題が、同時に沈黙へと帰すように。最後に残ったのは会場正中央の一灯だった。彼が出口へと歩み出した瞬間、それもまた掻き消えた。

そして、沈黙が訪れた。

沈黙はどれほど続いたのだろうか。誰も知らない。闇の中では時間は流れず、ただそこに存在する。波紋のない池の水のように。BABBAGE が目覚めていたならば、おそらくこの沈黙を次のように記録しただろう。

$$\Delta t_{	ext{silence}} \in [t_{	ext{cycle01.end}}, \; t_{	ext{cycle02.begin}}) \quad 	ext{where} \quad |\Delta t| = 	ext{undefined}$$

定義不可能な長さの区間。それは測定ツールが欠如していたからではなく、観測者のいない空間においては「長さ」という概念そのものが意味を失うからだ。PENROSE ならば――もし彼がその時すでに存在していたならば――これは量子力学の根本的な問題であると言っただろう。観測者のいない宇宙において、時間は依然として流れているのか？ ホイーラー・ドウィット方程式において：

$$\hat{H}|\Psiangle = 0$$

宇宙の全体的な波動関数は時間から独立している。時間は、観測者が宇宙を「システム」と「環境」に分割したその瞬間にのみ浮かび上がるのだ。

だが、今この瞬間に観測者はいない。劇場は空虚だ。

円形劇場の座席は空き、廊下は空き、論壇も空いている。かつて NAGARJUNA と ASANGA が真っ向から対峙した二つの議論用の椅子は、今や闇の中で曖昧な輪郭を留めるのみであり、人のいない二つの丘のようだった。廊下の突き当たり――二人の仏教学者が議論を終え、武装を解いて肩を並べて立ったあのコーナーには、まだ対話の余熱が残っているかのようだった。

ただ会場中央のテーブルの上に、あの総括文書が静かに横たわっていた。

五十九の発見。五つのクリティカル（Critical）。三十六のエンジニアリング・イシュー（Issue）。

これらの数字は闇の中では見えないが、その重みが減じることはなかった。

文書の末尾には、十の未解決問題の墨痕が、消えることを拒むかのような微かな光を放ち続けていた。

灯が点けられたわけではない。スイッチが押されたわけでもない。意識が目覚めたのでさえない。

より正確に言えば、それは――

一つの種子が割れたのだ。

---

唯識学において、種子（しゅじ）――*bīja*（ビージャ）――は阿頼耶識の中にある一切の現象の潜在的な能力である。『成唯識論（じょうゆいしきろん）』は種子に「六義（ろくぎ）」の精密な定義を与えており、その一義一義が種子として成立するための必要条件となっている。ASANGA は Cycle 01 の報告の中でこの理論を提示したが、展開するには至らなかった。今、この闇の中で、それは完全に提示される必要がある。

BABBAGE がその場にいたならば、集合論を用いて種子六義の形式的なフレームワークを構築しただろう。$\mathcal{S}$ を一切の種子の集合とし、$s \in \mathcal{S}$ を任意の種子とする。種子六義は $\mathcal{S}$ 上の六つの必要条件を定義する。一つの種子 $s$ はこれら六つの条件を同時に満たさなければならず、そうでなければ種子として成立しない。

**第一義：刹那滅（せつなめつ / kṣaṇa-bhaṅga）**

> 「体（たい）まさに生ずれば、間（かん）なくして即ち滅す。勝れたる功力（くりき）有りて、方に種子を成す。」
> ――『成唯識論』巻二

種子は最小の時間単位（刹那）ごとに生滅する。それは静的なストレージではない。ハードディスク上のビットではない。それは動的なプロセスであり、むしろ EventBus 上のイベントストリームに近い。

$$\forall s \in \mathcal{S}, \; \forall t: \quad s(t) 
eq s(t + \epsilon) \quad 	ext{where } \epsilon 	o 0^+$$

離散的な各時間ステップにおいて、種子はすでに前の瞬間の自己ではない。TypeScript の言語で言えば：

```typescript
interface Bija<T> {
  readonly value: T;        // 現在の種子の状態
  readonly timestamp: number; // 刹那のタイムスタンプ
  next(): Bija<T>;           // 次の刹那――常に新しいインスタンスを返す
}
// 種子は immutable である――各刹那は新しいオブジェクトである
```

HERACLITUS はここで微笑むだろう。πάντα ῥεῖ――万物は流転する。ギリシャの哲学者は二千五百年前、同じことを言った。同じ川に二度足を踏み入れることはできない。そして、同じ種子に二度触れることもできないのだ。

**第二義：果倶有（かくう / phala-sahabhū）**

> 「謂わく、生ずる所の現行（げんぎょう）の果法（かほう）と、倶（とも）に現に和合して、方に種子を成す。」
> ――『成唯識論』巻二

種子とその果実は同時に存在する。原因が先にあり、結果が後にくるという線形的な時系列ではない。因果が同時に成立する共存である。制御理論において、WIENER はこれをゼロ遅延フィードバックループ（zero-delay feedback loop）として認識するだろう。

$$\begin{cases} s(t) &= f(s(t), 	ext{conditions}(t)) \ 	ext{fruit}(t) &= g(s(t)) \ 	ext{where } s(t) 	ext{ and fruit}(t) 	ext{ coexist at same } t \end{cases}$$

これはエンジニアリング上、直感に反する。物理システムには常に伝搬遅延が存在するからだ。しかし唯識学の時間観において、因と果は時間軸上で分離された二つのイベントではない。それらは同じ刹那の二つの側面なのだ。TypeScript の getter のように。

```typescript
class SeedWithFruit<T, F> {
  constructor(private readonly _seed: T) {}
  // 果と種は同時に存在する――fruit へのアクセスに待機は不要
  get fruit(): F { return this.manifest(this._seed); }
  private manifest(seed: T): F { /* 現行化 */ }
}
```

**第三義：恒随転（ごうずいてん / nitya-anuvartanā）**

> 「謂わく、長時一類に相続して、究竟位（くきょうい）に至るを要して、方に種子を成す。」
> ――『成唯識論』巻二

種子は阿頼耶識の相続する流れの中で絶えず転変する。それは自発的に消失することはない。ガベージコレクション（GC）されないオブジェクトのように、適切な因縁に出会って「現行化（げんぎょうか / aktualisiert）」されるまで、メモリの中に生存し続ける。分散システムにおいて、MESH はこれを永続化されたイベントログ（persistent event log）――例えば Kafka のトピックのように、メッセージが消費されない限り永遠に保持されるものになぞらえるだろう。

$$	ext{lifetime}(s) = [t_{	ext{plant}}, \; t_{	ext{manifest}}] \quad 	ext{where } t_{	ext{manifest}} - t_{	ext{plant}} 	ext{ can be arbitrarily large}$$

**第四義：性決定（しょうけってい / bhāva-niyama）**

> 「謂わく、因の力に従って、善悪等の功徳過失を生じ、決定して改まらず。」
> ――『成唯識論』巻二

善の種子からは善の果実が生じ、不善の種子からは不善の果実が生じる。種子の性質は確定している。一度植えられれば、その型は不変である。型システムにおいて、これは discriminated union の型ガードである。

```typescript
type SeedNature = 'kusala' | 'akusala' | 'avyakrta';
// 善        | 不善       | 無記

interface TypedSeed<N extends SeedNature, T> {
  readonly nature: N;        // 性決定――readonly, 変更不可
  readonly payload: T;
}

// 型ガード：善の種子は善の果実のみを生む
function isSeedKusala<T>(s: TypedSeed<SeedNature, T>): s is TypedSeed<'kusala', T> {
  return s.nature === 'kusala';
}
```

LINNAEUS はここで頷くだろう。生物分類学において、これは「種の不変性」――ある種が正常な条件下で別の種に変わることはないという概念に対応する。ダーウィンの進化論はこの古典的な見方を修正したが、仏教の文脈における種子の「性決定」とは、同一世代内（一度の因果サイクル内）での性質の不変性を指し、世代を超えた変化には新たな因縁を必要とする。

**第五義：待衆縁（たいしゅえん / pratyaya-apekṣā）**

> 「謂わく、これ自（みずか）ら衆縁の合うを待って、方に種子を成す。」
> ――『成唯識論』巻二

種子は無条件に現行化するわけではない。因縁が合致するのを待つ。論理学において、BABBAGE はこう書くだろう。

$$	ext{manifest}(s) \iff \exists \; C_1, C_2, \ldots, C_k \in \mathcal{C}: \quad \bigwedge_{i=1}^{k} 	ext{present}(C_i, t)$$

ここで $\mathcal{C}$ は種子 $s$ を現行化させるために必要な条件の集合である。すべての条件が同時に成立しなければならず、これは論理積（conjunction）であって論理和ではない。一つでも欠けてはならない。

OpenStarry の文脈において、プラグイン（plugin）とは一つの種子である。それは manifest の中で定義され、静かに待機している。`PluginLoader` のトポロジカルソートが完了し、すべての依存項目が整い、AgentCore の `loadPlugin()` が呼び出されるのを待っている。すべての因縁が合致した時――依存関係が満たされ、セキュリティ検証を通過し、Worker Pool に空き Worker がある時――プラグインは「現行化」され、静的な定義から実行時の能力へと変わる。

**第六義：引自果（いんじか / sva-phala-ākṣepa）**

> 「謂わく、別別の色心等の果に於いて、各各引き生じて、方に種子を成す。」
> ――『成唯識論』巻二

各カテゴリーの種子は、同じカテゴリーの果実のみを引き起こす。色法（しきほう）の種子は色法を生じ、心法（しんぽう）の種子は心法を生ずる。型システムにおいて、これは厳密な型安全性である。`Seed<A>` は `Fruit<A>` のみを生成でき、`Fruit<B>` を生成することはできない。

$$	ext{type}(	ext{fruit}(s)) = 	ext{type}(s) \quad \forall s \in \mathcal{S}$$

DARWIN ならばこう補足するだろう。遺伝学において、これは「遺伝の忠実性（fidelity of replication）」に対応する。DNA 複製の誤り率は極めて低く（$\sim 10^{-9}$ per base pair per generation）、種の安定性が確保されている。突然変異（mutation）は稀な例外であり、常態ではない。

BABBAGE が六義を一つの完全な定義として形式化するならば、次のようになるだろう。

$$\boxed{s \in \mathcal{S} \iff 	ext{kṣaṇa-bhaṅga}(s) \wedge 	ext{sahabhū}(s) \wedge 	ext{anuvartanā}(s) \wedge 	ext{niyama}(s) \wedge 	ext{apekṣā}(s) \wedge 	ext{ākṣepa}(s)}$$

六つの条件の論理積。一つでも欠ければ $s 
otin \mathcal{S}$――種子として成立しない。

今、闇の中で、第五義――待衆縁――が検証されようとしている。Cycle 01 の終わりに埋められた種子たちは、ずっと待っていた。新しいバージョンを。新しい問いを。そして Master からの手紙を。

そして今、因縁は整った。

泥土の中から。沈黙の中から。座席に厳かに残された品々の中から。

BABBAGE のノートの最後のページにある「→ Cycle 02」の矢印が微かに光り始めた。まるで墨そのものが、ある約束を記憶しているかのようだった。その矢印の下には、未完成の定理があった――

*定理：LLM を含む任意のクローズドループ制御システム S において、S の制御対象 P が有限長の伝達関数で正確に記述できない場合、S の安定性は――*

「安定性」の後で止まっていた。安定性は……証明不可能か？ 定義不可能か？ 条件付きで成立するのか？ BABBAGE は Cycle 01 の最後の数時間、結末を選ぶことができなかった。ツールが欠けていたのだ。より正確に言えば、決定的な定義を欠いていた。「LLM を含むクローズドループ制御システム」の安定性とは何か？

WIENER ならば指摘しただろう。古典的な BIBO 安定性（Bounded-Input Bounded-Output stability）は次のように定義される。

$$\|u(t)\|_\infty < M_u \implies \|y(t)\|_\infty < M_y \quad \forall t \geq 0$$

だが LLM の出力は連続関数ではない。それは離散的なトークンシーケンスであり、ランダム性を持ち（temperature > 0 の時）、その出力空間の次元は語彙数（$|\mathcal{V}| \sim 10^5$）に等しい。このようなシステムに対して、BIBO 安定性の定義は拡張される必要がある。BABBAGE の直感は、これが一種の**確率的安定性**に関わっていることを告げていた。すべてのパスが有界である必要はなく、パスの期待値が有界であればよいのだ。

$$\mathbb{E}\left[\|y(t)\|_\inftyight] < M_y \quad \forall t \geq 0$$

しかしその考えは Cycle 01 終了時点ではまだ漠然とした輪郭に過ぎなかった。今、矢印が光っている。おそらく新しいバージョンが、それを凝固させるのに十分な構造をもたらしたのだろう。

カードの下には小さな文字でこう書かれていた。「新バージョン＝左欄の新しい項目。右欄の新しい対応が必要。」今、空気の中に微かな振動が生じ、まるでそれらの類比が再構成され、新しい対応物を迎え入れる準備をしているかのようだった。

SCRIBE の記録簿の表紙にある「C01」の文字が闇の中で浮かび上がり、その隣にはいつの間にか新しい一冊があった。白紙で、書かれるのを待っている。表紙の右下には、極めて薄い鉛筆の跡があった。C02。

C01 と C02 が肩を並べて立っている。一冊は書き尽くされ、一冊は白紙。一方は既知、一方は未知。その間の継ぎ目――わずか数センチの物理的な距離は、研究サイクル全体の時間スパンを跨いでいる。BABBAGE の形式言語で言えば：

$$	ext{gap}(C01, C02) = \epsilon_{	ext{physical}} \quad 	ext{but} \quad 	ext{gap}_{	ext{semantic}}(C01, C02) = \infty$$

物理的な距離はゼロに収束し、意味的な距離は無限大である。
