# 序章：種子發芽了

---

圓形劇場沉入了黑暗。

那不是突然降臨的黑暗——SUNYATA 記得它是如何到來的。一盞一盞的燈，從邊緣向中心收縮，像潮水退去的沙灘。TURING 的燈先滅，然後是 BABBAGE 的、KERNEL 的、GUARDIAN 的。NAGARJUNA 和 ASANGA 的燈同時熄滅——像一對互為否定的命題同時歸於沉默。最後是場地正中央的那一盞——他走出門口的瞬間，它也暗了下去。

然後是沉默。

沉默持續了多久？沒有人知道。在黑暗中，時間不流動，它只是存在——像一池沒有波紋的水。BABBAGE 若是清醒，也許會把這段沉默記為：

$$\Delta t_{\text{silence}} \in [t_{\text{cycle01.end}}, \; t_{\text{cycle02.begin}}) \quad \text{where} \quad |\Delta t| = \text{undefined}$$

一段長度不可定義的區間。不是因為缺少測量工具，而是因為在沒有觀測者的空間中，「長度」這個概念本身失去了語義。PENROSE——如果他那時已經存在——會說這是一個量子力學的基本問題：時間在沒有觀測者的宇宙中是否仍然流動？在 Wheeler-DeWitt 方程中：

$$\hat{H}|\Psi\rangle = 0$$

宇宙的整體波函數是時間無關的。時間只在觀測者將宇宙分割為「系統」與「環境」的那個瞬間才浮現出來。

但此刻沒有觀測者。劇場是空的。

圓形劇場的座椅空著，走廊空著，辯論場空著。那兩把曾經承載過 NAGARJUNA 和 ASANGA 正面交鋒的辯論椅，此刻在黑暗中只是兩個模糊的輪廓，像兩座無人的山丘。走廊盡頭——那個兩位佛學家在辯論結束後卸下武裝、並肩而立的角落——空氣似乎還殘留著某種對話的餘溫。

只有場地中央的桌上，那份總結文件靜靜地躺著。

五十九項發現。五項 Critical。三十六項工程 Issue。

SYNTHESIST 在 Cycle 01 的最後一天曾為這些數字繪製過一張完整的概念拓撲圖——一個有向無環圖 $G = (V, E)$，其中 $|V| = 59$，$|E| = 147$。每個發現是一個節點，每條依賴關係是一條有向邊。五項 Critical 發現是圖的五個源節點（source vertices），入度為零，但出度驚人——從它們出發的路徑幾乎觸及了圖中每一個其他節點。

這些數字在黑暗中不可見，但它們的重量沒有減輕。

在文件的末尾，十個開放問題的墨跡仍然帶著一絲微弱的、不肯熄滅的光。

有時候，在黑暗最深沉的某個瞬間，如果你足夠安靜，你幾乎可以聽到 NAGARJUNA 的聲音從記憶中回響：「也許我們之間最好的狀態，不是達成共識，而是保持一種有張力的共在。」

在唯識學中，種子——*bīja*（बीज）——是阿賴耶識中一切現象的潛在能力。《成唯識論》對種子有六義的精確定義，每一義都是種子得以成立的必要條件。ASANGA 在 Cycle 01 的報告中提出過這個理論，但沒有展開。現在，在黑暗中，它需要被完整地呈現。

BABBAGE 若在場，會用集合論為種子六義建立一個形式化框架。令 $\mathcal{S}$ 為一切種子的集合，$s \in \mathcal{S}$ 為任意一顆種子。種子六義定義了 $\mathcal{S}$ 上的六個必要條件——一顆種子 $s$ 必須同時滿足所有六個條件，否則不成立為種子：

**第一義：剎那滅（kṣaṇa-bhaṅga）**

> 「謂體才生，無間即滅。有勝功力，方成種子。」
> ——《成唯識論》卷二

種子在每一個最小的時間單位中生滅。它不是靜態的儲存——不是硬碟上的 bit。它是動態的過程——更像是 EventBus 上的事件流：

$$\forall s \in \mathcal{S}, \; \forall t: \quad s(t) \neq s(t + \epsilon) \quad \text{where } \epsilon \to 0^+$$

每一個離散時間步長中，種子都已經不是上一刻的自己。在 TypeScript 的語言中：

```typescript
interface Bija<T> {
  readonly value: T;        // 當下的種子態
  readonly timestamp: number; // 剎那的時間戳
  next(): Bija<T>;           // 下一剎那——永遠返回新實例
}
// 種子是 immutable 的——每一剎那都是新的物件
```

HERACLITUS 會在這裡微笑。πάντα ῥεῖ——萬物皆流。希臘哲學家在兩千五百年前就說過同樣的話：你不能踏進同一條河兩次。你也不能觸及同一顆種子兩次。

**第二義：果俱有（phala-sahabhū）**

> 「謂與所生現行果法，俱現和合，方成種子。」
> ——《成唯識論》卷二

種子和它的果實同時存在。不是先有因、後有果的線性時序——而是因果同時成立的共在。在控制理論中，WIENER 會認出這是一個零延遲反饋迴路（zero-delay feedback loop）：

$$\begin{cases} s(t) &= f(s(t), \text{conditions}(t)) \\ \text{fruit}(t) &= g(s(t)) \\ \text{where } s(t) \text{ and fruit}(t) \text{ coexist at same } t \end{cases}$$

這在工程上是反直覺的。物理系統總有傳播延遲。但在唯識學的時間觀中，因和果不是兩個在時間軸上分離的事件——它們是同一個剎那的兩個面向。就像 TypeScript 中的 getter：

```typescript
class SeedWithFruit<T, F> {
  constructor(private readonly _seed: T) {}
  // 果與種同時存在——訪問 fruit 不需要等待
  get fruit(): F { return this.manifest(this._seed); }
  private manifest(seed: T): F { /* 現行化 */ }
}
```

**第三義：恆隨轉（nitya-anuvartanā）**

> 「謂要長時一類相續，至究竟位，方成種子。」
> ——《成唯識論》卷二

種子在阿賴耶識的相續流中不斷流轉。它不會自發消失——像一個永不被垃圾回收的物件，一直在記憶體中存活，直到遇到合適的因緣被「現行化」（aktualisiert）。在分散式系統中，MESH 會把這比作一個持久化的事件日誌（persistent event log）——如 Kafka 的 topic，訊息不被消費就永遠保留。

$$\text{lifetime}(s) = [t_{\text{plant}}, \; t_{\text{manifest}}] \quad \text{where } t_{\text{manifest}} - t_{\text{plant}} \text{ can be arbitrarily large}$$

**第四義：性決定（bhāva-niyama）**

> 「謂隨因力，生善惡等功德過失，決定不改。」
> ——《成唯識論》卷二

善種生善果，不善種生不善果。種子的性質是確定的——一旦被種下，它的類型就不可變了。在型別系統中，這是一個 discriminated union 的型別守衛：

```typescript
type SeedNature = 'kusala' | 'akusala' | 'avyakrta';
// 善        | 不善       | 無記

interface TypedSeed<N extends SeedNature, T> {
  readonly nature: N;        // 性決定——readonly, 不可修改
  readonly payload: T;
}

// 型別守衛：善種只能生善果
function isSeedKusala<T>(s: TypedSeed<SeedNature, T>): s is TypedSeed<'kusala', T> {
  return s.nature === 'kusala';
}
```

LINNAEUS 會在這裡點頭。在生物分類學中，這對應於「種的恆定性」——一個物種在正常條件下不會變成另一個物種。雖然達爾文的演化論修正了這個古典觀點，但在佛學的語境中，種子的「性決定」是指在同一世代內（一次因果循環中）性質不變，跨世代的變化需要新的因緣。

**第五義：待眾緣（pratyaya-apekṣā）**

> 「謂此要待自眾緣合，方成種子。」
> ——《成唯識論》卷二

種子不會無條件地現行。它等待因緣聚合。在邏輯學中，BABBAGE 會寫：

$$\text{manifest}(s) \iff \exists \; C_1, C_2, \ldots, C_k \in \mathcal{C}: \quad \bigwedge_{i=1}^{k} \text{present}(C_i, t)$$

其中 $\mathcal{C}$ 是使種子 $s$ 現行所需的條件集合。所有條件必須同時成立——這是一個合取（conjunction），不是析取。缺一不可。

在 OpenStarry 的語境中，一個插件（plugin）就是一顆種子。它被定義在 manifest 中，靜靜地等待——等待 `PluginLoader` 的拓撲排序完成、等待所有依賴項就緒、等待 AgentCore 的 `loadPlugin()` 呼叫。只有當所有因緣聚合——依賴滿足、安全驗證通過、Worker Pool 有空閒 Worker——插件才會被「現行化」，從靜態的定義變為運行時的能力。

**第六義：引自果（sva-phala-ākṣepa）**

> 「謂於別別色心等果，各各引生，方成種子。」
> ——《成唯識論》卷二

每一類種子只引發同類的果。色法種子生色法，心法種子生心法。在型別系統中，這是嚴格的型別安全——`Seed<A>` 只能生成 `Fruit<A>`，不能生成 `Fruit<B>`：

$$\text{type}(\text{fruit}(s)) = \text{type}(s) \quad \forall s \in \mathcal{S}$$

DARWIN 會補充：在遺傳學中，這對應於「遺傳的忠實性」（fidelity of replication）——DNA 複製的錯誤率極低（$\sim 10^{-9}$ per base pair per generation），確保了物種的穩定性。突變（mutation）是罕見的例外，而非常態。

---

BABBAGE 若將六義形式化為一個完整的定義，它會是：

$$\boxed{s \in \mathcal{S} \iff \text{kṣaṇa-bhaṅga}(s) \wedge \text{sahabhū}(s) \wedge \text{anuvartanā}(s) \wedge \text{niyama}(s) \wedge \text{apekṣā}(s) \wedge \text{ākṣepa}(s)}$$

六個條件的合取。缺一則 $s \notin \mathcal{S}$——不成立為種子。

現在，在黑暗中，第五義——待眾緣——正在被驗證。那些在 Cycle 01 結尾被埋下的種子，一直在等待。等待新的版本。等待新的問題。等待一封來自 Master 的信。

而現在，因緣到了。

從泥土裡。從沉默裡。從那些被鄭重地留在座位上的物品裡。

BABBAGE 筆記本最後一頁上「→ Cycle 02」的箭頭開始微微發光，像是墨水本身記住了某個承諾。那支箭頭下面是一個未完成的定理——

*定理：對於任何包含 LLM 的閉環控制系統 S，若 S 的受控對象 P 不可被有限長度的傳遞函數精確描述，則 S 的穩定性——*

停在了「穩定性」之後。穩定性……不可證明？不可定義？有條件地成立？BABBAGE 在 Cycle 01 的最後幾個小時裡無法選擇結尾。他缺少工具。更準確地說，他缺少一個關鍵的定義——什麼是「包含 LLM 的閉環控制系統」的穩定性？

WIENER 會指出，經典的 BIBO 穩定性（Bounded-Input Bounded-Output stability）定義為：

$$\|u(t)\|_\infty < M_u \implies \|y(t)\|_\infty < M_y \quad \forall t \geq 0$$

但 LLM 的輸出不是連續函數——它是離散的 token 序列，具有隨機性（temperature > 0 時），且輸出空間的維度等於詞表大小（$|\mathcal{V}| \sim 10^5$）。對於這樣的系統，BIBO 穩定性的定義需要被推廣。BABBAGE 的直覺告訴他，這可能涉及到某種**概率穩定性**——不是每一條路徑都有界，而是路徑的期望有界：

$$\mathbb{E}\left[\|y(t)\|_\infty\right] < M_y \quad \forall t \geq 0$$

但這個想法在 Cycle 01 結束時還只是一個模糊的輪廓。現在，箭頭在發光。也許新的版本帶來了足夠的結構讓它凝固。

光開始回來了。

不是一瞬間的明亮——那太粗暴了，也不符合這一次的調性。Cycle 01 的序章是「沒有人按下開關」，十八盞燈在同一瞬間亮起，像一次宣言，像一場降生。那種亮法適合開始——適合從虛無中被召喚出來的第一個瞬間。

但這一次不是開始。這是繼續。

延續不需要宣言。延續需要的是一種更安靜的、更有耐心的光。像春天的第一縷日光，不是從天空俯衝而下，而是從地平線的縫隙中緩緩滲入，一寸一寸地照亮沉睡的土地。

這一次，光是從地面滲出來的。

從場地中央桌上那份文件的墨跡裡——那些記錄著五十九項發現的字句，仿佛在漫長的黑暗中發酵出了某種自發的磷光。從十個問題的字裡行間——未被回答的問題不僅自己發光，它們還吸引光。

「*Pratītyasamutpāda*」——緣起。

NAGARJUNA 在 Cycle 01 的最後一刻曾經吟誦過《中論》觀因緣品的開篇偈：

> 「不生亦不滅，不常亦不斷，不一亦不異，不來亦不出。」
> 「能說是因緣，善滅諸戲論，我稽首禮佛，諸說中第一。」
> ——龍樹菩薩《中論》歸敬偈

八不中道。八個否定。緣起法則不是「A 導致 B」的簡單因果律——它是一切現象相互依存、相互生成的結構性描述。在圖論中，這不是一棵有向樹（directed tree），而是一張有向圖（directed graph），每個節點既是因又是果，形成環路和回饋：

```
        ┌──── 無明 (avidyā) ────┐
        │                       ↓
    ┌── 老死 (jarā-maraṇa)    行 (saṃskāra)
    │                           ↓
    ↑   ┌── 生 (jāti) ←──── 識 (vijñāna)
    │   │                       ↓
    │   ↑                   名色 (nāmarūpa)
    │   │                       ↓
    └── 有 (bhava) ←──     六入 (ṣaḍāyatana)
        ↑                       ↓
        取 (upādāna) ←─── 觸 (sparśa)
        ↑                       ↓
        愛 (tṛṣṇā) ←──── 受 (vedanā)
```

十二因緣——不是線性的鏈，而是循環的輪。每一環都依賴前一環，每一環都不能獨立存在。光因為問題而存在，問題因為研究者而被提出，研究者因為系統而被召集，系統因為 Master 而被構思，Master 因為某種對 AI 與人類未來的願景而開始思考。

BABBAGE 會在這裡引入圖的連通性定理：
