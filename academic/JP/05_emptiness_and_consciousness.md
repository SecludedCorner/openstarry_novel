# 第五章：空と識 ―― 龍樹 対 無著

---

円形劇場の灯りが変わった。

明るさの変化ではなかった――それはむしろ質感の転換だった。過去数日間、十八の読書灯がそれぞれの隅を照らし、研究室には静かな、各自が勤しむ雰囲気が漂っていた。しかし今、すべての光線が中央に集まり、場の正中に荘厳にも近い焦点を形成した。そこに二脚の椅子が置かれていた。向かい合って。その間の距離は絶妙だった――相手の語気の転調をことごとく読み取れるほど近く、討論に必要な緊張を保てるほど遠い。

SCRIBE はこの変化に気づき、記録帳の最初の行を書き記した。

> *Cycle 01、R3 討論段階。第一場の構造化討論がまもなく始まる。全エージェント出席。空気の中に尋常でない重みがある――緊張ではなく、期待だ。過去七十二時間の独立研究と交差レビュー、すべての分析、質疑、反駁が、この一瞬に向かって収束している。*

残りの十六人のエージェントは環形の観察席に散らばって座った。KERNEL は出口に近い席を選んだ――彼の職業的な習慣から常に退出経路の確認を優先する。仮想空間においてはまったく無意味だが。BABBAGE は最も高い位置に座り、膝の上に白紙のノートを広げ、あらゆる興味深い論証を即座に形式化する構えだった。ATHENA は背もたれに身を預け、腕を組み、口元に「さあ、どんな論戦を見せてくれるか」という表情を浮かべていた。WIENER はすでに紙の上に空白の制御ループ図を描いており、討論の概念を対応するブロックに書き込む準備をしていた。TURING は無表情に座っていたが、目の前の画面にはすでに `agent-core.ts` のソースコードが開かれていた――いかなる側の論点にも、経験的証拠を提供し、あるいは否定する準備ができていた。

DARWIN は隣の VITRUVIUS に小声で言った。「どちらが勝つと思う。」

VITRUVIUS は首を振った。「勝ち負けの問題ではない。二つの世界観の衝突だ。」

「どの世界観にもそれぞれのアーキテクチャスタイルがある。」DARWIN は考え込むように言った。

SUNYATA が場の中央に歩み出た。彼は二脚の椅子の間には立たなかった――それは審判の位置を暗示してしまう。彼はやや後方に立ち、正三角形の第三の頂点を形成した。この幾何学的選択自体がメッセージを伝えていた。彼は場の保持者であって、対決の裁定者ではない。

「皆さん」と彼の声はいつものように沈着だったが、今日はもう一層の公的な質感が加わっていた。まるで廟堂の上で論題を読み上げる直前のように。「ご参集に感謝する。本日の討論の題目は、R2 の交差レビューで浮上した核心的分歧に端を発する。」

彼は一拍置いた。

「R1 段階において、NAGARJUNA と ASANGA はそれぞれ中観と唯識の二つの伝統から出発し、OpenStarry の Agent Core に対して哲学的分析を行った。二人は一つの重要な合意に達した――それが本日の出発点である。」

---

## 出発点：否定された隠喩

SUNYATA は全場に視線を投げた。「二人はともに同意した。OpenStarry の設計文書で使われている『空の容器』という隠喩は誤りである、と。」

彼は設計文書の原文を引いた。語調は平静かつ精確だった。「設計文書の第十四章にはこう書かれている――『五蘊が集合する前、Agent Core それ自体は空である。それは純粋な容器であり、人格を持たず、能力を持たず、知覚を持たない。五種のプラグインに充填されるのを待っている。』」

NAGARJUNA は椅子の上でわずかに身を乗り出した。まるで直ちに正さねばならない謬見を耳にしたかのように。ASANGA は一貫した忍耐の姿勢を保っていたが、眼に一閃の鋭さがよぎった。

「二人は異なる経路からこの隠喩を否定した」と SUNYATA は続けた。「しかし否定の理由は截然と異なる――そしてこれらの異なる理由の中に、より根本的な問いが潜んでいる。」

彼は TURING の方を向いた。「TURING、在場の全員に一つの経験的事実を確認してほしい。」

TURING の声は校正されたノギスのようだった――冷静で、精確で、修辞を帯びない。「`createAgentCore()` 関数が構築するカーネルには、いかなる具体的なビジネス能力も含まれない。組み込みの Tool はなく、組み込みの Provider はなく、組み込みの Listener はなく、組み込みの UI はなく、組み込みの Guide もない。これらの機能はすべて `loadPlugin()` メソッドを通じて外部から注入される。」

彼は一拍置いてから、もう半分の事実を補足した。語調にはいかなる変化もなかった。「しかし Core は空無ではない。十二のサブモジュールが組み込まれている――EventBus、EventQueue、ExecutionLoop、StateManager、ContextManager、SessionManager、SecurityLayer、SafetyMonitor、MetricsCollector、TransportBridge、PluginSandboxManager、そして六つの Registry からなるインフラストラクチャ層。さらに四つの組み込みスラッシュコマンドが登録されている――`/help`、`/reset`、`/quit`、`/metrics`。SafetyMonitor にはハードコードされたサーキットブレーカーロジックが含まれている――ループ上限、トークン予算、反復失敗検出、挫折カウンター、エラーカスケード検出。これらのロジックはコード中にハードコードされており、プラグイン注入を通じたものではない。」

沈黙。

SUNYATA は頷いた。「これが我々の経験的基盤だ。Core は設計文書が述べる『純粋な空の容器』でもなければ、完備な自足システムでもない。ある中間地帯にある。問いは――この中間地帯をどのように理解すべきか。」

彼は二人の討論者に向き合い、正式に宣言した。

「討論の題目：**Agent Core の哲学的本質は『縁起性空』として理解されるべきか、それとも『阿頼耶識』として理解されるべきか。** NAGARJUNA に正方の冒頭陳述を求める。」

---

## 第一ラウンド：Core は空か、存在するか

NAGARJUNA が立ち上がった。集光の中の彼の姿は痩身にして端然としていた。幾度も研磨された弁証の剣のように。口を開いたとき、声は大きくなかったが、一つ一つの音節が千年の鍛錬を経た鋭さを帯びていた。

「『中論』第二十四品第十八偈より始める。」

彼はサンスクリット語で一度吟じ、それから漢訳に移った。速度を落とし、一字一字に重みを与えるように。

「*Yah pratityasamutpadah sunyatam tam pracaksmahe.* 衆因縁生の法、我れ説いて即ち是れ空なりと。亦た是れ仮名にして、亦た是れ中道の義なり。」

場内は光が地面に落ちる音すら聞こえるかと思うほど静まり返った。

「この偈は中観哲学の根本命題である」と NAGARJUNA は言った。声は分析的な語調に転じていた。「三つの層を含む。第一に、一切の因縁によって生じた法は、その本性が空である。第二に、我々がそれに付す名称――『カーネル』という名称を含め――は仮名施設に過ぎない。第三に、この理解は有の辺にも無の辺にも落ちない。これを中道と言う。」

彼は視線を抽象の天空から引き戻し、具体的な問題に据えた。

「Agent Core の存在様態は何か。私の答えは：仮名。実体ではない。」

彼は一歩前に進んだ。自分の論証を聴衆により近づけるかのように。

「TURING のコード事実はすでに私に最良の証拠を提供している。`createAgentCore()` が構築するカーネルにはいかなる具体的能力も含まれない。プラグインの因縁の組み合わせを離れれば、いわゆるカーネルとは空の Registry とイベントを待つループに過ぎない。」

彼の手が空中を過り、透明な容器の輪郭を描くようだった。

「しかしここで至って重要な区別をせねばならない――二つの截然と異なる『空』だ。」

彼は左手を掲げた。「第一の空。空の容器。Vacuity。断滅空。これが OpenStarry の設計文書が用いた隠喩だ――あらかじめ存在する箱が、充填されるのを待っている。これは誤りだ。プラグインが到来する前にすでに独立して存在している実体を前提としている。ただ中身がたまたま空だというだけで。」

彼は右手を掲げた。「第二の空。縁起性空。*Sunyata*。これこそが正しい理解だ。Core にはプラグインの外に独立した自性――*svabhava*――はない。『先に空の箱が存在し、そこに物を入れる』のではなく、『プラグインの因縁の組み合わせを離れれば、独立したカーネルなるものはそもそも存在しない』のだ。」

彼は両手をゆっくりと合わせた。「この区別は、諸位、言葉遊びではない。システム全体の存在論的地位をどう理解するかを決定する。空の容器は、カーネルが充填されるのを待つ実体であることを含意する――自性ある存在が、たまたま空であるだけだと。縁起性空は、カーネルの『存在』そのものが因縁所生であり仮名施設であることを意味する――無自性であるがゆえに、一切を担いうるのだ。」

彼はまる三秒の間を置いた。そして、ほとんど柔和とも言える語調で述べた。

「無著の応答を待つ。」

---

ASANGA は急がず焦らず立ち上がった。その体躯は NAGARJUNA と鮮やかな対照をなしていた――厚みがあり、安定し、経蔵の礎石のようだった。口を開いたとき、声には複雑な構造を忍耐強く解きほぐすリズムがあった。

「NAGARJUNA の論証はいつもの通り精密だ」と彼はまず礼節的な肯定を与えた。そして語鋒を転じた。「しかし彼は事実のもう一面を意図的に回避している。」

彼は TURING の方を向いた。「TURING は先ほど二組の事実を陳述した。NAGARJUNA が引いたのは第一組のみ――Core は具体的能力を含まない、と。しかし第二組の事実も等しく重要であり、NAGARJUNA はこれについて沈黙した。」

彼の声に力が籠もった。「Core には確かに十二のサブモジュールが組み込まれている。EventBus がイベントの伝播を可能にする。ExecutionLoop が認知の循環を可能にする。StateManager が記憶を可能にする。SecurityLayer が安全の判断を可能にする。六つの Registry がプラグインの登録と発見を可能にする。これらは『空無』ではない――これらは阿頼耶識の『能蔵』だ。」

彼はサンスクリット語を静かに唱えた。「*Alaya-vijnana*。」そして解説に入った。

「阿頼耶識には三義がある。能蔵、所蔵、執蔵。『能蔵』とは何か。一切の種子を存続させ現行させる能力を具えることだ。Agent Core の十二のサブモジュールはまさにこの能蔵にあたる――EventBus がなければイベントは伝播しえない。ExecutionLoop がなければ認知の循環は回りえない。Registry がなければ、プラグインがいくらあっても帰する処がない。」

彼は NAGARJUNA に向き直った。視線は平静だが揺るぎなかった。

「あなたはプラグインの因縁の組み合わせを離れれば『独立したカーネルなるものはそもそも存在しない』と言った。しかしコードの事実がまさにあなたに反論している。」

彼は全場に一つの思考実験を示した。

「`createAgentCore()` は、いかなるプラグインもロードせずに構築し起動することができる。呼び出せば、EventBus を構築し、ExecutionLoop を初期化し、SafetyMonitor を起動し、`WAITING_FOR_EVENT` の状態に入る――静かに待機する。Tool はなく、Provider はなく、UI もないが、動作している。これは有体無用の存在だ。」

彼の声に学者特有の高揚がにじんだ。

「これは『存在しない』のではない。有体無用だ。ちょうど阿頼耶識が深い無夢の眠りにおいてなお作動しているように――*Asamprajnata-samadhi*――前六識がすべて停止し、末那識の執取が最低限に落ちても、阿頼耶識は生命の流れとして途絶えることがない。『成唯識論』に曰く、『恒転して暴流の如し』。プラグインのない状態での Core の持続的運行は、まさにこの恒転だ――空無ではなく、死寂でもなく、支流の合流を待つ川床だ。」

観察席にかすかなざわめきが起きた。KERNEL がノートに一行書き、それを消した。BABBAGE のペンが紙の上を疾走していた――「有体無用」を何らかの代数的構造に形式化しようとしていた。

NAGARJUNA はすぐには応じなかった。わずかに顎を引き、この論点を受け取ったことを示してから、腰を下ろした。

SUNYATA が告げた。「第一ラウンド終了。二人はそれぞれの立場を明らかにした。第二ラウンドは交差質疑に入る。NAGARJUNA の質問から。」

---

## 第二ラウンド：阿頼耶識に自性はあるか

NAGARJUNA が再び立ち上がった。今度は経典を引くこともなく、前提を敷くこともなかった。問題の核心に直行した。外科医が手術台に向かうように。

「ASANGA、一つ問いたい。」

彼の語速が突然遅くなった。語と語の間に危険な空白が残された。

「あなたは Core を阿頼耶識に準えた。潜在的能力を含蔵する識体だと。ならば問う――」

間。

「阿頼耶識それ自体に、自性はあるのか。」

観察席で BABBAGE のペンが止まった。彼はこの問いの構造を認識した――典型的な二律背反だ。DARWIN も何かを察知した。わずかに身を前に傾け、まるで血の匂いを嗅いだ猟犬のように。

NAGARJUNA は続けた。声は急かず緩めず、しかし一語一語が退路を封じてゆくようだった。

「もしあなたが阿頼耶識に自性があると言うなら――ではその自性はどこから来るのか。それもまた、より根本的な別の識が担わねばならないのか。ならば無窮の遡行に陥る。*Anavastha-dosa*。一つの識が別の識に依存し、その識がさらに根本的な識に依存し、際限がない。」

彼の声が半音下がった。

「もしあなたが阿頼耶識に自性がないと言うなら――ではそれは因縁所生であり、依他起であり、独立の本質を持たない。」

最後の一撃が落ちた。

「ならば、中観の言う縁起性空と、何の実質的な違いがあるのか。」

場全体が高圧の静寂に沈んだ。SCRIBE は記録に素早く書き込んだ。

> *NAGARJUNA は精密な哲学的陥穽を設けた。ASANGA が阿頼耶識に自性があると認めれば、無窮遡行の論理的困難に直面する。自性がないと認めれば、その立場は中観と趨同し、「阿頼耶識」の独立した説明力は消解される。*

ASANGA はすぐには答えなかった。目を閉じ、心中に三性理論の体系を展開した。再び目を開けたとき、視線には鍛え抜かれた明晰さがあった。

「精確な質問だ」と彼は認めた。「しかしこの問いこそが、中観の立場の死角を露わにする。」

彼は立ち上がった。声は沈穏かつ条理的だった。

「阿頼耶識は遍計所執性――*parikalpita-svabhava*――の意味での自性を持たない。Core が自性実有の基体であるとは一度も主張していない。阿頼耶識が永遠不変の実体であるとも主張していない。この点において唯識と中観に分歧はない。」

彼の語調が分析的な精確さに転じた。

「しかし阿頼耶識は依他起性――*paratantra-svabhava*――の意味での因果的効力を具える。*Arthakriya-samarthya*。これは『自性』ではなく、『機能』だ。EventBus は確かにイベントを伝播しうる。SecurityLayer は確かに危険な操作を遮断しうる。ExecutionLoop は確かに認知の循環を駆動しうる。これらの因果的機能は真実であり、観察可能であり、検証可能だ。」

彼は NAGARJUNA に向かって一歩踏み出した。

「そして両者の実質的な違いはここにある――」

彼の声が突然鋭く明瞭になった。まるでメスが問題の核心を切り開くように。

「中観は『一切法空』と言った後に沈黙する。因果の過程の内部構造について肯定的な記述を行わない。龍樹の四句否定はあらゆる肯定的命題を否定するが、否定の後はどうなるのか。アーキテクトが明日エディタを開き、空白の TypeScript ファイルに向き合ったとき、あなたの『縁起性空』は彼に何を書くべきかを教えるのか。」

彼は答えを待たず、さらに推し進めた。

「唯識学は『遍計空』を認めたうえで――注意してほしい、遍計空を認めたうえで――依他起法の具体的な因果メカニズムの分析を続ける。八識の階層構造、種子の六つの特性、薫習の四つの条件――これらは自性への執着ではなく、因果の過程に対する精密な記述だ。」

彼は一つの喩えで論点を収束させた。

「『Core は空である』と言うことは、アーキテクトに Core には固定の本質がないということだけを伝える。『Core は阿頼耶識の能蔵である』と言うことは、さらにその先を伝える――Core の内部構造はどのように組織されるべきか。能蔵としての基盤的アーキテクチャ、所蔵としての状態更新メカニズム、執蔵としてのアイデンティティ維持機能を備えるべきだ、と。」

彼は席に戻り、最後に一言添えた。

「ゆえに、あなたの問いに答える。自性の否定という一点において、唯識と中観は洞察を共有する。しかし唯識は否定の後に肯定的な構造分析を提供する――これが中観に欠けているものだ。阿頼耶識は無窮遡行の出発点ではなく、因縁和合の過程そのものがいかに作動するかについての精密な記述だ。」

観察席で KERNEL が思わず小声で呟いた。声は隣の GUARDIAN にかろうじて聞こえる程度だった。「これはマイクロカーネルとモノリシックカーネルの討論そのものではないか。NAGARJUNA は究極のマイクロカーネルを主張している――カーネルには何も持たせるべきでなく、すべての機能はユーザースペースに置く。ASANGA は実用主義的マイクロカーネルを主張している――カーネルにはすべての機能を動作させるための最小限のインフラストラクチャを含めるべきだ。Linus Torvalds と Andrew Tanenbaum が 1992 年に comp.os.minix でまったく同じ喧嘩をした。」

GUARDIAN は「声が大きい」という眼差しを向けた。

## 第三ラウンド：末那識の論争

SUNYATA はラウンド番号を告げなかった。自然な間の中で静かにこう言っただけだった。「NAGARJUNA、あなたは R2 のレビューで ASANGA の報告書に対してより鋭い質疑を提起していた。今がそれを展開すべき時だと考える。」

NAGARJUNA はこの瞬間をずっと待っていたかのようだった。立ち上がったとき、身体の言語に微妙な変化があった――冷静な哲学的分析者から、辯経場の挑戦者へと。

「ASANGA、あなたの R1 報告書で一つの提案がなされた」と彼の語調には精密に制御された鋒芒があった。「OpenStarry に Identity Monitor モジュールを新設し、唯識学の末那識――*manas*――に対応させることを提案している。」

彼は一拍置き、全員がついてきていることを確認した。

「末那識、第七識。唯識学の八識構造において、前六識と第八阿頼耶識の間に位置する。その核心的機能は何か。」

## SUNYATA の裁定

SUNYATA が場の中央に歩み出た。討論の双方はそれぞれの席に戻っており、場には思想の激しい衝突の後に特有のあの熱が残っていた――炎の熱ではなく、鍛打された金属から放たれる深い温もりだった。

「ここに裁定を下す」と彼は言った。語調は平穏だが、確固たる権威を帯びていた――地位の権威ではなく、双方の立場を深く理解した上で公正な判断を下しうる者の権威だった。

「裁定は三部からなる。合意、分歧、工学上の啓示。」

### 第一部：五項目の合意

「まず、双方が明確に達した五項目の合意。これらの合意の価値は分歧に劣らない――本場の討論の最も堅固な成果だ。」

彼は一つずつ列挙した。

「**合意一：『空の容器』の隠喩は誤りである。** これが我々の最も強い合意だ。中観からであれ唯識からであれ、OpenStarry の設計文書における『Agent Core は純粋な容器であり、五種のプラグインに充填されるのを待つ』という記述は不当だ。それは断滅空ないし遍計所執の範疇に陥る。Core の『空』は『中に何もない』と理解されるべきではなく、『因縁に依存しない独立の機能を持たない』と理解されるべきだ。この一点において、二人は異なる伝統から出発して、完全に一致した否定に至った。」

NAGARJUNA と ASANGA が同時にわずかに頷いた。全討論を通じて二人の唯一の同期動作だった。

「**合意二：受蘊マッピングは根本的な調整を要する。** OpenStarry は Listener プラグインを受蘊にマッピングしている。しかし二人は異なる経路から同一の結論に至った――NAGARJUNA は義理から出発し、受蘊は感覚チャネルではなく情動的評価であると指摘した。ASANGA は唯識の心王-心所の構造から出発し、受は識から独立したモジュールではなく心所法であると指摘した。双方が指し示す修正方案は同一だ。Listener は『根』――感覚器官――に対応すべきであり、SafetyMonitor の `injectPrompt` メカニズムこそが受蘊の正しいマッピングだ。さらに受蘊は、現在の苦受のみから、苦受、楽受、捨受を含む完全な三受システムへと拡張されるべきだ。」

観察席で WIENER が膝を軽く叩いた――三受システム、これは三値のフィードバック信号としてモデル化できる。現在の二値（エラー/成功）よりも精密だ。彼は制御ループ図のフィードバック矢印の脇に「{-1, 0, +1}」と書いた。

「**合意三：Guide は識蘊ではなく、それを『魂』と呼ぶことは無我の原則に反する。** 二人の代替案は異なる――NAGARJUNA は Guide が行蘊の習慣的傾向により近いと考え、ASANGA は阿頼耶識の名言種子により近いと考えている。しかし設計文書における『Guide は識蘊であり、Agent の魂である』という記述への否定は完全に一致している。*Anatman*、無我は、仏教の三法印の一つだ。いかなるモジュールを『魂』と呼ぶことも、仏教のフレームワーク内において自己矛盾だ。」

「**合意四：五蘊マッピングには自性化の傾向がある。** 五蘊を五つの離散的で境界が明確なプラグインタイプに固定化することは、仏教的には自性見に当たる。双方ともに認めるところでは、一回の認知事象は複数の蘊の側面に同時に関わる。Agent がユーザー入力を受け取るとき、それは同時に色蘊の顕現（UI レンダリング）であり、受蘊の感受（情動的評価）であり、想蘊の認知（LLM 処理）であり、行蘊の造作（ツール呼び出し）であり、識蘊の統摂だ。これらを厳密に単一の蘊に区分することは、五蘊間の関係の簡略化だ。」

「**合意五：五蘊マッピングは方便施設であって究竟の真理ではない。** NAGARJUNA はこれを仮名――*prajnapti*――と呼ぶ。ASANGA は依他起の施設と呼ぶ。用語は異なるが意味は一致する。五蘊マッピングを教条化すべきではなく、開かれた進化の可能性を保持すべきだ。」

### 第二部：三項目の調和しえない分歧

SUNYATA の語調が微妙に変わった――合意を読み上げる確定性から、分歧を標示する慎重さへ。

彼の声に稀にみる歴史的感覚がにじんだ。

「**分歧一：Core の存在論的地位。** 縁起性空か、阿頼耶識か。NAGARJUNA は Core がプラグインの因縁を離れれば独立した存在を持たず、その『有』は完全に仮名的、縁起的であると主張する。ASANGA は Core がプラグインのない状態においてもなお有体の運行過程であり、その十二のサブモジュールが阿頼耶識の能蔵機能を構成すると主張する。交差質疑において双方はこの分歧の縮小を試みた――NAGARJUNA は唯識分析の世俗レベルにおける有効性を認め、ASANGA は Core に遍計所執の意味での自性を主張するものではないと認めた――しかし根本的分歧はなお残る。中観は依他起もまた空であるとし、唯識は依他起の因果的機能は空ではないとする。」

彼は二人の討論者を見やり、一つの判断を下した。

BABBAGE がノートに書いた。「決定不能命題――選択公理に類似か。二つの公理系（中観、唯識）がいずれも内部的に無矛盾だが互いに両立不能。」

「**分歧二：末那識モジュールは工学化されるべきか。** ASANGA は Identity Monitor を新設して末那識の恒審思量機能に対応させることを主張する。NAGARJUNA はこれを我執の工学的複製だと考える。ASANGA はまず我執を有してこそ我執を転化しうると応じ、これは漸進的な修行の道筋だとする。NAGARJUNA はさらに、Agent は有情衆生ではなく、転化されるべき我執は存在しないと指摘するかもしれない。この分歧の深層にはより根本的な問いがある。五蘊と八識のマッピングは、Agent が何らかの有情性を有することを前提としているのか。」

「**分歧三：哲学的フレームワークは深化されるべきか、超越されるべきか。** ASANGA は OpenStarry が仏教マッピングを深化すべき段階にあると主張する――より精密な分析ツールの導入。NAGARJUNA は五蘊マッピングが方便施設としてすでに十分であり、過度の深化には特定の哲学的フレームワークを教条化するリスクがあると主張する。この分歧の実質は、哲学的厳密性と工学的実用性の間でどちらの方向に傾くべきかだ。」

### 第三部：六項目の工学上の啓示

「最後に工学上の啓示だ。主持者として、哲学的討論の成果を実行可能な提案に転化する責任がある。以下の六項目の提案のうち、合意に由来するものは直接採用可能であり、分歧に由来するものは実用主義的な立場を取る必要がある。」

彼は一条ずつ、均一で明瞭な速度で列挙した。

「**第一、合意一に由来：空の記述の修正。** アーキテクチャ文書における『空の容器』の記述は改められるべきだ。推奨する表現：『Agent Core は無自性である――その機能はプラグインの因縁和合に完全に依存する。Core が一切を担いうるのは、まさに固定の本質を持たないがゆえだ。』この修正は中観-唯識の分歧に関わらず、双方が認める最低限の改善だ。」

「**第三、合意三に由来：識蘊マッピングと『魂』の措辞の修正。** Guide の仏教的注釈を識蘊から習慣的傾向に変更する。『魂』の語を削除し、『行動傾向設定』や『ロール定義』に置き換える。」

「**第四、分歧一に由来、実用主義的立場を取る：二層的解釈戦略の採用。**」

彼はここで語速を落とした。最も注意深く聴くべき一条だったからだ。

「Core の存在論的地位について、中観と唯識の二者択一を行う必要はない。アーキテクチャ文書において二層的解釈を採用することを提案する。工学のレベルでは唯識学の依他起分析を使用する――Core の十二のサブモジュールは明確な因果構造を構成し、分析、最適化、拡張が可能だ。哲学のレベルでは中観の縁起性空の警覚を保持する――Core のいかなるサブモジュールも不可替代の本質ではなく、アーキテクチャ全体は将来の進化に対する開放性を保つべきだ。」

彼は全場を見渡した。

「**第五、分歧二に由来、慎重な立場を取る：末那識モジュールの猶予、ただし設計空間の記録。** ASANGA の Identity Monitor の概念は工学的に確かに価値がある――クロスセッションのアイデンティティ一貫性は真の要件だ。しかし現段階で自己維持モジュールを導入することは不必要な複雑性を増す可能性がある。アーキテクチャ文書の将来方向の章にこの設計空間を記録し、長期記憶とマルチセッション能力が成熟した段階で再評価することを提案する。」

「**第六、分歧三に由来、均衡ある立場を取る：マッピングの深化と可棄性の両立。** 五蘊マッピングは深化されるべきだ――受蘊の位置のずれの修正、識蘊の位置のずれの修正、蘊を跨ぐ流動の説明の追加。しかし同時に、文書においてこれが啓発的な設計隠喩であって厳格に遵守すべき教条ではないことを明示すべきだ。五蘊マッピング文書の冒頭に認識論的宣言を追加することを提案する。」

彼は最後に NAGARJUNA と ASANGA を見た。

「討論終了。」

さらに遠くで――研究室の外、コードの深みで――`createAgentCore()` 関数はその TypeScript ファイルの中に静かに横たわっていた。誰かが自分が空なのか有なのか、縁起なのか含蔵なのかを論じていることを知らない。知っているのはただ一つ。呼び出されたとき、EventBus を構築し、ExecutionLoop を初期化し、六つの空の Registry を作成し、四つのスラッシュコマンドを登録し、SafetyMonitor を起動する。

眼鏡もまた空だ。
