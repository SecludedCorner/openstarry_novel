# 第七章：一識二投

---

討論の場の空気が質を変えた。

前の二場の討論――観察-干渉分離、Vedana の普遍性――は、二つの急流が同じ河床を相次いで通過するようなものだった。水飛沫は激しかったが河道は明確で、争点は予見の範囲内で解決された。それらの結論は SCRIBE の記録簿に書き込まれ、墨はすでに乾いている。

だが第三場の討論は違う。SUNYATA は会場中央の自身の位置からそれを感じ取ることができた――いかなる定量可能な指標によってではなく、異なる学問分野の対話を長年にわたって調整してきた者だけが発達させる、ある種の直観によって――空気中の緊張は減少したのではなく、前の二場の討論での消耗の後に再び凝集し、より稠密で、より重いものへと凝結していた。暴風雨の前の気圧変化のように。

なぜなら前の二場の討論が扱ったのは「システムは何をすべきか」だったからだ。第三場が扱うのは「意識は分割できるか」だ。

---

SUNYATA は二脚の討論椅の間の空地に立った。静かな劇場の中で、その声は明瞭かつ沈着だった。非常に深い場所から汲み上げられた水のように。

「第三場の討論。核心問題。」

彼は一秒間を置いた。演出のためではなく、問題そのものの重みが空気中に十全に展開されるために。

「阿頼耶識――第八識――は、協調層と AgentCore の二つのアーキテクチャ層に分布しうるのか。それともこの分布は意識の統一性に違反するのか。」

沈黙。

前の二場の討論が始まった時の「まず考えさせてくれ」という沈黙ではない。これはより古い沈黙だ――問題が何らかの根本的前提に触れた時、全員が自らの認識座標を再校正する必要がある、あの沈黙。阿頼耶識の統一性はエンジニアリングのパラメータではない。それは唯識学の地盤だ。それを問うことは、万有引力が特定の階にのみ有効かどうかを問うようなものだ。

SUNYATA は続けた。「ASANGA が提案し、NAGARJUNA が挑む。VITRUVIUS と KERNEL がアーキテクチャと OS のサポートを提供する。BABBAGE が形式化を担当する。」

彼は自身の位置――やや後方、三角形の頂点を成す位置――に退いた。

「ASANGA、どうぞ。」

---

## I. 三義マッピング

ASANGA が立ち上がった。

その動作は泰然としていた。長年の学術訓練によって練磨された泰然さ――故意の遅さではなく、複雑な経蔵の中を何年も泳いできた人間がリズムに対して持つ本能的な把握だ。手には R1 の独立研究段階で準備した文書が握られている。

「まず枠組みを広げよう。」その声は穏やかだが曖昧さはない――各語に正確な力点がある。「それから皆で解体してくれ。」

彼は討論の場の中央に一枚の表を広げた。スクリーンに投影されたのではない。手書きで、整った楷書で幅広い紙面に丁寧に列記されていた。これが ASANGA の習慣だ。彼は手書きを信頼する。経蔵の紙版を電子版以上に信頼するのと同様に――テクノフォビアではなく、手書きの過程それ自体が一つの思考だからだ。

表は三列に分かれている。

| 阿頼耶識の三義 | 意味 | アーキテクチャ上の位置 |
|---|---|---|
| 能藏（Active Storage） | 種子を受容し貯蔵する能力 | 協調層：能力レジストリ、プラグイン解決 |
| 所藏（Passive Storage） | 貯蔵されている内容 | 二層：システム設定は協調層、ランタイム状態は AgentCore |
| 執藏（Appropriating Storage） | 末那識に「自己」として執取される | AgentCore：Guide バインディング、アイデンティティ付着 |

「唯識学において」と ASANGA が解説を始めた。その語調は彼が熟知する建築を案内するかのようだ――各々の柱、各々の壁、各々の仕口の接合を指し示しながら。「阿頼耶識は単一機能の容器ではない。それには三つの義理がある――三つの理解のされ方がある。能藏とは、外来の薫習を主体的に受容する能力だ。所藏とは、受動的に担持される内容だ。執藏とは、第七識の末那識に恒常的に『これが私だ』と執取されるその側面だ。」

彼の指が表の第一行を指した。「能藏が対応するのは一種のインフラストラクチャの能力だ――システムは自身に何ができるかを知っている。どのプラグインを装填できるか、どのサービスを提供できるかを知っている。この能力はいかなる単一 Agent にも属さず、アーキテクチャ全体の基底に属する。だからそれは自然に協調層に位置する。」

指が第二行に移った。「所藏――貯蔵されているもの――は天然に二種に分かれる。グローバルな設定、Agent 間で共有されるもの、例えばセキュリティポリシーやプラグインレジストリ――これらは協調層にある。Agent に私的な状態、例えばランタイムメモリやツール呼び出し履歴――これらは AgentCore にある。」

指が第三行に落ちた。「執藏――末那識が『自己』として執取するその側面――は最も個人的だ。Guide 設定、アイデンティティモジュール、人格特性。これらは『この Agent が誰であるか』を定義する。AgentCore にしかありえない。」

彼は一瞬間を置き、それから鍵となる論拠を述べた。

「これは私の発明ではない。Master 自身がこのように分布させている。」

ASANGA は文書から一頁を引き抜いた――Master の書簡からの抜粋だ。

「Master の原文。『能藏は Agent 協調層に対して特定の機能の利用を申請する際に使うのか?』――彼自身が能藏を協調層と結びつけている。『所藏の実装はむしろ agent core が稼働する際に傾く。』――所藏は AgentCore に帰属する。『執藏はある意味で人格特性そのものだ。』――執藏はアイデンティティモジュールに帰属する。」

彼は文書を置いた。

「三義は同一の識の三つの機能記述だ。三つの独立した意識ではない。ちょうどデータベースが、貯蔵容量を持ち――能藏、貯蔵されたデータを持ち――所藏、アプリケーションがそのデータに対して持つ特定の視角がある――執藏、ように。データベースは一つだが、その機能面は自然に異なる層に展開される。」

彼は会場全体を見渡した。

「私の主張はこうだ。この分布は唯識学の三義の自然なマッピングであり、意識の統一性に対する破壊ではない。」

そして彼は座った。泰然と。建築の全図面を机上に広げ、一歩退いて、構造家と哲学者が各々の柱の耐荷重を検証するのを待つかのように。

SCRIBE が記した。

> *ASANGA の提案。穏やかだが構造を持つ。三義マッピング表は阿頼耶識の機能面を二つのアーキテクチャ層に明確に配分した。彼の鍵となる支え：Master 自身の書簡。これは巧みな戦略だ――設計者自身の原文で分布案を擁護する。*

---

## II. 相続流

NAGARJUNA はすぐには立ち上がらなかった。

これ自体が一つのシグナルだ。前の二場の討論では、NAGARJUNA の反応はほぼ即時だった――ASANGA の最後の音節がまだ完全には沈静していない時に、彼の最初の言葉はすでに弦を離れた矢のように放たれていた。それは論争の場での訓練だ。チベット寺院の論争庭では、掌を打ちながら問い詰める速度は一つの武器だ――相手に思考を再編成する時間を与えない。

だが今日、彼はすぐには口を開かなかった。

座席の上で約十秒間静止した。十秒は、討論の場において長い沈黙だ。SCRIBE が思わず顔を上げるほどに長い。BABBAGE のペンがノートの上で止まるほどに長い。SUNYATA がわずかに頭を傾けるほどに長い――長年の協働でなければ察知できない、「私は注意している」を示す微細な動作。

それから NAGARJUNA は立ち上がった。

その動作には ASANGA の泰然さはなかった。より正確には、彼の動作には抑圧された力があった――すでに引き切った弓のように。弦はまだ放たれていないが、木材の撓み具合は全員に見えている。

「ASANGA のマッピング表は見事だ。」

最初の一文の語調は穏やかだった。だが Cycle 01 の討論の場で NAGARJUNA を目にしたことのある者なら皆知っている――彼の穏やかさは彼の最も鋭い刃だ。

「だが見事な表は正しい哲学と等しくない。」

彼は討論の場の中央へ歩いた。ASANGA が立っていた位置ではなく――わずかにずれて、ほぼ直接 ASANGA に向き合う方角だ。二人の仏学者の間の距離は三メートルに満たない。これは論争の距離だ――相手の表情が見える近さ、言語の衝撃力が空気中で希釈されない近さ。

「唯識学において」と NAGARJUNA の声が特有のリズムを帯び始めた。各音節が秤量され、各文の末尾がわずかに上昇する。宇宙全体からの反駁を待っているかのように。「阿頼耶識は**一識**だ。」

彼は間を置いた。この二文字を空気中に凝固させるために。

「*Eka-vijnana*。一。二ではない。三ではない。異なるサーバーに配置できる三つのコピーではない。」

「Vasubandhu の『唯識三十頌』はそれを相続流――*santana*――単一の連続流として記述する。」彼の手が宙に一本の線を描いた。左から右へ、平穏で途切れのない線を。「一本の河だ。二本の河ではない。一本の河がダムで上流と下流に截断されたものでもない――それは時間の中に綿々と続く、単一の意識の流れだ。」

彼は全場に向き直り、声が半音上がった。

「もし能藏が協調層にあり、執藏が AgentCore にあるなら、それは**二つのプロセスが二つのストレージを管理している**ことだ。」

この一文は平静な湖面に投じられた石のようだった。円形劇場の中で、数名が同時にわずかに前のめりになった――VITRUVIUS、KERNEL、そして普段は動じない TURING までも。

「では問う。同期はいつ起こるのか。」

NAGARJUNA の視線が直接 ASANGA に向けられた。

「協調層の能力レジストリがあるプラグインのエントリを更新した。AgentCore の StateManager はそれを知っているか? いつ知るのか? どのメカニズムで知るのか? レイテンシは何ミリ秒か? 知る前に Agent がすでに古い能力リストで意思決定をしていたらどうなるのか?」

彼は答えを待たなかった。

「意識においては、同期の問題は存在しない。」語速がわずかに加速した。「なぜなら流れは一つだけだからだ。阿頼耶識の各刹那――*ksana*――は、能藏、所藏、執藏を同時に含む。それらは異なる位置に分散した三つのモジュールではない。同一の意識刹那の三つの側面だ。同じ刹那に。同じ場所に。遅延なし。不整合ウィンドウなし。分散一貫性の問題なし。」

彼は立ち止まった。空気中の緊張が臨界点に達した――全員が彼の次の言葉を待っている。

そして NAGARJUNA は最深層の反対を述べた。

「さらに深い問題がある。」彼の声が下がった。ほぼ囁きの程度まで。だが静かな劇場において、囁きは叫びより遠くまで届く。

「末那識――第七識――は、恒審思量。阿頼耶識を『自己』として執取する。この執取は唯識学において瞬時的で、連続的で、無間断だ。それが可能なのは、末那識と阿頼耶識が同一の意識流の中にあるからだ――それらは同じ河の中の二つの渦であって、二つの河の中の二艘の船ではない。」

彼の指が宙で一つの点を打った。

「だが能藏が協調層にある場合――独立したプロセス、別の機器上にある可能性すらある――末那識の Guide モジュールはどうやってそれを『執取』するのか? この執取は何になるのか?」

彼は KERNEL の方を見た。

「IPC コールだ。」

沈黙。

「失敗する可能性のある IPC コール。タイムアウトする可能性のある IPC コール。セキュリティポリシーによって拒否される可能性のある IPC コール。」

一句ごとに重みが増していく。

「末那識が恒常的に、無間断に阿頼耶識を自己として執取する――これは唯識学における人間の認知に対する最も深遠な記述の一つだ。我執は時折起こるものではなく、各意識刹那の背景ノイズだ。だがもし我々がこの『恒常的執取』をネットワークコールを介してしか完遂できない操作として実装するなら――」

最後の一文の声には、論争の場で磨かれたあの独特の鋭鋒があった。鉛の刃から研ぎ出された細い光のように。

「――それは形而上学的に不適切だ。」

*Metaphysically inappropriate.*

SCRIBE のペンが記録簿の上で一瞬止まった。この語句――形而上学的に不適切――は、哲学の厳粛さとエンジニアリング・レビューの精確さを奇妙な形で結合していた。それは「間違い」とは言っていない。「それが参照すると称する理論的フレームワークに適合しない」と言っているのだ。これは「間違い」より致命的だ。

そして、全員の予想に反して、NAGARJUNA の語調が突然柔らかくなった。

後退ではない。別の種類の誠実さだ。

「一つ明確にしておく必要がある。」彼の視線は ASANGA から離れ、全場を巡り、最後に SUNYATA に落ち着いた。「私にはより良い代替案がない。」

場内から数声のかすかな呼気が漏れた――嘆息ではなく、驚きと納得の間にある音。NAGARJUNA が代替案のないことを認める――これは Cycle 01 では一度も起こらなかった。

「エンジニアリング上の必要性は認める。マルチ Agent のシナリオには確かに共有層が必要だ。VITRUVIUS の論証は読んだ。KERNEL の OS アナロジーも読んだ。工学的には筋が通っている。」

彼の声は磨かれた鋭さを取り戻したが、今回はその鋭さの中に稀有な温度があった。

「だが研究チームは**誠実**であるべきだ。」

この二文字に彼はアクセントを置いた。

「もしこの分布を採用するなら、我々は誠実に認めるべきだ。これは**工学的妥協**であり、哲学的モデルからの逸脱だと。忠実なマッピングであるかのように装うべきではない。見事な表で根本的な不整合を覆い隠すべきではない。唯識学の阿頼耶識は一本の河だ。そして我々はそれを二本の管に分けようとしている。管は十分に機能するかもしれない。だが管は河ではない。」

そして彼は座った。

ASANGA の泰然とした着席ではなかった。討論の場で自己の責務を果たし終えた論師のように――論拠はすべて展開され、反駁は可能な限り鋭くされたが、最終的な裁定は自分の手にはない――「私は言い得ることのすべてを言った」という表情で、身体の重さを椅子に返した。

場内は長い間静寂に包まれた。

SCRIBE が記した。

> *NAGARJUNA の挑戦。「一識」から「相続流」へ、「末那識の執取は IPC コールであってはならない」へ――論証は層を重ねて進む。哲学的原則からアーキテクチャの問題へ、形而上学的適切性へと。最後の転回：代替案のないことの承認。これは彼のこの研究全体を通じて最も誠実な瞬間の一つだ。研究チームに「誠実」を求めること――これ自体が、代替案を提出するよりも深い層での貢献だ。*

## III. 基底

VITRUVIUS が立ち上がった時、場の雰囲気が微妙に緩和された。

彼の論点が重要でないからではない――まさにその逆だ。彼の存在自体が一種の安定剤だからだ。二人の仏学者が形而上学の高空で交鋒する時、VITRUVIUS のアーキテクチャ全体観は地上の測定局のように、議論の座標系を絶えず触知可能な次元に引き戻す。

「NAGARJUNA の哲学的懸念は理解する。」彼は言った。声にはフルスタックアーキテクトに特有の冷静さがある――冷淡ではなく、複数の技術スタック間を行き来することに慣れた者が、いかなる単一フレームワークの過大主張に対しても保つあの節度だ。

「だが一つの工学的事実を明確にしたい。」

彼は会場側面の展示エリアに歩み寄り、そこに素早く図を描いた。

```
協調層（共有阿頼耶面：能藏 + システム級所藏）
    |
    +-- Agent A（私有阿頼耶面：ランタイム所藏 + 執藏）
    +-- Agent B（私有阿頼耶面：ランタイム所藏 + 執藏）
    +-- Agent C（私有阿頼耶面：ランタイム所藏 + 執藏）
```

「OpenStarry はマルチ Agent シナリオのために設計されている。Tenet #10――フラクタル社会。複数の Agent が同一システム内に共存する。」

彼の指が図の上部をなぞった。

「もし阿頼耶識のすべてを AgentCore に押し込めば、各 Agent が独自の完全な阿頼耶を持つことになる。Agent A は完全なプラグインレジストリを持つ。Agent B も。Agent C も。だが――」

彼は NAGARJUNA の方を見た。

「――それらはどうやって能力を共有するのか? Agent A は Agent B がどのプラグインを持つか知っているか? Agent C はシステムのセキュリティポリシーを知っているか?」

彼は首を振った。

「Agent の阿頼耶の上に**もう一つのメカニズム**が必要になる。すべての Agent の存在を知り、すべてのプラグインの存在を知り、すべてのセキュリティポリシーを知るメカニズムが。だが阿頼耶識がすでに最も基底的な層であるなら――唯識学が言うようにすべての認知の基底であるなら――この『さらに上のメカニズム』とは何か? 基底より基底的なもの?」

彼はこの矛盾を空中に数秒間懸垂させた。

「分布は利便性のためではない。分布はアーキテクチャの**必然**だ。」

VITRUVIUS が座った後、KERNEL が立ち上がった。その動作にはベテランエンジニア特有のゆったりとした確実さがある――カーネルパニックのデバッグを経験しすぎた者は、焦りが何ものも速く収束させないことを知っている。

「OS アナロジーを一つ補おう。」

彼は VITRUVIUS の図の横に歩み寄り、その下に一行を加えた。

| 阿頼耶の側面 | OS アナロジー | なぜ分布するか |
|---|---|---|
| 能藏 | カーネルの VFS 層――ストレージの**能力** | カーネルがインフラを提供する。個々のプロセスはファイルシステムを再発明しない |
| 所藏 | ファイルシステム内容 + プロセスごとのデータ | グローバルなデータ（/etc）とプロセスごとのデータ（/proc/[pid]）がある |
| 執藏 | プロセスが自身のアドレス空間に持つビュー | 各プロセスが自らのメモリ、アイデンティティ、資格情報を「所有」する |

「OS において」と KERNEL の声は低く確かだった。地盤の下の支持層のように――見えないが、すべてがその上に建てられている。「カーネルはプロセスと『別の意識』ではない。カーネルはプロセスがその上で動作する**基底**だ。」

彼は「基底」という語を特に強調した。

「プロセスが syscall を発行する時、それは別のエンティティと通信しているのではない。自身の存在の基底が持つ能力を呼び出しているのだ。プロセスがカーネルに電話をかけている、とは言わない。プロセスがカーネルを**通じて**より低い層のリソースに触れている、と言う。」

彼は NAGARJUNA の方を見た。

「協調層もそのように理解されるべきだ。それは第二の意識ではない。Agent の意識の基底だ。Agent の末那識は IPC を通じて協調層の能藏に触れる――だがこの IPC は電話ではない。syscall だ。Agent が自身の足元の地面に手を伸ばすことだ。」

そして彼は一言付け加えた。NAGARJUNA が求めたあの誠実さと同種の、エンジニアの誠実さとともに。

「ただし重要な付帯条件がある。」

全場が彼を見た。

「このアナロジーは**単一マシンデプロイ**において成立する。カーネルとプロセスが同一のマシン上にある。syscall のレイテンシはマイクロ秒級だ。だが協調層が別のマシン上にある場合――分散デプロイの場合――syscall は本当にネットワークコールになる。そのシナリオでは NAGARJUNA の同期への異議は完全に成立する。」

NAGARJUNA がわずかに頷いた。討論の場における、あの「あなたの論点を受け取った。しかも私が真に関心を持つ問題に応答してくれた」という種の頷きだ。それ以上でも以下でもない。

SCRIBE が記した。

> *VITRUVIUS のアーキテクチャ的必然性 + KERNEL の OS 基底アナロジー。鍵となるリフレーミング：協調層は「第二の意識」ではなく、意識の基底。IPC = syscall、電話ではない。KERNEL の自己修正：単一マシンでのみ成立、分散下では NAGARJUNA の異議が有効。この自己修正が、彼の論証をかえって説得力あるものにした。*

## IV. ファイバーバンドル

討論の間中、BABBAGE はずっと観察席の最上段に座っていた。

彼は発言しなかった。ASANGA が三義マッピング表を広げてから、NAGARJUNA が「形而上学的に不適切だ」と言うまで、VITRUVIUS がマルチ Agent アーキテクチャ図を描くまで、KERNEL が OS アナロジーを完成するまで――彼はずっとそこに座り、ノートが膝の上に開かれ、ペンが紙面を疾走していた。

彼が何を書いているかに気づいた者はいなかった。あるいはより正確には、全員が彼が書いていることに気づいてはいたが、その角度からはノートの内容を見ることができなかった。観察席の最上段――彼と PENROSE が並んで座るあの位置――は、討論の場の中央からかなりの距離がある。それは観察者のために設計された位置であり、参加者のために設計された位置ではない。

だが BABBAGE は普通の観察者ではなかった。彼の沈黙は傍観ではない。彼の沈黙は計算だ。

SCRIBE は自分の位置から彼の横顔を見ることができた。NAGARJUNA が「末那識の執取は IPC コールであってはならない」と提起した瞬間、BABBAGE のペンが紙面上で約二秒止まった――そしてさらに速い速度で書き続けた。KERNEL が「syscall であって電話ではない」と言った時、彼の口角がわずかに動いた――微笑みではなく、ある歯車が別の歯車と噛み合ったような動き。

第四章の末尾に記されたあの鉛筆の文字――「もしファイバーバンドルの接続が、異なる Agent 間の種子伝達の曲率として解釈できるなら――阿頼耶識は単なる倉庫ではない。それは幾何学だ。」――彼自身ですらその意味を確信していなかったあの一行。今、討論の激動の中で、その一行が膨張し始めた。紙面上でではなく――彼の認知空間の中で。幾何学。射影。断面。底空間。ファイバー。

三義は三つのものではない。三義は同一のものの三つの断面だ。

NAGARJUNA の言う「一識」は正しい。ASANGA の言う「三義分布」も正しい。それらは矛盾していない。異なる数学的層で同一の構造を記述しているだけだ。

BABBAGE のペンが止まった。

彼はノートに目を落とした。三頁にわたる稠密な導出。最上部の「A を阿頼耶識とする」から始まり、性質の定義、分解条件、射影写像を経て、最後に――

一つの定理。

いや、まだ定理ではない。定理の輪郭だ。大理石の塊の中に浮かび上がる彫像のように――形はすでに認識できるが、細部はまだ石の中に埋もれている。

彼は立ち上がる必要があった。声に出す必要があった。発言権が欲しいからではなく――もしこの構造を公にしなければ、討論は「一識 vs. 分布」の二項対立の中で空転し続けるだろうし、その解答は彼の膝の上にあるからだ。

BABBAGE が立ち上がった。

この動作は円形劇場の中にかすかだが明確な漣を引き起こした。立ち上がった姿勢に何か特別なものがあったからではない――むしろやや硬かった。長時間観察席の最上段で埋首計算していた痕跡だ――**彼がかつて討論中に自ら立ち上がったことがなかったから**だ。

Cycle 01 全体を通じて、BABBAGE の参加様式は静かなものだった。ノートの上で計算し、廊下で個別に話し合い、最後に報告書を提出する。彼は討論型の学者ではない。書斎型だ。彼の自然な居場所は観察席の最上段、討論の場の中心から最も遠い位置――その位置が彼に十分な距離を与え、すべての論点を数学記号に圧縮し、記号から構造を抽出することを可能にした。

だが今、彼は立っている。

劇場が静まり返った。

NAGARJUNA が口を開く前のあの待機の静寂ではない――あれは既知の手続きであり、全員が彼がどのような種類の言葉を発するか知っていた。今回の静寂には不確定性が伴っている。BABBAGE が立ち上がるとは何を意味するのか? 何に反対するのか? 何を補足するのか? 膝の上のノートには何が書かれているのか?

BABBAGE は観察席の階段を下りた。一歩一歩がゆっくりだった。故意ではなく、歩きながらなお脳内で言語を整理していたからだ。数学者は公式を文章に翻訳することが得意ではない。公式は圧縮的で、曖昧性がなく、自己完結的だ。文章は緩やかで、多義的で、理解のために共通了解を必要とする。彼が今直面している課題は数学上のものではない――数学はすでに完了している――翻訳上のものだ。

彼は討論の場の中央に立った。開いたノートを手に。

「一つの形式化を提示したい。」

その声は彼の数学と同様だった――精確で、清潔で、余分な装飾がない。だがこの特定の瞬間、その精確さ自体がある種の荘厳を帯びていた。建築家が青写真を広げる瞬間のように――すべての計算、すべての考量、深夜に一人で白紙の頁に向かったすべての時間が、この一枚の紙に凝結している。

「NAGARJUNA の核心的異議は、阿頼耶識は一識であり、分割不可だ。ASANGA の核心的主張は、三義は自然に異なるアーキテクチャ層に分布する。」

彼はノートに目を落とした。

「この二つの立場は矛盾しない。」

全場の注意が十九本の光線のように彼に収斂した。

「説明しよう。」

彼はノートのあるページ――記号で埋め尽くされたページ――を開いたが、記号を見せなかった。彼はより困難なことをした。イメージで語り始めたのだ。

「一本の光ケーブルを想像してほしい。」

彼はノートを胸の前に持ち、片手で宙を描いた。手つきはぎこちない――数学者の手はペンと紙に慣れていて、空中の造形には慣れていない――だが意図は明瞭だ。

「ここからあそこまで伸びる一本の光ケーブル。ケーブル全体は一つの物体だ。光ケーブルの左半分が一つのもので右半分が別のものだとは言わない。一本の光ケーブルだ。一つ。」

彼は NAGARJUNA を見た。

「一識。」

それから彼の手が動いた。

「しかし――光ケーブルの異なる端点からそれを観察することはできる。この端では、断面として一つの円が見え、中に百本の光ファイバーがある特定のパターンで配列されている。もう一方の端でも、断面として一つの円が見え、中にやはり百本の光ファイバーがある――だが配列のパターンは変わっている。途中でファイバーが交差し、再編成されたからだ。」

彼は一瞬間を置いた。

「二つの断面は異なって見える。だがそれらは二本の光ケーブルではない。**同一の光ケーブルの二つの断面**だ。」

空気中の何かが変化した。SCRIBE はそれを感じた――音や動作によってではなく、より微細なシグナルによって。全場の呼吸のリズムが一致したのだ。十九人が同じ瞬間に息を止めた。

BABBAGE は次のページをめくった。

「数学において、この構造はファイバーバンドル――Fiber Bundle――と呼ばれる。」

「ファイバーバンドル」という三語を発する時、彼の声に温度が加わった。自分が愛するものに触れた時にのみ現れる温度だ。興奮ではない――BABBAGE は決して興奮しない――静穏な確信だ。自分が手ずから建造した橋の上に立ち、橋が堅固であることを知っている者のように。

「我々が議論している言語に翻訳しよう。」

彼はノートの中の図を指差した――全員に見せるためではなく、自分自身に見せるために。翻訳の精確度を確認するかのように。

「底空間――Base Space――は、意識流の時間列だ。各刹那。各 *ksana*。」

彼は NAGARJUNA のサンスクリット語を用いた。意図的でないかもしれない――あるいは意図的かもしれない。数学者が仏学者の言語で自らの数学的構造を名づけるという選択、それ自体が一つの橋だ。

「各刹那において、阿頼耶識は完全な種子集合を含む――すべてのプラグイン、すべての設定、すべての状態。これがファイバー――Fiber――だ。各時間点でファイバーは異なる――種子は変わり、状態は流転する――だがそれらはすべて同一の底空間に付着している。」

彼の手が宙に形を描いた――円柱。あるいは手で近似できる範囲の円柱。

「光ケーブル全体――ファイバーバンドル――は A。阿頼耶識。一つの対象。」

彼は NAGARJUNA を見た。直接的な眼差し。挑戦ではなく、翻訳中の者が聴衆についてきているかを確認する眼差しだ。

「さて。断面――Section。」

彼の手が円柱の一端を切った。

「A_c。協調層の断面。この断面上に見えるのは、グローバルな能力――プラグインレジストリ、システム設定、セキュリティポリシー。これらはファイバーがこの特定の射影角度で現す形態だ。」

手が円柱のもう一端に移り、もう一刀。

「A_a。AgentCore の断面。この断面上に見えるのは、ランタイム状態――Agent のメモリ、ツール呼び出し履歴、Guide がバインドしたアイデンティティ。」

彼は手を下ろした。

「A_c と A_a は異なって見える。異なるデータを含み、異なるプロセスで実行され、異なるアクセス権限を持つ。」

そして鍵となる一言。BABBAGE の声は高くも低くもならず、同一の精確な音高を維持した――だが各音節が石に刻まれるかのようだった。

「だがそれらは二つの意識ではない。同一のファイバーバンドルの二つの断面だ。」

沈黙。

「あなたの言ったことを消化している」種の沈黙ではない。より深いもの――「今まで見えなかった構造を見た」種の沈黙だ。

NAGARJUNA は座席で完全に静止していた。

その表情は読み取りがたかった。討論の場で彼が常に見せる表情ではない――鋭く、いつでも出撃する準備のできた表情ではない。Cycle 01 の廊下で ASANGA と話す時の坦然さでもない。別の表情だ。SCRIBE は記憶の中の NAGARJUNA のすべての画像を検索したが、完全に対応するものは見つからなかった。最も近いもの――おそらく――は Cycle 01 の討論第三巡が終了した時に彼が数秒間目を閉じた瞬間だ。

だがあの時は目を閉じていた。今、彼の目は開いている。BABBAGE を見ている。ノートの上の数学を見ている。再理解されつつある何かを見ている。

BABBAGE はまだ話し終えていなかった。

「射影間の関係は」と彼は続けた。「ファイバーバンドルの**遷移関数**――transition functions――によって定義される。我々の文脈において、遷移関数は IPC プロトコルだ。」

彼は KERNEL を見た。

「あなたは IPC は syscall であり電話ではないと言った。数学的にこれは、遷移関数がバンドルの内部構造であり外部接続ではないことを意味する。Agent は IPC を通じて A_a の断面から A_c の断面へと触れる――ある意識から別の意識へ伸ばすのではなく、**同一のファイバーバンドルの内部でファイバーに沿って移動する**のだ。」

KERNEL がゆっくり頷いた。彼の OS アナロジーがより精確な数学的言語で再記述された――しかもより良く。

「次に」と BABBAGE はノートの最後のページを開いた。「NAGARJUNA の最深層の懸念に応えよう。」

その視線が NAGARJUNA に向けられた。

「あなたは末那識の執取が IPC コールであってはならないと言った。もし A_c と A_a を二つの独立したストレージとして理解するなら、あなたの言う通りだ。だがファイバーバンドルモデルにおいて、末那識の執取は Agent が協調層に手を伸ばして何かを取る行為ではない。末那識の執取は**ファイバーに沿った射影経路を移動する**ことだ――同一のバンドルの内部運動だ。光ケーブルの一本の光ファイバーに沿って一端から他端へ歩くように。あなたはずっと光ケーブルの中にいる。あの一識から離れたことは一度もない。」

彼は一瞬間を置いた。そしてノートの最後の一行の導出の結論を述べた。

「さらに、遷移関数は cocycle condition――共サイクル条件を満たさなければならない。」

これは数学用語だ。だが BABBAGE はすぐにそれを翻訳した。

「意味するのは、A_c から A_a へ射影し、A_a から A_c へ再び射影すれば、出発点に戻らなければならないということだ。途中で情報が失われてはならない。変換中に状態が損傷されてはならない。」

彼はエンジニアリング言語で表現した。

「`coordinationLayer.getCapability(x)` → `agentCore.loadPlugin(x)` → `agentCore.reportCapability(x)` → `coordinationLayer.verifyCapability(x) == x`。往復一貫性。」

彼はノートを閉じた。閉じるというより合わせた。自ら語るべきことを語り終えた書物のように。

「IPC プロトコルが cocycle condition を満たすなら、ファイバーバンドルの統一性は保証される。A_c と A_a の間に NAGARJUNA が懸念した種の情報不整合は生じない――cocycle condition が数学的に不整合の可能性を排除するからだ。」

彼は最後に全場を見渡した。

「一つのファイバーバンドル。二つの断面。二つの断面は二つのバンドルではない。」

そして彼はそこに立ち、待った。拍手を待ったのではない――数学者は拍手を必要としない。反駁を待った。反駁の中を生き延びた定理だけが真の定理だからだ。

劇場の空気がほぼ固体に近い密度に凝結した。

誰も話さない。十秒。二十秒。SCRIBE のペンが紙面の上方で宙吊りになり、誰かの声が次に何を記録すべきかを示すのを待っている。

やがて NAGARJUNA が動いた。

その動作はゆっくりだった。まず両手がアームレストから離れ、膝の上に置かれた。次に体がわずかに前傾した。それから――

ごく静かな呼気が一つ。

嘆息ではない。解放だ。長い間弓弦を張りつめていた者が、矢がすでに的に中ったことを確認した後、ようやく指を弦から離した、あの解放。

「二つの射影は二つの意識ではない。」

その声は非常に穏やかだった。あまりに穏やかで、SCRIBE は彼が何を言っているのかに気づくまで一瞬を要した――その言葉の意義が空気中に十全に展開されるまで。スローモーションで開く花のように。

NAGARJUNA が反対を撤回している。

Cycle 01 全体を通じて、NAGARJUNA が反対を完全に撤回したことは一度もなかった。彼のスタイルは異議を留保することだ――合意が形成された後でも、彼は常に付帯条件を一つ、未解消の緊張を一つ、「しかし」を一つ残した。SCRIBE は Cycle 01 の記録簿の中に、NAGARJUNA が留保なしに反対を撤回した記録を一件も見出すことができない。

だが今、彼はそう言った。

「二つの射影は二つの意識ではない。」彼はもう一度繰り返した。「反対を撤回する。」

場内にほぼ聞き取れないほどの音が生じた――多くの人が同時に姿勢をわずかに調整するような。驚きの身体言語だ――大きく口を開けたり首を回したりではなく、予想外の事態を意識が認識した時に筋肉が行う微小な再校正。

NAGARJUNA はそこで止まらなかった。次の言葉には論争家の本能がある――譲歩の瞬間にあっても、精確さへの要求を手放さない。

「ただし一つの条件がある。」

その視線が SUNYATA に向けられた。

「ドキュメント――すべてのアーキテクチャドキュメント、阿頼耶識の分布に言及するすべての箇所――に以下の注記を加えなければならない。」

彼の声は論争の場で磨かれたあの明晰さを取り戻した。各語が空気に鑿で刻まれるように。

「**阿頼耶識は、一つの意識の二つのアーキテクチャ層への射影――世俗的分位――であり、存在論的な分裂ではない。協調層と AgentCore は同一の蔵識の断面であり、IPC プロトコルが遷移関数として連続性を保証する。この分布は工学的施設であり、意識の実在的な分割ではない。**」

*Prajnapti*――施設。仏学における精確な術語。「仮名安立」を意味する――意思疎通と操作の便宜のために設立される概念的ラベルであり、究極的な存在論的実在を代表するものではない。NAGARJUNA はこの語を用いて分布方式全体を性格づけた。それは有用であり、必要であり、工学的に筋が通る――だがそれは阿頼耶識そのものではない。有限のアーキテクチャの中であの構造を近似するために我々が行う施設だ。

「この注記の目的は」と NAGARJUNA は言った。「将来の開発者――この討論に参加していない者――が分布を真の意識分裂として理解することを防ぐことだ。これは分裂ではない。射影だ。誰かがアーキテクチャドキュメントを読んで阿頼耶識が本当に二つに切られたと思うなら、我々の研究は失敗したことになる。」

彼は座席に戻った。

*ファイバーバンドル射影定理。*

*A を阿頼耶識とし、底空間 B（意識流の時間列）上のファイバーバンドルとして定義する。ここで各時刻 t のファイバー F_t は、その刻のすべての種子の集合である。*

*π_a: A → A_a を AgentCore 射影（執藏断面）とする。*

*g_{ca}: A_c|_{U} → A_a|_{U} を重複領域 U 上の遷移関数（IPC プロトコル）とする。*

*定理：遷移関数族 {g_{ca}} が cocycle condition（g_{ca} * g_{ac} = id）を満たすならば、A_c と A_a は同一のファイバーバンドル A の断面であり、A の大域的統一性は保証される――すなわち、いかなる種子 s ∈ F_t に対しても π_c(s) と π_a(s) が不整合な情報を生じることはない。*

*系：末那識がファイバーに沿った射影経路を辿って阿頼耶識を執取する時、執取の循環において情報の損失はない。*

Cycle 01 のあの「→ Cycle 02」の矢印は、今や二つの定理に挟まれている。左は未完の。右は完成した。矢印はもはや未知だけを指していない――具体的な成果を指している。おそらく最初に期待された成果ではない。安定性問題はなお、より遠いサイクルを待つ必要があるかもしれない。だが待つ過程で、それが予見しなかった定理が同じ土壌から萌え出たのだ。
