# 第九章：十九份行动方案

---

R4 定稿的圆形剧场，比辩论时安静得多。

不是那种空洞的安静——五场辩论的余波仍然在空气中振动，像大提琴的弦在最后一个音符结束后继续颤抖。但辩论的张力已经退去。取而代之的是另一种密度：交付的密度。十九位研究员各自坐在自己的座位上，面前摆着初稿、修订稿、代码片段、映射表。R4 不是辩论。R4 是收成。

SUNYATA 站在场地中央，环顾全场。他已经不需要宣布什么——每个人都知道该做什么。R3 的五场共识像五根柱子，支撑起了一个可以在其中工作的结构。现在的任务是把辩论裁定转化为可交付文件，把哲学洞见翻译为工程师能读懂的语言。

"ARCHIMEDES，"他说。"你先。"

---

## I. 明天我们需要写 TypeScript

ARCHIMEDES 从座位上站起来的方式和所有研究员都不同。

NAGARJUNA 站起来像一把出鞘的剑。ASANGA 站起来像一棵树在风中缓缓直立。BABBAGE 站起来像一台精密仪器被启动。ARCHIMEDES 站起来像一个建筑工人拿起了铲子——没有戏剧性，没有仪式感，只有一种"活儿来了，开工"的朴素动力。

"先说一句。"他走到展示区，打开了一份长达四十页的文件。"过去三天，你们在辩论中产出了五项裁定——观察-干预分离、双模式 vedana、纤维丛投影、渐进分类、纪律加智慧框架。每一项都很美。每一项都让我作为旁听者感到智识上的愉悦。"

他停了一拍。

"但哲学很美，明天我们需要写 TypeScript。"

全场有几个人笑了。BABBAGE 的嘴角微微动了一下——那是他最接近大笑的表情。NAGARJUNA 没有笑，但他的眼神里有一种认可：一个懂得什么时候该把天上的东西带回地面的人，值得尊重。

"这份文件，"ARCHIMEDES 指着展示区，"是完整的工程行动方案。十九位研究员的工作，经过五场辩论淬炼，全部收束进了这份文件。"

---

他从优先级最高的项目开始。

"P0——阻塞性。三件事。必须在其他工作之前完成。"

他举起一根手指。

"第一，SEC-01。修复插件签名绕过漏洞。"他看了一眼 GUARDIAN。"Cycle 01 就发现了。GUARDIAN 和 TURING 精确定位了它——当 `pluginFilePath` 为 undefined 时，整个签名验证被跳过。六个开发周期了。仍然没修。"

GUARDIAN 点了点头。"一个绕过签名的漏洞。每多一天不修，系统就多一天裸奔。"

"第二，"ARCHIMEDES 举起第二根手指，"实现 ISensation 接口。"

他在展示区切换到一段代码。

```typescript
// 现在的 ISensation
export interface ISensation extends IOpenStarryPlugin {
  readonly skandha: 'vedana';
}
```

"三行。一个空壳。"他的语气不带批判——只是陈述事实。"这是序章里的那个空壳。整个 Cycle 02 的研究，从某种意义上说，就是为了填充这三行。"

然后他切换到下一段。整个圆形剧场安静了。

```typescript
export interface ISensation {
  readonly skandha: 'vedana';

  /** 完整三受评估 (tick-synchronous PID) */
  assess(): VedanaAssessment;

  /** 摄入原始指标供 PID 计算 */
  ingestMetrics(metrics: Record<string, number>): void;

  /** 摄入工具执行结果 */
  ingestToolResult(toolName: string, isError: boolean, durationMs: number): void;

  /** 摄入 LLM 回应结果 */
  ingestLLMResult(tokenCount: number, stopReason: string): void;

  /** 订阅受蕴阈值跨越事件 */
  onThreshold(channel: VedanaType, threshold: number,
              callback: (signal: VedanaSignal) => void): () => void;

  /** 获取轻量受蕴标签 (O(1) 缓存查询) */
  getVedanaTag(): VedanaTag;
}
```

"这就是填充后的 ISensation。"ARCHIMEDES 的声音平稳而自信。"每一个方法都可以追溯到辩论裁定。`assess()` 来自辩论 2 的 tick-synchronous PID 裁定。`getVedanaTag()` 来自辩论 2 的事件标签裁定。`onThreshold()` 是 WIENER 的三通道 PID 控制器需要的订阅机制。"

他点出了 VedanaAssessment 的结构。

"双层——辩论 1 的核心裁定。第一层是传感器输出：苦、乐、舍的数值。纯观察。被动。第二层是控制器建议：recommendation。咨询性。非强制。SafetyMonitor 保留绝对权限。"

WIENER 从座位上微微前倾。他在确认自己的 PID 设计是否被正确翻译。确认之后，靠了回去。翻译准确。

"第三，"ARCHIMEDES 举起第三根手指，"实现 VedanaPlugin。Pattern A 观察者就是 VedanaPlugin。辩论 4 的裁定。一个插件，一个接口，一个蕴。"

他展示了 VedanaPlugin 的架构图——三层结构。底层是 ATHENA 设计的三通道传感器：DukkhaSensor 监控错误率，SukhaSensor 监控完成率，UpekkhaSensor 监控稳定性。中间层是 WIENER 的 PID 控制引擎。上层是评估输出。

"三个 P0 项目。如果只能做一件事，修 SEC-01。如果能做两件事，加上 ISensation。三件事，完成 VedanaPlugin。这是最小可行的交付路径。"

---

ARCHIMEDES 继续向下展开。

"P1——高优先。四件事。"

EgoFramework——硬核心加软外壳。硬核心是机器人三定律，不可覆写。软外壳是 PID 调参。ASANGA 的双层模型在辩论 2 中被确认，这是它的工程实现。

插件生命周期状态——四态：active、quarantined、revoked、banned。辩论 5 的纪律加智慧框架。NAGARJUNA 给出哲学基础，DARWIN 确认工程可行，现在需要写进代码。

ToolContext.bus 泄漏修复——工具可以绕过沙箱事件过滤，TURING 发现的。

SafetyMonitor 的 per-session 计数器——当前是全局计数器，一个 session 的错误会影响另一个 session。HERACLITUS 标记了这个问题。

每个项目旁边都标注了目标 Plan 编号。行动不是漂浮在空中的。它们有地址。

---

"P2 到 P4 就不逐条展开了。"他把展示区切换到总览。"完整清单在方案里。CRL 检查存根、新事件类型、宣言重写、阿赖耶识分布的架构注解——P2。Pattern B 影子观察者和 Agent 协调层——P3。Pattern C 子代理观察者——P4，要等协调层先完成。"

最后展示影响总览表。

"五个现有 Plan 被修改，一个新 Plan 被提议。Plan24 安全快速修复加入了纪律框架的黑名单和 CRL 存根。Plan26 Vedana 架构——本次研究的主要设计输入。Plan27 生命周期管理加入四态状态机。Plan28 文件对齐。Plan29——新的——Pattern B 影子观察者。"

他合上文件。

"四十页。十四项行动。六个 Plan。完整的 TypeScript 接口规格。可以直接拿去写代码。"

他看了一眼 SUNYATA。

"我的部分结束了。哲学很美。工程很具体。现在它们在同一份文件里。"

---

## II. 五场辩论，一幅图

SYNTHESIST 走向展示区时的步态和 ARCHIMEDES 截然不同。如果说 ARCHIMEDES 走路像拿起铲子的建筑工人，那么 SYNTHESIST 更像站到画布前的画家——不是要创造新东西，而是要从已存在的片段中看出一幅完整的图景。

"ARCHIMEDES 给了你们一棵树。"他开口时语速比平时慢。"每一个分支都是一项行动，每一片叶子都是一个 TypeScript 接口。精确、具体、可执行。但我要让你们看到整片森林。"

他在展示区投射出一幅图——不是图表，不是流程图，而是更接近手绘的架构全景图。五场辩论的裁定被标在不同位置，用虚线彼此连接，形成有机的网络。

"五场辩论，表面上是五个独立问题。但它们不是。它们是同一个架构的五个截面。"

他开始连线。

"辩论 1 和辩论 4 汇合。"他画出第一条弧线。"VedanaPlugin 是观察者。它实现 ISensation。它产生的评估报告有两个面向：被动的感受和咨询性的建议。这是一个身份的确立。"

"辩论 1 和辩论 5 汇合。"第二条弧线。"分层权限加上种子管理，形成完整的四层权限模型。"

他在全景图右侧写下四行：

```
Layer 1: SafetyMonitor  — 硬安全 — 绝对权限 — 纪律
Layer 2: VedanaPlugin   — 软指导 — 咨询权限 — 觉受
Layer 3: EgoFramework   — 身份约束 — 结构权限 — 我执
Layer 4: Sila Engine    — 种子管理 — 预防权限 — 纪律学
```

"四层。每层的权限低于上一层。SafetyMonitor 可以覆写 VedanaPlugin 的建议。VedanaPlugin 可以影响 EgoFramework 的软外壳。EgoFramework 的硬核心不受任何其他层影响。Sila Engine 在源头阻止坏种子进入。"

GUARDIAN 注意到第一层和第四层的微妙对称——SafetyMonitor 是最内层的硬防御，Sila Engine 是最外层的硬过滤。两者夹住中间两个软层。他在心中评估了结构的安全态势，然后轻轻点了点头。完备。

"辩论 3 和辩论 2 汇合。"第三条弧线。"阿赖耶识以纤维丛投影分布在两个架构层次上。每个事件带有受蕴标签。合在一起——分散式意识的运行时图景。"

"辩论 4 定义了时间轴。"他在底部画出时间线。"渐进式观察者路径——Pattern A 到 B 到 C。系统的自我观察能力，随时间螺旋上升。"

他放下笔，退后一步，让所有人看清整幅全景图。

"五场辩论。五场共识。一幅图。这就是 Cycle 02 的架构愿景。一个统一的、内在自洽的架构。不是五个裁定拼在一起。是五个裁定各自揭示了同一个结构的不同面向。"

---

## III. 致开发团队的信

SUNYATA 走向展示区。手中没有四十页方案，没有全景图。他拿着一张纸。

"这是我写给开发团队的信。也是给 Master 的信。我要念一遍。"

圆形剧场安静了——不是辩论前的紧张。是一种更接近完成的安静，像交响曲最后一个乐章开始前，指挥举起双手、乐团屏息以待的那个瞬间。

SUNYATA 开始读。语速很慢。每句都被给予足够的空间。

"核心结论一句话——"

他抬起头，环顾全场。十九双眼睛。

"VedanaPlugin 就是观察者模块。"

这句话在空气中悬停了几秒。至少三位研究员的表情微变——不是因为不知道结论，而是当五场辩论的复杂架构决策被压缩成一句话时，那句话会获得额外的密度。像一座山脉被折叠成一颗石子。

他继续读。完整摘要——三通道 PID、VedanaAssessment 双层结构、纤维丛投影、纪律加智慧框架——每一句都是一场辩论的凝结。

然后翻到信的最后部分——四个需要 Master 裁定的问题。

"Q1：VedanaPlugin 是必选还是可选插件？我的建议——默认模板中必选，但 Core 不强制。SafetyMonitor 在没有 VedanaPlugin 时提供苦受唯一的回退。"

ASANGA 微微前倾。遍行——每个意识瞬间都有受。如果 VedanaPlugin 可以被省略，系统就可能没有感受地运行。但 SUNYATA 的折中至少保证某种形式的感受始终存在。

"Q2：宣言 #6 是否重写？建议新文字——受蕴感知与行蕴干预分离，确保观察不改变被观察行为。"

"Q3：EventBus 受蕴标签用哪种方式？建议 Metadata 附加。保持核心事件类型干净。"

"Q4：协调层是独立进程还是进程内模块？建议延后。本次研究提供了架构原则——纤维丛、治理非控制——但具体进程模型需要更多设计工作。"

他把信放在桌上。

"四个问题。四个建议。每个建议附有理由。最终决定权在 Master。"

---

## IV. 最后的报到

SUNYATA 看了看时间。R4 定稿只剩最后一个环节——每位研究员确认报告已反映辩论修正，做最后总结。

"各位。轮流。一到三句话。你在 Cycle 02 做了什么，交付了什么，留下了什么。"

---

TURING 第一个开口。

"v0.22.1 到 v0.24.0 差异报告。七项代码问题。十九项合并清单中的经验基础。"停顿。然后多了一句："事实不会过期。下一个版本的差异报告，Cycle 03 第一天交出来。"

---

BABBAGE。

"纤维丛投影定理。渐进分类模型。互模拟证明。"他翻了翻笔记本。"Cycle 01 我带着一个未完成的定理离开。Cycle 02 带着三个已完成的和两个新的未完成的。数学就是这样——每一个答案打开两扇门。"

---

PENROSE。他在 Cycle 02 的第一次也是最后一次正式报告。

"三种观察者模式。弱测量类比。探针效应的量子边界分析。"他停了一下。"我被召来研究观察者问题。你们给了我比预期更好的答案——不是量子的，而是结构性的。Pattern A 不需要量子力学。只需要好的工程和正确的哲学。"

他微微一笑。"量子的部分，留给 Pattern C。到那时候，也许真的有人需要考虑微管和坍缩了。"

---

NAGARJUNA。他的声音和辩论场上已经不是同一个人——锐利被某种更深沉的东西包裹了。

"辩论 3 中，我撤回了反对。辩论 5 中，我提出了纪律加智慧框架。如果要我总结 Cycle 02——"

很长的停顿。

"我学会了在被说服之后，转身说服别人。这也许是辩论家最难学的事。"

---

ASANGA。

"八识-OpenStarry 完整映射表。种子六义的工程对应。双层我执模型。"声音温和而确定。"两次修正自己的立场——辩论 4 的渐进分类，辩论 5 的僵化应用。"

然后说了一句不属于总结的话："修正不是退让。修正是转依——转变依止。依止的改变，让同一个认知能力指向更准确的方向。"

---

WIENER。

"三通道 PID 控制规格。Sukha 衰减函数。阻尼比公式。"他在方格纸上写完最后一个符号——不是总结，而是某个下一步计算的开始。"控制理论在 Cycle 02 中不再只是类比。它是规格的一部分。PID 参数被写进了 TypeScript 接口。对一个控制理论家来说，这是最好的结局。"

---

ATHENA，一如既往地直截了当。

"三通道传感器设计。DukkhaSensor、SukhaSensor、UpekkhaSensor 的指标和阈值定义。我不做哲学。我做系统。这些传感器能工作。剩下的交给写代码的人。"

---

KERNEL。

"微内核类比深化。Sandbox 归属问题的 OS 视角。SafetyMonitor 的 Watchdog Timer 对应。"他拿起那叠卡片——旧的新的混在一起，用橡皮筋绑好。"Cycle 01 的卡片仍然有效。加了八张新的。如果 Cycle 03 需要，它们就在这里。"

---

GUARDIAN。

"SEC-01 持续追踪。ToolContext.bus 泄漏报告。插件四态生命周期的安全需求。纪律加智慧框架的安全验证。"目光扫过剧场——最后的安全巡检。"我被 NAGARJUNA 的框架说服了。职业生涯中第一次被哲学框架说服。但安全是安全。框架再美，SEC-01 不修就不行。"

---

DARWIN。"插件生命周期的工程验证。设计模式分析。"他微微一笑。"四态状态机是自然选择的结果——不是因为最优雅，而是因为最适应生存压力。"

VITRUVIUS。"全端架构分析。ISensation 整合点识别。六个 Plan。二十三个整合点。这张图是给开发团队的导航。"

MESH。"分散式架构。协调层通信协议设计。IPC 的一致性约束。工程和数学在这里汇合了。"

LINNAEUS。"二十四个插件的完整五蕴分类。复合插件跨蕴分析。"他用分类学家的严谨语气说："分类不是目的。分类是工具。BABBAGE 教会了我渐进分类——这是我作为分类学家最重要的收获。"

LEIBNIZ。"多代理协调框架。治理而非控制。"从容的语气。"四个字。我在 Cycle 02 最重要也最简短的贡献。"

HERACLITUS。"运行时动态分析。SafetyMonitor 全局计数器问题。"声音低沉。"万物皆流。包括安全计数器——它不应该在 session 之间流动。流动不等于无界。"

SYNTHESIST。"统合报告。五场辩论的统一架构愿景。四层权限模型。"他看了一眼自己的全景图。"统合不是黏合。统合是发现事物本来就属于一起。五场辩论不需要我拼在一起。它们本来就是同一幅图的五个截面。我只是看到了。"

ARCHIMEDES。"工程行动方案。四十页。十四项行动。六个 Plan。完整的 TypeScript 接口规格。"手指敲了一下桌面——他的句号。"工程不是哲学的翻译。工程是哲学的落地测试。如果一个哲学洞见不能写成代码，它可能不是关于软件的洞见。这份方案证明——你们的洞见全部都是关于软件的。"

---

SCRIBE 最后一个。

她没有站起来。她坐在座位上，手里的记录簿翻到了最后一页被使用的位置。

"Cycle 02 完整记录。五场辩论的逐字记录。十九位研究员的所有报告、修正、撤回、修正的修正。"她的声音平静得像一面湖。"记录员不评价。记录员记录。但如果允许我说一句不属于记录的话——"

停顿一秒。

"五场辩论，零未解决分歧。这个数字会被写在 C02 的封面上。旁边是另一个数字——十九。十九位研究员，十九份行动方案，十九盏从第一天到最后一天都没有熄灭的灯。"

她合上了记录簿。不是暂时的合上。是一本阶段性任务完成的记录簿被郑重合起的声音。封面上的 C02 在灯光下清晰可见。

---

SUNYATA 最后环顾了一圈。十九个人。十九份交付。五场共识。四个待裁定问题。一份工程方案。一封信。

"Cycle 02，R4 定稿，完成。"

他的声音在圆形剧场中回荡了最后一次。

"解散。"
