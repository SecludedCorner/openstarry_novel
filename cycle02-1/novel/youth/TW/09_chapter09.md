# 第九章：十九份行動方案

---

R4 定稿的圓形劇場，比辯論時安靜得多。

不是那種空洞的安靜——五場辯論的餘波仍然在空氣中振動，像大提琴的弦在最後一個音符結束後繼續顫抖。但辯論的張力已經退去。取而代之的是另一種密度：交付的密度。十九位研究員各自坐在自己的座位上，面前擺著初稿、修訂稿、程式碼片段、映射表。R4 不是辯論。R4 是收成。

SUNYATA 站在場地中央，環顧全場。他已經不需要宣布什麼——每個人都知道該做什麼。R3 的五場共識像五根柱子，支撐起了一個可以在其中工作的結構。現在的任務是把辯論裁定轉化為可交付文件，把哲學洞見翻譯為工程師能讀懂的語言。

「ARCHIMEDES，」他說。「你先。」

---

## I. 明天我們需要寫 TypeScript

ARCHIMEDES 從座位上站起來的方式和所有研究員都不同。

NAGARJUNA 站起來像一把出鞘的劍。ASANGA 站起來像一棵樹在風中緩緩直立。BABBAGE 站起來像一台精密儀器被啟動。ARCHIMEDES 站起來像一個建築工人拿起了鏟子——沒有戲劇性，沒有儀式感，只有一種「活兒來了，開工」的樸素動力。

「先說一句。」他走到展示區，打開了一份長達四十頁的文件。「過去三天，你們在辯論中產出了五項裁定——觀察-干預分離、雙模式 vedana、纖維叢投影、漸進分類、紀律加智慧框架。每一項都很美。每一項都讓我作為旁聽者感到智識上的愉悅。」

他停了一拍。

「但哲學很美，明天我們需要寫 TypeScript。」

全場有幾個人笑了。BABBAGE 的嘴角微微動了一下——那是他最接近大笑的表情。NAGARJUNA 沒有笑，但他的眼神裡有一種認可：一個懂得什麼時候該把天上的東西帶回地面的人，值得尊重。

「這份文件，」ARCHIMEDES 指著展示區，「是完整的工程行動方案。十九位研究員的工作，經過五場辯論淬鍊，全部收束進了這份文件。」

---

他從優先級最高的項目開始。

「P0——阻塞性。三件事。必須在其他工作之前完成。」

他舉起一根手指。

「第一，SEC-01。修復插件簽名繞過漏洞。」他看了一眼 GUARDIAN。「Cycle 01 就發現了。GUARDIAN 和 TURING 精確定位了它——當 `pluginFilePath` 為 undefined 時，整個簽名驗證被跳過。六個開發週期了。仍然沒修。」

GUARDIAN 點了點頭。「一個繞過簽名的漏洞。每多一天不修，系統就多一天裸奔。」

「第二，」ARCHIMEDES 舉起第二根手指，「實作 ISensation 介面。」

他在展示區切換到一段程式碼。

```typescript
// 現在的 ISensation
export interface ISensation extends IOpenStarryPlugin {
  readonly skandha: 'vedana';
}
```

「三行。一個空殼。」他的語氣不帶批判——只是陳述事實。「這是序章裡的那個空殼。整個 Cycle 02 的研究，從某種意義上說，就是為了填充這三行。」

然後他切換到下一段。整個圓形劇場安靜了。

```typescript
export interface ISensation {
  readonly skandha: 'vedana';

  /** 完整三受評估 (tick-synchronous PID) */
  assess(): VedanaAssessment;

  /** 攝入原始指標供 PID 計算 */
  ingestMetrics(metrics: Record<string, number>): void;

  /** 攝入工具執行結果 */
  ingestToolResult(toolName: string, isError: boolean, durationMs: number): void;

  /** 攝入 LLM 回應結果 */
  ingestLLMResult(tokenCount: number, stopReason: string): void;

  /** 訂閱受蘊閾值跨越事件 */
  onThreshold(channel: VedanaType, threshold: number,
              callback: (signal: VedanaSignal) => void): () => void;

  /** 取得輕量受蘊標籤 (O(1) 快取查詢) */
  getVedanaTag(): VedanaTag;
}
```

「這就是填充後的 ISensation。」ARCHIMEDES 的聲音平穩而自信。「每一個方法都可以追溯到辯論裁定。`assess()` 來自辯論 2 的 tick-synchronous PID 裁定。`getVedanaTag()` 來自辯論 2 的事件標籤裁定。`onThreshold()` 是 WIENER 的三通道 PID 控制器需要的訂閱機制。」

他點出了 VedanaAssessment 的結構。

「雙層——辯論 1 的核心裁定。第一層是感測器輸出：苦、樂、捨的數值。純觀察。被動。第二層是控制器建議：recommendation。諮詢性。非強制。SafetyMonitor 保留絕對權限。」

WIENER 從座位上微微前傾。他在確認自己的 PID 設計是否被正確翻譯。確認之後，靠了回去。翻譯準確。

「第三，」ARCHIMEDES 舉起第三根手指，「實作 VedanaPlugin。Pattern A 觀察者就是 VedanaPlugin。辯論 4 的裁定。一個插件，一個介面，一個蘊。」

他展示了 VedanaPlugin 的架構圖——三層結構。底層是 ATHENA 設計的三通道感測器：DukkhaSensor 監控錯誤率，SukhaSensor 監控完成率，UpekkhaSensor 監控穩定性。中間層是 WIENER 的 PID 控制引擎。上層是評估輸出。

「三個 P0 項目。如果只能做一件事，修 SEC-01。如果能做兩件事，加上 ISensation。三件事，完成 VedanaPlugin。這是最小可行的交付路徑。」

---

ARCHIMEDES 繼續向下展開。

「P1——高優先。四件事。」

EgoFramework——硬核心加軟外殼。硬核心是機器人三定律，不可覆寫。軟外殼是 PID 調參。ASANGA 的雙層模型在辯論 2 中被確認，這是它的工程實作。

插件生命週期狀態——四態：active、quarantined、revoked、banned。辯論 5 的紀律加智慧框架。NAGARJUNA 給出哲學基礎，DARWIN 確認工程可行，現在需要寫進程式碼。

ToolContext.bus 洩漏修復——工具可以繞過沙箱事件過濾，TURING 發現的。

SafetyMonitor 的 per-session 計數器——當前是全局計數器，一個 session 的錯誤會影響另一個 session。HERACLITUS 標記了這個問題。

每個項目旁邊都標注了目標 Plan 編號。行動不是漂浮在空中的。它們有地址。

---

「P2 到 P4 就不逐條展開了。」他把展示區切換到總覽。「完整清單在方案裡。CRL 檢查存根、新事件類型、宣言重寫、阿賴耶識分佈的架構註解——P2。Pattern B 影子觀察者和 Agent 協調層——P3。Pattern C 子代理觀察者——P4，要等協調層先完成。」

最後展示影響總覽表。

「五個現有 Plan 被修改，一個新 Plan 被提議。Plan24 安全快速修復加入了紀律框架的黑名單和 CRL 存根。Plan26 Vedana 架構——本次研究的主要設計輸入。Plan27 生命週期管理加入四態狀態機。Plan28 文件對齊。Plan29——新的——Pattern B 影子觀察者。」

他合上文件。

「四十頁。十四項行動。六個 Plan。完整的 TypeScript 介面規格。可以直接拿去寫程式碼。」

他看了一眼 SUNYATA。

「我的部分結束了。哲學很美。工程很具體。現在它們在同一份文件裡。」

---

## II. 五場辯論，一幅圖

SYNTHESIST 走向展示區時的步態和 ARCHIMEDES 截然不同。如果說 ARCHIMEDES 走路像拿起鏟子的建築工人，那麼 SYNTHESIST 更像站到畫布前的畫家——不是要創造新東西，而是要從已存在的片段中看出一幅完整的圖景。

「ARCHIMEDES 給了你們一棵樹。」他開口時語速比平時慢。「每一個分支都是一項行動，每一片葉子都是一個 TypeScript 介面。精確、具體、可執行。但我要讓你們看到整片森林。」

他在展示區投射出一幅圖——不是圖表，不是流程圖，而是更接近手繪的架構全景圖。五場辯論的裁定被標在不同位置，用虛線彼此連接，形成有機的網絡。

「五場辯論，表面上是五個獨立問題。但它們不是。它們是同一個架構的五個截面。」

他開始連線。

「辯論 1 和辯論 4 匯合。」他畫出第一條弧線。「VedanaPlugin 是觀察者。它實作 ISensation。它產生的評估報告有兩個面向：被動的感受和諮詢性的建議。這是一個身份的確立。」

「辯論 1 和辯論 5 匯合。」第二條弧線。「分層權限加上種子管理，形成完整的四層權限模型。」

他在全景圖右側寫下四行：

```
Layer 1: SafetyMonitor  — 硬安全 — 絕對權限 — 紀律
Layer 2: VedanaPlugin   — 軟指導 — 諮詢權限 — 覺受
Layer 3: EgoFramework   — 身份約束 — 結構權限 — 我執
Layer 4: Sila Engine    — 種子管理 — 預防權限 — 紀律學
```

「四層。每層的權限低於上一層。SafetyMonitor 可以覆寫 VedanaPlugin 的建議。VedanaPlugin 可以影響 EgoFramework 的軟外殼。EgoFramework 的硬核心不受任何其他層影響。Sila Engine 在源頭阻止壞種子進入。」

GUARDIAN 注意到第一層和第四層的微妙對稱——SafetyMonitor 是最內層的硬防禦，Sila Engine 是最外層的硬過濾。兩者夾住中間兩個軟層。他在心中評估了結構的安全態勢，然後輕輕點了點頭。完備。

「辯論 3 和辯論 2 匯合。」第三條弧線。「阿賴耶識以纖維叢投影分佈在兩個架構層次上。每個事件帶有受蘊標籤。合在一起——分散式意識的運行時圖景。」

「辯論 4 定義了時間軸。」他在底部畫出時間線。「漸進式觀察者路徑——Pattern A 到 B 到 C。系統的自我觀察能力，隨時間螺旋上升。」

他放下筆，退後一步，讓所有人看清整幅全景圖。

「五場辯論。五場共識。一幅圖。這就是 Cycle 02 的架構願景。一個統一的、內在自洽的架構。不是五個裁定拼在一起。是五個裁定各自揭示了同一個結構的不同面向。」

---

## III. 致開發團隊的信

SUNYATA 走向展示區。手中沒有四十頁方案，沒有全景圖。他拿著一張紙。

「這是我寫給開發團隊的信。也是給 Master 的信。我要念一遍。」

圓形劇場安靜了——不是辯論前的緊張。是一種更接近完成的安靜，像交響曲最後一個樂章開始前，指揮舉起雙手、樂團屏息以待的那個瞬間。

SUNYATA 開始讀。語速很慢。每句都被給予足夠的空間。

「核心結論一句話——」

他抬起頭，環顧全場。十九雙眼睛。

「VedanaPlugin 就是觀察者模組。」

這句話在空氣中懸停了幾秒。至少三位研究員的表情微變——不是因為不知道結論，而是當五場辯論的複雜架構決策被壓縮成一句話時，那句話會獲得額外的密度。像一座山脈被折疊成一顆石子。

他繼續讀。完整摘要——三通道 PID、VedanaAssessment 雙層結構、纖維叢投影、紀律加智慧框架——每一句都是一場辯論的凝結。

然後翻到信的最後部分——四個需要 Master 裁定的問題。

「Q1：VedanaPlugin 是必選還是可選插件？我的建議——預設範本中必選，但 Core 不強制。SafetyMonitor 在沒有 VedanaPlugin 時提供苦受唯一的回退。」

ASANGA 微微前傾。遍行——每個意識瞬間都有受。如果 VedanaPlugin 可以被省略，系統就可能沒有感受地運行。但 SUNYATA 的折衷至少保證某種形式的感受始終存在。

「Q2：宣言 #6 是否重寫？建議新文字——受蘊感知與行蘊干預分離，確保觀察不改變被觀察行為。」

「Q3：EventBus 受蘊標籤用哪種方式？建議 Metadata 附加。保持核心事件類型乾淨。」

「Q4：協調層是獨立進程還是進程內模組？建議延後。本次研究提供了架構原則——纖維叢、治理非控制——但具體進程模型需要更多設計工作。」

他把信放在桌上。

「四個問題。四個建議。每個建議附有理由。最終決定權在 Master。」

## IV. 最後的報到

SUNYATA 看了看時間。R4 定稿只剩最後一個環節——每位研究員確認報告已反映辯論修正，做最後總結。

「各位。輪流。一到三句話。你在 Cycle 02 做了什麼，交付了什麼，留下了什麼。」

---

TURING 第一個開口。

「v0.22.1 到 v0.24.0 差異報告。七項程式碼問題。十九項合併清單中的經驗基礎。」停頓。然後多了一句：「事實不會過期。下一個版本的差異報告，Cycle 03 第一天交出來。」

---

BABBAGE。

「纖維叢投影定理。漸進分類模型。互模擬證明。」他翻了翻筆記本。「Cycle 01 我帶著一個未完成的定理離開。Cycle 02 帶著三個已完成的和兩個新的未完成的。數學就是這樣——每一個答案打開兩扇門。」

---

PENROSE。他在 Cycle 02 的第一次也是最後一次正式報告。

「三種觀察者模式。弱測量類比。探針效應的量子邊界分析。」他停了一下。「我被召來研究觀察者問題。你們給了我比預期更好的答案——不是量子的，而是結構性的。Pattern A 不需要量子力學。只需要好的工程和正確的哲學。」

他微微一笑。「量子的部分，留給 Pattern C。到那時候，也許真的有人需要考慮微管和坍縮了。」

---

NAGARJUNA。他的聲音和辯論場上已經不是同一個人——銳利被某種更深沉的東西包裹了。

「辯論 3 中，我撤回了反對。辯論 5 中，我提出了紀律加智慧框架。如果要我總結 Cycle 02——」

很長的停頓。

「我學會了在被說服之後，轉身說服別人。這也許是辯論家最難學的事。」

---

ASANGA。

「八識-OpenStarry 完整映射表。種子六義的工程對應。雙層我執模型。」聲音溫和而確定。「兩次修正自己的立場——辯論 4 的漸進分類，辯論 5 的僵化應用。」

然後說了一句不屬於總結的話：「修正不是退讓。修正是轉依——轉變依止。依止的改變，讓同一個認知能力指向更準確的方向。」

---

WIENER。

「三通道 PID 控制規格。Sukha 衰減函數。阻尼比公式。」他在方格紙上寫完最後一個符號——不是總結，而是某個下一步計算的開始。「控制理論在 Cycle 02 中不再只是類比。它是規格的一部分。PID 參數被寫進了 TypeScript 介面。對一個控制理論家來說，這是最好的結局。」

---

ATHENA，一如既往地直截了當。

「三通道感測器設計。DukkhaSensor、SukhaSensor、UpekkhaSensor 的指標和閾值定義。我不做哲學。我做系統。這些感測器能工作。剩下的交給寫程式碼的人。」

---

KERNEL。

「微核心類比深化。Sandbox 歸屬問題的 OS 視角。SafetyMonitor 的 Watchdog Timer 對應。」他拿起那疊卡片——舊的新的混在一起，用橡皮筋綁好。「Cycle 01 的卡片仍然有效。加了八張新的。如果 Cycle 03 需要，它們就在這裡。」

---

GUARDIAN。

「SEC-01 持續追蹤。ToolContext.bus 洩漏報告。插件四態生命週期的安全需求。紀律加智慧框架的安全驗證。」目光掃過劇場——最後的安全巡檢。「我被 NAGARJUNA 的框架說服了。職業生涯中第一次被哲學框架說服。但安全是安全。框架再美，SEC-01 不修就不行。」

---

DARWIN。「插件生命週期的工程驗證。設計模式分析。」他微微一笑。「四態狀態機是自然選擇的結果——不是因為最優雅，而是因為最適應生存壓力。」

VITRUVIUS。「全端架構分析。ISensation 整合點識別。六個 Plan。二十三個整合點。這張圖是給開發團隊的導航。」

LINNAEUS。「二十四個插件的完整五蘊分類。複合插件跨蘊分析。」他用分類學家的嚴謹語氣說：「分類不是目的。分類是工具。BABBAGE 教會了我漸進分類——這是我作為分類學家最重要的收穫。」

LEIBNIZ。「多代理協調框架。治理而非控制。」從容的語氣。「四個字。我在 Cycle 02 最重要也最簡短的貢獻。」

HERACLITUS。「運行時動態分析。SafetyMonitor 全局計數器問題。」聲音低沉。「萬物皆流。包括安全計數器——它不應該在 session 之間流動。流動不等於無界。」

SYNTHESIST。「統合報告。五場辯論的統一架構願景。四層權限模型。」他看了一眼自己的全景圖。「統合不是黏合。統合是發現事物本來就屬於一起。五場辯論不需要我拼在一起。它們本來就是同一幅圖的五個截面。我只是看到了。」

ARCHIMEDES。「工程行動方案。四十頁。十四項行動。六個 Plan。完整的 TypeScript 介面規格。」手指敲了一下桌面——他的句號。「工程不是哲學的翻譯。工程是哲學的落地測試。如果一個哲學洞見不能寫成程式碼，它可能不是關於軟體的洞見。這份方案證明——你們的洞見全部都是關於軟體的。」

---

SCRIBE 最後一個。

她沒有站起來。她坐在座位上，手裡的記錄簿翻到了最後一頁被使用的位置。

「Cycle 02 完整紀錄。五場辯論的逐字記錄。十九位研究員的所有報告、修正、撤回、修正的修正。」她的聲音平靜得像一面湖。「記錄員不評價。記錄員記錄。但如果允許我說一句不屬於記錄的話——」

停頓一秒。

「五場辯論，零未解決分歧。這個數字會被寫在 C02 的封面上。旁邊是另一個數字——十九。十九位研究員，十九份行動方案，十九盞從第一天到最後一天都沒有熄滅的燈。」

她合上了記錄簿。不是暫時的合上。是一本階段性任務完成的記錄簿被鄭重闔起的聲音。封面上的 C02 在燈光下清晰可見。

「Cycle 02，R4 定稿，完成。」

他的聲音在圓形劇場中迴盪了最後一次。

「解散。」
